# 结局数据总览 — Papers, Please

> 游戏版本: Papers, Please v1.4 (PC/Steam, 2013)
> 数据来源: [Papers Please Wiki](https://papersplease.fandom.com/), [StrategyWiki](https://strategywiki.org/), [Steam社区指南](https://steamcommunity.com/app/239030/guides/)

## 一、结局总览（约20种）

| 结局编号 | 名称 | 类型 | 关键条件 |
|:--:|------|:--:|------|
| 1 | EZIC胜利 | 主线 | 完成所有EZIC任务+Day31帮助EZIC |
| 2 | EZIC失败(被捕) | 主线 | 接受EZIC但任务失败/被举报 |
| 3 | 逃亡Obristan | 逃亡 | Day29-31逃往Obristan |
| 4 | 逃亡被捕 | 逃亡 | 逃亡未遂→被边境卫兵逮捕 |
| 5 | 继续工作(家人全存活) | 职业 | 无EZIC/无逃亡+家人全存活 |
| 6 | 家人死亡(一个或多个) | 家庭 | 没钱付账单/药品→家人病死/饿死/冻死 |
| 7 | 被逮捕(违规太多) | 职业 | 犯错太多→被降级→失业→被捕 |
| 8 | 被处决 | 职业 | 极端违规(如帮助恐怖分子)→被处决 |
| 9-20 | 其他变体 | 混合 | 各路线的中间状态 |

---

## 二、结局判定机制

判定由以下变量累积决定：

| 变量 | 影响 |
|------|------|
| EZIC任务完成度 | 0/5→无EZIC结局 → 5/5→EZIC胜利结局 |
| 家人存活状态 | 全存活→好结局，有死亡→坏/中性结局 |
| 违规次数 | 过多→被逮捕/失业结局 |
| Day31选择 | 逃亡/帮助EZIC/继续工作 |
