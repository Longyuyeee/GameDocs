# 选择与分支架构 — Papers, Please

> 游戏版本: Papers, Please v1.4 (PC/Steam, 2013)
> 数据来源: [Papers Please Wiki](https://papersplease.fandom.com/), [Lucas Pope GDC演讲](https://www.gdcvault.com/), [Steam社区](https://steamcommunity.com/app/239030/)

## 一、"盖章"——将道德选择压缩为一个动作

Papers, Please 的选择架构是品类中最极简的——**所有选择都通过"盖章"完成**：

| 动作 | 叙事意义 |
|------|------|
| **盖绿色章(批准)** | "我选择帮助这个人——我选择承担风险" |
| **盖红色章(拒绝)** | "我选择遵守规则——我选择拒绝这个人" |
| **不盖章(拖延)** | 你无法决定——但时钟在走，你赚不到钱——家人会挨饿 |

**设计革命**: "盖章"这个动作将"道德选择"从传统的"阅读选项→点击A或B"压缩为一个**单一动作**——但每一次盖章都包含了**规则/同情/恐惧/贪婪**四重考量。

## 二、选择密度

| 游戏日 | 申请人数量 | 选择次数 | 累计选择 |
|:--:|:--:|:--:|:--:|
| Day 1 | ~5-8 | ~5-8 | 5-8 |
| Day 10 | ~8-12 | ~8-12 | ~80-100 |
| Day 20 | ~12-15 | ~12-15 | ~200-250 |
| Day 31(终) | ~10-15 | ~10-15 | **~300-400次选择/每局** |

**每一次盖章=一次选择**。Papers, Please 在一个月（游戏时间）内让你做出300-400次道德选择——**远超任何其他叙事游戏的选择密度**。

## 三、分支拓扑——网状后果系统

Papers, Please 不使用传统的"分支树"——所有选择的后果累积成一个**网状**：

```
Day 1: 盖了10个章 → 赚了X钱 → 家人吃了饭
                        ↓
Day 2: 规则改变——"现在需要检查入境许可"
                        ↓
        你犯了错误 → 被罚钱 → 家人没饭吃 → 病人生病
                        ↓
Day 5: 一个妇女求你让她通过——她的孩子在外面
        ├── 让她通过(违反规则) → 你被罚钱 → 但她的孩子活了
        └── 拒绝她(遵守规则) → 你保持记录干净 → 但她可能...
```

## 四、"真选择"vs"假选择"——Papers, Please的独特之处

Papers, Please **没有"假选择"**——每一个章都是真选择：

| 选择类型 | 为什么是"真选择" |
|------|------|
| 放行 vs 拒绝 | **任何一方都有真实后果**——放行=风险+同情，拒绝=安全+内疚 |
| 加快速度 vs 仔细检查 | 加快=多赚钱但多犯错，仔细=少犯错但少赚钱 |
| 举报可疑 vs 不举报 | 举报=忠诚+风险，不举报=沉默+可能的后患 |

**与苏丹的游戏的对比**: 苏丹的游戏用卡牌承载选择——Papers, Please 用**印章**。两个游戏都将复杂的道德选择压缩为一个极简动作。但苏丹的游戏的"抽卡"是**被迫**的——Papers, Please 的"盖章"是你**主动的**。
