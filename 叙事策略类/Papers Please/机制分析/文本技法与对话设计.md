# 文本技法与对话设计 — Papers, Please

> 游戏版本: Papers, Please v1.4 (PC/Steam, 2013)
> 数据来源: [Papers Please Wiki](https://papersplease.fandom.com/), [Lucas Pope开发者博客](https://dukope.com/), [Steam社区](https://steamcommunity.com/app/239030/)

## 一、"官僚语言"——将人性压缩为表格

Papers, Please 的文本策略是品类中最独特的——**几乎所有文本都是"官僚语言"**：

| 文本类型 | 风格 | 功能 |
|------|------|------|
| **规则手册** | 冰冷的官方语言——"根据Arstotzka劳动法第4条第2款..." | 脱人性化——将"拒绝一个人"变成"违反第4条第2款" |
| **申请人对话** | 简短——破碎的英语——"Please...my wife...she is sick..." | 在官僚语言的海洋中——人性偶尔浮出水面 |
| **家庭账单** | 数字+项目——"房租: $15 / 食物: $10 / 暖气: $5" | 将"爱"量化为"钱"——你爱你的家人=你付账单 |
| **EZIC消息** | 密码+暗语——在官方文件中隐藏的"地下语言" | 系统中有另一层语言——反抗的语言 |
| **报纸** | 宣传语调——"Arstotzka Glory to the Highest!" | 官方叙事的"正确用语" |

## 二、"沉默的文本"——没有写的内容

Papers, Please 最有力量的文本是**没有写的文本**：

| 你没读到的 | 你感受到的 |
|------|------|
| 你拒绝的人后来怎么了 | **你自己填充了最坏的可能** |
| 你的家人长什么样子 | 你没有画面——只有名字和"需要"的列表 |
| "外面的世界"是什么样的 | 你不知道——报纸说"很好"，但那是宣传 |
| EZIC的真正目的 | 你不知道他们"好"还是"坏"——你只知道他们**在反抗** |

---

## 三、"反乌托邦的语调"——通过官僚语言创造

Papers, Please 不告诉你"这是反乌托邦"——它通过**官僚语言本身**让你感受到：

| 官僚语言的元素 | 反乌托邦效果 |
|------|------|
| 规则没有"为什么"——只有"什么" | 规则不需要理由——因为你是公务员，不是质疑者 |
| 每天的新规则=每天的新"不应该做什么" | 不确定性+恐惧——"我昨天做的事今天会害我吗？" |
| "Glory to Arstotzka" | 强制性的爱国口号——不真诚但必需的 |
