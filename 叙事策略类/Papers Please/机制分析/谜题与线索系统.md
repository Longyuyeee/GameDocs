# 谜题与线索系统 — Papers, Please

> 游戏版本: Papers, Please v1.4 (PC/Steam, 2013)
> 数据来源: [Papers Please Wiki](https://papersplease.fandom.com/), [Lucas Pope GDC演讲](https://www.gdcvault.com/), [Steam社区](https://steamcommunity.com/app/239030/)

## 一、无传统谜题——"文件验证"作为替代系统

Papers, Please **无谜题/非推理游戏**。谜题驱动叙事的替代设计是将**"文件真实性验证"**转化为一个非传统的"解谜"系统：

| 传统推理游戏 | Papers, Please |
|------|------|
| 谜题: "谁杀了人？" | **"这张文件是真的吗？"** |
| 线索: 证物/证词 | **线索: 护照号码/出生日期/印章/照片/指纹** |
| 推理: 逻辑演绎 | **推理: 逐项比对——"这个日期和那个日期矛盾"** |
| 真相揭示: 指认真凶 | **真相揭示: 拒绝/放行——一个生命的命运** |

---

## 二、文件验证的"解谜"深度

| 验证项目 | 复杂程度 | 常见矛盾 |
|------|:--:|------|
| 照片 vs 本人 | ★☆☆☆☆ | 申请人说"我胖了"——你信吗？ |
| 日期（护照有效期/入境许可日期） | ★★☆☆☆ | 日期格式不同——"1986-05-12 vs 12/05/1986" |
| 姓名拼写 | ★★☆☆☆ | "Grzegorz vs Grigori"——拼写错误还是不同的人？ |
| 指纹匹配 | ★★★☆☆ | 指纹扫描仪显示"MATCH"或"MISMATCH"——但设备可能出错 |
| 多重文件交叉验证 | ★★★★☆ | 护照说来自Arstotzka——但入境许可说Obristan——"你在说谎" |
| 体重/身高 vs 本人 | ★★★☆☆ | "他们说180cm——但他看起来没有" |

**设计意图**: 文件验证不是你"赢"的游戏——它是你"**承担道德责任**"的方式。每一次"我发现了矛盾"同时意味着"我需要拒绝这个申请人"。

## 三、EZIC线索——系统中的"腐烂"

游戏通过EZIC(反抗组织)传递**隐藏信息**——这些信息是线索，但不是"推理"用的——它们是**选择**用的：

| EZIC信息 | 内容 | 你的选择 |
|------|------|------|
| "明天有一个叫XXX的人会来——让他通过" | 你不是在"调查"——你在**被招募** | 帮助EZIC(风险) vs 报告EZIC(忠诚) |
| 密码/暗号 | 在文件中隐藏的密码——"我知道你是EZIC的人" | — |

**与逆转裁判/弹丸论破的对比**: Papers, Please 不提供"谜题→推理→真相"的满足感。它提供的是"**我发现了矛盾→我选择相信还是不相信→我承担后果**"的道德重量。
