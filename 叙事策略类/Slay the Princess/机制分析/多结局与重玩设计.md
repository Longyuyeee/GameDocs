# 多结局与重玩设计 — Slay the Princess

> 游戏版本: Slay the Princess v2023 (PC/Steam)
> 数据来源: [Slay the Princess Wiki](https://slay-the-princess.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/1989270/guides/)

## 一、结局体系——"许多版本的公主"

Slay the Princess 约5-8种主要结局——由你在所有循环中与公主的互动累积决定：

| 结局 | 核心条件 |
|------|------|
| 杀了公主（永远） | 服从叙述者——在所有循环中选择"杀" |
| 救了公主 | 在所有循环中选择"救"——违抗叙述者 |
| 与公主离开 | 你和她一起走出小屋——面对未知的世界 |
| 成为公主 | 你在某个循环中与她"交换位置" |
| 无限循环 | 你既不完全杀也不完全救——循环永不停止 |
| "真实"结局 | 揭示了叙述者/小屋/世界的本质（meta-narrative层） |

---

## 二、重玩驱动——"我想要看到每一个版本的公主"

| 周目 | 目标 |
|:--:|------|
| 一周目 | 默认路径——大多数玩家选择"不完全服从也不完全不服从" |
| 二周目+ | "如果我每次都带刀会怎样？"→不同的公主版本 |
| 全公主收集 | 10+种公主形态——每种需要不同的互动策略 |
| 探索meta层 | "如果我在这个节点做完全不同的选择——世界会揭示什么？" |

**与命运石之门的重玩对比**: 命运石之门: "我想看到不同的世界线结局"——改变关键选择。Slay the Princess: "我想看到公主变成什么"——**每个循环是一个新的"她"**。
