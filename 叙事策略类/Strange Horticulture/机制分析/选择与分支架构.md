# 选择与分支架构 — Strange Horticulture

> 游戏版本: Strange Horticulture v2022 (PC/Steam/Switch/Xbox)
> 数据来源: [Wikipedia](https://en.wikipedia.org/wiki/Strange_Horticulture), [Game Developer设计访谈](https://www.gamedeveloper.com/design/finding-right-plant-job-strange-horticulture), [Steam中文深度评测](https://steamcommunity.com/id/weinima233/recommended/1574580), [Nintendo Life评测](https://www.nintendolife.com/reviews/switch-eshop/strange-horticulture), [TVTropes](https://tvtropes.org/pmwiki/pmwiki.php/VideoGame/StrangeHorticulture)

## 一、"植物识别=叙事选择"——品类中唯一的物品识别驱动叙事引擎

Strange Horticulture的核心突破：**将"物品识别"从副系统提升为唯一的叙事引擎**：

| 传统冒险游戏 | Strange Horticulture |
|------|------|
| "找钥匙→开门" | "识别出Clavilium→给顾客→打开锁(或毒死他)" |
| 物品=工具——"使用" | **植物=选择**——"给了什么植物=你选择了什么" |

**开发者Rob Donkin**: "Plants be the way you interact with the game world."——植物不只是"道具"——植物是**你与世界的全部交互界面**。

---

## 二、"给了什么植物"=16天×多分支=8结局

| 选择节点 | 内容 |
|------|------|
| **顾客请求** | 描述症状或需求——"我需要一种'能让人忘记悲伤'的植物" |
| **你的识别** | 查图鉴→比对外观/气味/效果→选择货架上的植物→给出 |
| **给正确的植物** | 顾客被帮助→解锁新图鉴页/植物→主线推进 |
| **给错误的植物** | "Rising Dread"(恐惧值)增加→满值=心智破碎→需要特殊解谜恢复 |
| **给"正确但是..."的植物** | 你斟酌后给了某种植物——"这能缓解他的症状——但是副作用..."——道德灰度 |

**设计精妙之处**: 开发者刻意设计为**"每个顾客来访必须有叙事理由"**——顾客不能"只是来聊天的"——他们**必须**需要一种植物。这迫使植物系统与叙事完全绑死。

---

## 三、8结局——植物=命运

| 结局分支 | 决定因素 |
|------|------|
| 3条主要分支 | 16天中关键顾客的植物选择——"你给了Sisterhood什么？你给了Draer什么？" |
| 8个结局 | 主要分支上的微调——"你选择'安抚'还是'加剧'顾客的状况？" |

**与Papers, Please的对比**: Papers, Please=印章=决定谁通过/不通过。Strange Horticulture=**植物=决定谁被治愈/谁中毒**。两者都将"一个简单动作"转化为"道德选择"——但Papers, Please的印章是主动的官僚权力，而Strange Horticulture的植物选择是**知识=权力**——"我识别出的植物=我有治愈或毒害的权力。"

---

## 四、"图鉴=知识=选择空间"

| 图鉴状态 | 你能做的选择 |
|------|------|
| 图鉴页空白(植物未识别) | **你根本不知道这个植物能做什么**——"你无法选择'用这个植物'——因为你不认识它" |
| 图鉴页解锁 | "现在你知道了——你可以选择它了" |

**设计意图**: "知识=选择的先决条件"——你不是在"已有的选项"中选择——你在**"通过识别来创造选项"**。
