# 多结局与重玩设计 — 这是我的战争

> 游戏版本: This War of Mine Final Cut v2021 (PC/Steam)
> 数据来源: [This War of Mine Wiki](https://this-war-of-mine.fandom.com/), [Steam社区指南](https://steamcommunity.com/sharedfiles/filedetails/?id=2524871302)

## 一、结局系统概述

这是我的战争的结局不是"你选择了A路线→结局A"——它是**系统根据你在整个战争期间的累积行为自动生成的个性化尾声**。每个角色有一个独立的结局段落，由一个算法根据角色的状态和行为历史来撰写。

### 1.1 结局生成逻辑

| 输入因子 | 如何影响结局 |
|------|------|
| 存活/死亡 | 死了就没有结局 |
| 精神状态(是否崩溃过) | 崩溃过→更悲观/负面的结局 |
| 道德行为累计 | 偷窃/杀害平民/拒绝帮助→结局更阴暗 |
| 帮助他人的记录 | 救助邻居/帮助医院→结局更温暖 |
| 成瘾管理 | 成瘾得不到满足→结局中提及负面影响 |
| 同伴关系 | 同伴死亡→所有幸存者的结局变差 |

## 二、角色结局的典型类型

### 2.1 好结局示例

| 角色 | 好结局内容 |
|------|------|
| Pavle | "战争结束后，Pavle找到了他的家人。成为当地一所学校的足球教练。" |
| Katia | "Katia成为了一名记者，致力于报道冲突地区的平民故事——确保这个世界的Pogorens不会被遗忘。" |

| 角色 | 坏结局内容 |

## 三、"没有好结局"的设计

与命运石之门不同——这是我的战争**没有"真结局"**。即使是最好的结局，也带着战争的痕迹。你无法让所有人获得完美结局——角色们的尾声总是混合了"他们活下来了"和"战争从他们身上夺走了某些东西永远无法挽回"。

| 对比 | 命运石之门 | 这是我的战争 |
|------|------|------|
| 有真结局？ | 是(1.048596%) | **没有**——只有"更好"或"更坏"的尾声 |
| 完美结局的条件 | 完成6个精确FLAG | **不存在"完美"**——总有人失去了什么东西 |
| 结局的情感基调 | "我改变了命运" | **"我活下来了...但这代价是什么？"** |

## 四、重玩设计：随机性驱动的重玩

### 4.1 每次游戏的变量

| 变量 | 随机性 |
|------|:--:|
| 初始角色组合 | 3-4人，从12+个角色中随机(可根据剧本自定义) |
| 季节开始时间 | 春季/夏季→标准；冬季开始→地狱难度 |
| 可用搜寻地点 | 每天刷新的可用地点不同 |
| 随机事件 | 敲门者、暴风雪、犯罪浪潮、停火 |
| 结局长度 | 战争在25-45天之间随机结束 |

### 4.2 重玩的价值

| 重玩方式 | 新体验 |
|------|------|
| 不同角色组合 | Boris+Roman vs Katia+Bruno——完全不同的挑战 |
| 不同道德路线 | "完全利他"vs"完全自私"——看结局如何变化 |
| 不同剧本(Stories DLC) | 3个手工编写的线性故事——每个有不同的角色和挑战 |
| 冬季开始 | 极限难度——没有时间准备 |
