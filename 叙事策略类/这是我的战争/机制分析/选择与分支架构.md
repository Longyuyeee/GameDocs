# 选择与分支架构 — 这是我的战争

> 游戏版本: This War of Mine Final Cut v2021 (PC/Steam, 原版2014)
> 数据来源: [This War of Mine Wiki](https://this-war-of-mine.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/282070/guides/), [Game Developer访谈](https://www.gamedeveloper.com/design/the-inspirational-stories-behind-i-this-war-of-mine-i-)

## 一、选择架构概述

这是我的战争的选择系统与命运石之门或逆转裁判完全不同——它**没有对话树、没有A/B菜单、没有"关键选择节点"**。它的选择嵌入在**每一个生存行为**中。出去搜寻时拿了最后一块面包——你做出了一个"选择"。开门收留了一个陌生人——你做出了一个"选择"。制作了一把斧头而不是一个炉子——你做出了一个"选择"。

### 1.1 "没有选择菜单的选择系统"

| 传统叙事游戏 | 这是我的战争 |
|------|------|
| 显式的"选A或选B" | **隐式的"做或不做"** |
| 选择=菜单操作 | **选择=游戏行为本身** |
| 阅读选项→预计后果→选择 | **做了一件事→后果在几天/几周后才显现** |
| 选择有"正确的" | **没有"正确选择"——所有选项都有代价** |

## 二、选择的类型体系

### 2.1 生存选择（每天每夜都在做）

| 选择类型 | 典型场景 | 直接后果 | 延迟后果 |
|------|------|------|------|
| **资源分配** | 3个人2块面包——谁吃？ | 没吃的人饥饿度上升 | 长期饥饿→疾病→死亡 |
| **建造优先级** | 先造炉子还是先造床？ | 决定了接下来几天的生存质量 | 影响了整个游戏进程 |
| **搜寻目标选择** | 今晚去超市(食物)还是废弃公寓(材料)？ | 带回来不同类型的资源 | 决定了短期能否撑过冬天 |
| **工具使用** | 用仅剩的武器零件做刀还是做斧头？ | 刀=战斗/斧头=收集木材 | 影响了应对威胁的能力 |

### 2.2 道德选择（游戏的核心）

| 道德困境场景 | 选项A | 选项B | 选项C |
|------|------|------|------|
| **安静的小屋**(一对老夫妇) | 偷走他们的食物和药品 | 只拿一些，留下一点 | 什么都不拿 |
| **超市**(士兵骚扰女孩) | 帮助女孩→与士兵战斗 | 无视→保全自己 | 从背后偷袭士兵→抢武器 |
| **医院**(受伤的陌生人) | 救助他→消耗药品 | 拒绝→他可能死 | |
| **敲门者**(深夜有人敲门求助) | 开门让他进来→多一张嘴 | 拒绝→他可能死在外面 | |
| **狙击手交叉点**(受伤的人求救) | 冒险救他→可能自己受伤 | 无视→保全自己和物资 | |
| **军队前哨**(大量武器和食物) | 偷窃→物资充裕 | 不偷→保持清白 | 偷窃+杀士兵→物资充裕且道德负担较轻 |

### 2.3 角色管理选择

| 选择 | 后果 |
|------|------|
| 让谁去搜寻？ | Boris(17格背包但最慢) vs Pavle(最快但12格) vs Roman(战斗最强但背包10格) |
| 让谁守夜？ | Roman(最强守夜) vs 其他人(不同守夜能力) |
| 让谁睡床？ | 疲劳的人优先——但床的位置有限 |

## 三、分支拓扑分析

### 3.1 "涌现式分支"而非"预设式分支"

这是我的战争没有预设的"分支树"——它的分支是**系统生成的**：

```
[初始条件] — 随机3-4个角色 + 随机季节开始 + 随机避难所状态
     │
     ▼
[每天] ──→ [白天选择] — 建造/制作/休息/治疗
     │         │
     │         └──→ [夜晚选择] — 谁去搜寻/去哪里/如何应对
     │                   │
     │                   └──→ [搜寻中的即时选择] — 打/跑/偷/帮/交易
     │                              │
     └──────────────────────────────┘
                ▼
         [后果累积] → 角色状态变化 → 结局
```

### 3.2 "蝴蝶效应"式的因果

与命运石之门的"世界线收束"不同，这是我的战争的因果是**累积式的**——没有"物理法则"强制某个结果，而是**你的每一个选择都在累积**：

| 因果链示例 |
|------|
| 初期选择造斧头而非床 → 有更多木材造炉子 → 冬天前准备好→ 熬过冬天 → 但一直缺床→ 疲劳累积→ 某个角色生病→ 需要药品→ 冒险去危险地区→ 角色受伤→ ... |
| 初期选择救助受伤的陌生人 → 消耗了药品 → 但获得了新角色(额外劳动力) → 可以更快建造 → ... |

## 四、"没有好选择"的设计哲学

### 4.1 这是核心设计原则

11 bit studios的首席设计师Pawel Miechowski强调：游戏中的每一个道德困境都是**故意设计的没有好答案**。

> "There is no good or bad decision. You just have to survive. And dealing with the consequences." — 这不是关于对错——是关于生存。

### 4.2 为什么"所有选项都是坏的"？

| 困境 | 选项A的坏处 | 选项B的坏处 | 没有选项C |
|------|------|------|:--:|
| 老夫妇 | 偷=老人可能死+角色负罪感 | 不偷=你和你的角色可能饿死 | ❌ |
| 士兵骚扰女孩 | 帮忙=战斗风险 | 无视=道德负担 | ❌ |
| 孩子来敲门 | 开门=更多资源消耗 | 不开=孩子在雪地里冻死 | ❌ |

### 4.3 后果的"延迟支付"

这是我的战争最残酷的设计之一是：你做了一件"坏事"（偷了老夫妇的药），**当时没有立即的惩罚**——但几天/几周后，角色开始说"我睡不着"、"我不能忘记那对老人"——最终崩溃或离开。

## 五、与本品类其他游戏的对比

| 维度 | 这是我的战争 | 命运石之门 | 逆转裁判 |
|------|------|------|------|
| 选择方式 | 隐式(行为即选择) | 半隐式(手机回复) | 显式(指证/追问) |
| 分支类型 | 系统涌现式(无数微选择) | 预设式(6路线) | 伪分支(单线) |
| 核心体验 | "我在绝境中做出了选择...我承受了后果" | "我的小选择改变了世界" | "我推理出来了" |
| "正确"选择？ | **无**——只有"不那么坏的" | 有(6个明确结局路径) | 有(每个案件一个真相) |
| 后果时间 | 延迟(几天到几周) | 立即(世界线跳转) | 即时(扣血/推进) |
