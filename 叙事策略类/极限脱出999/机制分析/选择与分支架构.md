> 游戏版本: Zero Escape: Nine Hours, Nine Persons, Nine Doors v2017 (PC/Steam, 原版2009 NDS, Fami通36/40, IGN 9.0)

## 一、"数字根"分支系统——数学驱动的命运选择

999的分支架构由**数字根(Digital Root)**驱动——这是品类中唯一将"数学规则"作为分支机制的案例：

| 规则 | 内容 |
|------|------|
| 参与者编号 | 9人各自手腕有1-9的号码——固定不变 |
| 门的编号 | 每扇门标有1-9的数字 |
| 数字根计算 | 进入门的参与者的腕号之和→求数字根→必须等于门号才能进入 |
| 组队限制 | 3-5人/组——"谁和谁组队=谁可能死——谁可能活" |

### 数字根计算规则
数字根=将一个整数的各位数字相加，若结果≥10则继续相加直到<10为止。例如: 5+8+7=20→2+0=2

### 关键门选择节点

| 门号 | 示例参与者组合 | 走向路线 | 对应结局 |
|:--:|------|------|:--:|
| 门4 | 淳平(5)+紫(6)+八代(4)+一宫(1)=16→7 | →门7→门6 或 →门7→门1 | Sub Bad End / True End |
| 门5 | 淳平(5)+サンタ(3)+尼尔斯(2)+ユウキ(9)=19→1 | →门8→门6 或 →门8→门1 | Knife Bad End / Gas Bad End |

**与逆转裁判的对比**: 逆转裁判的分支是"伪分支+惩罚循环"——每个案件只有一个真相。999的分支是"真分支+信息积累"——不同的门选择=看到不同的真相碎片。逆转裁判的核心满足感="我找到了唯一正确的答案"；999的核心满足感="我收集了所有碎片——现在全图清晰了"。

## 二、分支拓扑——6条路线→1个真结局

```
开始(9人被困)
    ├── 门4 → 门7 → 门6 → Sub Bad End
    ├── 门4 → 门3 → 门2 → Axe Bad End
    ├── 门5 → 门8 → 门6 → Knife Bad End
    ├── 门4 → 门8 → 门1 → Coffin Bad End
    ├── 门5 → 门8 → 门1 → Gas Bad End
    └── 门4 → 门7 → 门1 → True End(形态形成场揭示)
```

**设计精妙之处**: 真结局路线(门4→7→1)的前两个门(4→7)与第一条Bad End路线相同——直到最后一个门(1 vs 6)才分道扬镳。这意味着你**必须**先经历Bad End中的信息——然后回到分歧点——选择另一个门——使用你在Bad End中"学到的信息"来解锁真结局。跨路线知识=形态形成场=玩家自身的体验。

## 三、"剧情锁"——信息积累式解锁

| 锁 | 内容 | 解锁方式 |
|------|------|------|
| 真结局锁 | 需要完成至少一个Bad End | 经历"错误"路线——获取跨路线信息 |
| 信息锁 | 你在路线A中知道"船长是谁"——但路线B中你没有这个信息 | **玩家自己**携带了跨路线知识——这就是形态形成场的叙事隐喻 |

**与命运石之门的对比**: 命运石之门使用"D-Mail"作为跨世界线的信息传递——999使用**玩家本身**作为跨路线信息传递的媒介。命运石之门=角色在"记忆"另一条世界线；999=**玩家**在"知道"另一条路线——这两种设计都是"meta-narrative"的核心技法。

## 四、DS版vs PC版——双屏设计的叙事损失

| 版本 | 双屏处理 | 叙事影响 |
|------|------|------|
| **NDS(2009)** | 物理双屏——上屏=Akane视角(过去)/下屏=淳平视角(现在) | **完美**——双屏分离=时间分离，最终反转利用硬件特性 |
| **PC(2017)** | 单屏——对话/叙事交替显示 | **部分损失**——双时间线的视觉分离不再存在——反转的冲击力降低了 |

**设计启示**: 999是"媒介特性驱动叙事"的教科书——NDS的双屏不是UI便利——它是**叙事的硬件基础**。PC版的移植不可避免地削弱了原设计中最精妙的维度。这是叙事策略品类中"机制=媒介=叙事"三位一体的经典案例。
