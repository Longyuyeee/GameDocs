# 团队与开发历程 — 海市蜃楼之馆

> 开发商: NOVECT(旧称Novectacle, 日本同人社团) | 核心创作者: Keika Hanada(剧本/导演)
> 发行年份: 2012(Comiket) / 2016(Steam) | 销量: 10万+(Steam)
> 数据来源: [automaton-media](https://automaton-media.com/en/news/20231123-23261/), [automaton-media(2)](https://automaton-media.com/en/news/it-used-to-sell-only-70-copies-a-month/)

## 1. 开发团队

| 角色 | 姓名 | 贡献 |
|------|------|------|
| 剧本/导演/脚本/视觉指导 | **Keika Hanada** | 一人承担多个核心角色——撰写全部剧本、指导视觉方向、编写游戏脚本 |
| 美术 | **Moyotaro** | 水彩/油画风格的背景与角色——哥特美学 |
| 音乐 | 多名作曲家(同人音乐圈) | 歌剧风格配乐+葡萄牙语吟唱 |

团队最初是**大学朋友组成的同人社团**——没有游戏开发经验，在学业和工作之余制作。引擎使用KiriKiri(同人视觉小说常用引擎)，**全程无配音**——纯粹的文字驱动。

## 2. 开发历程

| 时间 | 事件 |
|:--:|------|
| 2012年12月 | Comiket 83首发——同人摊位 |
| 2016年5月 | Steam英文版上线(MangaGamer发行) |
| 2016-2020 | **"至暗时刻"**——很多月份Steam销量不到100份。Hanada对MangaGamer感到抱歉 |
| 2021 | Switch版发售——**Metacritic评分96/100**，一度Switch最高分游戏 |
| 2023年11月 | **Steam销量突破10万份** |
| 2025年3月 | Steam视觉小说节——**一周售出10,000份**（发行9年后！） |

## 3. "70份/月"到"10万+"——口碑的胜利

Hanada将游戏的成功完全归功于**玩家口碑**：多语言翻译志愿者(7种语言)、Reddit/Discord社区的持续推荐、Steam评测的极高好评率(96%)。没有任何大型营销——是一个"纯粹的玩家驱动的成功"。

### 3.1 为什么"没有营销"还能成功？

| 因素 | 说明 |
|------|------|
| **叙事品质的"传教效应"** | 通关的读者成为"传教士"——自发在社区推荐、写长评、劝说朋友购买 |
| **翻译志愿者的贡献** | 7 种语言的志愿者翻译——让游戏能够触及非日/英语玩家群体 |
| **媒体评分作为"信任信号"** | Switch 版 Metacritic 96 分——在潜在玩家搜索"这个游戏好吗？"时提供了即时答案 |
| **Steam 视觉小说社区的口碑传播** | VN 社区规模小但高度集中——少数高质量推荐就足以覆盖核心受众 |
| **"9 年长尾"** | 2016 年发售，2025 年仍能在一周内售出 10,000 份——证明了叙事品质不受时间衰减影响 |

## 3b. 开发技术与工具

| 工具 | 用途 | 说明 |
|------|------|------|
| **KiriKiri (吉里吉里)** | 游戏引擎 | 同人视觉小说社区的标准引擎——免费、轻量、文本驱动 |
| **手绘水彩/油画** | 美术 | Moyotaro 的非动漫风格在 VN 市场中极为稀有——介于 Amano Yoshitaka 和 Vampire Hunter D 之间 |
| **葡萄牙语歌词创作** | 音乐 | 作曲家与 Hanada 合作——歌词内容与故事主题紧密关联 |

项目无配音的根本原因不是预算（虽然后来预算也有限）——是**审美选择**：Hanada 认为"文字本身的声音"比声优的表演更适配哥特文学的质感。

## 5. 设计启示

1. **纯线性叙事+极致文学质量=可以成功**——在"多结局=标配"的时代，海市蜃楼之馆证明了"一个好故事不需要分支"
2. **"门"的框架是完美的悲剧结构**——每扇门=一个独立完整的悲剧单元，但通过"馆"连接成更宏大的叙事
3. **文字本身的视觉化设计**——颜色、速度、屏幕位置的动态变化让"纯文字"有了电影般的质感
4. **长期口碑>短期营销**——从月销70份到10万+，用了7年——但"好故事"最终会被发现
