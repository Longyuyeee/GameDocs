# 文本规模数据总览 — 命运石之门

> 游戏版本: Steins;Gate v2016 (PC/Steam, 原版2009)
> 数据来源: [Committee of Zero Script Analysis](https://github.com/CommitteeOfZero), [VNDB Steins;Gate](https://vndb.org/v217), [Bangumi 命运石之门页面](https://bangumi.tv/game/2166)

## 一、文本总量统计

命运石之门是科学ADV系列中文本量最大的作品之一。以下数据基于游戏脚本分析(Steam版+原版2009):

| 统计项 | 数据 | 备注 |
|------|:--:|------|
| 总字数(日文原版) | 约91万字 | 包含所有路线和分支文本 |
| 总字数(中文翻译版) | 约100万字 | 中文化后的估计文字量 |
| 总角色字符数(日文) | 约200万字符 | 含标点和TIPS文本 |
| 语音台词数 | 约2.5万条 | 主要角色的全语音覆盖 |
| TIPS词条数 | 约120条 | 科学概念/术语的解释弹出 |
| CG枚数 | 约50枚(含差分) | 包含事件CG和背景差分 |
| 邮件总数(玩家可收) | 约80封 | 不同分支不同邮件组合 |

> 与**《幽灵诡计》**对比：幽灵诡计的全文本量约15万字(日文)——只有命运石之门的约1/6。幽灵诡计靠"精炼的对话+核心反转"在极短篇幅内完成叙事；命运石之门则选择"大量的日常对话建立情感→密集的反转引爆"。两者代表了视觉小说叙事的两种极端风格——"轻量爆发型"和"厚重沉浸型"。

### 2.2 文字量分布特点

- **最长的三章**: Ch.10(9万字)、Ch.4(8万字)、Ch.6(8万字)——分别对应"高潮执行""角色关系建立""转折触发"
- **最短的章节**: 序章(2万字)、Ch.11(4万字)——"开场冲击"和"收束解决"都相对简短
- **节奏控制**: 前半(序章-Ch.5)占约40%，后半(Ch.6-Ch.11)占约60%——后半更长，因为包含了多条分支路线的文本

## 四、文本类型分布

| 文本类型 | 占比估计 | 功能 | 示例 |
|:--:|:--:|------|------|
| **日常对话** | 约45% | 角色关系建立、世界观暗示、笑点提供 | 未来道具研究所的日常互动、秋叶原探索 |
| **剧情叙述** | 约25% | 推动主线、事件描述、世界线切换描写 | 冈部的内心独白、关键事件的叙述 |
| **科学解说/教学** | 约10% | 时间机器原理、世界线收束理论、科学概念 | 红莉栖的学术解说、TIPS的弹出词条 |
| **D-Mail/邮件文本** | 约8% | 分支触发、角色性格展示、线索提供 | 所有可回复/不可回复的邮件正文+回复 |
| **情感独白/内心戏** | 约7% | 冈部的心理状态描述、决策时的内心挣扎 | "我该怎么办"系列独白、崩溃时的自白 |
| **TIPS系统** | 约3% | 科学概念解释、术语定义 | "Kerr黑洞""世界线""牵牛星"等词条 |
| **Phone Trigger对话** | 约2% | 分支关键、角色关系值影响 | 电话中对不同角色的回应 |

### 4.1 文本类型分布的叙事功能

- **45%日常对话**是"慢热"的直接原因——命运石之门花了大量篇幅建立"普通的秋叶原日常"，让玩家爱上角色，为后期的"失去"做准备
- **10%科学解说**不是"说教"——是通过红莉栖的学术性格自然呈现，同时为最终"欺骗世界"方案提供理论基础
- **8%邮件文本**看似是"附加内容"——但实际上是**真正的分支系统UI**——玩家在回复邮件中完成的"选择"构成了FLAG系统

## 五、邮件系统数据

| 邮件类型 | 数量 | 占总量比例 |
|:--:|:--:|:--:|
| 玩家可回复的关键邮件 | 约12封 | 决定路线和FLAG |
| 玩家可回复的普通邮件 | 约30封 | 影响角色关系值但不变动世界线 |
| 不可回复的剧情邮件 | 约38封 | 推进剧情或提供背景信息 |
| D-Mail(发送到过去) | 6封 | 造成世界线变动的最重大选择 |
| **全邮件合计** | **约80封** | |

> 与**《极限脱出：善人死亡》**对比：善人死亡使用"对话选项"作为分支机制——每次对话有2-3个选项，但"错误"选择会直接导致死亡结局；命运石之门的分支不惩罚玩家——选项错误只是让你走向不同的结局路线，没有"Game Over"。
