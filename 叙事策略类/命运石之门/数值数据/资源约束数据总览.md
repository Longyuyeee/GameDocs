# 资源约束数据总览 — 命运石之门

> 游戏版本: Steins;Gate v2016 (PC/Steam, 原版2009)
> 数据来源: [Steins;Gate Wiki Mechanics](https://steins-gate.fandom.com/wiki/Gameplay), [SciADV Wiki System](https://sciadv.fandom.com/wiki/Steins;Gate/System)

## 一、命运石之门的约束体系概述

命运石之门作为一款视觉小说，没有传统意义上的"HP/MP/金钱"等资源管理——它的"资源约束"是完全叙事化的：**时间约束、信息约束和情感约束**共同构成了玩家在决策时的"不可随意行为"框架。

> 与**《这是我的战争》**对比：我的战争使用显性的数值资源管理(食物、药品、材料)来制造生存压力。命运石之门则使用隐性的叙事约束"你不能同时救两个人""你不能做所有选择"——前者的约束可见可计算，后者的约束不可见但更沉重。

## 二、D-Mail发送次数限制

### 2.1 可发送D-Mail的总数

| 统计项 | 数据 |
|:--:|:--:|
| **D-Mail总数(剧情中)** | 6封剧情强制发送 |
| **玩家可主动发送的D-Mail** | 6封——但全部由剧情触发的"确认框"决定 |
| **D-Mail的撤回机会** | 每封1次撤回机会(在后续章节中) |
| **36字节限制** | 每封D-Mail正文不得超过36字节(约18个中文字) |
| **发送的目标时间限制** | D-Mail只能发送到"过去"——但不能超过电话微波炉的可调范围(理论上约48小时——但实际上可以发送到更远的过去) |

### 2.2 为什么是6封？

D-Mail的数量是刻意设计的——不是越多越好：

| 原因 | 解释 |
|:--:|------|
| **叙事可管理** | 6封D-Mail=6个"可以改变的过去"——每一个都足够重要且独立 |
| **"6">"7"** | 6封刚好可以让玩家"记得每一个"——更多的D-Mail会让玩家忘记"这封邮件改变了什么" |
| **对应6条路线** | 6封D-Mail的发送/撤回决定了6条路线——每一封对应一条路线的"要不要撤回"的选择 |
| **叙事节奏** | 6封D-Mail分布在序章到Ch.6(前中期)——每章1封——保持了稳定的节奏 |

## 三、时间约束——β世界线的倒计时

### 3.1 α场内的收束日倒计时

在α吸引场中，冈部面临的是**"真由理死亡倒数"**：

| 时间节点 | 距离"收束日" | 事件 |
|:--:|:--:|------|
| 7/28 (进入α场) | 16天 | α场开始——真由理的"必死"被锁定了 |
| 8/1 | 12天 | 萌郁拿走IBN5100——收束机制加速 |
| 8/3 | 10天 | 红莉栖FLAG链——但真由理的命运不变 |
| 8/5 | 8天 | 菲利斯D-Mail——秋叶原文化改变——但真由理的死亡日期只推迟了1天 |
| 8/10 | **3天** | 真由理第一次死亡(Chapter 6)——"收束开始了" |
| 8/11 | 2天 | Time Leap多次尝试——死亡推迟到8/13 |
| 8/12 | 1天 | 最后的机会——撤回所有D-Mail=跳出α场 |
| **8/13** | **0天(收束日)** | α场收束——真由理100%死亡 |

> 倒计时的叙事功能：它不是游戏系统强制显示的(没有"距离真由理死亡还有X天"的进度条)，但玩家的推进感完全被这个倒计时驱动——"我必须在这之前做点什么"。

### 3.2 β场内的第三次世界大战倒计时

在β吸引场中，约束变成了更宏大的尺度：

| 时间节点 | 事件 | 倒计时类型 |
|:--:|------|:--:|
| 2010/7/28 | 红莉栖"死亡"(收束) | 触发后→冈部可以随意"在β场内待多久" |
| 2010/8/21 | 铃羽到达(带来D-Video) | "执行窗口"——如果冈部不在这天执行欺骗世界…… |
| 2010-2011(推测) | 中钵论文被各国知晓 | "论文导弹"——时间机器竞赛开始 |
| 2015(目测) | WW3的早期阶段开始 | 世界大战——核武+时间机器 |
| 2025 | 冈部在β场的死亡 | 未来的冈部完成D-Video后死亡 |
| 2036 | 铃羽出发回到2010 | 最后一次机会——没有之后了 |

> 与**《极限脱出999》**对比：999的核心约束是"9小时"——玩家必须在9小时内通过所有房间并解开谜题。命运石之门没有这么显性的时间限制——它的时间约束是"叙事内在的"——你知道真由理将死、你知道WW3将来——但游戏不会说"你还有X小时"。999的约束是游戏系统强制的，命运石之门的约束是玩家自己感受到的。

## 四、信息约束——不同世界线知道的信息不同

命运石之门最独特的"资源约束"是**信息约束**——你在不同世界线上"知道"和"不知道"的东西完全不同。

### 4.1 信息约束矩阵

| 玩家在α场知道 | 但在α场被约束的信息 | 需要什么才能获得 |
|------|------|:--:|
| 真由理会死 | "怎么救她"的答案 | 必须跳回β线→看D-Video |
| SERN是反派 | FB的真实身份(=布朗先生) | 需要铃羽揭示 |
| D-Mail能改变过去 | "撤回D-Mail能跳出α场" | 需要尝试→验证→绝望→找到答案 |
| Time Leap能回到过去 | "Time Leap在α场内无法改变收束" | 需要几十次Time Leap才能确认 |

| 玩家在β场知道 | 但在β场被约束的信息 | 需要什么才能获得 |
|------|------|:--:|
| 红莉栖"死"了 | "她其实可以活下来(只要欺骗世界)" | D-Video给出答案 |
| 真由理活着 | "真由理在α线死了很多次"(冈部记得) | RS能力——只有冈部知道 |
| WW3会发生 | "怎么阻止WW3" | 必须执行欺骗世界 |
| D-Video存在 | "D-Video的内容" | Ch.10才能看到 |

### 4.2 信息约束的叙事功能

| 功能 | 在游戏中的体现 |
|:--:|------|
| **驱动探索** | "我不知道怎么救人→所以我必须继续推进"——信息缺失=行动理由 |
| **制造对称性** | α场知道A不知道B——β场知道B不知道A——只有在Steins Gate上的冈部是"全知的" |
| **提高重玩价值** | 第二次玩的时候——你知道一切了——但"知道"本身改变了你体验文本的方式 |
| **与Reading Steiner绑定** | 冈部的RS是唯一的信息传递机制——其他角色在不同世界线中"不知道" |

## 五、情感约束——"每一次选择都是一次伤害"

### 5.1 撤回D-Mail的情感代价

命运石之门最残酷的约束是情感约束——每撤回一封D-Mail，就等于"放弃了一个人的幸福"：

| D-Mail撤回 | 放弃的"幸福" | 情感代价 |
|:--:|------|:--:|
| 撤回萌郁的彩票D-Mail | 萌郁的"有钱+不加入SERN"的幸福 | 她重新被SERN控制——一个朋友重新变成敌人/工具 |
| 撤回IBN5100 D-Mail | (无直接角色愿望——这是功能性撤回) | 萌郁被迫交出电脑——与SERN的冲突升级 |
| 撤回琉华子的性别D-Mail | 琉华子"成为女性"的个人愿望 | 琉华子变回男性——她的表白/感情被驳回 |
| 撤回菲利斯的救父D-Mail | 菲利斯的"父亲存活+秋叶原文化变化"的幸福 | 菲利斯的父亲"/"去世——秋叶原恢复但菲利斯失去父亲 |
| 撤回铃羽的D-Mail | 铃羽"留在这个时代"的可能 | 铃羽离开——"再见"的重复 |

### 5.2 "正确"选择的代价

| 选择类型 | "正确"选择 | 代价 | 为什么"正确的选择"也是昂贵的 |
|:--:|------|------|------|
| 进真结局 | 完成所有FLAG | 需要"放弃"其他4个分支结局的世界线——你选择了Steins Gate，意味着你没有选择"琉华子作为女性的世界"和"菲利斯父亲存活的世界" | 真结局不是"所有愿望都实现了"——是"只有最重要的愿望实现了" |
| 救真由理(α线) | 撤回所有D-Mail→跳出α场 | 放弃红莉栖(β线=她死了) | "你爱的人活下来了——但你爱的人死了" |
| 救红莉栖(欺骗世界) | 执行欺骗世界 | 放弃真由理在α线的"安全"——需要承担"她不知道你为她做了什么"的孤独 | "你救了所有人——但你是一个人永远记得一切" |

## 六、约束系统的综合评估

| 约束类型 | 显性/隐性 | 对玩家的压力 | 叙事功能 |
|:--:|:--:|:--:|------|
| D-Mail发送次数限制 | 显性(确认框) | 中等——"发还是不发？" | **选择即命运**——每次发送改变世界线 |
| 时间约束(倒计时) | 隐性——没有进度条 | 高——玩家感知到的紧迫感 | **驱动力**——让玩家不能停下来 |
| 信息约束 | 隐性 | 极高——"我不知道怎么救她" | **求知欲**——驱动推进剧情 |
| 情感约束 | 显性(撤回D-Mail时的痛苦) | 极高——"我不得不伤害朋友" | **情感深度**——让选择有意义 |
| Time Leap的"无效"约束 | 隐性 | 极高——"几千次都失败了" | **绝望体验**——让"欺骗世界"成为真正的突破 |

> 命运石之门的约束系统最独特的地方是：**几乎所有约束都是"内在的"(叙事驱动)而不是"外在的"(系统强制)**。游戏不告诉你要在X天内救真由理——但你在玩的过程中自然感受到"我必须快点行动"。这是叙事型游戏约束设计的最高境界——让玩家自己给自己压力，而不是系统施加压力。
