## 一、后台物流——Parkitect的核心创新

### 供应链流程

```
配送中心(Distribution Center)→[货物]→地下管道→仓库(Depot)→搬运工(Hauler)→商店
                                                                        ↓
                                                          垃圾→地下管道→配送中心/垃圾槽
```

### 物流设施

| 设施 | 位置 | 功能 |
|------|------|------|
| **配送中心(Deliveries)** | 公园入口(基础2格) | 所有货物的进出枢纽+垃圾收集 |
| **仓库(Depot)** | 靠近商店(后台) | 存储货物→搬运工从仓库送到商店 |
| **垃圾槽(Trash Chute)** | 可选 | 专门收集垃圾(配送中心本身也收垃圾) |
| **地下管道** | 连接仓库/垃圾槽→配送中心 | **游客看不见的物流通道** |

## 二、沉浸感(Immersion)——前台vs后台的分离

### 什么会降低沉浸感

| 后台元素 | 如果被游客看到 |
|------|------|
| 仓库(Depot) | 沉浸感↓ |
| 搬运工(Hauler)抱着箱子 | 沉浸感↓ |
| 员工通道 | 沉浸感↓ |
| 员工休息室 | 沉浸感↓(游客会说"竟然有人在这里工作?") |
| 厕所/急救室 | **大量降低——必须隐藏** |

### 隐藏策略

| 策略 | 效果 |
|------|------|
| 墙壁/围栏/树丛遮挡 | 基础隐藏——阻挡视线 |
| **地下通道** | **完全消除装饰惩罚——地下的任何东西都不会影响沉浸感** |
| "禁止游客进入"标志 | 将通道设为员工专用 |
| 商店后面建员工通道 | 搬运工从后台补货(游客看不到) |

## 三、装饰系统

### 装饰评分(Decoration Rating)

| 特性 | 说明 |
|------|------|
| **尺寸无关** | **1/4比例的时钟和全尺寸时钟提供相同的装饰评分！** |
| 价格越贵=评分越高 | 但可以用缩小技巧降低成本 |
| 微型隐藏 | 在景观中隐藏微型时钟/石头→廉价提升评分 |
| 负评分范围 | 8格（厕所/员工设施/搬运工产生负面评分） |

### 装饰策略

游客在装饰评分>65%且沉浸感>65%的地块上有10%概率触发"这里的景色真棒!"的正面评价。**摄影点(Photo Spot)**将触发概率提升到30%并将门槛降低到32.5%。

## 四、设计特点分析

**后台物流=游乐园经营中的"第四面墙":** Parkitect是游乐园经营游戏中唯一将"后台"作为核心系统设计的作品。在过山车之星中，所有设施都是"前台"——没有隐藏供应链的概念。Parkitect的后台系统创造了一个独特的空间设计挑战: **你不仅要设计游客看得到的"乐园"，还要设计游客看不到的"运营后台"**。这是游乐园经营游戏中最接近"真实乐园运营"的设计。

**尺寸无关的装饰评分=隐藏微型物品的元策略:** 1/4比例的时钟和全尺寸时钟提供相同的评分——这意味着你可以在景观中"埋入"大量微型高价值物品来廉价提升评分。这种"隐藏优化"是Parkitect独有的——过山车之星的景观评分是基于"游客视线范围内"的可见物品。

**地下通道=终极后台解决方案:** 地下的所有东西都不会影响沉浸感——这是Parkitect后台系统的最强技巧。将整个物流网络建在地下=游客完全看不到供应链=完美沉浸感。

---

> 跨品类对比: 与过山车之星的"所有设施=前台"(无后台概念)不同——Parkitect独有的后台物流系统是品类中独一无二的。与双点医院的"房间=功能单元"不同——Parkitect的后台是**游客不能看到的隐藏空间**——这是"第四面墙"在经营游戏中的实现。
