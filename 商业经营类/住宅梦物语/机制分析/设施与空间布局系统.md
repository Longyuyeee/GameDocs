## 二、公寓楼布局原则

### 垂直分区

| 楼层 | 推荐户型 | 原因 |
|------|:--:|------|
| 底层 | 小户型 | 方便老年租客(不需爬楼) |
| 中层 | 大户型 | 核心展示层——最高租金 |
| 顶层 | 中户型 | 安静——家庭居住 |

### 九宫格扩建法

将公寓楼分成**3×3九宫格**，每次扩建一个区域。扩建前存档→确认扩建方向正确。

### 走廊社交设计

| 社交设施 | 放置位置 | 效果 |
|------|------|------|
| 长椅(Bench) | 两个小公寓之间 | **鼓励邻里互动→恋爱触发率↑** |
| 自动售货机 | 每层公共区域 | 满足基本需求 |
| 绿植 | 走廊两端 | 提升公共区域美观 |

## 三、房间内布局

### 家具配置原则

| 原则 | 说明 |
|------|------|
| **同一家具不超过4件** | >4件=租金变为负数(系统反堆叠) |
| 专门房间优先 | 家具+地板组合=舒适度大幅加成 |
| 地板是组合触发器 | 大多数专门房间需要特定地板类型 |
| 功能分区 | 厨房区(冰箱+橱柜+厨房台)/休息区(床+灯+沙发) |

### 不同生命阶段的房间配置

| 阶段 | 必须家具 | 推荐专门房间 |
|------|------|------|
| 单人(青年) | 床+灯+基础家具 | 卧室+兴趣房间 |
| 夫妻 | +厨房台+冰箱+橱柜+厕所+浴室+洗脸台 | 卧室+厨房 |
| 有婴儿 | **+婴儿床** | 儿童房(绿之床) |
| 有儿童 | +儿童床+学习桌 | 儿童房+学习室 |
| 老年 | 基础家具(不需要发展空间) | 卧室(简洁) |

## 四、搬迁策略

| 场景 | 推荐操作 |
|------|------|
| 用高舒适大房间招募租客后 | **搬移到小户型**(节省大房间给下一个高薪租客) |
| 两大一小家庭 | 搬入中户型 |
| 4人+1宠物(成年/学生为主) | 大户型(方便发展兴趣转职) |
| 家中有老人 | 搬回小/中户型(老人不能转职→不需要发展空间) |

> 搬迁消耗5票(Ticket)——票是稀有货币，不要滥用。

## 五、设计特点分析

**户型差异化=不是"越大越好"的经济学**：大户型=更高租金上限=但也更高维护费。如果一个大户型中住着一个低薪青年，维护费可能超过租金=亏损。最优策略不是"所有房间都最大"——而是**房间大小匹配租客收入水平**。小户型给低薪租客(正现金流)/大户型给高薪租客(利润最大化)。

**走廊社交=空间设计影响游戏行为**：长椅放在两个小公寓之间=增加邻里互动=触发恋爱。这不是"随机事件"——这是**空间布局对游戏结果的直接影响**。走廊设计在住宅梦物语中不是"美观"——它是"婚姻策略"的一部分。

**同一家具>4件=负租金的反meta设计**：如果游戏允许无限堆叠同一种家具，最优解就是"找到最强家具→在64格大房间放64件"。>4件的惩罚机制强制玩家配置多样化——这是对抗"meta碾压"的优雅设计。
