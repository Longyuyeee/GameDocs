# 顾客/居民系统 — 都市大亨物语

> 游戏版本: Dream Town Island (都市大亨物语) Steam版 v2022 (Steam/PC, 2022)
> 数据来源: [9k9k攻略站](https://m.9k9k.com/czdsdwy/), [百家号全攻略](https://baijiahao.baidu.com/s?id=1763040022050096412)

## 一、居民——不是"顾客"而是"租客"

与传统商业经营游戏不同，都市大亨物语的核心"顾客"不是一次性消费者——他们是**长期入住的居民(租客)**。居民支付租金(持续收入)、参与竞赛(获得奖金)、升级住宅(提升租金倍率)、结婚生育(产生新居民)。

| 居民生命周期 | 说明 |
|------|------|
| 入住 | 选择首个居民→自动入住可用住宅 |
| 工作 | 办公楼发现职业→居民转职→工资提升→储蓄增加 |
| 升级住宅 | 储蓄达标+条件满足→升级到更高倍率住宅 |
| 联姻(Matchmaking) | 配对两个单身居民→合并储蓄+共同升级住宅 |
| 生育/子女回归 | 子女长大后离开→作为高储蓄成年居民回归 |

|:--|:--|------|

| **44-45** | **上限** | **第45个居民需要等200年** |

## 三、联姻(Matchmaking)——居民经济的倍增器

| 联姻效应 | 说明 |
|------|------|
| **储蓄合并** | 两人储蓄叠加→更容易达到住宅升级门槛 |
| **共同升级** | 夫妻共同升级到更高级住宅→更高租金倍率 |
| **子女回归** | 子女离开后作为高储蓄居民回归→直接入住高级住宅 |

> 核心策略: **定期使用联姻**(Matchmaking措施)——不联姻=后期人口/储蓄/租金全部停滞。联姻是都市大亨物语中最容易被忽视但回报最高的投资。

## 五、设计特点分析

**"居民=长期资产"而非"一次性顾客"**: 区别在于——居民不是"消费完就走了"——他们是你城市的**长期投资者**。你今天投入资源提升地价的回报不是"今天卖了更多东西"——而是"这个居民未来50年会支付更高的租金"。这种"长期资产"的逻辑完全不同于拉面店的"单次消费"模型。

**联姻=居民经济的"复利效应"**: 联姻不仅是"浪漫"——它是经济增长的催化剂。两个居民合并储蓄→更快升级住宅→更高租金倍率→更快积累下一笔储蓄→孕育子女→子女回归→循环。这是一个"代际复利"的经济模型。

---

> 跨品类对比: 与双点医院/博物馆的"顾客=一次性消费"不同——都市大亨物语的居民是**持续的租金来源**。与温泉物语的"顾客=浴池消费"不同——居民在都市大亨物语中的角色是"资产"(你投资地价→他们支付租金)，而非"消费者"。
