## 一、每日时间结构

### 1.1 单日完整时间线

| 时间段 | 状态 | 可操作内容 | 时间流速 |
|:------|:---:|-----------|:--------:|
| **8:00 AM - 9:00 AM** | 开店前准备 | 员工开始补货；上架商品；整理货架；下单进货；**时间冻结** | 静止 |
| **9:00 AM - 9:00 PM** | 营业时间 | 顾客进店；收银；拆包；定价调整；交易谈判；**时间流动** | 正常 |
| **9:00 PM (准时)** | 强制关店 | 最后一位顾客被送走；当日营业结束；自动日结算 | — |
| **9:00 PM - 次日8:00 AM** | 关店后 | 无法操作；游戏进入结算状态 | 快进 |

### 1.2 "8AM准备窗口"策略分析

| 维度 | 描述 |
|------|------|
| **时间冻结** | 8:00AM-9:00AM期间虽然时间显示在流动，但游戏内部计时器并未推进 |
| **员工行为** | 所有已雇用的员工在此时间段自动补货(从库存→货架) |
| **玩家行为** | 可自由整理货架、调整定价、通过手机APP下单进货 |
| **免费补货窗口** | 员工在此时补货不计入"工作时间成本"——实质上这是系统给予的免费补货期 |
| **策略价值** | ★★★★★——这是每日最关键的策略窗口，利用得好可节省大量营业时间 |

> "免费补货"的设计意图: 让玩家在开店前完成所有准备工作，避免营业时间因补货而分心。这与《潜水员戴夫》开店前的备菜阶段有相同设计哲学——经营压力应集中在营业期间的顾客服务上，而非后勤准备。

## 二、营业周期与高峰期

### 2.1 单日客流分布(估算)

| 时间段 | 客流量 | 特征 |
|:------|:-----:|------|
| 9:00-10:00 AM | 低 | 开店初期，顾客逐步进入 |
| 10:00-12:00 PM | 中 | 逐渐上升 |
| 12:00-2:00 PM | **高峰** | 午餐时段，客流量最大 |
| 2:00-5:00 PM | 中 | 下午平稳期 |
| 5:00-7:00 PM | **小高峰** | 下班时段，客流量回升 |
| 7:00-9:00 PM | 低→归零 | 关店前逐渐减少 |

> 高峰期收入约占全日收入的50%-60%。高峰期的收银效率(员工数量+收银台数量)决定了能否最大化日收入。

### 2.2 营业速度控制

| 功能 | 快捷键 | 效果 |
|------|:-----:|------|
| 正常速度 | 1x | 1秒=1秒，标准营业节奏 |
| 2倍速 | 2x | 1秒=2秒，加速无聊时段 |
| 暂停 | Space | 时间暂停，可操作但不能推进时间 |

## 四、失败条件与破产机制

### 4.1 破产判定流程

```
每日营业结束 → 检查余额
    ↓
余额 ≥ 0 → 继续营业(无警报)
余额 < 0 → 进入负债状态
    ↓
下一营业日 → 收入自动抵扣债务
    ↓
债务持续扩大 → 系统警告(待确认具体触发条件)
    ↓
持续负余额 → 游戏结束(Game Over)
```

### 4.2 破产压力分析

| 压力维度 | 具体表现 |
|---------|---------|
| **触发条件** | 付不起7天账单(余额 < 账单金额) |
| **缓冲机制** | 账单日扣款后余额为负不会立即破产——玩家可在后续经营中赚回 |
| **硬性红线** | 持续负余额达到某个阈值(具体数值待补充) |
| **破产前信号** | 屏幕提示/账单警告(待确认具体表现形式) |
| **恢复方式** | 出售库存卡牌、降低扩张节奏、等待市场价回暖 |
| **难度相关性** | 前期容易破产(现金储备不足)，后期不容易(收入来源多样化) |

## 五、与其他游戏的时间节奏对比

### 5.1 商业经营品类时间节奏对比表

| 游戏 | 时间单位 | 节奏密度 | 核心压力 | 是否有季节/年份 |
|------|:-------:|:-------:|---------|:--------------:|
| **卡牌店模拟器** | 天 | **高**(实时营业) | 7天账单+市场波动 | **否**(纯线性等级) |
| 双点医院 | 年(关卡制) | 中(加速可调) | 关卡星级目标+疫情 | 有(年度评星) |
| 温泉物语(开罗) | 月(约30天) | 低(回合制推进) | 客流满意度+评级 | 有(春夏秋冬) |
| 游戏开发物语(开罗) | 月(12月/年) | 低(指令式) | 游戏销量+评分 | 有(12月年度结算) |
| 冒险村物语2 | 年(4季/年) | 中(非实时) | 怪物跃升+换村清零 | 有(年度表彰) |

### 5.2 关键差异分析

| 差异点 | 卡牌店模拟器 | 开罗/双点系列 |
|--------|------------|--------------|
| **时间流动方式** | **实时**(类似《双点医院》) | 回合制/加速可调 |
| **年度循环** | **无**——没有"年"的概念，纯等级推进 | **有**——年度结算/评级/季节变化 |
| **经营节奏** | 每日相同→逐日积累 | 季度/年度波动→周期性忙碌期 |
| **等级vs时间** | 店等级决定进度，而非玩了多少天 | 年份/月数决定进度 |
| **对玩家素养的要求** | 需要"每日必做"的纪律感 | 可随时暂停、随时加速 |

> **设计洞察**: 卡牌店模拟器摒弃了季节/年份循环，采用纯线性推进——这意味着游戏没有"春季大酬宾"或"年底结算"等周期性事件。所有进度都与"店等级"绑定，而非"时间经过"。这种设计简化了游戏的时间管理复杂度，但也让游戏缺乏周期性新鲜感——每天都是"类似的一天"。

**与双点医院的对比**: 双点医院有明确的关卡制——每关3年(游戏内)，有星级评定压力，需要在截止日期前达到目标。卡牌店模拟器则没有任何时间限制——你可以无限期停留在低级，想经营多久就经营多久。这种无压时间设计更适合"放松型经营"玩家，但可能缺少"冲刺目标"的成就感。

## 六、产品补货节奏

### 6.1 补货流程

| 步骤 | 操作 | 耗时 |
|:---:|------|:---:|
| 1 | 手机APP→下单(选择商品+数量) | 即时 |
| 2 | 等待次日送达 | 第2天8:00AM到货 |
| 3 | 到货形式：纸箱出现在店内地面 | 第2天8:00AM |
| 4 | 打开纸箱→物品进入库存 | 手动操作 |
| 5 | 员工(或玩家)将库存商品上架到货架 | 营业时间或8AM窗口 |

### 6.2 补货策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **每日补货** | 每天闭店前下单，次日到货 | 高销量卡包，保持货架不空 |
| **批量下单** | 一次下大量订单，涵盖数日销量 | 周末无法长时间游戏时 |
| **依赖员工补货** | 员工全自动处理上架，玩家只负责下单 | 后期有足够员工时 |
| **急单** | 是否有加急选项？待确认 | 缺货紧急情况 |

## 七、设计特点分析

### 7.1 "低日均压+高周期性压力"的节奏设计

卡牌店模拟器的压力设计遵循"低日常压力+高周期性压力"的模式：

```
日常(低压力)
├─ 顾客自然进店 → 售出商品 → 收入增长
├─ 市场价小波动 → 可忽略或主动应对
└─ 员工自动补货 → 日常运营几乎无需干预
        ↓
账单日(高压力)
├─ 一次性扣除大额费用 → 余额骤降
├─ 若余额不足 → 恐慌/危机管理
└─ 度过账单日 → 回到低压状态
```

这种"锯齿状"压力曲线让玩家在"轻松"和"紧张"之间循环，避免持续的单调或持续的焦虑。

### 7.2 与牧场物语类的对比

| 维度 | 卡牌店模拟器 | 牧场物语/星露谷物语 |
|------|------------|------------------|
| **每日节奏** | 开店→营业→关店 | 起床→干活→睡觉 |
| **季节系统** | **无** | 有(春夏秋冬，作物受季节影响) |
| **时间压力** | 账单日(可预测) | 季节更替(不可预测的天气) |
| **玩家自主权** | 可随时2x加速 | 时间固定速度 |
| **熬夜惩罚** | 强制关店(9PM) | 体力耗尽晕倒 |

卡牌店模拟器相比牧场物语类的时间控制更宽松——没有"错过农作物收割期就损失一季"的高额惩罚。其时间设计的核心理念是"可预见的压力循环"：玩家知道账单日什么时候来（每7天），可以提前准备。

### 7.3 线性推进的利弊

| 优势 | 劣势 |
|------|------|
| 学习成本低——无需管理季节性策略 | 长期游玩产生"重复感"——每天一样 |
| 适合碎片化游玩——随时拿起放下 | 缺乏"年度目标"的仪式感 |
| 与店等级系统契合——始终有"升级"目标 | 没有"春季准备"等策略维度 |

> **最终洞察**: 卡牌店模拟器的时间系统设计选择反映了其"休闲但上瘾"的定位——它不需要玩家投入大量时间去理解季节循环或周年活动，而是通过"每天开店→卖卡→关店"的简单循环，加上"7天一次账单危机"的周期性刺激，创造一个低门槛但持续驱动的经营体验。这种设计在Steam上获得了97%好评率，证明了"极简时间系统"在商业经营品类中的有效性。
