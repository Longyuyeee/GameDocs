## 一、动态市场价格系统

本游戏的核心经济特征在于**所有商品没有固定售价**——每件商品都有"起始市场价"（Base Market Price），每日有 40% 概率发生波动，形成动态变化的价格环境。

### 1.1 价格波动参数

| 参数 | 数值 | 说明 |
|:--|:--:|:--|
| 每日波动概率 | 40% | 每天每个商品独立判定是否波动 |
| 波动幅度范围 | 0.08% ~ 2% | 单次价格上下波动的百分比 |
| 价格硬上限 | 起始市场价 × (1 + 200%) | 价格最多涨到起始价的 3 倍 |
| 价格硬下限 | 起始市场价 × (1 − 80%) | 价格最多跌到起始价的 20% |
| 波动方向 | 上涨/下跌 各 50% | 随机决定 |
| 波动触发时机 | 每日开店时 | 实际为每天 8AM 前结算 |

> 注：波动幅度为当前价格的百分比，而非起始价的百分比。例如价格已下跌 50% 后，再次下跌 2% 的绝对值远小于起始时的 2%。

### 1.2 市场崩溃与暴涨机制

当价格偏离起始市场价达到或超过 ±25% 时，有概率触发极端事件：

| 触发条件 | 偏离 ≥ 25%（上涨为暴涨 / 下跌为崩溃） |
|:--|:--|
| 触发概率 | = 当前偏离百分比（偏离 50% 则 50% 概率触发） |
| 崩溃幅度 | 5% ~ (偏离% ÷ 2.1) 之间随机 |
| 方向 | 崩溃=继续下跌 / 暴涨=继续上涨 |
| 连续触发 | 每日重新判定，可连续多日触发 |
| 是否可以提前预测 | 不可预测，纯概率驱动 |

**极端事件示例**：

- 某卡牌起始市场价 $100，经过多日下跌至 $60（偏离 40%）
- 次日判定：40% 概率触发崩溃，若触发则再跌 5% ~ (40/2.1 ≈ 19%) 之间
- 若触发 19% 暴跌，价格降至 ≈ $48.6，偏离达 51.4%
- 次日继续以 51.4% 概率判定是否再次崩溃

### 1.3 崩溃/暴涨后的恢复

| 机制 | 说明 |
|:--|:--|
| 崩溃见底 | 价格跌至硬下限（起始价 × 20%）后不再继续下跌，维持低位 |
| 暴涨封顶 | 价格涨至硬上限（起始价 × 300%）后停止上涨 |
| 自然恢复 | 不发生波动时，价格不会自动向起始价回归 |
| 人工干预 | 玩家可通过大量购买/暂停进货来影响市场（效果待补充） |
| 恢复速度 | 价格只能通过每日 40% 的波动概率向上或向下移动，无均值回归机制 |

## 二、进价与成本系统

### 2.1 进价计算公式

| 商品类型 | 进价公式 | 示例 |
|:--|:--|:--|
| 卡包（单包） | Base Cost × Random(1, 1.3) | Basic Pack Base $1.00 → 进价 $1.00 ~ $1.30 |
| 卡盒（盒装） | 卡包进价 × 8 × Random(1, 1.15) | 约等于 8 包进价的 1~1.15 倍 |
| Battle Deck | Base Cost × Random(1, 1.1) | 待补充具体基础成本 |
| 配件 | Base Cost × Random(1, 1.2) | 待补充具体基础成本 |

> Random(1, 1.3) 表示 1 到 1.3 之间的均匀随机分布，即进价在基础成本的 100%~130% 之间浮动。

### 2.2 起始市场价

起始市场价 = 进价 × 产品倍率

| 产品类型 | 产品倍率范围 | 起始市场价示例 |
|:--|:--:|:--|
| Basic Card Pack | 1.5× ~ 2× | Base $1.00 → 起价 $1.50 ~ $2.00 |
| Rare Card Pack | 1.8× ~ 2.5× | 待补充 |
| Epic Card Pack | 2× ~ 3× | 待补充 |
| Legendary Card Pack | 2.5× ~ 3.5× | 待补充 |
| 配件类 | 1.2× ~ 2× | 待补充 |
| 周边类 | 1.5× ~ 2.5× | 待补充 |

### 2.3 一次进价 vs 二次进货成本

| 进货类型 | 价格特征 | 说明 |
|:--|:--|:--|
| 首次进货 | 基础成本 × Random(1, 1.3) | 需先购买许可证 |
| 二次进货 | 同上，每次独立随机 | 无进货量折扣或涨价惩罚 |
| 大批量 | 无批量折扣 | 买 1 箱和买 10 箱单价相同 |

## 三、收入来源构成

### 3.1 收入来源权重

| 收入来源 | 优先级 | 预计占比 | 说明 |
|:--|:--:|:--:|:--|
| 零售卖货（架上商品） | 主力 | ~50%~60% | 卡包/配件/周边/桌游等日常销售 |
| 卡牌单品销售 | 高利润 | ~20%~30% | 稀有单卡以远高于进价的价格出售，利润率极高 |
| 游戏桌费用 | 稳定被动 | ~5%~10% | 按时间或按局收费，无需补货 |
| 顾客卖卡差价 | 高利润 | ~5%~10% | 从红叹号顾客手中低价收卡，高价卖出 |
| 自动拆包机产出 | 间接 | 待补充 | 自动拆包所得卡牌可用于销售 |

### 3.2 零售利润分析

| 定价策略 | 利润率范围 | 风险 |
|:--|:--:|:--|
| 市场价 + 0% | 0%（按起始利润） | 无竞争力问题，但零利润 |
| 市场价 + 5%~10% | ~15%~50% | 顾客接受度高 |
| 市场价 + 10%~15% | ~30%~80% | 推荐区间，平衡利润与销量 |
| 市场价 + 15%~30% | ~50%~150% | 高利润但可能滞销 |
| 市场价 + 30%+ | ~80%~300% | 仅对稀有单卡有效 |

## 四、7 天账单与运营成本

### 4.1 账单构成

| 费用项目 | 计算方式 | 说明 |
|:--|:--|:--|
| 租金（Rent） | 基于店面扩张次数和面积 | 主店面 A 区 + 侧室 B 区分别计算 |
| 电费（Electricity） | 基于通电时间和设备数量 | 照明+设备（除臭机、拆包机等）持续耗电 |
| 员工日薪 | 按雇佣员工数量 × 日薪 | 每晚 9PM 结算当天工资 |
| 账单周期 | 每 7 天结算一次 | 固定周期，每日收入自动累积 |

### 4.2 账单通知与破产

| 状态 | 条件 | 后果 |
|:--|:--|:--|
| 正常 | 余额 ≥ 账单金额 | 自动扣除 |
| 余额不足（负余额） | 余额 < 0 | 进入风险状态 |
| 破产 | 连续付不起账单 | 游戏结束（待确认具体条件） |
| 预防手段 | 预留现金流 | 建议始终预留至少 2 期账单金额 |

### 4.3 费用递增趋势

| 店等级区间 | 租金水平 | 电费水平 | 员工工资 |
|:--|:--:|:--:|:--:|
| Lv1~10 | 低（$50~$200/期） | 低（无设备） | 无员工 |
| Lv11~25 | 中（$500~$2000/期） | 中（1-2 台设备） | $30~$80/天 |
| Lv26~50 | 中高（$2000~$5000/期） | 高（3-5 台设备） | $80~$150/天 |
| Lv51~100 | 高（$5000+/期） | 高（5+ 设备） | $150~$250/天 |

## 五、定价策略分析

### 5.1 不同产品类型的推荐定价

| 产品类型 | 推荐加价率 | 策略原因 |
|:--|:--:|:--|
| Basic Card Pack | 市价 +10%~15% | 走量商品，薄利多销 |
| 稀有卡包 | 市价 +15%~20% | 需求刚性，可适度加价 |
| 配件（卡套/骰子） | 市价 +10%~20% | 低频购买，加价空间大 |
| 普通单卡（白边/黑边） | 市价 +20%~30% | 玩家按需收集，价格敏感度低 |
| 稀有单卡（金边/EX以上） | 市价 +30%~50% | 稀缺性决定，收藏家愿意支付溢价 |
| 周边（毛绒/手办） | 市价 +15%~25% | 礼品属性，中高溢价 |

### 5.2 整数定价加速机制

| 定价方式 | 结账速度 | 说明 |
|:--|:--:|:--|
| 整数定价（如 $5.00） | 快速 | 员工无需找零，立即完成交易 |
| 非整数定价（如 $5.49） | 慢速 | 员工需要计算和找零，产生停顿 |
| 多位小数（如 $5.4987） | 极慢 | 复杂找零导致显著延迟 |

### 5.3 价格自动调整

| 调整方式 | 操作途径 | 说明 |
|:--|:--|:--|
| 手动定价 | 店内编辑商品价格 | 每个商品可单独定价 |
| 按市价百分比自动定价 | 员工手机 APP 设定 | 设置 Markup% 后自动跟随市价波动 |
| 批量调价 | 待确认 | 当前版本是否支持批量操作待补充 |

## 六、设计特点分析

### 6.1 设计意图

**动态市场价格系统**是本游戏区别于同类经营游戏（如《Supermarket Simulator》《Gas Station Simulator》）的核心设计。它的设计意图包括：

1. **创造"投机"玩法维度** — 玩家不只是"低价进货→高价卖出"的线性经营，还需要关注市场行情走势，判断何时囤货、何时清仓。这为经营游戏增加了原本属于股票交易的策略层。

2. **模拟实体 TCG 市场的不确定性** — 现实中的卡牌二级市场确实存在剧烈的价格波动（例如一张卡因比赛获胜而一夜暴涨，或因禁卡表而暴跌）。用 40% 每日波动 + 崩溃/暴涨机制，以简化的方式再现了这种不确定性。

3. **消除"最优解"** — 如果价格固定，玩家很快会计算出"卖什么最赚钱"，然后只卖那一种。动态价格使得最优策略随时间变化，延长了游戏的策略寿命。

4. **崩溃/暴涨的"赌博"设计** — 崩溃/暴涨机制创造了一种"追涨杀跌"的心理体验，玩家会在下跌时恐慌抛售，在上涨时贪婪囤货——这与现实市场中投资者的行为偏差一致。

### 6.2 跨游戏对比

与 **《Supermarket Simulator》** 相比，TGC Card Shop Simulator 的经济系统更加动态和复杂：

| 对比维度 | TCG Card Shop Simulator | Supermarket Simulator |
|:--|:--|:--|
| 价格系统 | 每日 40% 概率波动，有崩溃/暴涨 | 固定进销差价，无价格波动 |
| 定价自由度 | 完全自由定价 | 受限于固定利润率 |
| 二级市场 | 有——顾客卖卡、卡牌鉴定增值 | 无 |
| 市场风险 | 存在——价格下跌可能亏损 | 无——进价固定，利润稳定 |
| 投机空间 | 大——囤货/抄底/套利 | 无——线性经营 |

这种经济系统的复杂度提升是一把双刃剑——它增加了深度和重玩性，但也提高了新手玩家的认知门槛。对于目标受众（TCG 爱好者），这种复杂度反而是吸引力所在，因为他们本身就在现实中进行卡牌交易。

### 6.3 本游戏独有特色

- **无均值回归机制**：价格不自动回归起始价，意味着崩溃后的商品可能长期低位运行，或者暴涨后的商品持续高位——这比"价格总是回归均值"的设计更真实地模拟了市场惯性。
- **进价随机性**：每次进货价格不同（1~1.3 倍随机），增加了成本管理的变量，不同于大多数模拟游戏中固定的批发价。
- **整数定价加速结账**：将现实中的"定价 $9.99 还是 $10.00"的心理博弈转化为有实际游戏机制效果的抉择——不仅仅是玩家偏好，而是影响运营效率的系统级机制。

### 6.4 策略建议

| 阶段 | 推荐策略 | 理由 |
|:--|:--|:--|
| 前期（Lv1~10） | 卡包 +10% 走量，单卡 +30% | 现金流紧张，需要快速周转 |
| 中期（Lv11~30） | 关注市场波动，低价囤货 | 现金流充裕后可承担库存风险 |
| 后期（Lv31+） | 多样化定价，利用自动定价 APP | 商品种类多，手动管理成本高 |
| 应急 | 崩溃时暂停进货，暴涨时适当囤货 | 避免价格进一步下跌造成损失 |
