# 平台生命周期数据总览 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013) | 来源: [Fandom Wiki](https://gamedevtycoon.fandom.com/)

## 一、平台生命周期表

| 平台 | 诞生(约) | 成长 | 成熟 | 衰退 | 死亡 | 最佳进入时机 |
|------|:--:|:--:|:--:|:--:|:--:|:--:|
| PC | 永久 | — | **永久成熟** | 永不 | 永不 | 任何时候 |
| Gameling(G3) | 80s | 80-90s | 90-2000s | 2000s+ | 约15-20年 | 成长期(80-90s) |
| Playsystem | 90s | 90-2000s | 2000s | 2000s+ | 约12-15年 | 成长初(90s) |
| mBox Next | 2000s | 2000-2010s | 2010s | 2010s+ | 约10-15年 | 成长期 |
| 移动平台 | 2010s | 2010s | 2010-2020s | — | — | 成长期 |

## 二、多平台策略

| 配置 | 说明 |
|------|------|
| PC独占(初期) | 80-90%游戏放在PC上——最安全 |
| PC+1个成长平台 | 成熟期——扩展市场覆盖 |
| 全平台 | 后期——研究解锁后一个游戏发布到所有存活平台 |

---

> PC永不死亡——是任何时期的"安全锚"平台。

## 三、平台选择决策树

```
需要最大市场份额？ → Gameling(休闲)/PC(全类型)
需要核心玩家口碑？ → Playsystem(动作/冒险)
需要稳定的全年龄市场？ → mBox Next(全类型)
需要最安全的锚？ → PC(永不死亡)——任何情况都能靠PC存活
```

## 四、平台死亡的处理

当平台进入"死亡"阶段时：当前该平台的所有游戏**停止产生销售**。你需要在平台死亡前至少6-12个月开始将游戏迁移到新平台。未及时迁移的后果是：依赖该平台的大部分收入突然归零→可能触发破产。

## 五、跨平台开发的经济学

研究解锁"多平台发布"后，一个游戏可以同时发布到多个平台——**一份开发成本，多份销售收入**。这是游戏中期最大的收入增长引擎。但需要权衡：每个平台的受众偏好不同——一个"PC全年龄动作游戏"可能畅销PC但滞销于休闲主导的Gameling。
