# 开罗公式的PC进化：Game Dev Story vs Game Dev Tycoon

> 两款游戏共享同一个核心概念(经营游戏开发公司)，但代表了"同一设计DNA在两种不同技术生态中的表达"。

---

## 一、核心维度对比

| 维度 | Game Dev Story(开罗/1997) | Game Dev Tycoon(2013) |
|------|------|------|
| 开发者 | Kairosoft(26人团队) | Greenheart Games(2人兄弟) |
| 视角 | 像素俯视2D/设施布局 | 等距2D/无空间建造 |
| 开发机制 | 选择类型+题材→自动开发 | **滑块分配**→主动优化 |
| 平台系统 | 主机固定发布 | **平台生命周期(诞生→死亡)** |
| 员工系统 | 等级+职业+转职树 | 设计/技术专家+培训 |
| 技术系统 | 等级制(提升→更大规模) | **自定义引擎+研发树** |
| 续作 | 发布新游戏(无惩罚) | **续作机制(间隔40周/新引擎)** |
| 时间跨度 | ~20年 | **35年+(产业代际)** |
| 产业链终点 | — | **自定义主机(从开发商到平台方)** |
| 盗版 | 无此机制 | **Copy Protection(不是投资就破产)** |
| 定价 | — | 自动(由平台和规模决定) |

---

## 二、两个"进化层级"

GD Tycoon在原公式基础上增加了四个维度的深度：
1. **平台生命周期**(Game Dev Story没有——平台是静态的)
2. **滑块分配开发**(Game Dev Story是"自动开发"——GD Tycoon是"手动优化")
3. **自定义引擎/主机**(Game Dev Story没有产业链纵向整合)
4. **盗版/保护经济学**(Game Dev Story完全没有)

---

## 三、设计启示

同一设计DNA在不同技术生态中的表达：Game Dev Story是开罗公式在移动/轻量环境下的表达(量小/轻快/2-8小时一周目)；GD Tycoon是同一公式在PC环境下的深度扩展(更长/更深/更复杂的经济系统)。这证明了好的核心概念可以在不同技术平台和设计复杂度下进行**自适应的缩放**。

---

> 核心观察: 两兄弟(Greenheart Games)在Kairosoft的公式基础上做了"PC深度版"——这不是抄袭，而是对同一设计理念在不同媒介中的再创作。正如牛蛙的《主题医院》→Frontier的《双点医院》的"精神续作"关系。
