# 扩张与升级系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/)

---

## 一、办公室扩张

| 阶段 | 月租金 | 员工容量 | 搬入时机 |
|:--:|:--:|:--:|------|
| 车库(Garage) | ~$8K | 1-2人 | 初期默认 |
| 小型办公室 | ~$30-50K | 3-4人 | 现金充足+需要更多员工 |
| 中型办公室 | ~$80-120K | 5-8人 | 稳定收入+稳定团队 |
| 大型办公室 | ~$200-500K+ | 8-12人 | 3A时代/MMO/自定义主机 |

**关键警告**: 不要在现金不足时搬离车库——租金从$8K跳到$100K+可以瞬间摧毁现金流。推荐搬离时机: 现金1500-2000万+粉丝15-25万。

## 二、游戏规模扩张

| 规模 | 所需最低引擎 | 开发时间 | 成本 |
|:--:|------|:--:|:--:|
| 小型(S) | 基础 | 短 | 低——初期主力 |
| 中型(M) | 2D V3+ | 中 | 中——有稳定粉丝后 |
| 大型(L) | 2D V5+或3D V3+ | 长 | 高——需要强力团队 |
| AAA | 3D V5+ + 多人 | 极长 | 极高——需要全员专家 |

## 三、续作与系列化

| 策略 | 说明 |
|------|------|
| 续作间隔 | **原作发布至少40周后**——过早发布=粉丝疲惫+评分惩罚 |
| 续作引擎 | **必须使用新引擎版本**——使用旧引擎="原地踏步"惩罚 |
| 重复类型+题材 | 不能连续两款——必须有间隔或不同类型 |
| 系列化 | 每2-3款游戏回归一个成功配方——保持新鲜感 |

## 四、设计特点分析

**办公室扩张=戏剧性的成本跳变**：从车库到办公室的租金跳跃是最具戏剧性的经济时刻之一——许多新手玩家在此破产。这不是"设计缺陷"——它是游戏内关于"过早扩张"风险的商业教育。

## 五、系列化策略

| 策略 | 最优间隔 | 引擎要求 |
|------|:--:|------|
| 独立作品 | 无间隔限制 | 无要求 |
| 续作 | **原作40周后** | **必须新引擎** |
| 三部曲 | 每作间隔40周+ | 每作新引擎 |
| 年货化(不推荐) | 短间隔 | 评分逐作递减 |

## 六、多类型扩张

第12年解锁多类型(Multi-Genre)后——游戏可以混合2个类型(如"动作+RPG")。这极大地扩展了有效组合数——从~60到~3600。但这需要更复杂的滑块分配(两个类型的最优分配需要兼顾)。

## 七、从开发商到平台方

自定义主机是产业链的最高点——你不再是为别人的平台开发游戏的公司，你是拥有一个平台的发行商。其他"开发商"会为你的平台开发游戏——你从他们的销售中抽成。这种纵向整合是GD Tycoon中最宏大的策略转型，也是游戏对"任天堂/Sony/微软"式的平台商业模式的致敬。

## 八、扩张风险的最优节奏总结

```
车库(1-2人/小型游戏) → 积累现金1500万+/粉丝15万+ →
小办公室(3-4人/中型游戏) → 积累现金5000万+/粉丝50万+ →
中办公室(5-8人/大型游戏) → 积累现金2亿+/粉丝200万+ →
大办公室(8-12人/AAA+自定义主机) → 终局自由
```

**核心原则**: 先确保当前规模的盈利能力，再扩张。永远不要在"我觉得下个游戏会大卖"的预期上提前扩张——Game Dev Tycoon的经济设计正是为了惩罚这种"过度乐观"的创业心态。
