## 一、游戏开发流程

### 三阶段开发

| 阶段 | 时间 | 可分配滑块 | 关键决策 |
|------|:--:|------|------|
| **阶段1(概念)** | 短 | 引擎/玩法/故事 | 选择类型+题材+平台+目标受众 |
| **阶段2(制作)** | 中 | 对话/关卡设计/AI | 确定技术/设计投入比 |
| **阶段3(打磨)** | 短 | 世界设计/图形/音效 | 最终质量调整 |

## 二、类型×题材组合系统

### 组合评价

| 搭配 | 评分影响 | 示例 |
|------|:--:|------|
| 绝佳组合(+++) | +30-40% | 动作+科幻/奇幻/军事; RPG+奇幻/中世纪 |
| 好组合(++) | +10-20% | 模拟+城市/经营; 策略+历史/军事 |
| 中性(+) | 0% | 大多数一般组合 |
| 差组合(—) | -10-20% | 动作+词汇/舞蹈 |
| 极差组合(———) | -30-50% | 模拟+僵尸?/休闲+恐怖? |

### 最佳组合速查

| 类型 | 最佳题材 |
|------|------|
| 动作(Action) | 科幻/异形/赛博朋克/奇幻/中世纪/军事/末日/超级英雄/僵尸 |
| 冒险(Adventure) | 间谍/侦探/奇幻/中世纪/海盗/神秘 |
| RPG | 异形/赛博朋克/侦探/地牢/奇幻/中世纪/末日/科幻/吸血鬼 |
| 模拟(Simulation) | 航空/城市/地牢/经营/进化/医院/军事/赛车/太空/手术 |
| 策略(Strategy) | 航空/异形/城市/经营/政府/黑客/历史/中世纪/军事/太空 |
| 休闲(Casual) | 喜剧/舞蹈/时尚/音乐/赛车/体育/词汇 |

## 三、滑块分配系统

### 按类型的最优分配

| 类型 | 阶段1 | 阶段2 | 阶段3 |
|------|------|------|------|
| **动作** | 引擎100%/玩法80%/故事0% | 对话0%/关卡80%/AI100% | 世界0%/图形100%/音效80% |
| **冒险** | 引擎0%/玩法0%/故事100% | 对话100%/关卡0%/AI0% | 世界0%/图形100%/音效80% |
| **RPG** | 引擎0%/玩法80%/故事100% | 对话100%/关卡80%/AI0% | 世界100%/图形80%/音效0% |
| **模拟** | 引擎80%/玩法100%/故事0% | 对话0%/关卡80%/AI0% | 世界100%/图形80%/音效0% |
| **策略** | 引擎80%/玩法100%/故事0% | 对话0%/关卡100%/AI80% | 世界100%/图形0%/音效80% |
| **休闲** | 引擎0%/玩法100%/故事0% | 对话0%/关卡100%/AI0% | 世界0%/图形100%/音效80% |

### 不良分配的影响

- 偏离最优分配→技术和设计评分分别下降
- 极端偏离(如在RPG中投入大量引擎而忽略故事)→严重扣分

## 四、游戏规模与技术/设计比

| 规模 | 所需引擎 | 开发成本 | 推荐时机 |
|:--:|------|:--:|------|
| 小型(Small) | 基础 | 低 | 初期(积累粉丝和资金) |
| 中型(Medium) | 2D V3+ | 中 | 中期(有稳定粉丝后) |
| **大型(Large)** | 2D V5+或3D V3+ | 高 | **不要过早——只做能够承受的** |
| AAA | **3D V5+ + 3+专家投入** | 极高 | 后期(数百万粉丝后) |

### 技术和设计的类型比率

| 类型 | 技术:设计比率 |
|------|:--:|
| 动作/模拟/策略 | 60:40(偏技术) |
| 冒险/RPG | 40:60(偏设计) |
| 休闲 | 30:70(偏设计) |

## 五、设计特点分析

**滑块分配=将"游戏开发"抽象为多维资源分配**：GD Tycoon没有让玩家"逐行写代码"或"逐块建轨道"——它用9个滑块的分配抽象了整个开发过程。每种游戏类型有独特的"最优滑块配置"——这创造了一个类似"配方发现"的学习过程：玩家通过试错找到每种类型的最佳分配。

**类型×题材组合=创造性的轻度约束**：不是所有组合都可行——动作+词汇是糟糕的，RPG+奇幻是经典的。这种"约束"让选择有了重量——你不能随便选一个类型和题材然后期待高分。它强制玩家思考"什么类型和题材搭配是有意义的"。

**三阶段开发=时间分配的权衡**：概念/制作/打磨三个阶段各自分配不同的滑块——这意味着你不能"全投所有维度"。你必须在有限的滑块精力中选择投资哪些方面——RPG需要大量故事投入但基本不需要引擎投入。

---

> 跨品类对比: GD Tycoon的"滑块分配"与过山车之星的"EFN三指标热力图"相似——都是"多维度的质量优化问题"。但GD Tycoon是抽象的(滑块位置→隐藏分数)，PC是物理的(轨道曲率→实时G力→热力图颜色)。
