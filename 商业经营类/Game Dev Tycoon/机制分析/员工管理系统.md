# 员工管理系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/239820)

---

## 一、员工角色

| 角色 | 功能 | 推荐数量 |
|------|------|:--:|
| **设计专家(Design Specialist)** | 提升游戏设计评分 | 2人 |
| **技术专家(Tech Specialist)** | 提升游戏技术评分 | 2人 |
| **平衡型(Balanced)** | 均衡提升 | 2人 |
| **玩家角色** | 开发+管理决策 | 1人(可培训为任意方向) |

### 推荐团队配置

游戏中后期推荐：2设计专家 + 2技术专家 + 2平衡 + 玩家(培训为某方向) = 7人团队。每个阶段(概念/制作/打磨)可以分配不同组合的员工来最大化该阶段的技术/设计分数。

## 二、培训系统

| 培训效果 | 说明 |
|------|------|
| 设计培训 | 提升设计技能→更高设计评分 |
| 技术培训 | 提升技术技能→更高技术评分 |
| 全员培训 | 包括玩家自己——所有员工都应持续培训 |

## 三、设计特点分析

**专家vs平衡的资源配置**：平衡型员工适应性更强但单项不突出——专家型在特定阶段能产生更大影响。你的团队配置定义了你公司的开发能力特征——技术导向型公司vs设计导向型公司。

## 四、阶段分配策略

| 开发阶段 | 推荐分配员工 | 原因 |
|------|------|------|
| 概念(阶段1) | 设计专家 | 故事/玩法取决于设计 |
| 制作(阶段2) | 技术专家+设计专家 | 关卡+AI需要双方投入 |
| 打磨(阶段3) | 技术专家 | 图形/音效更依赖技术 |
| AAA游戏 | **全员投入每个阶段** | AAA需要所有维度的极致投入 |

## 五、培训的长期规划

| 阶段 | 培训组合 |
|------|------|
| 初期(1-3人) | 玩家培训为全能(可承担任何开发角色) |
| 中期(4-6人) | 2设计+2技术+玩家补位 |
| 后期(7-12人) | 全专家化→所有员工培训到最高等级 |

## 六、员工系统与开罗游戏开发物语的对比

GD Tycoon的员工系统比Game Dev Story更精简但更聚焦——没有"转职树"和"职业路线"，而是用了更抽象的"设计专家/技术专家/平衡型"三元分类。这种简化让玩家将注意力集中在"培训投资"和"阶段分配"上，而非复杂的职业规划。GD Tycoon的设计哲学是"让员工系统支持游戏开发决策"，而非"让员工系统本身成为一个复杂的小游戏"。

## 七、团队规模的隐性约束

游戏不会直接限制你雇佣多少员工——但通过**固定月度薪资成本**施加了隐性约束。每增加一个员工都永久提高了月度固定支出，而游戏开发收入是浮动的(依赖评分和市场)。这种"固定成本膨胀"vs"浮动收入"的结构性张力是GD Tycoon最优雅的规模控制机制——你不需要"人口上限"规则，经济本身就会告诉你"现在不能再招人了"。
