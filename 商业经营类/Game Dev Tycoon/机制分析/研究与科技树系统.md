# 研究与科技树系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/239820)

---

## 一、研究系统

| 研究类型 | 负责方 | 投入 | 解锁 |
|------|------|------|------|
| 技术研发 | 技术专家 | 研发点数 | 新技术/引擎升级 |
| 设计研发 | 设计专家 | 研发点数 | 新游戏类型/题材 |
| 平台研发 | 研发实验室 | 大量资金 | 自定义主机 |

## 二、关键研究优先级

| 优先级 | 研究项目 | 原因 |
|:--:|------|------|
| **#1** | **自定义引擎+Copy Protection V1** | 盗版保护——没有它=70%+销售损失 |
| #2 | 目标受众(Target Audience) | 可以定向游戏客群→更高评分 |
| #3 | 新引擎版本(2D V3→V5) | 解锁更大游戏规模→更高潜在评分 |
| #4 | 3D引擎 | 解锁新游戏类型(3D动作/冒险/RPG) |
| #5 | 多平台发布 | 一个游戏可以发布到多个平台→收入翻倍 |

## 三、自定义引擎系统

| 引擎特性 | 说明 |
|------|------|
| 一次性投入(巨资) | 开发需要大量研发点+时间 |
| 长期回报 | 所有后续游戏使用该引擎=降低每游戏开发成本 |
| 提升评分上限 | 高级引擎=更高技术评分上限 |
| 续作用新引擎 | 续作必须使用新引擎版本——否则评分惩罚 |
| 发布旧引擎源代码 | 开源旧引擎→粉丝好感提升 |

## 四、设计特点分析

**Copy Protection的生存级优先级**：研发列表中最优先的不是"更好的画面"或"新游戏类型"——是**反盗版保护**。没有它的惩罚(70%+销售损失)大到足以强制玩家在享受任何其他研发福利之前先投资它。这是一个巧妙的设计——它通过游戏机制说明了盗版对开发者的真实经济影响。

**自定义引擎=技术投资的战略选择**：是否研发自定义引擎是一个重大的战略决策——它需要大量前期投入，但长期降低开发成本。这模拟了真实游戏产业中的"自研引擎vs第三方引擎"的选择(Epic的Unreal vs EA的Frostbite vs Unity的普及)。

## 五、研究vs开发的资源分配

研发和游戏开发**争夺同一批员工的时间**——派员工去研发意味着他们不能参与游戏开发。这种资源竞争迫使玩家在两个优先级之间不断权衡：
- 短期: 研发→解锁更好的技术→未来的游戏受益
- 短期: 开发→赚取现金→当前生存

最优策略是"脉冲式"研发:在游戏发布后/新游戏开始前的空窗期集中研发，在游戏开发期间全员投入开发。这模拟了真实游戏公司中"预研阶段→生产阶段"的节奏。

## 六、引擎开源的粉丝效应

发布旧引擎的源代码到社区——虽然"免费放弃了技术资产"——但会获得粉丝好感度提升。这是一个"商业理性"vs"社区好感"的权衡，反映了游戏产业中关于"开源的商业价值"的持续讨论。
