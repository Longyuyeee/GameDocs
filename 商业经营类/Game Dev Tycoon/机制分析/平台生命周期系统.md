# 平台生命周期系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/239820)

---

## 一、平台生命周期五阶段

| 阶段 | 特征 | 市场份额 | 策略 |
|------|------|:--:|------|
| **诞生(Launch)** | 新平台刚发布/市场极小 | 极低 | 谨慎观望(等待市场成长) |
| **成长(Growth)** | 市场份额快速扩张 | 中-高 | **最佳进入时机** |
| **成熟(Maturity)** | 市场份额稳定/最大 | 最高 | 主力开发平台 |
| **衰退(Decline)** | 市场份额下降 | 中-低 | 逐步退出/迁移到新平台 |
| **死亡(Discontinued)** | 平台退出市场 | 零 | 停止开发(游戏无法再在该平台销售) |

## 二、平台类型与特征

| 平台 | 优势受众 | 生命周期 | 市场份额特征 |
|------|------|:--:|------|
| **PC** | 全年龄/全类型 | **永不死亡** | 稳定——永远存在的安全锚 |
| Gameling(G3) | 休闲/年轻 | 中-长 | 休闲游戏市场份额最大 |
| Playsystem(PS) | 核心玩家(动作/冒险) | 中 | 高竞争/高回报 |
| mBox Next | 全年龄 | 中-长 | Xbox等价物 |
| Nintendo等价物 | 家庭/年轻 | 长 | 独特的第一方市场 |
| 自定义主机 | 全类型 | 取决于投资 | 后期终局内容 |

## 三、平台迁移策略

### 多平台同时开发的逻辑

| 策略 | 说明 |
|------|------|
| **PC为主(80-90%)** | PC永不消亡——是永远安全的开发平台 |
| **跟风成长平台** | 在成长阶段进入——享受市场扩张红利 |
| **避免衰退平台** | 市场份额下降后停止开发——资源浪费 |
| **多平台+同游戏** | 一个游戏可以发布到多个平台(研发解锁多平台能力) |

### 平台进入时机

| 时机 | 策略 |
|------|------|
| 刚发布(0-1年) | 观望——市场太小 |
| 成长中(1-3年) | **黄金进入期** |
| 成熟期(3-7年) | 主力开发——最大市场份额 |
| 衰退(7-10年) | 退出——寻找新平台 |
| 即将死亡(10年+) | 立即停止——不要浪费资源 |

## 四、自定义主机（终局内容）

| 要素 | 说明 |
|------|------|
| 解锁条件 | **现金2亿+** / 研发解锁 |
| 投资需求 | 巨额一次性投入 |
| 长期回报 | 拥有自己的平台=平台生命周期中所有游戏的利润归你所有 |
| 风险 | 巨额投入——如果平台寿命短于预期→亏损 |
| 策略 | 确保有足够的粉丝基础和IP储备来支持自定义主机的"发布游戏阵容" |

## 五、设计特点分析

**平台生命周期=整个游戏的时间骨架**：GD Tycoon的"时间"不是抽象的"第X年"——时间是由平台的生命周期定义的。你从80年代的PC/Gameling开始→90年代的Playsystem→2000年代的mBox→2010年代的移动平台。每个平台的兴衰定义了你在每个阶段应该做什么——这是一种将"真实游戏产业历史"编码为游戏机制的优雅方式。

**"PC永不消亡"=安全锚的设计智慧**：在所有平台都会衰退的世界中，PC是唯一永不死亡的平台。这给玩家提供了一个"永远可以回来"的安全锚——避免了"所有平台都死了=游戏无法继续"的死局。这也反映了真实游戏产业中PC平台的独特地位。

**自定义主机=产业链终点**：从"为别人的平台开发游戏"到"发行你自己的平台"——这是GD Tycoon中最宏大的产业升级。自定义主机在游戏早期是不可能的(需要2亿+现金)，但在后期它代表了"从开发商到平台持有者"的产业跃迁。

---

> 跨品类对比: GD Tycoon的平台生命周期是商业经营品类中**最独特的时间系统**——大多数经营游戏的时间是周期的(月/年/季节循环)，但GD Tycoon的时间是**线性的平台代际更替**。PC永不消亡的设计类似于Elona中"家是永远的锚点"——在变化的世界中保留一个不变的安全空间。
