# 评分与口碑系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/239820)

---

## 一、双重评分体系

### 游戏分数(Game Score)——隐藏值

游戏分数是一个**隐藏的内部值**，由以下因素决定：

| 因素 | 影响 | 说明 |
|------|:--:|------|
| 类型×题材组合 | ±30-50% | 绝佳组合→大幅加分；极差组合→大幅扣分 |
| 滑块分配 | ±20-40% | 偏离最优分配→技术和设计分别扣分 |
| 技术与设计比率 | ±10-30% | 与类型预期比率不匹配→扣分 |
| 平台匹配度 | ±10-20% | 游戏类型与平台受众不匹配→扣分 |
| 引擎等级 | +0-30% | 高级引擎=更高基础分数 |
| 整体质量(Bug/打磨) | -0-20% | 缺少打磨→引入Bug→降低分数 |
| 趋势(Trending) | 临时加成 | 当前流行的类型/题材有额外加分 |

### 评分数(Review Score)——公开显示

评分数(1-10)是玩家看到的，计算方式为：**当前游戏分数 vs 历史最高游戏分数**的比较：

| 相对关系 | 评分数 |
|:--:|:--:|
| 超过历史最高10+分 | **9-10**(杰作) |
| 超过历史最高5-10分 | 7-9(优秀) |
| 接近历史最高 | 5-7(良好) |
| 低于历史最高 | 3-5(平庸) |
| 远低于历史最高 | 1-3(差评) |

## 二、评分的动态性

### "与过去的自己比较"

GD Tycoon的评分系统与传统经营游戏"绝对质量评分"完全不同——它是**相对于你自己历史记录的**。这意味着：

- 你的第一个游戏即使质量一般也能得6-7分(因为没有历史最高来比较)
- **重复同样类型/同样题材的组合**→分数递减(缺乏创新感)
- **续作需要在原作基础上超越**——否则"吃老本"会被惩罚
- 要持续获得高分，必须**不断创新**(新类型/新题材/新技术/新平台)

## 三、销量影响因素

| 因素 | 影响 |
|------|:--:|
| 评分数 | **核心**(高分=高销量+长尾销售) |
| 粉丝数 | **强**(粉丝多=基本销量有保底) |
| 平台市场份额 | 强(大平台=更多潜在买家) |
| 营销投入 | 中(可以拉升初期销量) |
| 盗版率 | 负(**无Copy Protection=70%+销售被盗版侵蚀**) |
| 游戏规模 | 中(AAA=更高潜在销量但更高成本) |

## 四、设计特点分析

**相对评分=永无止境的创新驱动力**：大多数游戏中的评分是"绝对质量"的反映——你达到某个质量标准就能获得某个评分。GD Tycoon的相对评分意味着**你永远不能躺在过去的荣光上**——你必须不断创新(新类型/新题材/新技术)才能持续获得高分。这与真实游戏产业的"创新驱动"逻辑惊人地一致。

**重复惩罚=反刷子机制**：相同类型×相同题材的连续游戏会获得递减评分——这是一个优雅的反刷子设计。它禁止了"找到一个最优组合→无限刷同款游戏"的玩法，强制玩家尝试不同的类型和题材组合。

**历史最好成绩=评分的相对锚点**：这个机制将"进步"从"绝对数值的增长"变成了"超越自己的过去"。一个9分游戏不是在与其他游戏比较——它是在告诉你"这是你迄今为止做得最好的"。

---

> 跨品类对比: GD Tycoon的相对评分与游戏开发物语(Game Dev Story)的"类型×题材组合评分公式"相似但更复杂——GD Tycoon多了"平台生命周期"和"引擎技术等级"两个额外维度。与过山车之星的"Prestige=绝对质量指标"不同——GD Tycoon的质量始终是相对的。
