# 时间节奏与里程碑系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区](https://steamcommunity.com/app/239820)

---

## 一、时间结构

| 时间单位 | 说明 |
|------|------|
| 周(Week) | 游戏开发的基本单位 |
| 月/年 | 基于时间的里程碑解锁 |

GD Tycoon的"时间"不是循环的——它是**线性推进的游戏产业发展史**，从80年代一直延伸到未来。

## 二、产业年代演进

| 年代 | 新平台/技术 | 产业特征 |
|:--:|------|------|
| 80年代 | PC/Gameling(初期) | 车库时代——单人开发 |
| 90年代 | Playsystem/CD-ROM | 团队化——中型工作室 |
| 2000年代 | mBox/3D图形 | 3A时代——大型团队 |
| 2010年代+ | 移动/独立/数字发行 | 多元化——多种商业模式 |

## 三、关键解锁时间线

| 解锁 | 时间(年份) | 意义 |
|------|:--:|------|
| 目标受众(Target Audience) | 第3年 | 可以定向游戏客群 |
| 休闲游戏(Casual Games) | 第3年 | 新游戏类型 |
| 营销(Marketing) | 第4年 | 可以推广游戏 |
| 续作(Sequels) | 第8年 | 可以为成功游戏开发续作 |
| 多类型(Multi-Genre) | 第12年 | 游戏可以混合多个类型 |
| MMO | 互联网机遇后 | 持续订阅收入 |
| AAA游戏 | 大型游戏评分10后 | 最高制作规模 |
| 自定义主机 | 2亿+现金 | 产业终局成就 |
| 自有展会 | 100万粉丝或第23年 | 最大粉丝触达 |

## 四、设计特点分析

**线性推进=自然的难度曲线**：游戏早期(80年代)竞争少/平台少/类型少——是自然的"教程期"。随时间推进，更多平台/更多类型/更高技术要求的出现自然地提高了复杂度。这种"产业演化=难度曲线"的设计无需显式的难度选择。

**解锁时间线=期待驱动的进度系统**：与大多数经营游戏的"无限循环"不同，GD Tycoon有明确的"未来会解锁什么"的期待感。

## 五、产业阶段与策略适配

| 产业年代 | 团队规模 | 主流游戏类型 | 最佳策略 |
|:--:|:--:|------|------|
| 80年代(车库) | 1人 | 小型动作/冒险 | 专注PC+掌握1-2个类型×题材组合 |
| 90年代(小办公室) | 2-4人 | 中型动作/RPG/模拟 | 扩展平台(Playsystem)+培训员工 |
| 2000年代(中办公室) | 4-8人 | 大型3D游戏 | 3A准备+多平台发行 |
| 2010年代+(大办公室) | 8-12人 | AAA/MMO | 自有引擎+续作系列+自定义主机准备 |

## 六、时间压力与"错失恐惧"

由于解锁基于年份(而非玩家行为)，玩家有隐性的"错失恐惧"(FOMO)——如果你在第3年时还没有足够的技术准备好使用"目标受众"功能，你会在第4年(营销解锁)之前已经落后于产业趋势。这种基于时间的解锁系统创造了一种"必须在窗口期内准备好"的紧迫感——接近真实游戏产业中"错失平台周期=无法挽回"的现实。
