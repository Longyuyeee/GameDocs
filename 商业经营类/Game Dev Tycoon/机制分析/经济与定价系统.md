# 经济与定价系统 — Game Dev Tycoon

> 游戏版本: Game Dev Tycoon v1.8+ (Steam, 2013)
> 数据来源: [Game Dev Tycoon Fandom Wiki](https://gamedevtycoon.fandom.com/), [Steam社区指南](https://steamcommunity.com/app/239820)

---

## 一、收入来源

| 来源 | 占比 | 特征 |
|------|:--:|------|
| **游戏销售** | 70-90% | 核心收入——评分×粉丝基础×平台份额 |
| 发行合同收入 | 早期10-30% | 为发行商做代工→稳定但利润低 |
| MMO订阅(后期) | 持续 | 互联网机遇→MMO研发→持续月收入 |
| 自定义主机平台费(终后期) | 极高 | 拥有平台=所有在该平台上的游戏利润归你 |

## 二、成本结构

| 成本 | 说明 |
|------|------|
| 开发成本(游戏规模决定) | 小型:低成本/大型+AAA:数十万到数百万 |
| 营销费用(可选) | 拉升初售销量 |
| 员工薪资(固定月度成本) | 随团队规模增长 |
| 办公室租金(随扩张上涨) | 车库$8K/月→大办公室$100K+/月 |
| 研发投入 | 一次性——解锁新技术/新游戏类型 |
| 自定义引擎开发 | 极贵但长期回报高 |
| 自定义主机开发 | **数亿级一次性投入** |

## 三、关键经济决策

| 决策 | 影响 |
|------|------|
| **何时搬离车库** | 过早=成本膨胀→现金流断裂。推荐: 现金1500-2000万+粉丝15-25万后 |
| **Copy Protection投资** | 没有它→70%+销售被盗版侵蚀→**第一优先研发** |
| **发行合同vs自主发行** | 合同=稳定收入(无风险)/自主=高利润(有风险) |
| **引擎重复使用vs开发新引擎** | 续作用新引擎=更高的评分上限 |
| **拒绝投资邀请** | 投资=别人拿走你公司的股权=长期损失 |

## 四、定价系统

GD Tycoon的游戏**玩家不直接定价**——价格由游戏规模和平台市场自动决定。这简化了定价管理，将经济焦点转移到"做什么游戏"而非"怎么定价"的游戏性决策上。

## 五、设计特点分析

**"不自己定价"的设计选择**：与大多经营游戏不同，GD Tycoon没有定价滑块。这个选择将玩家的注意力从"定价策略"转移到"产品质量策略"——你不是通过调整价格来赚钱，而是通过做出更好的游戏来赚钱。这是一种"产品质量为王"的设计哲学。

**盗版率=经济防线的突破**：没有Copy Protection的游戏会损失70%+的销售额给盗版——这是一个巨大的经济惩罚，足以毁灭任何游戏公司。盗版不是"麻烦"而是"如果不投资反盗版机制就会破产"的生存威胁。

---

> 跨品类对比: GD Tycoon的无手动定价与过山车之星的"Prestige→自动公式定价"类似——都是将定价权抽象化以减少微观管理。

## 六、拒绝投资邀请的设计意图

游戏中会收到"投资人"的投资邀请——接受意味着获得大量现金注入但失去公司股权。**社区共识是永远拒绝**——因为游戏后期的利润远超投资金额，而失去的股权是永久的。这个设计选择传递了一个微妙的"反VC bootstrapping"哲学——独立开发者在不受外部资本干扰的情况下创造最大价值。
