# RPG融合与竞争AI设计分析 — Big Ambitions

> 游戏版本: Big Ambitions EA (PC/Steam, Hovgaard Games, 2023+)

## 1. "人"作为经营变量——RPG融合的设计哲学

### 传统经营游戏中"人"的缺席

在分析 Big Ambitions 的 RPG 融合之前,有必要先理解传统经营游戏中"人"的缺位:

| 游戏类型 | 玩家代表 | 经营者的"身体"存在 |
|:--:|:------|:--:|
| 传统经营模拟(开罗等) | 无实体的管理者 | 不存在——玩家是"上帝视角" |
| 超市模拟器 | 第一人称角色 | 存在但无个人需求——工具人 |
| 双点医院 | 院长 | 存在但管理需求≈0 |
| 模拟城市/天际线 | 市长 | 不存在——纯粹的上帝视角 |
| **Big Ambitions** | **18岁创业青年** | **有饥饿/精力/幸福/身体的物理存在** |

### "人"作为经营变量的设计实现

Big Ambitions 中的角色有三大核心状态,每一个都对经营产生直接影响:

| 状态 | 上限 | 衰减机制 | 恢复方式 | 降至0的后果 |
|:--:|:--:|:--:|------|:--:|
| 精力 (Energy) | 100% | 每小时消耗约8%(活动时) | 睡眠(约20%/小时) | **昏倒→送医院→浪费时间+金钱** |
| 饥饿 (Hunger) | 100% | 每小时消耗约5% | 进食(约50%/餐) | 持续掉幸福度→经营效率下降 |
| 幸福度 (Happiness) | 100% | 每天衰减约3% | 娱乐活动(10-30%/次) | 抑郁→**经营决策质量显著下降** |

### "老板的幸福 = 公司的生产力?"——一个被验证的设计假设

Big Ambitions 做了一件传统经营游戏中几乎没人做过的事: **把"玩家的个人状态"绑在"公司的经营效率"上**。

**正反馈链**:
```
你的角色幸福度高
    → 决策质量高 (选址/定价/人事决策更理性)
    → 公司运营效率提升
    → 更多利润 → 更好的生活品质
    → 你的角色更幸福
```

**负反馈链**:
```
你的角色过度工作、忽视个人需求
    → 精力下降/幸福度下降
    → 决策质量下降 (错过关键决策/误判市场)
    → 公司效率受损
    → 利润下降 → 焦虑
    → 你的角色更不幸福
```

这一设计在传统管理游戏中是反直觉的——大多数管理游戏的默认假设是"你越努力经营,公司越好"。但 Big Ambitions 暗示了一个更现实的结论:**疲惫的老板做不好决策,再多的努力也无法弥补决策质量的下降。**

### 对比: 卡牌店模拟器 vs Big Ambitions

| 设计维度 | 卡牌店模拟器的玩家 | Big Ambitions的玩家 |
|:--:|:------|:------|
| 身体 | "无身体的管理者" | "有物理存在的创业者" |
| 经营决策 | 基于系统提示 | 基于角色状态的约束 |
| 失败惩罚 | 公司破产(从头再来) | 公司亏钱+角色健康受损 |
| 成功标准 | 收入最大化 | 收入+角色幸福度双达标 |
| 情感连接 | 与公司的连接 | 与角色+公司的双重连接 |

> Big Ambitions 让经营游戏第一次有了"你"而不是"它"——当你需要决定"今天是去店里工作还是去赌场放松"时,这个游戏就超出了传统经营模拟的范畴。它变成了一个关于"平衡生活与工作"的叙事工具。

## 2. 竞争AI的"人味"——从NPC到对手

### 传统经营游戏的竞争AI: 背景板

在分析 Big Ambitions 的AI之前,先看看传统经营游戏的"竞争"是什么样的:

| 游戏 | 竞争形式 | AI行为 | 玩家感受 |
|:--:|:------|:--:|:--:|
| 开罗系列 | 年度排名 | 分数驱动的被动系统 | "排名只是数字" |
| 双点医院 | 无直接竞争 | 无 | "我一个人在经营" |
| 模拟城市 | 邻城发展 | 被动对比 | "邻居与我无关" |
| 超市模拟器 | 无 | 无 | "我是唯一超市" |

### Big Ambitions 的竞争AI: 会记仇的对手

EA 0.5 "The Rivalry Begins" 更新引入了真正具有"行为智能"的竞争对手AI。对手的行为不是随机的,而是基于玩家行为的**响应式策略**。

### 对手AI的行为模型

```
触发条件 → 检测 → 评估 → 决策 → 执行 → 反馈
```

**具体流程**:

1. **触发条件**: 玩家在某区域开设同业态第 X 家店铺
   - X=1-2: 对手不反应(视为正常竞争)
   - X=3+: 触发"进入对手领地"警告
   - X=5+: 触发主动反击

2. **检测阶段**: 对手AI扫描玩家的经营数据(可见部分)
   - 玩家在该区域的店铺数
   - 玩家店铺的预估利润率
   - 玩家核心员工的熟练度排名

3. **评估阶段**: 对手计算反击"性价比"
   - 反击成本(削价/挖角/开店的投入)
   - 预期效果(能多大程度削弱玩家)
   - 反击风险(玩家是否可能反制)

4. **决策阶段**: 选择反击手段
   - 轻量反击: 削价竞争(幅度5-15%)
   - 中等反击: 挖角玩家高熟练度员工(成功率取决于薪资差距)
   - 重量反击: 在玩家最赚钱的区域开设同业态店铺
   - 组合反击: 同时使用多种手段

5. **执行阶段**: AI按计划执行反击

6. **反馈阶段**: 根据反击效果调整后续策略

### 反击手段详细参数

| 反击手段 | 触发条件 | 对玩家影响 | 持续时间 | 玩家反制方法 |
|:--:|:--:|:--:|:--:|------|
| 削价 (Price Cut) | 玩家某店利润超标 | 该店客流-10~20% | 持续(对手随时可调) | 用定价工具+服务品质对抗 |
| 挖角 (Poaching) | 玩家有熟练度>75%员工 | 可能失去核心员工 | 单次(成功即永久) | 提高薪资+改善福利 |
| 对等开店 | 玩家在对手领地开店 | 对手在你最强区域开店 | 永久 | 用品牌/服务/效率碾压 |
| 人才截胡 | 玩家发布招聘需求 | 抢走高潜力候选人 | 单次 | 多开猎头渠道 |

### "报复"机制的设计哲学

Big Ambitions 引入竞争AI的报复机制,不是为了"让游戏更难"——它的真正目的是**让世界感觉是"活"的**。

**为什么报复机制是必要的?** 因为如果AI对手不对玩家行为做反应,玩家会逐渐感到"我做什么都不会引起世界的变化"——这是大多数经营游戏在20-30小时后玩家流失的核心原因。

**报复机制创造的叙事**:
- "我在默里山开了第3家店,第二天发现对手在加门特区我也开了家店"
- "我的金牌员工被Ingrid Schneider挖走了!"
- "Huang Guo的价格战让我的利润降了15%,我需要重新定价"

这些不是预设事件,而是**玩家行为触发的结果**。每个玩家的"竞争故事"都是不同的。

## 3. 四大区域对手深度档案

| 对手 | 区域 | 风格 | 攻击偏好 | 防御弱点 | 应对策略 |
|:--:|:--:|:--:|:--:|:--:|------|
| Thierry Laurent Moreau | 默里山 | 防守反击型 | 在你店旁开同品类店 | 反应慢(开店决策需5-7天) | 快速扩张让他跟不上 |
| Huang Guo | 中城区 | 激进攻击型 | 削价竞争、挖角 | 价格战消耗自身利润 | 提升服务品质而非拼价格 |
| Jessica Johnson | 地狱厨房 | 扩张侵略型 | 大量开店抢占市场 | 快速扩张导致服务质量下降 | 用高品质单店碾压她大量低质店 |
| Ingrid Schneider | 加门特区 | 隐蔽挖角型 | 猎取你的高技能员工 | 自身员工培养能力弱 | 培养多个高水平员工分散风险 |

### 对手AI的"个性"差异

Big Ambitions 的四个区域对手不只是换了个名字和头像——它们在**行为模式上有本质差异**:

- **Thierry (防守型)**: 他的行为像是在说"这是我的地盘,你最好别进来"。反应机制是被动的——玩家不惹他,他也不惹玩家。但如果玩家进入他的区域,他会用"复制开店"的方式回应。

- **Huang (进攻型)**: 他不管玩家做什么,都会主动扩张和削价。他代表的是"市场竞争的残酷面"——即使你什么都没做错,竞争对手也会主动攻击。

- **Jessica (扩张型)**: 她追求市场份额而非利润。她的店面可能利润不高,但数量多、覆盖广。这迫使玩家思考"数量 vs 质量"的经典商业选择题。

- **Ingrid (隐蔽型)**: 她不会直接降价或开店竞争,而是专注于挖走玩家的核心人才。这对应了现实中"人才竞争比价格竞争更隐蔽但也更致命"的商业常识。

> 四个对手各有独特的"商业人格",这在策略游戏中是常见的(如文明系列的不同领袖风格),但在商业经营品类中几乎是**独一无二的设计**。

## 4. 竞争AI设计的局限性(EA状态)

Big Ambitions 的竞争AI虽然品类领先,但仍处于早期阶段(v0.5引入)。以下是已知的局限性:

| 局限性 | 说明 | 社区反馈 | 开发者计划 |
|:--:|------|:--:|:--:|
| 触发条件过于线性 | 主要根据"店铺数量"触发,忽略了营收/品牌等维度 | 玩家希望更多元的触发条件 | 路线图中已标注"Market Simulation" |
| 反击手段不够丰富 | 目前仅削价/挖角/开店三种 | 希望加入广告战、供应链截断等 | EA 0.6+可能扩展 |
| 对手之间无互动 | 四个对手彼此不竞争,只针对玩家 | 希望AI之间也有竞争关系 | 未确认 |
| 长期缺乏变化 | 对手行为模式固定,后期可预测 | 希望AI能学习和进化 | 未确认 |
| 竞争后果不够严重 | 被反击的损失容易恢复 | 希望竞争"真正有意义" | EA 0.5仅是"开始" |

## 6. 设计启示——Big Ambitions对商业经营品类的贡献

### 贡献一:证明了"人"可以成为经营游戏的约束条件

Big Ambitions 证明了角色系统(饥饿/精力/幸福)不是RPG游戏的专属。在经营游戏中加入"有身体的管理者"不仅增加了代入感,更重要的是创造了**自然的决策摩擦**:你不能无限工作,你需要平衡生活。这种约束让经济系统不再是单纯的数字游戏。

### 贡献二:证明了竞争AI是"防止后期无聊"的关键

大多数经营游戏的"后期无聊"问题,根源在于**玩家的一举一动都不会引起世界的响应**。Big Ambitions 的竞争AI(即使还在早期阶段)已经展示了方向:当玩家开店时,对手会报复——玩家的每一个决策都有后果,这种"后果感"是维持长期游玩的核心理念。

### 贡献三:证明了"商业模拟"可以拥有角色叙事

Big Ambitions 证明了一件事:你不需要写一本小说来创造叙事。**系统本身就可以产生叙事**:你因为过度工作昏倒被送进医院——这是一个故事。你的金牌员工被对手挖走——这是一个故事。你在赌场赢了一大笔钱然后用它扩张——这是一个故事。系统驱动的叙事比脚本驱动的叙事更强大,因为它属于每一个玩家的独一无二的游戏过程。
