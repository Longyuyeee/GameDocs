## 一、客流与选址系统

### 1.1 Traffic Index（人流指数）

选址是Big Ambitions中最关键的决策之一，而**Traffic Index（人流指数）**是衡量选址质量的核心指标。

| Traffic Index等级 | 含义 | 客流量预期 | 租金水平 |
|:--:|:--:|:--:|:--:|
| 极低(1-2/10) | 偏僻区域，几乎无人经过 | 极低 | 极低 |
| 低(3-4/10) | 居民区次级街道 | 低 | 低 |
| 中(5-6/10) | 普通商业街区 | 中等 | 中等 |
| 高(7-8/10) | 主要商业街道/购物区 | 高 | 高 |
| 极高(9-10/10) | 曼哈顿核心地段/交通枢纽 | 极高 | 极高 |

> **设计特点分析**: Traffic Index是Big Ambitions对"现实选址逻辑"的游戏化映射。在真实商业中，"选址×业态错位"是最常见的失败原因——一个卖鲜花的店铺开在工业区即使租金再低也没有客流。游戏通过Traffic Index让玩家像真实创业者一样思考：高客流=高租金，低客流=低租金，玩家需要在"支付高租金换取高销量"和"低成本运营等待客源"之间做权衡。这与《温泉物语》中"景观辐射"的空间逻辑形成鲜明对比——温泉物语关注的是建筑间的视觉协同，Big Ambitions关注的是纯地理位置的客流动线。

### 1.2 选址综合评价要素

| 考量因素 | 权重 | 说明 |
|------|:--:|------|
| **Traffic Index** | ★★★★★ | 最核心的指标 |
| **租金** | ★★★★ | 高客流地段租金高，ROI需计算 |
| **店铺面积** | ★★★ | 面积影响货架数量/商品展示量/员工容量 |
| **停车位** | ★★ | 是否有停车位影响开车顾客的光顾意愿 |
| **区域类型** | ★★★ | 商业区/住宅区/工业区各有不同的需求特征 |
| **附近竞争** | ★★★★ | 同品类竞争店铺数量直接影响分流 |

## 二、顾客需求系统

### 2.1 区域需求热度

每区域有不同商品的需求热度和竞争密度。玩家可以通过**每周一的需求报表**了解各区的最新情况：

| 区域 | 居民类型 | 核心需求品类 | 消费力 | 竞争程度 |
|:--:|------|------|:--:|:--:|
| **上东区(Upper East)** | 高收入 | 珠宝/服装/电子产品 | 高 | 高 |
| **中城(Midtown)** | 混合(白领+居民) | 礼品/花卉/餐饮 | 中-高 | 中 |
| **金融区(Financial)** | 上班族 | 快餐/便利店/办公服务 | 中 | 中-高 |
| **下东区(Lower East)** | 中低收入 | 折扣商品/二手 | 低-中 | 低-中 |
| **哈莱姆/外围** | 待补充 | 待补充 | 待补充 | 待补充 |

> **设计特点分析**: 需求报表系统是Big Ambitions引导玩家"做市场调研"的机制。在传统商业经营游戏中，玩家通常可以"想开什么店就开什么店"，只需考虑资金和空间。Big Ambitions则要求玩家像真实的商人一样——先看市场数据，再做业态决策。这种"数据驱动决策"的设计在商业经营品类中较为罕见，更多地出现在硬核模拟游戏（如《金融帝国2》）中。

### 2.2 需求品类的动态变化

需求不是一成不变的——它是动态系统，受以下因素影响：

| 影响因素 | 影响方向 | 时效 |
|------|------|:--:|
| **季节/节假日** | 特定品类需求上升(如节假日礼品) | 短期 |
| **竞争对手入驻/退出** | 新入驻→分流需求；退出→释放需求 | 中期 |
| **区域发展** | 玩家开店越多→该区商业氛围越浓→客流量上升 | 长期 |
| **玩家广告/营销** | 扩大知名度→提升该区对本店的需求 | 中期 |

## 三、顾客行为系统

### 3.1 进店到购买的完整流程

```
顾客进入店铺 → 浏览商品(多品类) → 决定购买(单件或多件) → 排队结账 → 离开
          ↓                    ↓                      ↓
    受Traffic Index影响   受商品选择和价格影响   受收银效率影响
```

### 3.2 顾客行为特征

| 行为特征 | 细节 | 设计意义 |
|------|------|------|
| **品类浏览** | 顾客会浏览店铺内的**多个品类**，不限于"主营"商品 | 鼓励店铺商品多样化 |
| **多件购买** | 客单价高——顾客倾向于一次购买多件商品 | 合理摆放可提升ARPU |
| **价格敏感度** | 不同品类/不同顾客对价格敏感度不同 | 定价策略有差异化空间 |
| **排队容忍度** | 排队时间过长→顾客不满→流失 | 需合理配置收银员 |
| **满意度影响回头率** | 满意→回头客(高频消费)；不满意→流失且可能差评 | 长期运营的关键 |

### 3.3 顾客满意度公式(估算)

```
顾客满意度 = f(员工熟练度 × 权重 + 店内环境/氛围值 × 权重 + 价格合理度 + 排队等待时间 × (-权重))
```

| 满意度因子 | 权重(估值) | 优化方式 |
|:--:|:--:|------|
| **员工熟练度** | ~40% | 培训收银员/客服至100%熟练度 |
| **店内环境** | ~25% | 优质地板/墙面/灯光提升氛围值 |
| **价格** | ~20% | 定价在顾客容忍范围内 |
| **排队时间** | ~15% | 充足收银员/高效排班 |

## 四、竞争AI反击系统

这是Big Ambitions**最独特、最具深度的系统之一**——其他商业经营游戏中几乎不存在如此激进的竞争AI。

### 4.1 竞争AI的反击方式

| 反击方式 | 触发条件 | 效果 | 应对策略 |
|:--:|------|------|------|
| **抢地盘(开同类店)** | 你在某区域开了高利润店铺 | AI在你旁边或同区开同类店分流你的客流 | 垄断市场或跨品类经营 |
| **削价竞争** | AI检测到你在该品类利润过高 | AI降低售价压缩你的利润空间 | 优化供应链降低成本以维持利润 |
| **挖角员工** | AI检测到你的某些员工高熟练度 | 你的员工可能被AI以更高薪资挖走 | 提高员工待遇/满意度减少流失 |
| **竞争排名系统** | 持续经营 | AI之间的竞争力有排名，你在排名中的位置影响AI行动强度 | 排名越高反击越激烈 |

### 4.2 竞争激烈度等级

| 等级 | 玩家状态 | AI反击强度 | 市场状态 |
|:--:|:--:|:--:|------|
| **Lv.1 无人问津** | 无店或1家在低需求区 | 无 | 市场平静 |
| **Lv.2 新玩家** | 1-2家店铺在盈利 | 弱——AI开始关注 | 市场略有波动 |
| **Lv.3 搅局者** | 3-5家店铺，跨品类经营 | 中——AI可能开1-2家竞争店 | 局部竞争 |
| **Lv.4 区域霸主** | 5-10家店铺，某个品类市占率高 | 强——AI集中在该品类反击 | 价格战/抢人战 |
| **Lv.5 垄断威胁** | 10+家店铺，全品类覆盖 | 极强——全品类AI反击 | 全面商业战争 |

### 4.3 竞争AI的实际案例分析

```
场景: 玩家在Midtown开了一家高端服装店，利润率高
第1周: 没有竞争，顾客络绎不绝
第3周: 附近的AI竞争对手注意到你的成功
第4周: AI在Midtown开了第二家服装店，价格比你低10%
第5周: 你的顾客流失20%，利润下降
第6周: AI试图挖走你的熟练店员(涨薪30%的offer)
```

> **设计特点分析**: 竞争AI系统是Big Ambitions区别于其他商业经营游戏的核心差异点。在大部分同类游戏中（如双点医院、冒险村物语2、温泉物语），竞争要么不存在，要么只是数值化的背景设定。Big Ambitions的AI会**主动反击**——不仅是被动应对，而是根据玩家行为做出策略性反应。这种设计：
> 1. 营造了"真实的商业战争"氛围——玩家不是在一个真空环境中经营
> 2. 迫使玩家不断优化（一旦停止进步就会被AI蚕食市场份额）
> 3. 增加了游戏的重复可玩性——每次开局的竞争格局不同
> 4. 唯一不足：AI的决策模式可能存在可预测性，高级玩家可能利用AI的反应模式获利

## 六、跨游戏对比：竞争与顾客系统

| 对比维度 | Big Ambitions | 双点医院 | 冒险村物语2 | 游戏开发物语 |
|------|:--:|:--:|:--:|:--:|
| **竞争是否存在** | **是(核心系统)** | 否(无竞争) | 否(无竞争) | 否(无竞争) |
| **AI反击机制** | **抢地盘/削价/挖角(三重)** | 无 | 无 | 无 |
| **顾客行为模拟** | 深度(浏览→购买→满意度→回头) | 中等(排队→治疗→离开) | 中等(消费→打怪) | 简单(评分→销量) |
| **客流系统** | Traffic Index(选址核心) | 关卡固定(不受选址影响) | 景观辐射(建筑布局影响) | 无(固定粉丝增长) |
| **选址自由度** | 全曼哈顿自由选(受租金/客流约束) | 关卡内固定位置 | 村庄内自由放置 | 无(固定格子) |
| **品类需求差异化** | **是(区域+动态报表)** | 否(所有患者一样) | 部分(职业偏好) | 否 |

> **跨游戏洞察**: Big Ambitions的顾客与竞争系统体现了"真实商业模拟"的设计哲学——它不把顾客视为被动的"消费机器"，也不把竞争视为可忽略的背景。顾客有行为模式、有价格敏感度、有满意度留存；竞争对手则会主动出击保护自己的市场份额。这种"相互博弈"的动态比"玩家单向经营"的传统模式提供了更丰富的策略空间。唯一的代价是学习曲线更陡——新手需要理解客流/区域需求/竞争AI等多层系统才能做出正确决策。

## 七、竞争中的反制策略

| 情况 | 应对策略 | 实施方法 |
|------|------|------|
| AI在你旁边开了竞争店 | 差异化经营 | 调整品类/价格避免正面冲突 |
| AI发动价格战 | 成本优化 | 升级供应链→降低进货成本→保持利润 |
| AI挖角你的员工 | 提升员工满意度 | 加薪/优化排班/设备升级 |
| AI多品类包围 | 跨品类突围 | 进入AI尚未涉足的品类/区域 |
| AI垄断某个品类 | 寻求合作或等待 | 暂避锋芒，先做其他品类积累资金 |
