## 一、商品品类总览

Big Ambitions提供了**商业经营品类中最丰富的商品品类系统**——涵盖10+个品类，每个品类有独立的定价基线、需求周期和供应链特征：

| 品类 | 典型商品 | 利润率(估) | 存储条件 | 需求稳定性 |
|:--:|:--:|:--:|:--:|:--:|
| **服装(Clothing)** | 衬衫/裤子/外套/鞋 | 中-高 | 常温 | 中等 |
| **电子产品(Electronics)** | 手机/电脑/耳机/配件 | 高 | 常温(防潮) | 中等(更新迭代快) |
| **珠宝(Jewelry)** | 项链/戒指/手表 | 极高 | 展示柜(安全) | 中等(价格敏感) |
| **酒品(Alcohol)** | 葡萄酒/烈酒/啤酒 | 中-高 | 常温/恒温 | 中等 |
| **花卉(Flowers)** | 鲜花/盆栽 | 中 | 冷藏/保鲜 | 低(过期风险高) |
| **食品饮料(Food & Drink)** | 零食/饮料/生鲜 | 低-中 | 冷库(生鲜) | 高(刚性需求) |
| **家居(Home Goods)** | 家具/厨具/装饰 | 中 | 常温 | 中等 |
| **书籍(Books)** | 图书/杂志 | 低-中 | 常温 | 低-中 |
| **礼品(Gifts)** | 礼盒/玩具/纪念品 | 中-高 | 常温 | 波动(节日驱动) |
| **运动用品(Sports)** | 器材/服装/配件 | 中 | 常温 | 中等(季节性) |

> **设计特点分析**: Big Ambitions的商品品类系统不是简单的"换皮"——每个品类有独立的**供应链路径、存储条件、利润率、需求周期**。例如：花卉需要冷库存储且保鲜期短，但利润率不错；珠宝利润率极高但需要展示柜和安保；食品是刚性需求但利润率较低且需要冷链。这意味着玩家需要像真实零售商一样思考："我的店铺适合卖什么？我的仓库能支持什么品类？"

## 二、五阶段供应链进化(核心系统)

这是Big Ambitions**最有深度、最具渐进性**的设计——玩家的采购方式从简单的"去批发商买东西"逐步进化到"自建工厂全产业链控制"。

### 2.1 五阶段总览

| 阶段 | 采购方式 | 单位成本 | 品类齐全度 | 解锁条件 | 自动化程度 |
|:--:|:--:|:--:|:--:|:--:|:--:|
| **Phase 1 批发商** | 直接向批发商下单 | **最贵(基准价100%)** | 低(品类不全) | 游戏开局 | 低(手动下单) |
| **Phase 2 批发仓库** | 开车去批发仓库采购 | 中等(80-90%) | **全(品类齐全)** | 购买车辆 | 低(手动采购) |
| **Phase 3 进口商/港口** | 通过进口商从港口进货 | 便宜(60-75%) | 高(需采购代理对接) | 雇佣采购经理+建立合作关系 | 中(代理自动) |
| **Phase 4 Hub&Spoke仓库网络** | 中心仓→专业仓→店铺 | 较便宜(见下方) | 全(自行调配) | 拥有多个仓库+物流团队 | 高(配送自动化) |
| **Phase 5 工厂制造** | 自建产线生产 | **最低(材料成本+人工)** | 有限(仅自产品类) | 大量资金+工业用地 | **完全自主可控** |

### 2.2 各阶段详解

#### Phase 1: 批发商(Wholesaler)

```
优势: 开局即用，无需任何前置条件
劣势: ①价格最贵 ②品类不全 ③需手动下单 ④有最小起订量
适用: 游戏第一天，首店启动
```

| 属性 | 说明 |
|:--:|------|
| 解锁 | 自动(游戏初始) |
| 价格水平 | 基准价(100%，最贵的采购方式) |
| 品类范围 | 有限——可能缺少部分小众品类 |
| 配送方式 | 按合同配送至店铺 |
| 管理水平 | 手动——玩家需逐个下单 |

#### Phase 2: 批发仓库(Wholesale Warehouse)

```
优势: ①品类齐全(一次买全) ②价格中等 ③可电话订购
劣势: ①需要车辆(前期投资) ②仍需要手动操作 ③需要仓库空间
适用: 中期扩张，品类多样化
```

| 属性 | 说明 |
|:--:|------|
| 解锁 | 购买车辆(约$2,000-$10,000) |
| 价格水平 | 中等(批发价的80-90%) |
| 品类范围 | **全品类**(所有品类都能找到) |
| 配送方式 | 玩家/司机开车去拉→存仓库→分配至店铺 |
| 管理水平 | 半自动——可电话订购 |

#### Phase 3: 进口商/港口(Importer/Port)

```
优势: ①价格便宜(大量采购) ②采购代理自动对接 ③长期合同稳定供应
劣势: ①需雇佣采购经理(一人对应一进口商) ②最小批量大 ③有清关周期
适用: 中后期想要降低成本的玩家
```

| 属性 | 说明 |
|:--:|------|
| 解锁 | 雇佣采购经理(Procurement Manager) |
| 价格水平 | 便宜(批发价的60-75%) |
| 品类范围 | 高——不同进口商覆盖不同品类 |
| 配送方式 | 代理对接→港口提货→入仓→分拨 |
| 管理水平 | 中——经理自动对接但需玩家设定采购计划 |

#### Phase 4: Hub&Spoke仓库网络

详见[设施与店铺布局系统](./设施与店铺布局系统.md)中关于仓库网络的说明。这一阶段将前三个阶段整合为一个完整的物流体系。

#### Phase 5: 工厂制造(Factory)

```
优势: ①成本最低(材料+人工) ②完全自主控制产能 ③不受供应商断货影响
劣势: ①初始成本极高(设备+场地) ②需要大量工人 ③生产线复杂
适用: 终局阶段，追求全产业链控制和最大利润
```

| 属性 | 说明 |
|:--:|------|
| 解锁 | 大量资金+工业用地 |
| 初始投资 | ~$50,000-$200,000+ |
| 产线类型 | 传送带系统→组装线(珠宝/电子产品等) |
| 工人需求 | 大量工厂工人(多个工种) |
| 生产品类 | 有限——只能生产自己工厂设定的品类 |

### 2.3 供应链成本对比图(估算)

```
成本水平:   Phase 1 > Phase 2 > Phase 3 > Phase 4 ~ Phase 5
             100%     85%       70%        60-65%    50-60%

自动化:    Phase 1 < Phase 2 < Phase 3 < Phase 4 < Phase 5
              低       低        中         高         极高

控制力:    Phase 1 < Phase 2 < Phase 3 < Phase 4 < Phase 5
              低       低        中         中         极高
```

> **设计特点分析**: 五阶段供应链是Big Ambitions**最核心的经济设计**。它不是让玩家一步到位建立工厂，而是通过5个阶段逐步升级采购方式——每个阶段都比前一个阶段"更便宜但更复杂"。这种设计：
> 1. **自然的学习曲线**: 玩家不用一次性理解整个供应链，而是随着业务扩张逐步接触每个阶段
> 2. **渐进的成本优化**: 每一阶段都在降低单位成本，给玩家持续的"优化正反馈"
> 3. **管理复杂度递增**: 从"手动下单"到"自动配送"到"自产自销"，管理复杂度和游戏深度同步增长
> 4. **战略抉择**: 玩家需要决定"要不要升级到下一阶段?"——更低的成本背后是更高的管理复杂度
>
> 在商业经营品类中，这是**最深度的供应链系统**。大多数同类游戏（如开罗系列、双点系列）的"采购"仅仅是"点击购买"的抽象操作。Big Ambitions把供应链从一个抽象的黑盒变成了一个可感知、可优化、可掌控的管理系统。

## 三、采购合同体系

| 合同类型 | 合作对象 | 期限 | 灵活性 | 价格优惠 |
|:--:|:--:|:--:|:--:|:--:|
| **定期配送合同** | 批发商 | 可设定周期 | 中(可调整) | 一般 |
| **进口采购合同** | 进口商 | 长期 | 低(大批量) | **高** |
| **单次采购** | 批发商/批发仓库 | 一次性 | 高(按需) | 低 |

### 3.1 采购计划的设定

玩家设定采购计划时需要考虑：

| 决策因素 | 影响 | 策略 |
|:--:|------|------|
| **采购量** | 过多→库存积压(资金占用+过期)；过少→缺货损失 | 以历史销量为基准+安全库存 |
| **采购频率** | 高频→配送成本高；低频→断货风险 | 根据品类周转率设定 |
| **合同期限** | 长期→价格优惠但锁定；短期→灵活但价高 | 稳定品类用长期，波动品类用短期 |
| **供应商选择** | 不同供应商价格/品类/服务不同 | 品类匹配度>价格优势 |

## 四、进口清关系统

使用进口商(Phase 3)时需要经过清关流程：

| 清关环节 | 耗时(估) | 涉及角色 | 说明 |
|:--:|:--:|:--:|------|
| **下单** | 即时 | 采购经理 | 向进口商发出采购订单 |
| **备货** | 1-3天 | 进口商 | 进口商准备货物 |
| **运输** | 2-7天 | 物流公司 | 海运/空运至港口 |
| **清关** | 1-2天 | 海关 | 通关手续 |
| **提货** | 当日 | 采购经理/司机 | 从港口的仓库提货 |
| **入仓/分配** | 当日 | 仓库团队 | 存入仓库或直接配送至店铺 |

> 清关周期的存在意味着：通过进口商采购的商品**有较长的前置时间**，玩家必须提前规划库存，不能临时补货。这也增加了"预测需求"的重要性。

## 五、工厂制造系统

### 5.1 工厂建设要素

| 要素 | 说明 | 成本范围 |
|:--:|------|:--:|
| **厂房** | 工业区/外围区域的大面积空间 | 租金较低但需要大空间 |
| **传送带系统** | 连接各工站的物流通道 | $10,000-$50,000 |
| **装配线** | 具体的生产设备(依产品类型不同) | $5,000-$30,000/线 |
| **原料仓库** | 存储原材料的空间 | 含在厂房内 |
| **成品仓库** | 存储成品的空间 | 含在厂房内 |
| **工人工作区** | 工厂工人的站位和通道 | 含在布局中 |

### 5.2 可制造的品类

| 可制造品类 | 复杂度 | 利润(相比外购) | 说明 |
|:--:|:--:|:--:|------|
| **珠宝** | 高 | +30-50% | 高附加值制造 |
| **电子产品** | 高 | +25-40% | 组装型制造 |
| **服装** | 中 | +15-30% | 加工型制造 |
| **食品** | 中 | +10-20% | 需冷链配合 |
| **其他待验证** | 待补充 | 待补充 | 待补充 |

> **跨游戏对比 — 供应链深度**

| 维度 | Big Ambitions | 双点医院 | 冒险村物语2 | 游戏开发物语 |
|------|:--:|:--:|:--:|:--:|
| **供应链阶段数** | **5(最深)** | 1(直接购买) | 1(直接购买) | 1(直接购买) |
| **品类差异化** | **是(10+品类各有特征)** | 否(药品无品类概念) | 部分(装备/食物等) | 否(游戏类型≠商品品类) |
| **采购方式选择** | **多种(批发/进口/自产)** | 无(单一来源) | 无(单来源) | 无(单一来源) |
| **物流管理** | **是(仓库/配送/调度)** | 无 | 无 | 无 |
| **工厂/制造** | **是(传送带+装配线)** | 无 | 无 | 无 |
| **进口清关** | **是(有时间延迟)** | 无 | 无 | 无 |
| **库存管理** | **是(货架/仓库/冷库)** | 无 | 无 | 无 |

> **核心洞察**: Big Ambitions的产品与采购系统在商业经营品类中**领先所有同类游戏**。它不仅提供了品类差异化，还将"供应链管理"作为一个**完整的、可逐步升级的玩法系统**来设计。在大多数商业经营游戏中，"采购"只有2步(去商店→点购买)，而Big Ambitions把它变成了5个阶段、横跨数十小时游戏时间的持续优化主线。这一系统单独拿出来已经足够撑起一个"物流管理模拟"游戏。

## 六、设计特点分析总结

1. **渐进复杂度**: 玩家不必一开始就理解全部供应链——从批发商起步，逐步解锁更强的方式
2. **成本优化主线**: 从100%采购成本→50%自产成本，提供了一条清晰的"省钱"成长线
3. **管理深度与自由度**: 玩家可以选择停留在"简单的批发采购"阶段，也可以追求"全产业链控制"
4. **经济系统的基座**: 采购成本是零售利润的基石——供应链越优化，定价越灵活，利润空间越大
5. **现实主义设计**: 进口有清关周期、工厂有建设周期、不同品类有不同的存储条件——这些设计没有简化商业，而是忠实地映射了真实商业的物流逻辑
