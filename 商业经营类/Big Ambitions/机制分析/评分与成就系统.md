# 评分与成就系统 — Big Ambitions

> 游戏版本: Big Ambitions EA (PC/Steam, Hovgaard Games, 2023+)
> 数据来源: [Big Ambitions Wiki](https://big-ambitions.fandom.com) | [官方论坛](https://forum.bigambitionsgame.com) | [Steam社区](https://steamcommunity.com/app/1331550) | [Codecks访谈](https://www.codecks.io/blog/2025/how-to-stop-feature-creep-from-killing-your-game-with-hovgaard-games/)

## 一、系统概述

Big Ambitions的评分与成就系统是**开放式沙盒的自主目标体系**——与传统经营游戏的"线性通关"不同，Big Ambitions不预设"通关条件"，而是提供多维度排名(竞争对手排名、营业额排名)、Steam成就(里程碑记录)、顾客满意度(隐形评分)和玩家自定义的终局目标(全覆盖/收购/自给自足)。其评分系统的核心逻辑是: **"排名是你相对于竞争对手的位置，成就记录你走过的路，终局由你自己定义。"**

| 属性 | 参数 |
|:--|:--|
| 竞争排名 | 全类型/分业态/分区——三维度排名体系 |
| Steam成就 | 基于里程碑的自选成就 |
| 顾客满意度 | 隐形评分(无显式数值，通过回头客率和营业额间接感知) |
| 终局标准 | 开放式——由玩家自定义 |
| 排名更新频率 | 每周/每日(具体待确认) |
| 顾客满意度影响因素 | 员工熟练度、店铺环境、商品丰富度、价格合理性 |

## 二、竞争排名系统

### 2.1 排名维度

| 排名类型 | 覆盖范围 | 更新频率 | 意义 |
|:--|:--|:--|:--|
| **全类型总排名** | 全市所有商业类型 | 待补充 | 综合实力标杆——你的帝国在所有商人群落中的位置 |
| **分业态排名** | 按商业类型(服装/电子/食品/酒类/珠宝等) | 待补充 | 该业态的统治力——你在服装市场排第几? |
| **分区排名** | 按曼哈顿区域 | 待补充 | 区域影响力——你在上东区的综合商业实力 |

### 2.2 排名与顾客流量的关系

| 排名位置 | 对店铺的影响 | 竞争优势 |
|:--|:--|:--|
| **#1** | 最多自然客流——消费者优先选择第一名 | 品牌效应强，新店也容易吸引顾客 |
| **#2-3** | 稳定客流量，但仍需差异化竞争 | 第二选择——需要价格或服务优势 |
| **#4-5** | 客流量明显减少——需要更激进的营销/定价 | 靠特色竞争——不能靠品牌势能 |
| **#5+** | 客流量稀少——生存压力大 | 需要大幅降价或大幅提升服务质量 |

> **设计分析**: 排名系统不是"好看的数字"——它直接影响顾客流量。排名#1的店铺天然获得更多顾客，即使定价较高或服务一般。这种"排名→流量→收入→扩张→更高排名"的正反馈循环是Big Ambitions竞争动态的核心引擎。但正反馈也是双刃剑: 一旦排名下滑，流量减少→收入下降→更难竞争→排名继续下滑的负反馈循环同样成立。

### 2.3 排名影响因素

| 因素 | 对排名的影响权重 | 可控性 | 优化方式 |
|:--|:--:|:--:|:--|
| **店铺数量** | 高 | ★★★★★ | 开设更多店铺(简单粗暴) |
| **总营收** | 高 | ★★★★ | 提高单店效率+开新店 |
| **顾客满意度** | 中高 | ★★★ | 提升员工熟练度+优化购物环境 |
| **商品丰富度** | 中 | ★★★ | 拓展品类宽度+深度 |
| **定价合理性** | 中 | ★★★★ | 平衡利润率与顾客接受度 |
| **店铺位置** | 中 | ★★ | 选址受限于空置店铺和租金预算 |
| **广告/营销** | 中低 | ★★★★ | 投入广告费(具体ROI待确认) |

## 三、Steam成就系统

### 3.1 成就分类

| 成就类别 | 典型成就 | 解锁条件 | 难度 |
|:--|:--|:--|:--:|
| **创业里程碑** | "首店开业" | 开设第一家店铺 | ★ |
| | "第一桶金" | 达到指定累计收入 | ★ |
| | "百万富翁" | 资产/收入达到$1,000,000 | ★★★ |
| | "千万帝国" | 资产/收入达到$10,000,000 | ★★★★★ |
| **雇佣里程碑** | "招聘狂人" | 雇佣第X名员工 | ★★ |
| | "团队壮大" | 雇佣满X名员工(不同级别) | ★★★ |
| | "全面人才" | 解锁全部9种员工角色 | ★★★ |
| **营收里程碑** | "日进斗金" | 单日收入达到指定金额 | ★★★ |
| | "周入百万" | 单周收入达到指定金额 | ★★★★ |
| **扩张里程碑** | "跨区经营" | 在2个以上区域开店铺 | ★★ |
| | "帝国雏形" | 建成第一个仓库+物流系统 | ★★★ |
| | "工业巨头" | 建成第一个工厂 | ★★★★ |
| | "五区霸主" | 曼哈顿5区全覆盖 | ★★★★★ |
| **特殊成就** | "收购之王" | 收购一家竞争对手公司 | ★★★★★ |
| | "自给自足" | 完全通过自有工厂供应链运营(不依赖进口) | ★★★★★★ |
| | "满熟练度" | 某员工熟练度达到100% | ★★★ |

> 注: 以上成就列表基于已知Steam成就子集，完整成就列表及精确解锁条件待补充。Big Ambitions仍在EA阶段，成就列表可能随版本更新调整。

### 3.2 成就与游戏进程的关系

| 游戏阶段 | 可解锁的主要成就 | 成就密度 |
|:--|:--|:--:|
| 初期(1-20h) | 首店开业、第一桶金、第一个员工 | 高(每小时约1-2个) |
| 中期(20-100h) | 百万富翁、跨区经营、全面人才、仓库建造 | 中(每5-10小时1个) |
| 后期(100h+) | 千万帝国、五区霸主、收购之王、自给自足 | 低(每10-20小时1个) |

> **设计分析**: 成就密度随游戏进程递减的设计是合理的——初期需要频繁的正反馈激励玩家度过"难熬的手动期"，中后期玩家的内在动机(自己设定的目标)替代了外在动机(成就解锁)。这与Game Dev Tycoon的成就设计相似(初期密集里程碑，后期稀疏)，但Big Ambitions的后期目标更多样化——没有一个固定的"通关"终点。

## 四、顾客满意度系统

### 4.1 满意度影响因素

| 影响因素 | 影响程度 | 优化方式 | 优先级 |
|:--|:--:|:--|:--:|
| **员工熟练度** | ★★★★★ | 培训(HR连锁培训)或雇佣高熟练度员工 | ★★★★★ 核心 |
| **店铺环境/整洁度** | ★★★★ | 定期清洁(清洁工)、良好装修 | ★★★★ |
| **商品丰富度** | ★★★ | 拓展品类、保持库存充足 | ★★★ |
| **价格合理性** | ★★★ | 适量加价(过度加价降低满意度) | ★★★ |
| **设备品质** | ★★ | 购买更高级的货架/收银设备 | ★★ |
| **装修/氛围** | ★★ | 选择更高档的装修方案 | ★★ |

### 4.2 满意度影响链

```
高员工熟练度 ──→ 快速/准确的服务 ──→ 顾客满意度↑
店铺整洁干净 ──→ 舒适的购物环境 ──→ 顾客满意度↑
商品丰富充足 ──→ 顾客一次买齐所需 ──→ 顾客满意度↑
价格公平合理 ──→ 性价比感知 ──→ 顾客满意度↑
         ↓
满意度↑ → 回头客↑ → 客单价↑ → 营收↑ → 排名↑ → 更多新客
         ↓
满意度↓ → 回头客↓ → 营收↓ → 排名↓ → 客户流失
```

### 4.3 满意度vs其他经营游戏的对比

| 对比维度 | Big Ambitions | 双点医院 | 开罗系列 | 超市模拟器 |
|:--|:--|:--|:--|:--|
| **满意度显式显示** | 否(隐形评分) | 是(显示病人满意度) | 是(各类满意度条) | 否(间接反馈) |
| **核心影响因素** | 员工熟练度+环境+价格 | 诊断准确率+等待时间+治愈率 | 设施品质+员工等级 | 结账速度+商品齐全度 |
| **满意度后果** | 回头客率→营收→排名 | 声誉→病人数量→收入 | 客流→收入 | 客流量变化 |
| **可操作性** | 间接(通过管理员工/环境) | 直接(改善诊室等待时间等) | 直接(升级设施) | 间接 |

> **品类洞察**: Big Ambitions选择"隐形满意度"而非显式数值的设计，实质上是将满意度从"一个需要被填满的进度条"转变为"一个通过经营结果间接优化的指标"。好处是更真实(真实商业中顾客满意度的确是个模糊概念)，坏处是玩家缺少了"看到满意度涨了"的即时满足感。这是"模拟仿真"与"游戏化"之间的设计取舍。

## 五、终局标准——开放式沙盒的自我目标

### 5.1 社区常见的"通关"标准

| 目标类型 | 具体目标 | 难度 | 预计游戏时长 | 说明 |
|:--|:--|:--:|:--:|:--|
| **全覆盖** | 曼哈顿5区全部业态覆盖 | ★★★ | 150-300h | 最标准的"通关"目标——在每个区开所有类型的店 |
| **排名制霸** | 全类型/全业态排名#1 | ★★★★ | 200-400h | 需要持续对抗竞争AI和区域市场饱和 |
| **营收目标** | 年收入$X百万/亿 | ★★★★ | 200-500h | 纯粹的规模追求——需要最大化所有渠道收入 |
| **自给自足** | 全品类自行制造(不依赖进口) | ★★★★★ | 300h+ | 最难的运营目标——需要建立完整的工厂生产线 |
| **完全收购** | 收购所有竞争对手 | ★★★★★ | 400h+ | 终极垄断——将AI竞争对手全部收入囊中 |
| **跨城帝国** | 曼哈顿+其他城市全覆盖 | ★★★★★★ | 500h+ | 需要完成法律/移民系统，征服多个城市 |

### 5.2 不同终局目标的体验差异

| 目标 | 核心玩法 | 管理风格 | 适合玩家类型 |
|:--|:--|:--|:--|
| 全覆盖 | 运营优化 | 注重细节和标准化 | 完美主义者/收集党 |
| 排名制霸 | 竞争博弈 | 进攻型(积极开店+压制对手) | PvE竞技型玩家 |
| 营收目标 | 效率最大化 | 数据驱动(ROI/单店产出) | 数值优化型玩家 |
| 自给自足 | 供应链管理 | 保守型(减少外部依赖) | 模拟/建造型玩家 |
| 完全收购 | 战略并购 | 激进型(金融操作) | 战略/统治型玩家 |
| 跨城帝国 | 全方位管理 | 全能型(运营+合规+扩张) | 硬核模拟玩家 |

> **设计分析**: 开放式终局是Big Ambitions"沙盒"定位的必然结果——当游戏没有预设结束条件时，玩家必须自己定义"什么时候算赢了"。社区自发形成的6种通关目标反映了玩家群体对"这款游戏最好玩的部分是什么"的不同认知。这种"千人千面的终局"是高质量沙盒游戏的标志性特征——与之类似的是《环世界》(RimWorld)和《星露谷物语》——玩家自己决定故事何时结束。

## 七、设计特点分析

1. **排名即流量——实质化的竞争压力**: 大多数经营游戏的"排名"是装饰性的(好看的数字或排行榜评分)。Big Ambitions的排名直接决定顾客流量——排名#1的店铺天然获得更多顾客。这意味着排名差异转化为了可感知的营收差异，迫使玩家认真对待竞争排名。排名不是目标，而是竞争的结果——这个结果反过来又影响竞争。

2. **隐形满意度——更接近真实的顾客管理**: 选择"隐形满意度"而非显式数值的满意度条，是Big Ambitions追求"模拟真实"而非"游戏化"的又一例证。玩家通过营收变化、顾客密度等间接指标感知满意度水平，而不是看一个数值条。这更接近真实商业中的满意度管理——你永远不知道顾客有多满意，只能从营收变化推测。

3. **开放式终局——玩家自定义成功**: 没有预设通关条件是所有高端沙盒游戏的共同特征。Big Ambitions不告诉你"什么时候赢了"，而是提供一套工具(扩张、收购、制造、排名)和随你定义的目标。这让游戏的"寿命"完全由玩家的内在动机决定——想玩多久就玩多久，想怎么赢就怎么赢。

4. **成就密度递减——从外在动机到内在动机的过渡**: 初期高密度成就提供持续的正反馈(每小时1-2个成就)，帮助玩家度过最艰难的手动期。中后期成就密度降低，玩家逐渐转向自己设定的目标(内在动机)。这种"外在动机→内在动机"的平滑过渡是长寿命游戏的共性设计——与之类似的还有《星露谷物语》的成就设计。

5. **三维排名体系——多维度竞争认知**: 全类型排名(综合实力)、分业态排名(专精度)、分区排名(区域统治力)三个维度相互独立又相互关联。玩家可能在全类型排名中游，但在电子业态排名第一(专精路线)，也可能在某区排名第一但全市排名靠后(区域深耕路线)。这种多维排名制造了比"单一总分排名"更丰富的策略叙事。
