## 一、系统概述

物流与仓储系统是Big Ambitions区别于所有同类商业经营游戏的最核心差异化机制。它**不是简单的"点按钮补货"**，而是要求玩家设计并运营一张多层级的实体物流网络——从进口商到中心仓库(Hub)、从中心仓库到专业仓库(Spoke)、从专业仓库到各零售店铺的完整配送链路。系统涉及3种后勤角色、2种配送车辆、库存安全策略和仓库间转运调度。

| 属性 | 参数 |
|:--|:--|
| 系统复杂度 | 品类最复杂——五层级供应链中的第3-4阶段 |
| 管理层级 | 进口商→中心仓→专业仓→店铺(4层) |
| 所需员工角色 | 采购代理(Purchasing Agent)、物流经理(Logistics Manager)、司机(Driver)共3种 |
| 配送窗口 | 每日凌晨2:00自动触发 |
| 车辆类型 | 大货车(8配送点/次)、面包车(4配送点/次) |
| 库存基准 | 社区建议安全库存=15天需求量 |
| 仓库类型 | Hub(中心仓)、Spoke(专业仓)共2级 |
| 存储设施 | 大储物架(8格/60空间)、托盘货架(Pallet Rack)等 |
| 转运机制 | Hub→Spoke可设定定期自动转运 |

## 二、Hub&Spoke仓库网络设计

### 2.1 网络拓扑结构

| 层级 | 仓库类型 | 功能 | 库存策略 | 位置策略 |
|:--|:--|:--|:--|:--|
| **L1 - 进货层** | 进口商/批发商 | 外部供应源，非自有资产 | N/A(外部实体) | 无需管理 |
| **L2 - 中心仓(Hub)** | 大型综合仓库 | 接收所有进口商来货，承担对下游Spoke的集中分拨 | 高库存(覆盖全品类) | 中央位置，靠近港口/主干道 |
| **L3 - 专业仓(Spoke)** | 按业态划分的专业仓库 | 按业态存储(珠宝仓/食品仓/电子仓/服装仓)，对下游店铺配送 | 按业态分区存储 | 靠近所服务店铺集群 |
| **L4 - 终端层** | 各零售/服务店铺 | 面向消费者的最终销售点 | 少量周转库存+展架陈列 | 按市场区域分布 |

### 2.2 Hub&Spoke与传统直配的对比

| 对比维度 | Hub&Spoke网络 | 传统直配(无中间仓) |
|:--|:--|:--|
| **配送效率** | 进口商→Hub→Spoke→店铺，集中运输降低单点配送成本 | 进口商→每个店铺单独配送，空驶率高 |
| **库存总成本** | 中央集中库存+分散库存，总量较高但风险分散 | 各店自持库存，缺货与积压并存 |
| **灵活性** | 高——可动态调整Spoke间的库存调拨 | 低——每店库存独立，无法快速调拨 |
| **管理复杂度** | 高——需要管理转运计划和配送分配 | 低——点对点配送，无中转环节 |
| **适用阶段** | 多店+跨区(3-4阶段起) | 1-2家店铺的初期阶段 |

### 2.3 仓库容量管理

| 存储设施 | 占用面积 | 存储容量 | 适用场景 |
|:--|:--:|:--|:--|
| 大储物架(Large Shelf) | 待补充 | 8格/60空间 | 中心仓通用存储 |
| 托盘货架(Pallet Rack) | 待补充 | 按托盘计 | 专业仓大件/批量存储 |
| 店铺托盘货架 | 店铺内 | 按托盘计 | 决定店铺可接收的送货量上限 |
| 普通货架 | 店铺内 | 按SKU排列 | 前端零售陈列 |

> **容量管理核心逻辑**: 店铺的托盘货架空间直接决定了该店能接收的配送量上限。如果仓库配送的货物超过店铺货架容量，多余的货物将无法上架。因此仓库配送计划必须与各店铺的货架容量匹配——这也是物流经理需要人工配置的核心参数之一。

## 三、物流角色链

### 3.1 角色分工与职责

| 角色 | 招聘解锁条件 | 核心职责 | 管理对象 | 对其他角色的依赖 |
|:--|:--|:--|:--|:--|
| **采购代理(Purchasing Agent)** | 完成"进口"商业课程 | 向进口商下达采购订单、谈判价格、签订采购合同 | 采购合同(品类×数量×频率) | 不依赖——独立对外采购 |
| **物流经理(Logistics Manager)** | 租用仓库后 | 设计配送路线、分配配送任务、管理仓库间转运、监控库存水位 | 配送路线表、司机任务分配 | 依赖采购代理(有货可运才有物流) |
| **司机(Driver)** | 租用仓库后 | 执行具体配送任务、驾驶车辆运输货物 | 配送车辆(面包车/货车) | 依赖物流经理(分配任务才能执行) |

### 3.2 角色链协作流程图

```
采购代理 ────── 签订采购合同 ──────→ 进口商发货 ──────→ Hub中心仓到货
     ↑                                                    │
     │                                                    │ 物流经理调度
     │                                                    ↓
     │                                            Hub→Spoke转运
     │                                                    │
     │                                              Spoke专业仓
     │                                                    │
     │                                             物流经理分配
     │                                                    ↓
     │                                            司机执行配送
     │                                                    │
     └──────────────────────────────────────────── 店铺签收 ←─┘
```

> **设计分析**: 三条角色形成了完整的"计划→调度→执行"分工链。采购代理负责"买什么、买多少"的计划决策，物流经理负责"怎么分配、走哪条路线"的调度决策，司机负责"开哪辆车、送到哪家店"的执行操作。每个角色的职位数量直接影响对应环节的吞吐能力——采购代理不够则进货跟不上销售速度，物流经理不够则配送调度滞后，司机不够则货物滞留仓库。这三者的数量比需要根据业务规模动态调整。

## 四、配送机制

### 4.1 配送时序

| 时间节点 | 事件 | 说明 |
|:--|:--|:--|
| 凌晨 2:00 | 每日配送执行 | 系统自动触发，所有已分配的配送任务开始执行 |
| 凌晨 2:00-黎明 | 配送完成 | 各司机按分配路线执行配送，货物到达各店铺 |
| 白天营业时段 | 无配送活动 | 避免配送干扰正常营业和顾客出入 |
| 配送当日 | 店铺库存更新 | 配送完成后店铺货架库存即时更新 |

### 4.2 车辆与配送能力

| 车辆类型 | 单次配送店铺数 | 单次运力 | 购置成本 | 日常费用 | 适用场景 |
|:--|:--:|:--:|:--:|:--|:--|
| **面包车(Van)** | 4个配送点 | 较低 | 待补充 | 待补充 | 小批量配送、Spoke→少量店铺 |
| **大货车(Truck)** | 8个配送点 | 较高 | 待补充 | 待补充 | Hub→Spoke大批量转运、多店配送 |

> **配送能力管理要点**: 每辆车的配送点数是物流经理分配路线时的硬约束。如果1家物流经理管理的店铺超过8家(使用大货车时)，需要规划两条配送路线或使用多辆车。路线规划的合理性和配送频率直接影响店铺库存稳定性。

### 4.3 配送优先级与缺货处理

| 情况 | 处理机制 | 连锁反应 |
|:--|:--|:--|
| **正常配送** | 按物流经理设定的配送路线和频率执行 | 无 |
| **部分缺货(Partial Stock)** | 配送现有库存，缺货部分标记为Backorder | 店铺部分商品缺货→顾客满意度下降→营业额损失 |
| **完全断供(Out of Stock)** | 配送任务失败，货物未送达 | 店铺无货可卖→全天零收入→大额利润损失 |
| **Backorder(缺货订单)** | 等待下一批采购到货后补发 | 需求积压→延迟满足→可能流失顾客 |

### 4.4 安全库存策略

| 策略 | 建议库存天数 | 适用阶段 | 风险 |
|:--|:--:|:--|:--|
| **最小库存** | 3-5天 | 资金极度紧张期 | 配送延迟即断货 |
| **标准库存(社区推荐)** | 15天 | 稳定运营期 | 库存占用资金适中 |
| **高安全库存** | 30天+ | 资金充裕+高需求品类 | 资金占用大、仓储空间需求高 |

> **骨牌效应**: Big Ambitions的缺货不是局部问题——如果中心仓缺货，所有依赖它的Spoke都会缺货，进而所有依赖Spoke的店铺都会缺货。这就是"供应链骨牌效应": 上游一个品类断供→下游整条链路瘫痪。与超市模拟器不同——超市模拟器的断货只影响单店，而Big Ambitions的断货可以沿着Hub&Spoke网络蔓延至整个商区。

## 五、仓库间转运

### 5.1 转运机制

| 转运方向 | 触发条件 | 操作方式 | 频率 |
|:--|:--|:--|:--|
| **Hub→Spoke** | Spoke库存低于设定阈值 | 自动/手动触发转运单 | 可设定定期(每日/隔日/每周) |
| **Spoke→Spoke(调拨)** | 某Spoke缺货+另一Spoke有冗余 | 手动操作 | 应急处理 |
| **Spoke→Hub(退货)** | 进货过多/业态调整 | 手动操作 | 按需 |

### 5.2 转运效率影响因素

| 因素 | 影响 | 优化方式 |
|:--|:--|:--|
| Hub与Spoke的距离 | 距离越远→转运时间越长→运输成本越高 | 合理选址(Spoke靠近所服务店铺，Hub居中) |
| 转运频率 | 频率越高→库存越新鲜→运输成本越高 | 根据品类销售速度设定(快消品高频，耐用品低频) |
| 转运批量 | 批量越大→单次运输成本越低→库存积压风险越高 | 平衡运输效率与库存周转率 |
| 车辆类型 | 大货车转运成本低于多次面包车转运 | Hub→Spoke尽量用大货车 |

## 六、与同类游戏的供应链对比

### 6.1 供应链深度对比矩阵

| 对比维度 | Big Ambitions | 超市模拟器 | 卡牌店模拟器 | 开罗系列经营游戏 |
|:--|:--|:--|:--|:--|
| **供应链层级数** | 5层(批发→进口→Hub→Spoke→店铺) | 2层(批发→店铺 / 配送→店铺) | 2层(下单→到货) | 1层(直接购买/生产) |
| **仓库层级** | 有(Hub+Spoke二级网络) | 无(直配到店) | 无 | 无 |
| **物流角色** | 3种(采购/物流经理/司机) | 无(自动补货) | 无(自动到货) | 无 |
| **缺货骨牌效应** | 有(沿Hub&Spoke链蔓延) | 有限(仅影响单店) | 无 | 无 |
| **配送路线规划** | 有(物流经理手动分配) | 无 | 无 | 无 |
| **车辆类型选择** | 有(面包车vs大货车) | 无 | 无 | 无 |
| **安全库存策略** | 有(玩家自行决定) | 无(默认定量) | 无 | 无 |

### 6.2 超市模拟器 vs Big Ambitions 供应链设计哲学对比

```
超市模拟器:
  批发商 ───→ 店铺           (管理核心: 单店库存水位)
     或
  送货服务 ─→ 店铺           (管理核心: 选择更贵的送货or更便宜的批发)

Big Ambitions:
  进口商 ──→ Hub中心仓 ──→ Spoke专业仓 ──→ 店铺   (管理核心: 整张网络的设计与优化)
                                     ↕
                                Spoke间调拨
```

> **核心差异**: 超市模拟器的供应链管理停留在"是否选择更贵的送货服务"的二元决策上。Big Ambitions将其扩展为一个需要持续投入管理精力的独立子系统——玩家不是在"管理店铺"的同时顺便管一下进货，而是在"经营一个物流公司"的同时顺便开几家店铺。这个定位差异是所有商业经营游戏中最激进的。

## 七、设计特点分析

1. **物流是独立游戏层而非附属系统**: 在绝大多数商业经营游戏中，"进货"是一个2-3步操作(选择商品→确认数量→等待到货)。Big Ambitions将其做成了一个需要多角色协作、多层级网络设计、持续监控优化的独立子系统。物流不再是为店铺服务的附属功能——它本身就是核心玩法内容。

2. **角色链设计将抽象物流具体化**: "管理物流"在传统经营游戏中是个抽象按钮。Big Ambitions通过采购代理(对外谈判)、物流经理(对内调度)、司机(执行配送)三个具体角色，将抽象的物流管理"拟人化"为3个独立的管理维度。每个维度都有可优化空间: 采购代理的合同谈判能力、物流经理的路线规划效率、司机的人数与车辆匹配。

3. **Hub&Spoke结构=自然的地理扩张约束**: Hub&Spoke不仅是物流效率工具，也是地理扩张的自然约束——在曼哈顿开第一家店不需要仓库，但扩张到更多区域时必须建立Hub和Spoke网络。这创造了"规模不经济"的真实商业现象: 小型阶段不需要物流管理，但扩张到一定规模后物流成本呈非线性增长。

4. **缺货骨牌效应制造紧迫感**: 缺货不是"某一商品缺了→等补货就行"——而是连锁式的供应链崩溃。这种设计将"库存管理"从低成本优化的被动行为，变成了需要主动防御的战略任务。社区建议的"15天安全库存"规则就是玩家社群对这一设计压力的自发回应。

5. **与真实物流系统的映射**: Big Ambitions的Hub&Spoke网络是真实世界物流(如亚马逊的FC→配送站→最后一公里配送)的游戏化简化版。采购代理对应真实企业的采购部(Sourcing)，物流经理对应供应链计划部(Supply Chain Planner)，司机对应配送团队。这种真实映射让玩家在游戏中学到的"安全库存"、"转运策略"等概念具有一定的教育价值。

## 八、优化策略建议

| 阶段 | 物流配置建议 | 核心关注点 |
|:--|:--|:--|
| **1-2店阶段** | 不需要仓库，直接从批发商采购 | 无物流管理压力 |
| **3-5店阶段(同区)** | 租1个小型仓库做Hub，面包车配送 | 学习基础配送路线规划 |
| **5-10店阶段(跨区)** | Hub+按业态分Spoke，增加大货车 | 建立Hub→Spoke转运节奏 |
| **10+店+工厂阶段** | 多Hub+多Spoke+工厂直接供应 | 全局供应链优化，减少进口依赖 |
