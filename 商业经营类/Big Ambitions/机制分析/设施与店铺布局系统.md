## 一、店铺类型总览

Big Ambitions提供多种类型的商业场所，每种类型的设施需求、运营逻辑和目标客群不同：

| 店铺类型 | 经营模式 | 核心设施 | 利润水平 | 复杂程度 |
|:--:|:--:|:--:|:--:|:--:|
| **零售店(Retail)** | 进货→上架→销售 | 货架/收银台/展示柜 | 中-高 | 低 |
| | 含礼品/服装/电子/珠宝/酒/花卉/超市等 | | | |
| **服务类(Service)** | 提供服务→收费 | 服务设备/等候区 | 中 | 中 |
| | 含理发店/健身房 | | | |
| **办公类(Office)** | 接项目→交付→收钱 | 办公桌/电脑/服务器 | 高 | 高 |
| | 含律师事务所/网页开发 | | | |
| **工厂(Factory)** | 材料→生产线→成品 | 传送带/装配线/仓库 | 高(初始成本大) | 极高 |

> **设计特点分析**: Big Ambitions的店铺类型覆盖了**零售、服务、专业服务、制造**四大商业形态，每种形态的运营流程完全不同。这不像开罗游戏那样所有店铺共享一套经营逻辑——Big Ambitions要求玩家理解不同商业模式的区别：零售店靠商品差价赚钱，服务店靠服务时间收费，办公室靠项目制收入，工厂靠制造效率。这种"业态多样性"增加了游戏的深度和重玩价值。

## 二、零售店设备系统

### 2.1 货架与展示系统

| 设备名称 | 容量 | 成本 | 适用场景 | 空间效率 |
|:--:|:--:|:--:|:--:|:--:|
| **小型货架(Shelf)** | 少(2-3格) | 低 | 小店/精选商品 | 低 |
| **大型储物架(Large Shelf)** | 多(8格/60空间) | 中 | **仓库/零售通用(最常用)** | **高(最优解)** |
| **托盘货架(Pallet Rack)** | 大量 | 中-高 | 仓库大量存储 | 高(但占用空间大) |
| **展示柜(Display Case)** | 中 | 中 | 珠宝/贵重/小件商品 | 中 |
| **收银台(Counter)** | — | 低 | 所有零售店必备 | 必需 |

> 空间效率提示: 大型储物架(8格/60空间)是性价比最高的存储方案。在同一面积内，用大型储物架比用小型货架多装2-3倍的商品。这反映了"仓库式零售"vs"精品零售"的策略选择。

### 2.2 装修与氛围系统

| 装修类别 | 可选项目 | 对顾客的影响 | 成本范围 |
|------|------|:--:|:--:|
| **地板** | 多种材质/颜色 | 影响氛围值→影响顾客满意度 | $500-$5,000 |
| **墙面** | 壁纸/油漆/装饰 | 影响氛围值 | $200-$3,000 |
| **灯光** | 基础/高级/装饰灯 | 影响氛围值(最重要) | $300-$8,000 |
| **装饰品** | 植物/艺术品/招牌 | 小幅度提升氛围 | $100-$2,000 |

> **设计特点分析**: 装修系统看似是"美化功能"，实则是**战略性投资**——良好的装修提升氛围值→提升顾客满意度→增加回头客和客单价。但装修有成本，且不能直接产生利润。玩家需要在"把钱花在装修上"和"把钱花在进货/扩张上"之间做权衡。这种"经营深度"让Big Ambitions不同于那些装修仅为视觉效果的商业游戏。

## 三、蓝图系统(独有特色)

蓝图系统是Big Ambitions**最具创新性的系统之一**——它允许玩家将一家店铺的完整布局（含设备摆放、商品配置、装修风格、定价策略）保存为"蓝图"，然后应用到其他店铺。

| 蓝图功能 | 说明 | 效率提升 |
|:--:|------|:--:|
| **保存布局** | 将当前店铺的设备位置/类型/数量保存 | — |
| **保存商品配置** | 包含商品品类/具体商品/定价信息 | — |
| **复制到新店** | 新店一键应用蓝图的全部配置 | **减少90%重复操作** |
| **多店统一管理** | 连锁店使用同一蓝图，统一形象和定价 | 管理效率大幅提升 |

### 3.1 蓝图的战略价值

```
场景A(无蓝图):
开第1家服装店 → 手动摆放设备(1小时)
→ 手动配置商品(30分钟) → 手动设定价格(15分钟)
开第2家服装店 → 重复上述全部操作(1.75小时)
→ 第5家店同样重复 → 累计时间成本极高

场景B(有蓝图):
开第1家服装店 → 手动完成布局 → 保存为蓝图
开第2家服装店 → 应用蓝图 → 微调即可完成(10分钟)
→ 第5家店 → 应用蓝图 → 完成
```

> **跨游戏对比 — 蓝图系统的独特性**

| 游戏 | 自动布局/复制功能 | 品类 | 效率提升 |
|------|:--:|------|:--:|
| **Big Ambitions** | **蓝图系统(完整复制)** | **商业经营** | **极高** |
| 双点医院 | 无(每间房需手动放置设备) | 商业经营 | 无 |
| 冒险村物语2 | 无(每个建筑逐个放置) | 商业经营 | 无 |
| RimWorld | 蓝图系统(仅建筑结构) | 生存建造 | 中等 |
| Factorio | 蓝图系统(生产流水线) | 工厂自动化 | 极高 |

Big Ambitions是商业经营品类中**唯一具有店铺级完整蓝图复制功能**的游戏。这一设计直接解决了连锁经营中的"重复劳动"痛点——在现实中，连锁店也是标准化复制的。蓝图的引入使得规模化扩张变得可行。

## 四、总部设施

当玩家开设多家店铺后，需要建立**公司总部(Headquarters)**来管理运营：

| 设施 | 功能 | 需要量 | 说明 |
|:--:|------|:--:|:--:|
| **办公桌(Desk)** | 员工工作位 | 按员工数 | 小办公桌可塞更多员工(空间效率高) |
| **电脑(Computer)** | 办公必备 | 每员工一台 | 办公室员工需要 |
| **服务器(Server)** | 数据存储/管理 | 1-2台 | 支持公司整体IT需求 |
| **会议室** | 待补充 | 待补充 | 可能用于招聘/员工会议 |
| **休息区** | 员工休息 | 建议设置 | 提升员工满意度 |

办公室的空间规划策略：
- **小办公桌布局**：小办公桌占用面积小，可以在有限空间内容纳更多员工——虽然单个员工的工作空间减少，但总产出/面积比更高
- **服务器集中放置**：服务器可以集中放置在一个房间，不必分散

## 五、仓库设施

仓库是连接"供应链"和"零售端"的中间节点。在Big Ambitions的5阶段供应链中，仓库从第2阶段开始变得重要：

| 设施 | 存储容量 | 适用阶段 | 说明 |
|:--:|:--:|:--:|------|
| **大型储物架** | 8格/60空间 | 全程(最常用) | 综合性价比最优 |
| **托盘货架** | 大量 | 中期(批量存储) | 适合大批量进货后长期存储 |
| **装卸区(Loading Bay)** | — | 必需 | 货车装卸货物 |
| **冷库(Cold Storage)** | 中 | 食品经营必需 | 存储食品/生鲜品类 |
| **办公区(在仓库内)** | — | 建议 | 仓库管理员/物流经理工作位 |

### 5.1 仓库网络架构(Hub & Spoke)

```
中心仓库(Central Hub)
    │
    ├── 专业仓库1(服装) ──→ 服装店1/2/3
    ├── 专业仓库2(电子) ──→ 电子店1/2
    ├── 专业仓库3(食品) ──→ 超市1/2/3 + 便利店1
    └── 专业仓库4(综合) ──→ 礼品店/花店
```

> **设计特点分析**: Hub&Spoke仓库网络是Big Ambitions对"供应链管理"的游戏化呈现。玩家需要决定中心仓的位置、各专业仓的品类分配、配送路线优化。这一系统的深度足以支撑一个完整的"物流管理"游戏——实际上它是从第2阶段供应链到第5阶段的持续优化过程。这种"供应链可视化管理"在商业经营品类中几乎是独一无二的——大多数同类游戏要么简化了供应链（如开罗系列只有"买→卖"两步），要么完全省略了物流环节。

## 六、选址策略综合指南

### 6.1 选址与业态匹配矩阵

| 业态 | 最佳区域 | 最小面积 | 停车需求 | 装修需求 |
|:--:|:--:|:--:|:--:|:--:|
| **服装店** | 上东区/中城 | 中等 | 中 | 高 |
| **电子产品** | 中城/金融区 | 中等 | 低(目标客群多为通勤) | 中 |
| **珠宝店** | 上东区 | 小(精品) | 中 | 极高 |
| **花卉店** | 中城/居民区 | 小 | 低 | 中 |
| **超市** | 居民区 | 大 | 高 | 低 |
| **便利店** | 金融区/交通枢纽 | 小 | 低 | 低 |
| **理发店** | 居民区/中城 | 小 | 中 | 中 |
| **健身房** | 中城/金融区 | 大 | 中 | 中 |
| **律师事务所** | 金融区 | 中 | 低 | 高 |
| **工厂** | 工业区/外围 | 极大 | 高 | 极低 |

### 6.2 租金优化策略

| 策略 | 适用场景 | 优点 | 缺点 |
|------|:--:|:--:|:--:|
| **高客流高租金** | 初期快速盈利 | 流量大→销量高 | 租金压力大 |
| **中客流中租金** | 稳健经营 | 平衡的ROI | 需要较长回本周期 |
| **低客流低租金** | 工厂/仓库 | 空间大成本低 | 不适合零售 |

## 八、设计特点分析总结

Big Ambitions的设施与布局系统体现了几个核心设计理念：

1. **业态差异化**: 不同商业类型(C端零售/B端服务/办公室/工厂)的布局逻辑完全不同，不是"换皮"
2. **空间即策略**: 如何在一个有限的空间内最大化利用——用哪种货架、怎样布局动线、收银台放哪里——都是需要考量的经营决策
3. **标准化复制**: 蓝图系统使连锁店经营从"重复劳动"变为"标准输出"，与现实中的连锁商业模式对齐
4. **物流可视化**: 仓库→配送→店铺的物理链条让"供应链"不再是抽象概念，而是可见可管理的物理系统
5. **渐进复杂度**: 玩家从单独的零售店起步→进而需要总部→仓库→工厂，设施复杂度随商业规模同步增长，自然引导学习曲线
