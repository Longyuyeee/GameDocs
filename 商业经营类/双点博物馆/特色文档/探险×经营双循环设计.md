# 探险×经营双循环设计 — 双点博物馆

> 双点博物馆最核心的机制创新——将传统经营游戏的"确定性的产品制造"替换为"概率性的探险发现"。

---

## 一、双循环架构

### 循环图

```
                    ┌─── 探险端 ───┐
                    │              │
    资金支持 ──→ 直升机停机坪 → 选择POI → 派遣团队(N天)
       ↑                                    ↓
       │                              探险事件/风险
       │                                    ↓
       │                              获得展品+资源
       │                                    │
       │              ┌────────────────────┘
       │              ↓
   ┌────────── 经营端 ──────────┐
   │                            │
   │  布置展品 → 装饰 → Buzz↑   │
   │     ↓                     │
   │  游客参观 → 捐赠/消费      │
   │     ↓                     │
   │  收入→门票+捐赠+礼品店+赞助 │
   │     ↓                     │
   │  再投资→扩建/培训/探险 ────┘
   └──────────────────────────┘
```

### 两端的节奏对应

| 探险端 | 经营端 |
|------|------|
| 派遣探险(瞬间) | 等待探险返回期间持续经营 |
| 探险进行中(N天) | 布置上一批展品+应对日常运营 |
| 探险队返回(事件) | 收到新展品→兴奋峰值 |
| 展品劣化(持续) | 专家维护→再派遣探险获取新品 |

## 二、与传统经营游戏的对比

### 产品获取模式对比

| 游戏 | 产品获取方式 | 确定性 | 等待感设计 |
|------|------|:--:|------|
| **游戏开发物语** | 选择类型+题材→确定产出 | 100%确定 | 开发周期(固定) |
| **双点医院** | 病人进院→诊断→治疗 | 确定性流程 | 诊断链长度 |
| **温泉物语** | 建造设施→固定效果 | 100%确定 | 建造瞬间完成 |
| **双点博物馆** | **派遣探险→概率获取** | **不确定** | **探险天数(异步等待)** |

### 不确定性的设计价值

1. **"开箱"快感嵌入经营循环**：每次探险队返航是一个微型"开箱"时刻——你不知道这次会带回什么品质的展品。这种期待感是确定性制造无法提供的。
2. **延长内容寿命**：要收集到完美品质的特定展品可能需要10-20次探险——但每次都有"万一这次出完美"的期待。
3. **多部件收集=进度可视化**：即使这次没出完美品质，收集到恐龙骨架的第三个部件也是明确的进展。

## 三、探险参数设计

### POI分级设计

| POI等级 | 探险天数 | 费用 | 风险 | 品质预期 |
|:--:|:--:|:--:|:--:|:--:|
| 1-2级 | 5-8天 | 低($500-1,000) | 低 | 普通为主，偶出优秀 |
| 3-5级 | 8-15天 | 中($1,000-5,000) | 中(需生存技能) | 优秀常见，偶出史诗 |
| 6-8级 | 15-25天 | 高($5,000-15,000) | 高(需多技能) | 史诗常见，偶出完美 |
| 9-10级 | 25-40天 | 极高($15,000-25,000) | 极高(需满技能) | 完美出现率最高 |

### 调查等级机制

- **渐进式提升**：每个POI有独立的调查等级(1-10)
- **品质关联**：调查等级3→优秀保底；调查等级7→史诗几率大幅提升；调查等级10→完美几率最高
- **投入回报**：10次初级探险 vs 3次高级探险(时间成本相近)→后者品质期望更高

## 四、探险风险与技能博弈

### 事件×技能矩阵

| 探险事件 | 所需技能 | 无技能后果 | 有技能效果 |
|------|------|------|------|
| 攀爬事故 | **生存技能** | 员工受伤→长时间治疗 | 避免事件/+25%绷带恢复 |
| 直升机故障 | **飞行员之翼** | 探险延迟/展品损坏 | 避免事件/+5%速度 |
| 文物破损 | **快速修复** | 展品降品质 | +100%修复速度 |
| 毒气/毒害 | **生存技能** | 员工中毒→治疗 | 免疫 |
| 恶劣天气 | **飞行员之翼** | 探险延期 | 不受影响 |

### 设计意图

事件的技能门槛创造了一个自然的进度门：低技能=只能探索安全的初级POI→获得普通品质展品→Buzz提升慢→升级慢。这是一种比"等级锁"更优雅的进度限制。

---

## 五、异步等待的节奏设计

探险期间(5-40天)，经营端持续运转——这是该双循环最具设计智慧的地方：

- **等待≠无聊**：你有充足的时间重新布置展品、调整员工分区、优化装饰评级、处理突发事件
- **多任务并行**：一个专家在探险时，其他专家仍在馆内维护展品
- **多博物馆并行**：A馆的探险队在外面跑，B馆的运营照常进行

这种"探险=后台运行的长期任务"设计，将经营模拟从"盯着看下一个产品什么时候好"的等待感中解放出来。

---

## 六、设计总结

双点博物馆的探险×经营双循环给商业经营品类的一个重要启示是：**产品获取端可以有不确定性，且这种不确定性可以增强而非削弱游戏体验**。关键在于：

1. 不确定性的范围要可控(调查等级提供保底)
2. 等待要有意义(探险期间的经营不中断)
3. 失败要有补偿(重复探险提升调查等级=不是白跑)
4. 风险要有博弈(决定是否派最优秀专家→影响探险品质但影响馆内维护)

---

> 参考: 类似的"外出获取资源→带回基地经营"双循环也出现在夜勤人(地下城→商店)、潜水员戴夫(潜水→寿司店)中，但博物馆的探险端更深(调查等级/事件/技能要求/多部件收集)，且与经营的耦合更紧密。
