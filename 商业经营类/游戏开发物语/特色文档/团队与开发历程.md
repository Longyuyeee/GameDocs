# 团队与开发历程 — 游戏开发物语

> 开发商: 开罗社(Kairosoft Co., Ltd.)
> 发行年份: 1997(Windows日本) → 2010(iOS/Android全球) → 2022(Steam) | 平台: PC/Steam/Switch/PS4/Xbox/iOS/Android | 团队规模: ~9人(初创)/~26人(2025)

### 开罗社的开发哲学

开罗社是一个极其特殊的游戏开发商：
- **永不扩大团队**(26人维持了十余年)——刻意保持小团队以维持创意控制
- **同一套引擎+同一套公式**——90+款游戏共享核心代码和设计DNA
- **不融资、不上市**——臼井和之坚持独立运营，拒绝外部资本介入
- **移动端→Steam的10年大迁移**——2021年后大量将手机游戏移植到Steam(Switch/PS4/Xbox)

### 关键决策

1. **从Windows到iOS的转型(2010)** — 当日本PC游戏市场萎缩时，臼井选择了正在崛起的iPhone平台。这个决定让开罗从一个默默无闻的日本小团队变成了全球知名的"像素经营游戏"品牌。

2. **"不改变公式"的坚持** — 开罗20多年来坚持像素画风+深度数值的公式，从未跟风3D化或氪金化。在手游行业全面走向F2P+抽卡的时代，开罗坚持买断制。

3. **Steam大迁移(2021-2023)** — 在移动端市场日趋饱和后，开罗开始将全部90+款游戏移植到Steam。Game Dev Story是其中最早移植的几款之一。

## 3. 技术与工具

- **引擎**: 开罗社自研引擎(基于C++/Java，具体技术栈未公开)
- **美术**: 像素画风，美术资源高度可复用(同一套像素Asset在90+款游戏中混搭使用)
- **音频**: 简单电子风BGM，非重点投入领域
- **团队协作**: 极简流水线——设计→编程→美术→测试，小团队全栈制

## 5. 设计启示

### 对独立游戏开发者的启示

1. **"组合公式"是可迁移的设计模式** — Game Dev Story的类型×题材组合公式被开罗成功迁移到漫画物语(题材×画风)、拉面店(汤底×面条×配料)等90+款游戏中。找到一个好的核心公式后，更换"参数"比创造新系统更高效。

2. **26个人做90+款游戏的秘密** — 代码和设计模式的**超高复用率**。开罗不是"做90款不同的游戏"，而是"为同一套系统配置90套不同的参数表"。

3. **不去追3D，坚持像素** — 在3D泛滥的时代，开罗用像素画风省钱、省时、省人，同时建立了独特的品牌辨识度。

### 对商业经营品类的参考价值

- **"简化的复杂度"** — Game Dev Story提供了商业经营品类的一种极端成功公式：系统复杂度保持在一个下午能通关的水平，但系统深度让玩家反复玩10次还能发现新东西。
- **阶段门设计** — 用几个大额消费门槛划分发展阶段，是2-8小时轻量经营游戏的最佳节奏设计。
