## 一、组合公式的结构

```
产品品质 = 员工属性基础值 × 类型×题材默契度加成 × 随机爆发 × Boost加成
```

其中**类型×题材默契度**是整个公式的核心——它定义了某类型(玩法)与某题材(内容)的"天然适配性"。

### 五个默契等级

| 等级 | 评价文字 | 基础销售额倍率(约) | 数量(约) |
|:--:|------|:--:|:--:|
| Lv1 | 绝佳!(Amazing!) | 3.0-4.0x | 80+ |
| Lv2 | 有创意(Creative) | 2.0-2.5x | 150+ |
| Lv3 | 还不错(Not Bad) | 1.2-1.5x | 200+ |
| Lv4 | 嗯...(Hmm...) | 0.7-0.9x | 150+ |
| Lv5 | 不怎么样 | 0.3-0.5x | 100+ |

## 二、为什么这套公式有效（设计分析）

### 2.1 "半隐藏规则"驱动玩家社区

类型×题材的默契度不是直接显示在UI上的——玩家必须在开发前"试"才能知道是"绝佳！"还是"不怎么样"。这种**半隐藏信息**产生了三个效应：

1. **好奇心**: "这个组合会不会是绝佳？"(→尝试的动机)
2. **记忆游戏**: "我记得RPG+幻想是绝佳，RPG+飞机不行"(→玩家的隐性知识积累)
3. **UGC爆发**: 玩家自发整理"最佳组合表"发布在论坛/Wiki/Steam社区(→产生了游戏之外的社区内容)

> **对比**: Game Dev Tycoon的组合加成是隐式的(影响评分但不显示文字评价)，玩家无法直观感知"这个组合好不好"。游戏开发物语的5级文字反馈让玩家**知道自己走对了还是走错了**——这是关键的设计差异。

### 2.2 模拟是最"通用"的类型

绝佳！组合最多的类型是**模拟(Simulation)**——有16个题材符合"绝佳！"级别。这不是随机设计的——游戏的"模拟"本来就是最灵活的类型(可以模拟赛马/恋爱/电车/书店/游戏公司等几乎所有题材)，所以它拥有最多的适配题材。而RPG只有4个绝佳题材(幻想/恐怖/武少女/蘑菇)，因为"不是所有题材都适合做成RPG"。

> 开罗在这个维度上不是在做"游戏平衡"，而是在做**"常识模仿"**——模拟游戏确实在现实中更灵活，RPG确实更受限于题材。这种"与现实的对应"让玩家更容易直觉理解系统。

### 2.3 "首次尝试"加成鼓励探索

玩家第一次尝试一个新组合时，获得额外的方针点数奖励。这个设计让"实验新组合"比"反复用已知的最优组合"有短期收益——避免了玩家永远只用"模拟+恋爱"一个组合。

## 三、从Game Dev Story到开罗全系列：一个公式的迁移

```
Game Dev Story:    类型 × 题材 = 游戏评分
漫画物语:           题材 × 画风 = 漫画评分
杂志物语:           栏目 × 主题 = 杂志销量
开罗拉面店:          汤底 × 面条 × 配料 = 拉面评分
冒险村物语2:        建筑类型 × 居民职业 = 村庄发展
```

开罗社将"组合→评分→收入"这个**元公式**应用到了几乎所有游戏中。这解释了为什么26人的团队能产出90+款游戏——每款新游戏本质上是：
1. 选一个新的产业/主题
2. 定义该产业的"组合轴"(A×B)
3. 填入产业相关的数据表
4. 复用同一套培训/升级/评分框架

## 四、"组合"设计的通用规律

从开罗公式中可以总结以下规律：

| 规律 | 说明 | 例 |
|------|------|:--:|
| **少量"绝佳"→锚定价值** | 最多的绝佳组合也仅占总组合的10-15%，稀缺性维持"绝佳！"的成就感 | 模拟16/200+ |
| **大量"还不错"→兜底** | 大多数组合不至于"不怎么样"，让玩家不至于频繁受挫 | 200+组合 |
| **少量"垃圾组合"→惩罚错误** | 故意设置少量糟糕组合，让"试"有风险 | RPG+飞机 |
| **一个"万能类型"→新手友好** | 模拟是万能类型，保证新手即使随便试试也能找到路子 | 16个绝佳题材 |
| **"半隐藏"→触发好奇心** | 不直接展示所有最佳组合→驱动实验→产生社区UGC | — |

## 五、对游戏设计的启示

1. **"组合×评价"公式是商业经营品类的核心设计模式** — 不仅是开罗，从星露谷的"种子品质×加工方法"到双点医院的"病症×诊断链×治疗室"，商业经营游戏的深度往往来自于"将几个独立变量组合产生非线性结果"。

2. **半隐藏信息是社区驱动型游戏的最强杠杆** — 开罗没有花一分钱在社区运营上，但玩家自发为90+款游戏整理了完整的组合表。这种"让玩家有东西可发现、可整理、可分享"的设计，比任何社区经理都高效。

3. **公式的参数化 = 产品线的规模化** — 开罗不是"做了90款不同的游戏"，而是"将一套公式配置了90套不同的参数"。这是小团队高产出的终极设计方法。
