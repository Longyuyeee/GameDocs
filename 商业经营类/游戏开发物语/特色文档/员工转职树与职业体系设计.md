## 一、转职树全景

```
                         程序 Lv.5
                         /
        初始(4基础职业) —— 脚本 Lv.5
                         \
                         图像 Lv.5
                         /
                         音乐 Lv.5

        总监 = 程序Lv.5 + 脚本Lv.5
        制作人 = 图像Lv.5 + 音乐Lv.5
        硬件工程师 = 程序+脚本+图像+音乐+总监+制作人 全Lv.5
        超级黑客 = 硬件工程师 Lv.5
```

## 二、每个职业的设计意图

| 职业 | 设计意图 | 在团队中的角色 |
|------|------|------|
| 程序员 | 全阶段基础加成，最"泛用" | 编程阶段主力 |
| 脚本家 | 企划阶段品质保证 | 企划阶段主力 |
| 设计师 | 原画阶段品质保证 | 原画阶段主力 |
| 音乐家 | 作曲阶段品质保证 | 作曲阶段主力 |
| 总监 | 减少工位需求(1人做2人工作) | 企划+编程二合一 |
| 制作人 | 减少工位需求(1人做2人工作) | 原画+作曲二合一 |
| 硬件工程师 | 解锁终极系统(主机研发) | 主机开发 |
| 黑客 | 终极全能 | 什么都能做 |

## 三、为什么转职树是游戏开发物语最好的设计

### 3.1 "需要前瞻规划"

转职树不是"到了Lv.5就转"的自动流程——转职有代价。从设计师转到硬件工程师会**大幅降低图像和音乐属性**。如果你在转职前没有先把图像和音乐升到满意水平，转职后可能需要花数十倍的时间培训补回。这意味着玩家在Lv.1时就需要决定"这个人将来要做硬件工程师吗？如果要做，我先不着急转总监。"

> **对比**: Game Dev Tycoon没有职业系统——员工就是"程序员/设计师/音乐家"各一个槽位。转职树让游戏开发物语在不增加复杂度(仍是4-8人)的情况下，创造了远更深度的员工培养策略。

### 3.2 "隐藏角色的职业是捷径"

Kairo君自带所有职业满级，猩猩G-Force自带硬件工程师。这意味着玩家有两种路径：
- **慢路径**: 自己培养一个硬件工程师(需要6张转职卡+大量培训)
- **快路径**: 解锁隐藏角色(需要3-5次金奖+第14年后)

两种路径的"时间成本"大致相当——这是精心校准过的设计平衡。

### 3.3 "作为工具的员工"vs"作为人的员工"

游戏开发物语对员工的定位是极度工具性的——员工之间除了属性不同和名字梗(NPC名字大多是游戏行业的 pun)外，没有个性、没有情感、没有个人故事。这与培养模拟类(如火山的女儿)形成了最极端的对比。

> **设计洞察**: 商业经营品类的"员工"通常是工具性的(Software Inc.的员工有独立个性和行为AI是例外)，培养模拟品类的"角色"是情感性的。这是两个品类最根本的差异之一。

## 四、转职系统的数值张力

| 张力 | 描述 |
|------|------|
| **专精 vs 全能** | 程序特化Lv.5→总监→黑客的路径 vs 维持基础职业深挖单项属性 |
| **硬件工程师的代价** | 获得主机研发能力的代价是丢失图像/音乐能力——是否值得？ |
| **提前规划 vs 灵活应变** | 前期决定一个人的终极职业路线 vs 根据人才市场灵活调整 |
| **工位利用率** | 8人工位中总监+制作人各1人可省2个工位，但需要更多转职书 |

## 五、在其他开罗游戏中的演变

| 游戏 | 职业系统 | 与Game Dev Story的关系 |
|------|------|------|
| 温泉物语 | 无转职，只有线性升级 | 更简单的"员工变强"模型 |
| 冒险村物语2 | 居民职业(战士/法师等)影响战斗力 | 将"职业"从经营工具变为战斗能力 |
| 漫画物语 | 助手→漫画家的等级提升 | 无分支转职的线性版本 |
| 开罗拉面店 | 厨师等级+专精方向 | 单项深挖替代分支转职 |

> 游戏开发物语的6线转职树是开罗全系列中**最复杂的职业系统**——其他游戏要么是纯线性无分支，要么是简单的2选1。这可能是因为"游戏开发"这个主题天然适合"多种职业"的叙事(游戏行业确实有程序/策划/美术/音乐四种职能)。
