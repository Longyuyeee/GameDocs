### 广告疲劳机制

**同一广告类型重复使用会效果递减**——这是游戏开发物语营销系统的核心设计。例如：
- 第1次用杂志广告: 知名度+10
- 第2次连续杂志广告: 知名度+6
- 第3次连续杂志广告: 知名度+3
- ...趋近于零

> **设计分析**: "广告疲劳"是一个优雅的软限制——不禁止你反复用最便宜的广告，但让效果逐渐无效化。它迫使玩家要么**轮换广告**(每次换一种)，要么**等衰减结束后再用**(随时间恢复)。这种机制在现实中也有对应——同一广告Campaign的边际收益递减。

### 广告时机

| 投放时机 | 持续效果 | 最佳策略 |
|------|:--:|------|
| **发布前** | 增加首周知名度(首周销量) | 必投 |
| **发布后** | 延长销售长尾 | 评级高的游戏再投，评级低的不浪费 |

## 三、知名度(Hype)机制

知名度是介于0-100的值：

| 知名度来源 | 上限 |
|------|:--:|
| 广告投放(衰减制) | 约92% |
| Gamedex参展 | 可突破广告上限 |
| 获得金奖/GOTY | 直冲100% |
| 粉丝信/口碑 | 缓慢提升 |

知名度直接影响**首周销量**——知名度100时新游戏的首周销量是知名度0时的数倍。

## 四、竞争系统

游戏开发物语**没有显式的AI竞争对手**。这在商业经营品类中是一个有趣的选择：

| 设计选择 | 效果 |
|------|------|
| 无AI竞争对手公司 | 玩家的"敌人"是门槛(评分/GGA/破产)而非竞争者 |
| 有行业"环境"(GGA提名作品) | 暗示了其他公司的存在，但从不显式出现 |
| 有平台竞争(主机市场份额) | 不同主机平台的竞争是间接的——你开发游戏给哪个平台 |

> **设计分析-对比Game Dev Tycoon**: Game Dev Tycoon有隐式竞争——你的游戏评分需要超越行业基准。但游戏开发物语连这个都没有——你只需"比自己的上一款做得好"。这种**无外部竞争**的设计让游戏更"温暖"——失败的焦虑来自自身(要破产了/时间不够了/拿不到金奖)而非他人。在2-3小时的轻量经营游戏中，这比引入竞争压力更适合"放松+满足"的情感目标。
