## 一、时间结构

### 基本时间单位

| 单位 | 内容 |
|------|------|
| **1周** | 最小行动单位。每周可执行多种行动(开发/培训/合同等) |
| **1月** | 4周。每月有固定事件(第2周商人出现/第1周Gamedex等) |
| **1年** | 12个月(1月-12月)。每年3月发薪；7月Gamedex；12月GGA颁奖 |
| **游戏总长** | 21年(第21年第1月末结算最终成绩) |

### 时间流速
- 游戏为回合制，时间按"周"为单位前进(非实时)
- 开发一款游戏大约需要数周至十几周(取决于员工速度和是否外包)
- 培训员工消耗1周/次(可连续培训)

## 二、年度事件日历

| 月份 | 事件 | 策略意义 |
|:--:|------|------|
| **3月** | **发薪日**(部分版本4月) | 年度最大支出，第1年政府补贴免薪 |
| **5月第2周** | **商人出现** | 唯一购买转职书/道具的机会，错过等一年 |
| **7月第1周** | **Gamedex游戏博览会** | 提升知名度，吸引新粉丝。第1办公室后才出现 |
| **12月** | **GGA全球游戏大奖** | 年度评选。金奖(游戏最高奖)/GOTY(年度最佳)/优秀音乐奖/优秀美术奖 |
| **全年** | 粉丝来信、合同邀请、新主机授权机会 | 随机事件维持新鲜感 |

## 三、压力机制

### 时间压力

| 压力源 | 强度 | 后果 |
|------|:--:|------|
| **21年终点** | ★★★★★ | 第21年1月末结算后游戏结束(可继续自由游玩但不再结算) |
| **合同DDL** | ★★★ | 未按时完成合同→损失报酬+声誉下降 |
| **错过商人** | ★★★ | 每年仅5月第2周出现，错过=再等一年买不到转职书 |
| **错过Gamedex** | ★★ | 可跳过参展，但知名度提升机会永久失去 |

### 财务压力

| 压力源 | 强度 | 峰值时间 |
|------|:--:|------|
| **发薪日** | ★★★★(早期)/★(后期) | 第2-3年(尚无稳定收入时) |
| **办公室升级门槛** | ★★★★ | 第4年1月(需18,000G储备)；第4年10月(需更多) |
| **精英签约金** | ★★★ | 好莱坞招募期 |

### 竞争压力

游戏开发物语**没有显式的竞争AI**(竞争对手公司)。但存在**隐含竞争**——GGA颁奖在每年12月，入围作品来自整个"游戏行业"。这是一种"没有竞争对手实体，但有竞争结果"的设计。

## 四、时间管理的节奏设计

### 前4年的紧张感
```
第1年: 合同外包求生存 + 招聘 + 初次尝试游戏开发
第2年: 囤积转职书 + 第一款殿堂作品
第3年: 攒钱(目标>20,000G) + 拿GBA授权
第4年: 1月搬6人办公室 → 10月搬8人办公室(分水岭)
```

### 中期(5-10年)的效率优化
```
冲击金奖 → 招募精英 → 培养硬件工程师 → 续作链滚雪球
```

### 后期(11-20年)的终局冲刺
```
终极主机研发 → 霸占GOTY → Kairo君解锁 → 完美收官
```

## 五、目标与失败条件

| 类型 | 内容 |
|------|------|
| **短期目标** | 游戏进入殿堂(评分≥32)、获得金奖、招聘精英员工 |
| **中期目标** | 搬入第3办公室、研发自社主机、获得GOTY |
| **长期目标** | 在第21年结算时达成行业霸主 |
| **失败条件** | 破产(资金为负且无法偿还) |
| **软失败** | 21年结算时成绩平平(非正式失败，但玩家通常视为"打得不好") |

## 六、"压缩时间"设计

> **设计分析**: 21年的游戏总长在现实时间中大约2-8小时(取决于游戏速度设置)。开罗通过以下手段"压缩"时间感：
>
> 1. **加速按钮**: 开发/培训可在数秒内完成
> 2. **每周同时发生多件事**: 员工培训+合同外包+游戏开发可同步进行
> 3. **跳过非关键时间**: 无事发生时直接下一周
>
> 这种"时间压缩"让21年的历程可以在一个下午走完——保留了"长期经营"的叙事满足感，消解了"等待"的枯燥。与星露谷物语的"每一天都要亲自过"(不跳过)形成对比：前者是**CEO的时间**（只看关键决策），后者是**农场主的时间**（每一天都重要）。
