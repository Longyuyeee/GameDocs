## 一、产品结构：游戏=类型×题材

游戏开发物语的核心产品系统是最简洁的**二维矩阵**：

```
产品 = 类型(Genre) × 题材(Type/Content)
```

| 维度 | 数量(约) | 解锁方式 | 示例 |
|------|:--:|------|------|
| **类型(Genre)** | 30+ | 员工职业等级提升解锁 | RPG、模拟、动作、冒险、射击、竞速、养成、线上RPG... |
| **题材(Type)** | 50+ | 员工职业等级提升解锁 | 幻想、恐怖、忍者、恋爱、赛马、机器人、漫画、电影... |

> **设计分析**: 二维矩阵是视频游戏行业最自然的抽象——类型=玩法框架，题材=内容包装。开罗没有选择更复杂的N维系统(如Game Dev Tycoon的引擎/平台/类型/题材/目标受众)，而是用最简单的方式让玩家能直观理解。这种"可理解的复杂度"是开罗在全球成功的核心原因。

## 二、开发流程（四阶段制作）

每款游戏的开发分为四个阶段，每个阶段由一个员工负责：

| 阶段 | 负责属性 | 员工角色 | 说明 |
|------|:--:|------|------|
| **1. 企划(Proposal)** | **脚本(Script)** | 脚本家/总监 | 决定趣味性和独创性的基础值 |
| **2. 原画(Art)** | **图像(Graphics)** | 设计师/图像师 | 决定画面数值 |
| **3. 作曲(Sound)** | **音乐(Music)** | 作曲/音乐家 | 决定音乐数值 |
| **4. 编程(Coding)** | **编程(Program)** | 程序员/黑客 | 决定日常开发的基础加分数值，贯穿全过程 |

### 关键机制

#### 疲劳/重复惩罚
- **同一员工连续两次承担相同阶段** → 效率大幅下降(游戏内提示"效率很低")
- **同一类型连续三款游戏** → 粉丝流失(知名度下降)

#### 怒气爆发(Inspiration Burst)
- 开发过程中随机触发
- 爆发时将员工对应属性的数值**乘以爆发倍率**加到游戏对应分数上
- 高体力员工触发概率和持续时长更高

#### 研究数据/Boost
- 在开发阶段可使用研究数据(Research Data)执行Boost
- Boost固定提升本阶段分数，成功率最高80%
- 失败产生的Bug可以"修复"来获取额外研究数据——这是一个刻意的**正反馈循环**设计

#### 外包(Outsourcing)
- 可购买外包加速开发
- 消耗金币，减少开发周期
- 适用于财力充裕后的冲刺

## 三、品质/评分模型

每款游戏有四个关键属性：

| 属性 | 英文 | 决定因素 | 作用 |
|------|------|------|------|
| **趣味性** | Fun | 编程属性(编程阶段)+脚本属性(企划阶段)+爆发加成 | 决定游戏评分和销量的最重要因素 |
| **独创性** | Creativity | 脚本属性(企划阶段)+爆发加成 | 影响媒体评分和奖项 |
| **画面** | Graphics | 图像属性(原画阶段)+爆发加成 | 影响画面评分和优秀美术奖 |
| **音乐** | Sound | 音乐属性(作曲阶段)+爆发加成 | 影响音乐评分和优秀音乐奖 |

### 最终评分

游戏完成后，四家媒体给出1-10分的评分：
- **殿堂级(Hall of Fame)**: 总评≥32分(满分40)
- **年度最佳(Game of the Year)**: 总评≥36分 + 四项总分>800

> **设计特点**: 四个属性的"不对称重要性"——趣味性是销量的核心驱动力，而画面和音乐主要影响奖项。这种不对称性让员工培养有策略纵深：是做"全才团队"还是"极端特化"。

## 四、类型×题材组合系统（核心公式）

这是游戏开发物语最重要的设计机制。每个类型和题材的组合有不同的**默契度**：

| 默契度 | 效果 | 数量(约) |
|:--:|------|:--:|
| **绝佳!(Amazing!)** | 最高基础销售额+最高编辑评分潜力 | 80+组合 |
| **有创意(Creative)** | 中等基础销售额 | 150+组合 |
| **还不错(Not Bad)** | 一般基础销售额 | 200+组合 |
| **嗯...(Hmm...)** | 低基础销售额 | 150+组合 |
| **不怎么样(Not Good)** | 最低，建议避免 | 100+组合 |
| **无反应** | 无任何组合加成 | 未列入上述的所有组合 |

### 顶尖组合示例

| 类型 | "绝佳!" 题材 |
|------|------|
| **RPG** | 幻想、恐怖、武少女、蘑菇 |
| **模拟** | 赛马、恋爱、电车、书店、漫画、电影、游戏公司、城市建设、美少女、卡通、明星、蘑菇、摩托车、足球、F1 |
| **动作** | 忍者、相扑、武少女、恐怖、历史剧、格斗、篮球 |
| **冒险** | 解秘、侦探、时间旅行、卡通 |
| **射击** | 机器人、战争 |
| **竞速** | 摩托、F1 |
| **声音小说** | 美少女、电影、侦探、恐怖 |
| **养成** | 动物、偶像、卡通、足球、漫画 |
| **线上SLG** | 建筑、便利店、游戏公司、股票、偶像、蘑菇、赛马、游泳 |
| **体感游戏** | 健美、卡通、偶像、拳击、舞蹈、摔跤、滑冰、滑雪 |
| **桌面游戏** | 麻将、披风 |
| **动作RPG** | 狩猎、披风 |
| **线上RPG** | 中世纪、欧洲 |

> **设计分析-对比Game Dev Tycoon**: Game Dev Tycoon的评分公式更复杂——引擎(Technology)+平台适性+类型/题材+员工技能+研发投入的加权组合。而游戏开发物语用最简单的"组合默契度"五个等级来替代。事实证明，**简单的组合表比复杂的公式更容易让玩家产生"我记得哪个组合好用"的记忆**，这是开罗胜过Game Dev Tycoon的核心设计洞察。

### 首次尝试加成

当玩家第一次使用某个类型×题材组合时，会获得"首次尝试"的方针点数加成奖励——这个设计鼓励玩家实验新组合而非反复使用已验证的最优组合。

## 五、续作(Sequel)系统

- **触发条件**: 游戏进入殿堂(Hall of Fame, 总评≥32)
- **继承机制**: 续作继承前作的部分属性加成，起点高于全新IP
- **系列延续**: 续作再入殿堂→可做第三代；续作未入殿堂→系列终结
- **设计价值**: 续作系统是玩家从"单作思维"到"IP经营思维"的过渡——你开始考虑"这个游戏要不要留到能力更强时再做续作"

## 六、研发流程的节奏设计

```
选择类型×题材组合
    ↓
分配四阶段负责人(注意疲劳)
    ↓
开发进行中→随机爆发→使用Boost
    ↓
Bug出现(可选:修Bug换研究数据)
    ↓
完成→评分→销售
    ↓
殿堂级? →是→可做续作
         →否→总结经验，提升员工培养水平
```

> **跨品类对比**: 与星露谷物语的"作物→加工品"品质升级链不同，游戏开发物语的品质升级链是"员工培养→属性提升→游戏评分提升→殿堂级→续作加成"——本质上是将**人的培养**转化为**产品的品质**，而非将**原料加工**转化为**产品品质**。这种"人=生产资料"的模型与牧场经营有本质差异。
