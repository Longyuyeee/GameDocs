# 过山车编辑器：工程、艺术与游戏的融合

> 过山车之星的过山车编辑器可能是经营模拟游戏史上最深度的"产品创造"工具——它同时是工程模拟器、3D建模工具、和游戏机制的核心引擎。

---

## 一、编辑器的三层设计

### 第一层：工程层——轨道物理

编辑器中的**逐段轨道放置**本质上是工程问题：

- 每段轨道有独立的俯仰(Pitch)、偏航(Yaw)、滚转(Roll)参数
- 轨道的曲率半径和速度决定了作用于虚拟乘客的**G力**
- G力→转化为兴奋度/恐惧感/恶心度三项指标
- 这是真实的物理模拟，不是预设的数值配方

### 第二层：艺术层——视觉创造

- 轨道可以穿越地形、环绕景观、飞跃水面
- 隧道的灯光和内部装饰
- 轨道本身的外观材质和颜色
- 与"过山车照片"的结合(拍照点)

### 第三层：游戏层——经济产出

- 建设投入：建造时间和金钱成本
- 运营产出：Prestige→可定价范围→收入
- 长期回报：Classic→永恒吸引力

## 二、EFN三指标——工程优化的游戏化

### 指标关系图

```
增加倒转 → Excitement↑ + Nausea↑
增加速度 → Excitement↑ + Fear↑
降低高度 → Fear↓ + Excitement↓
保持平稳 → Fear↓ + 但 Excitement也↓(无聊)
```

三大指标的张力关系创造了一个**非线性的优化问题**：没有"最大化所有指标"的最优解——你必须找到Excitement/Fear/Nausea之间的最佳平衡点。

### 热力图的游戏化价值

- 将抽象的G力数值映射为**颜色**(绿/黄/红)
- 玩家不需要懂物理——看颜色就知道哪里需要调整
- 一段一段地修改，实时看到热力图变化
- **即时反馈 = 学习曲线被压缩到几分钟内**

## 三、"擦头而过"——PC的标志性设计词汇

PC独有的设计概念：过山车轨道**紧贴**某个物体（建筑/岩石/景观）通过——制造"快要撞上了！"的瞬间恐惧→转化为兴奋度。

| Head Chopper场景 | 效果 |
|------|------|
| 轨道穿越狭窄岩缝 | 恐惧+3/兴奋+5 |
| 贴近建筑物墙面的倒转 | 恐惧+4/兴奋+7 |
| 飞越水面(近距离) | 恐惧+2/兴奋+4 |
| 穿越环形景观雕塑 | 恐惧+3/兴奋+6 |

这可能是PC对过山车设计最独特的贡献——它将真实过山车的"擦头而过"刺激量化为可计算、可优化的游戏机制。

## 四、编辑器=无限Replayability

### 为什么编辑器让PC永不"通关"

- 每个过山车都是独一无二的——没有"最优过山车"的设计终点
- 玩家的技能会持续提升→更复杂/更精美的过山车
- Steam Workshop提供了无止境的灵感循环
- **建造行为本身是乐趣，不是通往"通关"的路上被迫做的事**

### 技能成长曲线

```
新手：用蓝图(Blueprint)放置预制过山车
中级：自己建造简单的来回式过山车
高级：设计结合地形+隧道+擦头而过的复杂轨道
大师：Workshop上被数千人订阅的过山车蓝图创作者
```

## 五、设计启示

1. **"产品研发"可以是创造性的，而非随机性的**——PC证明了经营游戏中的"新产品"不一定来自骰子，可以来自玩家的创造性劳动
2. **即时视觉反馈可以替代教程**——热力图比文字教程更有效地教会玩家"什么是好的过山车"
3. **编辑器是元游戏**——建造过山车本身的深度和乐趣足以支撑一个独立的游戏
4. **工程知识可以"游戏化"**——真实过山车工程师使用的概念(G力/曲率半径/势能转化)被转化为可玩机制

---

> 核心观察: 过山车编辑器是"玩家劳动→游戏内价值"转化的最优雅案例之一。你投入的不是金币或R&D点数——你投入的是**你的时间和创造力**，而游戏承认这一点并把它转化为Prestige。
