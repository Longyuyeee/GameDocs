# 时间节奏数据总览 — 过山车之星

> 游戏版本: Planet Coaster v1.13+ (Steam, 2016)
> 数据来源: [Frontier官方指南](https://www.planetcoaster.com/)

---

## 一、时间控制

| 速度 | 用途 |
|:--:|------|
| 暂停 | **建造模式**——设计过山车/布置景观 |
| 1× | 观察运营/检查客流 |
| 2× | 加速等待(等待月末结算/设施测试) |
| 3× | 最快等待 |

## 二、月度财务节奏

| 时段 | 事件 |
|------|------|
| 月初 | 营销活动新周期开始 |
| 月中 | 客流平稳/持续收入 |
| 月末 | 利润/损失结算→调整策略 |

## 三、设施检查周期

| 设施使用频率 | 建议检查间隔 | 忽略后果 |
|------|:--:|------|
| 高(持续排队) | 每10-20分钟 | 故障→退款→声誉降 |
| 中(正常使用) | 每20-30分钟 | 同上 |
| 低(少人坐) | 每30-60分钟 | 风险较低 |

## 四、设施声誉老化时间线(估)

| 阶段 | 持续时间(游戏内) | 触发条件 |
|:--|:--:|------|
| New→Established | ~1-3月 | 开放后自动 |
| Established→Dated | ~6-12月 | 低Prestige持续 |
| Dated→Aging | ~3-6月 | Prestige仍然低 |
| Aging→Old | ~1-3月 | 无人翻新 |
| **维持Classic** | **永久** | **只要Prestige保持高位** |

---

> 关键洞察: Classic不是一次性成就——是需要持续维护的状态。一旦Prestige下降，Classic也会滑入Aging。

## 五、日常运营节奏建议

| 时间 | 操作 |
|------|------|
| 开园 | 确认所有设施正常运行/员工到位 |
| 运营中(1×) | 观察客流/微调定价/处理突发事件 |
| 运营中(暂停) | 大规模建造/景观布置/重新规划 |
| 关园前 | 检查设施磨损/安排机修工夜间维护 |
| 月末 | 查看财务报告/调整策略/规划下月建造 |

## 六、紧急事件处理时间窗口

| 事件 | 最佳处理窗口 | 错过后果 |
|------|:--:|------|
| 设施故障 | 立即(1-2分钟内) | 每延误=更多退款 |
| 垃圾堆积 | 5分钟内 | 游客不适递增 |
| 破坏行为 | 2分钟内 | 景观毁坏→恢复需重建 |
| 游客投诉(定价) | 同一天内调价 | 当天剩余时间该游客不再消费
