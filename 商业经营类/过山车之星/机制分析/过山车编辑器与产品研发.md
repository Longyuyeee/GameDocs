## 一、过山车编辑器——核心创新的技术基础

### 编辑器的能力

PC的过山车编辑器是**逐段轨道编辑器**——玩家逐段放置轨道，每段的俯仰/偏航/滚转/长度都由玩家控制：

| 编辑维度 | 控制方式 | 对乘坐体验的影响 |
|------|------|------|
| 轨道高度 | 上下调整 | 势能→速度 |
| 轨道曲率 | 弯曲半径 | 横向G力→恐惧 |
| 轨道俯仰 | 上坡/下坡角度 | 加速度→兴奋 |
| 轨道滚转 | 翻滚角度 | 倒转→兴奋+恶心 |
| 轨道长度 | 每段延长/缩短 | 整体节奏 |
| 链式提升段 | 速度控制 | 初始势能 |

### 实时热力图测试

编辑器内置三种热力图可视化，随着建设实时更新：
- **兴奋度(Excitement)**: 绿色=高，红色=低——目标：全绿
- **恐惧感(Fear)**: 越红越高——目标：~5左右（有刺激但不吓人）
- **恶心度(Nausea)**: 越红越高——目标：最低

## 二、三大乘坐指标(EFN)

| 指标 | 物理来源 | 理想值 | 过高后果 |
|------|------|:--:|------|
| **Excitement(兴奋)** | G力变化/速度/倒转/"擦头而过"瞬间 | **越高越好** | 无(越高越好的核心指标) |
| **Fear(恐惧)** | 极端G力/陡坡/高速 | **~5.0** | >8→游客恐慌→Panic状态 |
| **Nausea(恶心)** | 持续倒转/过大横向G力/连续多次乘坐 | **越低越好** | 游客呕吐→需要清洁工打扫 |

### 提升兴奋度的关键技巧

| 技巧 | 效果 |
|------|------|
| **隧道(Tunnel)** | 在轨道段穿过隧道——额外兴奋度加成 |
| **"擦头而过"(Head Chopper)** | 轨道紧贴景观/岩石/建筑通过→恐惧+兴奋 |
| **速度变化** | 从低速到高速的突然加速→高兴奋 |
| **倒转(Inversion)** | 适当数量的翻滚→兴奋（过多→恶心） |
| **近地飞行** | 高速贴近地面→恐惧+兴奋平衡 |

## 三、过山车类型与研发

### 过山车类型

PC有**30+种过山车类型**（含DLC），按建造限制分类：

| 类型 | 代表 | 特征 | 难度 |
|------|------|------|:--:|
| 钢制轨道 | Hyper Coaster/Giga Coaster | 高速度/大落差 | 中 |
| 木质轨道 | Wooden Coaster | 复古感/粗糙振动→额外恐惧 | 高 |
| 弹射起步 | Launched Coaster | 无需提升段/直接加速 | 低(建造简单) |
| 倒挂式 | Inverted Coaster | 轨下悬挂/倒转视角 | 高 |
| 特殊 | 4D Coaster(座椅旋转)/Wing Coaster | 额外的移动自由度 | 极高 |

### 研发系统

| 研发方式 | 说明 |
|------|------|
| **机修工分配研发** | 将机修工(Mechanic)分配到研发任务→解锁新设施类型 |
| **研发点积累** | 随时间积累，可用于加速特定类型的研发 |
| **DLC解锁** | 部分过山车类型需要DLC包 |

## 四、平地游乐设施（Flat Rides）

### 与过山车的关系

| 维度 | 过山车 | 平地设施 |
|------|------|------|
| 建设时间 | 长(10-60分钟设计) | 短(选择→放置→完成) |
| Prestige上限 | 极高(1000+) | 中(200-600) |
| 占地面积 | 大 | 小-中 |
| 运营成本 | 高 | 低-中 |
| 游客吞吐量 | 中-高(每列火车载客多) | 高(快速周转) |
| 独家卖点 | 公园的明星设施 | 填充和多样化 |

## 五、设施声誉老化系统

### 声誉生命周期

| 阶段 | 说明 | Prestige影响 |
|------|------|:--:|
| **New(新)** | 刚建成 | 基准 |
| **Established(稳定)** | 正常运营 | 基准 |
| **Dated(过时)** | 低Prestige持续太久 | 渐降 |
| **Aging→Old** | 持续低Prestige | 大幅下降→游客不去 |
| **Resurging(复苏)** | 高Prestige维持 | 回升 |
| **Classic(经典)** | 极高Prestige持续 | 吸引远方游客！ |

> 关键洞察: **高Prestige的设施永远不会老化**——它们会从Established晋升为Classic，反而成为公园的永恒吸引力。

## 六、设计特点分析

**编辑器="研发"在PC中不是RNG骰子而是创造性劳动**：与传统经营游戏"投入金币→研发→随机产出"不同，PC的"产品研发"是你亲手逐段建造过山车。研发质量取决于你的**创造性劳动质量**而非骰子。一个精心设计的过山车（高兴奋/恐惧适中/低恶心）本身就是"高级产品"——不需要"研发点数"来解锁"更好的过山车"。

**EFN三角=工程优化游戏**：三大指标之间的张力构成了一个微型工程优化游戏。增加倒转→兴奋+但恶心也+。降低速度→恐惧-但兴奋也-。三项指标的平衡是所有过山车建造者的核心挑战。

**热力图=即时反馈消除"猜"**：编辑器中的三色热力图将抽象的数值(KB中的数字)转化为可视的反馈(哪一段轨道在"杀人")。这让非工程背景的玩家也能理解和优化过山车设计——将"工程知识壁垒"转化为了"可视化学习"。

---

> 跨品类对比: 与双点博物馆的"探险=不确定的产品获取"不同，PC的产品研发是**完全确定的创造性劳动**——你不会"随机"建造出一个过山车。与游戏开发物语的"类型×题材组合→评分"的公式化产品不同——PC的产品质量是一个纯创造力的游戏。
