# 设施与空间布局系统 — 过山车之星

> 游戏版本: Planet Coaster v1.13+ (Steam, 2016)
> 数据来源: [Frontier官方公园管理指南](https://www.planetcoaster.com/), [Planet Coaster Wiki](https://planetcoaster.fandom.com/), [Namu Wiki Tips](https://en.namu.wiki/w/플래닛_코스터/팁)

## 一、设施类型体系

### 四大设施类别

| 类别 | 子类型 | 数量(含DLC) | 收入模式 |
|------|------|:--:|------|
| **过山车(Coasters)** | 钢制/木质/弹射/倒挂/4D/飞行/水上等 | 30+ | 按次收费 |
| **平地设施(Flat Rides)** | 旋转类/升降类/摆动类/水上类 | 80+ | 按次收费 |
| **运输设施(Track Rides)** | 小火车/单轨/升降机等 | 10+ | 门票/免费 |
| **商店设施(Facilities)** | 食品/饮料/纪念品/洗手间/ATM | 20+ | 商品销售 |

## 二、空间布局：路径系统

### 路径类型

| 类型 | 用途 | 关键参数 |
|------|------|------|
| **普通路径(Path)** | 游客主要步行通道 | 宽度可调 |
| **自然路径(Natural Path)** | 景观区/非正式步道 | 不可建队列 |
| **排队路径(Queue Path)** | 连接游乐设施入口 | 必须从普通路径分出 |

### 布局核心原则

| 原则 | 说明 |
|------|------|
| **宽主干道** | 主干道至少4-5格宽(防止拥堵) |
| **环路而非死胡同** | 游客不会回头走——死胡同=设施无法被访问 |
| **分散热门设施** | 最高Prestige的设施放在不同区域→分散客流 |
| **设施出口=商店黄金位** | 游客离开设施时处于"刚被刺激→心情好→更愿消费"状态 |
| **排队路径不阻塞主干道** | 长队列可沿设施侧面延伸——不横穿主干道 |

## 三、景观与装饰系统

### 景观对游戏机制的影响

| 影响对象 | 效果 |
|------|------|
| 设施Prestige | 周围景观→**景观奖励分**(占总Prestige的30-50%) |
| 队列满意度 | 队列路径景观→Queue Scenery Rating(0-100%)——越高游客能忍受越长等待 |
| 游客心情 | 行走在高质量景观区→保持更长时间的好心情 |
| 公园评分 | 全园景观→Scenery Rating维度 |

### 景观效率最大化

| 技巧 | 说明 |
|------|------|
| **隧道加倍** | 过山车在隧道内=轨道上下都能放景观=**双倍景观密度** |
| **空中浮动物品** | 景观可以浮空放置——不需要物理支撑 |
| **主题一致性** | 同主题的景观组合→比混合主题更高的评级 |
| **特效物品** | 火/水/动画/灯光→更高景观分但需要月度维护费 |
| **蓝皮书(Blueprint)** | 保存装饰好的景观集合→一键复用到整个公园 |

## 四、地形工具

### 地形编辑

| 工具 | 功能 | 游戏性影响 |
|------|------|------|
| **升降(Push/Pull)** | 创建山丘/谷地 | 过山车利用自然地形→额外兴奋度 |
| **平滑(Smooth)** | 平滑地形表面 | 美观——但不直接影响评分 |
| **水体(Water)** | 创建湖泊/河流 | 景观加成——过山车飞过水面→场景分 |
| **洞穴/隧道** | 挖入山体 | 过山车穿越山洞→隧道兴奋度加成 |

## 五、设施布局策略

### 分区布局

| 区域类型 | 设施组合 | 目标游客群 |
|------|------|------|
| **家庭区** | 低恐惧平地设施+餐厅+纪念品店 | 家庭/儿童 |
| **刺激区** | 高Prestige过山车+饮料店+洗手间 | 年轻人/寻求刺激 |
| **入口区** | 信息亭+ATM+洗手间+礼品店 | 所有游客 |
| **景观区** | 景观密度极高+休息区+摄影点 | 所有游客(心情恢复区) |

### "两大一小"原则

```
每个主要区域: 1个旗舰过山车 + 1-2个平地设施 + 商店集群(食品+饮料+洗手间+长椅)
```

## 六、设计特点分析

**环路规则=反常识的动线约束**：游客AI"不走回头路"的设计强迫公园布局必须是环路——你不能把最好设施放在死胡同尽头。这在真实主题公园中也是核心设计原则（迪士尼的"hub and spoke"布局），PC将这条真实规则编码进了游客AI。

**出口=消费黄金位**：游客离开设施时处于"刚被刺激"的高心情状态——此时最愿意冲动消费。将商店放在设施出口是最大化收入的布局策略，这与真实购物中心/主题公园的"出口零售"原则一致。

**隧道的双倍景观bug/特性**：过山车轨道在隧道内时，上下两侧都能放置景观——这可能是设计上的意外，但成了社区公认的最优策略。Frontier选择保留而非修复它——因为它创造了有趣的建造挑战："如何在隧道中最大化景观密度？"

---

> 跨品类对比: 与双点医院的"房间式功能分区"(诊断室/治疗室)不同，PC的分区是基于"游客心情流动"而非"医疗流程"的。与温泉物语的"设施相性加成"(相邻设施产生组合效果)不同——PC的景观系统是全局累积的，不受"相邻关系"的硬性限制。
