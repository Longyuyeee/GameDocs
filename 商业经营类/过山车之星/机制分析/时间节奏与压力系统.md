# 时间节奏与压力系统 — 过山车之星

> 游戏版本: Planet Coaster v1.13+ (Steam, 2016)
> 数据来源: [Frontier官方指南](https://www.planetcoaster.com/), [Steam社区讨论](https://steamcommunity.com/app/493340)

---

## 一、时间结构

| 单位 | 说明 |
|------|------|
| 游戏日 | 完整的开园-闭园循环 |
| 游戏月 | 财务结算周期 |
| 速度控制 | 暂停/1×/2×/3×（建造时通常暂停） |

PC的"真正时间"在**暂停建造**时是停止的——玩家在无限暂停中精心设计过山车和景观，然后恢复正常速度来"看公园运转"。

---

## 二、运营节奏

| 阶段 | 活动 |
|------|------|
| 建造期(暂停) | 设计过山车/布置景观/规划布局——主要时间消耗 |
| 运营期(恢复速度) | 观察客流/调整定价/应对问题 |
| 月末(结算) | 查看利润/调整策略/规划下月 |

### 建造-运营循环

```
暂停→建新设施/景观→恢复→观察游客反应→微调定价→月末结算→暂停→新一轮建造...
```

---

## 三、压力/挑战

| 压力类型 | 触发 | 后果 |
|------|------|------|
| **财务压力** | 支出持续>收入 | 资金耗尽→破产 |
| **设施老化** | 低Prestige持续 | 游客不再乘坐→进一步降低Prestige→恶性循环 |
| **拥堵** | 路径太窄/太多人 | 游客不适→心情降→离园 |
| **设施故障** | 忽略维护 | 退款+游客不满+声誉降 |

---

## 四、失败条件

| 模式 | 失败条件 |
|------|------|
| 生涯模式 | 无法达成星级目标/破产 |
| 挑战模式 | 破产(资金归零且持续亏损) |
| 沙盒模式 | **无失败**(纯建造乐趣) |

---

## 五、设计特点分析

**"暂停建造"=对玩家时间的两种态度**：PC允许在暂停状态下进行所有建造——这等于承认了"建造"是一种不受时间约束的创造性行为，而"经营"是受时间约束的管理行为。这个设计选择尊重了两种不同的玩家动机。

**设施老化=温和的折旧机制**：设施不会突然"坏掉"——它缓慢地从New→Established→Dated→Old——这个过程足够慢，玩家有时间通过提升景观和翻新来阻止恶化。这种温和的折旧比"突然故障"更有效地驱动了持续的公园维护。

**沙盒模式=PC实际上有两个游戏**：一个是"经营模拟"(挑战/生涯模式)，一个是"创意工具"(沙盒模式)。大量玩家只玩沙盒模式——他们购买PC不是为了"经营公园"，而是为了"建造美丽的公园"。

---

> 跨品类对比: PC的时间节奏与双点医院的"暂停/2×/3×"一致，但PC的"暂停=建造时态"将建造成为了时间约束之外的活动。与温泉物语的"年度评分"压力不同——PC的挑战压力来自设施声誉的缓慢退化而非年度Deadline。

## 六、天气与昼夜的影响

| 天气/时段 | 对客流的影响 | 商业机会 |
|------|:--:|------|
| 晴天 | 客流正常，晒伤风险 | 防晒霜/遮阳棚需求↑ |
| 雨天 | 客流-20-50% | 雨伞/室内设施需求↑ |
| 高温 | 口渴加速/精力消耗快 | 饮料店生意↑ |
| 低温 | 寒冷不适 | 暖气/热饮需求↑ |
| 夜间 | 基础客流减少 | 灯光=额外兴奋度来源+夜间特有氛围 |

### 天气对设施运营的影响

雨天时户外过山车可能因安全原因自动关闭——这迫使公园不能"全户外"。室内设施（如暗室过山车/Dark Ride）在恶劣天气时成为公园唯一的收入来源。设计良好的公园需要**户外+室内设施的平衡**来应对天气波动。
