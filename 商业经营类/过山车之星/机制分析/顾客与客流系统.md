# 顾客与客流系统 — 过山车之星

> 游戏版本: Planet Coaster v1.13+ (Steam, 2016)
> 数据来源: [Frontier官方游客指南](https://www.planetcoaster.com/), [Namu Wiki](https://en.namu.wiki/w/플래닛_코스터/팁), [Steam社区指南](https://steamcommunity.com/)

---

## 一、游客心情（Mood）七层模型

| 心情 | 触发条件 | 行为影响 |
|------|------|------|
| **Content(满足)** | 所有需求满足 | 正常消费/正常行走速度 |
| **Happy(开心)** | 刚体验了好设施/吃了喜欢食物 | 奢侈品需求增加/更愿消费 |
| **Excited(兴奋)** | 排队等候心仪设施/高景观队列 | 最长停留时间 |
| **Bored(无聊)** | 长时间低景观队列/低兴奋度 | 疲劳加速/提前离园 |
| **Uncomfortable(不适)** | 淋雨/垃圾/脏厕所 | 疲劳加速/消费意愿降低 |
| **Unhappy(不开心)** | 多项需求未被满足/定价突变 | 减少消费/提前离园 |
| **Desperate(绝望)** | 极低需求 | 最快离园 |

## 二、游客需求系统

### 六大核心需求

| 需求 | 衰减速度 | 满足方式 | 不满足→ |
|------|:--:|------|------|
| **饥饿(Hunger)** | 中 | 食品店/餐厅 | 寻找食物→如找不到→心情降→离园 |
| **口渴(Thirst)** | 中快 | 饮料店/饮水机 | 寻找饮料→心情降 |
| **如厕(Bathroom)** | 慢 | 洗手间 | 寻找厕所→如脏→心情大幅降 |
| **精力(Energy)** | 慢 | 休息区/长椅 | 疲劳→离园 |
| **兴奋度(Excitement)** | 快 | 乘坐设施 | 无聊→离园 |
| **奢侈(Luxury)** | 中 | 纪念品/高端餐饮 | 开心后产生→额外消费 |

### 身体状态(Ailments)

| 状态 | 原因 | 预防/解决 |
|------|------|------|
| 恶心(Sick) | 高恶心度设施/连续坐多次 | 交替高低刺激设施/清洁工打扫 |
| 晒伤(Sunburnt) | 长时间阳光下 | 卖防晒霜/建遮阳棚 |
| 淋湿(Wet) | 雨天 | 卖雨伞/建遮蔽+烘干机 |
| 寒冷(Cold) | 低温 | 放置暖气 |
| 恐慌(Panic) | 高恐惧度设施 | 降低设施恐惧度 |

## 三、客流规律

### 门票制 vs 按次收费的客流差异

| 维度 | 按次收费 | 门票制 |
|------|------|------|
| **游客停留时间** | 较短(钱花完就走) | 极长(免费=赖着不走) |
| **新游客涌入** | 持续(旧游客离开→新游客进来) | 缓慢(旧游客占着不走) |
| **人均消费** | 高(每项设施都收费) | 低(只有门票) |
| **设施利用率** | 高 | 中等 |

### 影响客流的因素

| 因素 | 效果 |
|------|------|
| **公园评分(Park Rating)** | 基础客流量的核心决定因素 |
| **营销活动** | 短期客流大幅提升(+50-200%) |
| **新设施开放** | 开幕期客流峰值 |
| **季节/天气** | 客流波动 |
| **门票价格** | 过高→新游客望而却步 |

## 四、游客行为AI

### 决策循环

```
进入公园 → 寻找吸引人的设施(高Prestige优先) → 排队 →
→ 体验设施(心情变化) → 评估需求(饿/渴/厕所/累) →
→ 满足需求或寻找下一设施 → 重复 → 钱花完或太累 → 离园
```

### 排队行为

- 游客会评估**排队时间**是否值得（期望设施刺激度 vs 等待时长）
- **长队列+低景观** = 游客中途放弃 → 不满
- **高景观队列**(Queue Scenery Rating 100%) → 游客能忍受更长等待
- 优先通行证(Priority Pass) = 缩短排队时长 → 更多乘坐次数

## 五、设计特点分析

**七层心情模型=颗粒度适中的满意度体系**：不是"满意/不满意"的二元分，也不是20层的过度颗粒度——7层刚好提供了足够的策略空间，每层有明确的行为后果。Excited/Happy/Content/Bored之间的过渡给玩家留出了"发现问题→修复→游客恢复"的缓冲空间。

**"游客赖着不走"=门票制破产的原因**：这是PC最有趣的设计意外之一——门票制在理论上很美好(一次性收费+无限制乘坐)，但游客AI的"没钱了才走"逻辑导致旧游客无限停留→新游客进不来→没有新门票收入→破产。这个设计迫使玩家要么选择按次收费，要么用营销活动强行吸引新游客。

**队列景观=等待时间的软化剂**：游客对"等待"的满意度不是由等待时长本身决定的——而是由"等待期间看到了什么"决定的。一个100%景观评级的队列可以让游客忍受2-3倍的标准等待时间。这是主题公园设计中最真实的现实映射。

---

> 跨品类对比: 与双点医院的"病人流"（进→诊断→治疗→出院）的线性流程不同，PC的游客流是自由探索式的——游客自主决定下一个设施。与双点博物馆的"游客梦想之旅"目标驱动不同——PC的游客目标更简单（体验足够刺激→花钱→离开）。
