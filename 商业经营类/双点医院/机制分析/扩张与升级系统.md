## 一、三层面的扩张

| 层面 | 载体 | 机制 | 节奏 |
|------|------|------|:--:|
| **院区扩张** | 地块/建筑 | 购买新地块→建造新楼→布置新诊室 | 阶段性 |
| **服务扩张** | 诊室种类/数量 | 增加诊断室/治疗室种类 | 持续性 |

## 二、院区扩张

每个医院地图有多块可购买的土地。土地价格递增——需要选择正确的时机扩张：

| 扩张时机 | 信号 |
|------|------|
| **正确** | 现有诊室全部满负荷(GP排队>8人→需要更多GP) |
| **正确** | 员工培训到瓶颈(需要更多诊室利用已培训的员工) |
| **错误** | "我有闲钱所以买块地"——空地不产生收入 |
| **错误** | 不建新诊室只买地——地=沉没成本 |

## 三、服务扩张——诊室解锁链

### 3.1 诊断端扩张

```
初始: GP全科(唯一诊断)
    ↓ 研究/任务解锁
心脏病(Cardiology) —— 将部分诊断流转向护士
    ↓
X光(X-Ray) —— 最快诊断(22秒)，优先扩建
    ↓
体液分析(Fluid Analysis) / MEGA Scan / DNA Lab
    ↓
完整诊断链: GP → 专科诊断室 → GP确认 → 治疗室
```

### 3.2 治疗端扩张

每解锁一种新疾病→需要对应治疗室：

```
早期基础: 药房(Pharmacy) + 注射室(Injection)
    ↓
中期专科: 小丑诊所 / 颅疗室 / De-Lux Clinic / 染色体治疗
    ↓
后期高级: 手术室(Surgery) + 分辨率实验室 + DNA Lab + 病房升级
```

### 3.3 机器升级路径

```
Lv1机器(基础)
    ↓ 研究室研究
Lv2机器(+20%治疗能力)
    ↓ 继续研究
Lv3机器(+50%治疗能力) ← 天花板
    ↓ (超过Lv3无进一步效果)
```

## 五、扩张的风险管控

| 风险 | 症状 | 解决方案 |
|------|------|------|
| **过度诊断** | 建了5个专科诊断室但病人不够 | 拆掉多余的→节约维护费+员工薪资 |
| **扩张与收入脱节** | 地买了/诊室建了/员工招了→月利润变负 | 暂停扩张→等到月利润>月薪资支出再扩 |
| **单点瓶颈** | 只有2个GP但8个专科诊断室 | GP永远不够→加GP诊室→GP是唯一的"首诊入口" |

> **核心教训**: 扩张双点医院不是"建更多"——是**在正确的位置建正确的诊室**。最贵的错误是把资源投入一个永远不会被充分使用的诊室。开新诊室前问自己：这个诊室会每天有多少病人来？如果<2-3人/天，不值得。
