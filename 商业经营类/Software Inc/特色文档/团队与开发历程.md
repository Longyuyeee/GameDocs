### Coredumping品牌哲学

Kenneth Otto Larsen 是一个典型的"一人独立工作室"代表。他选择了Coredumping作为品牌名，在没有团队、没有投资的情况下，从零开始自学游戏开发，最终在Steam上获得了93%好评率的成绩。他的开发方式有几个显著特点：

- **自研引擎**：拒绝使用Unity或Unreal，自研2D/3D混合引擎。这个选择带来了短期的开发困难，但也让Software Inc.拥有了独特的自由建造系统，被社区认为是"用现成引擎难以实现的"。
- **完全独立**：拒绝Kickstarter众筹，全部开发资金来自个人积蓄和EA早期收入。这种"不欠债"的方式让他保持了完全的创意控制权。
- **超长EA周期**：从2015年EA到正式版，实际开发周期远超最初预计的2年，延续了近10年的持续更新。

### 2.2 关键决策点

**1. 拒绝Kickstarter，自费开发($2,000起步)**
Kenneth在开发初期已经投入了约$2,000的个人资金。当时许多独立开发者会选择Kickstarter来筹集资金，但他选择拒绝。这意味着他只能依靠自己的积蓄和EA早期收入来维持开发。这个决定让游戏保持了完全的创意独立性，但也让开发周期大大延长。

**2. 自研引擎 vs 现成引擎**
Software Inc.使用自研引擎，而非Unity或Unreal。这个选择在开发初期看起来"效率低下"，但最终让游戏获得了独特的自由建造系统——无网格的办公室建造、多层建筑、温度/噪音/光照的实时模拟。社区中有玩家评价："这种建造系统用现成引擎可能根本做不出来。"

**3. 渐进式EA定价策略**
Kenneth采用了"越早买越便宜"的定价模型：

| EA阶段 | 售价 |
|:-----:|:----:|
| Pre-Alpha | $10 |
| Alpha | $12 |
| Mid-Alpha | $14 |
| Beta | $16 |
| 正式版 | $18 |
| 当前(2026) | ~$23 |

这种策略鼓励早期支持者以更低价格获得游戏，同时随着游戏内容增加逐步提高价格。这是独立游戏EA的经典定价模型案例。

## 3. 技术与工具

### 3.1 引擎与平台

| 技术维度 | 详情 |
|---------|------|
| **游戏引擎** | 自研引擎(非Unity/Unreal)，使用C#/C++混合开发 |
| **渲染** | 3D俯视角，采用"平滑极简主义(Smooth Minimalism)"美术风格 |
| **支持平台** | Windows / Linux / Mac (跨平台) |
| **网络** | 自研网络代码(多人合作模式) |
| **Mod支持** | Steam Workshop集成，XML配置可修改 |
| **存档** | Steam Cloud云存档 |

### 3.2 自研引擎的代价与收获

| 代价 | 收获 |
|------|------|
| 开发周期大幅延长(原定2年→近10年) | 完全掌控技术栈，不受引擎更新/授权限制 |
| 需要实现所有基础功能(物理/渲染/网络) | 独特建造系统(无网格自由放置)成为核心竞争力 |
| 单人调试压力巨大 | 性能可深度优化，针对游戏需求定制 |
| 美术资源自绘(非专业美术师) | 形成了独特的极简主义视觉风格 |

### 3.3 技术演进

```
2014: 基于网格的墙体系(像Prison Architect)
    ↓ (发现占用空间过多)
2014末: 改为薄墙+网格边缘(像The Sims)
    ↓ (发现空间利用率大幅提升)
2015: 加入多层建筑支持
    ↓
2016: 加入温度/噪音实时模拟系统
    ↓
2017: 温度系统大改(从实时模拟→瞬时计算+绝缘统计)
    ↓
2018: 性格特质系统(图结构关系图谱)
    ↓
2019+: 硬件制造/股票/多人合作
```

## 5. 设计启示

### 5.1 对独立游戏开发者的启示

**单人10年持续开发的独立游戏案例**
Kenneth一人持续开发Software Inc.近10年，这在游戏行业是极为罕见的。他的经历证明：一个人+一个足够好的创意+持续更新，可以在Steam上获得商业成功。从学士论文期间做到现在还在更新，这种持久力本身就是一个传奇。

**从零自学到商业成功**
Kenneth并非计算机科班出身(虽然完成了学士学位)，他在开发过程中自学了编程、设计、美术、UI、市场和社区管理等全部技能。他的成功证明了独立开发中最重要的是"学习能力和坚持"，而非初始技能储备。

**自研引擎的代价与收获**
Software Inc.的自研引擎决策是一个高风险高回报的选择。如果Kenneth使用Unity/Unreal，游戏的开发周期可能会缩短到2-3年，但可能永远无法实现那个独特的自由建造系统。这是一个关于"技术栈选择如何塑造游戏设计"的经典案例。

### 5.2 EA定价递增模型

```
                    Pre-Alpha $10
                      ↓
                    Alpha $12
                      ↓
                  Mid-Alpha $14
                      ↓
                    Beta $16
                      ↓
                正式版 $18(→$23)
```

这种定价策略的设计逻辑：
- **早期支持者奖励**：以更低价格获得游戏，同时承受更多Bug和不完整内容
- **价值增长匹配**：随着游戏内容增加逐步提高价格，让每个阶段的购买者都觉得"值这个价"
- **心理锚定**：正式版$23相比EA时的$10，让玩家觉得"游戏在不断增值"
