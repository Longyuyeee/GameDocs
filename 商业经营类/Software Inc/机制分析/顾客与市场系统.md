## 一、市场结构总览

Software Inc. 的市场系统与其他经营模拟有本质区别——**没有"走进门的物理顾客"**，所有市场数据以统计/报表形式呈现：

| 市场维度 | 表现形式 | 更新频率 | 说明 |
|---------|---------|:-------:|------|
| **消费者触达(Reach)** | 百分比数字 | 月度 | 产品能触及的潜在用户比例。受发行渠道/营销/口碑影响 |
| **市场份额** | 饼图 | 月度 | 与AI公司在同一品类中的相对占有率 |
| **产品评分** | 1-100分 | 每款产品有独立评分 | 决定口碑→影响长尾销售 |
| **粉丝/追随者(Follower)** | 数字 | 随营销动作更新 | 发布前积累的关注度→首日销量转化 |
| **市场趋势** | 曲线图 | 逐年 | 品类兴衰(OS/2D/3D/Audio/Game等) |
| **竞争AI公司数据** | 列表 | 月度 | 对手的产品/价格/口碑/市场占有率 |

## 二、市场生成与竞争格局

| 机制 | 说明 |
|------|------|
| **随机生成市场** | 每次开局随机生成竞争AI公司及其初始策略——创造不同的市场格局 |
| **品类兴衰周期** | OS(操作系统)、2D编辑器、3D编辑器、音频工具、游戏引擎、游戏等品类各有独立的需求曲线 |
| **技术迭代** | 新技术出现→旧技术产品逐渐过时。类似真实科技史：如Windows崛起→DOS衰退 |
| **AI公司行为** | AI公司会自主开发/发布产品、设定价格、打营销战、申请专利 |
| **市场份额争夺** | 同一品类内与AI公司直接竞争。(产品评分+定价+营销)×渠道覆盖=竞争力 |

### AI竞争深度

| AI行为维度 | 说明 |
|-----------|------|
| 产品开发 | AI公司按自己的节奏开发产品，品质取决于其团队配置 |
| 动态定价 | AI公司可调整产品价格，形成价格战或避风港 |
| 营销活动 | AI公司发布新闻稿/媒体内容吸引关注 |
| 专利策略 | AI公司申请和授权专利，可能形成技术壁垒 |
| 收购兼并 | 大型AI公司可能收购小型竞争对手 |

## 三、粉丝/追随者(Follower)系统

| 积累方式 | 效果 | 成本 |
|---------|:----:|:----:|
| **新闻稿(Press Release)** | 少量Follower | 低 |
| **媒体图(Mockup)** | 中量Follower | 低(需设计师时间) |
| **媒体片(Trailer)** | 大量Follower | 中(需制作时间) |
| **试用版(Demo)** | 大量Follower+早期反馈 | 中(需额外开发时间) |
| **社交媒体运营** | 持续少量增长 | 低(需配置营销岗位) |

> Follower积累机制: 发布前积累的关注度 → 发布时按比例转化为首日销量。这是"预热期"的核心意义——没有预热的产品发布几乎注定失败。

## 四、产品口碑与长尾效应

| 评分区间 | 口碑等级 | 销量衰减速度 | 长尾效应 |
|:-------:|:-------:|:-----------:|:--------:|
| 90-100 | 神作 | 极慢 | 持续数年稳定销售 |
| 75-89 | 优秀 | 慢 | 1-2年稳定销售 |
| 50-74 | 中等 | 中等 | 数月后急剧下降 |
| 30-49 | 较差 | 快 | 短期销售后无人问津 |
| <30 | 差评 | 极快 | 发布即失败 |

> Review Score不仅是"质量标签"——它是驱动后续所有销售的核心杠杆。好评=长尾稳定收入，差评=需要立即出补丁/续作救场。

## 五、市场趋势与品类周期

| 品类阶段 | 特征 | 玩家策略建议 |
|---------|------|-------------|
| **新兴期** | 市场需求快速增长，竞争少 | 尽早进入→建立先发优势 |
| **成熟期** | 市场饱和，竞争激烈 | 品质/价格/营销综合竞争 |
| **衰退期** | 需求下降，旧技术产品过时 | 转型/推出替代品类 |
| **复兴期** | 待补充(某些品类可能有二次增长) | 待补充 |

> OS平台的兴衰周期模拟了真实操作系统的历史更迭——如从命令行的原始OS → GUI OS → 移动OS的演进路径。

## 六、渠道覆盖

| 发行渠道 | Reach影响 | 利润率 | 解锁条件 |

## 七、设计特点分析

**"无形市场"的设计哲学**：Software Inc. 没有"走进公司的顾客"——这是其与超市模拟器/餐厅模拟器最根本的差异。这种"企业级B2B/B2C"视角而非"店铺级零售"视角的设计选择，传递了一个清晰的信号：你经营的不是面向消费者的"商店"，而是一家真实的"科技企业"。市场以统计数据存在意味着玩家需要学会阅读报表、分析趋势、预测需求——而不是"摆放货架→顾客上门"的直观反馈循环。

**Follower系统的"预热教育"**：要求产品发布前积累关注度才可能有好的首日销量——这个机制模拟了真实软件行业的"发布前营销"必要性。与Game Dev Tycoon的"发布即靠评分"不同，Software Inc. 要求玩家在发布前就投入营销精力，否则高品质产品也可能首发失利。这是对"好产品自动卖得好"这一迷思的纠正。

**品类周期替代"玩家选择"**：市场趋势系统将产品类型的成功从"玩家选择"转移到"市场时机"——你可以在错误的时机做出完美的OS但依然失败，也可以在正确的时机推出平庸的产品而大获成功。这是一个更现实(也更有挑战性)的市场模型。

## 八、跨品类对比

**vs Game Dev Tycoon**: GD Tycoon 的市场是"类别+平台+品质"的三维引擎：选择游戏类型→选择平台→评分决定销量。没有Follower积累机制，没有品类兴衰周期，没有渠道覆盖管理。Software Inc. 的市场系统在深度和真实度上都远超GD Tycoon——尤其是Follower系统和对市场趋势的模拟。

**vs 双点医院**: 双点医院有"物理顾客"(走进医院的患者)，而Software Inc. 完全没有。双点医院是"医院经营"——每个患者是可见的个体，有诊断/治疗流程；Software Inc. 是"企业经营"——市场是统计数字，没有可见的消费者。这个差异定义了两种完全不同的经营模拟子类：店铺级vs企业级。

**vs 开罗游戏开发物语**: 开罗系列在市场中引入的是"评分+人气+大奖"的三角循环——玩家追求高分→赢得奖项→获得人气→更多销量。没有Follower系统，没有品类周期，没有AI竞争。Software Inc. 的市场深度和真实度在开罗系列的数十倍以上。
