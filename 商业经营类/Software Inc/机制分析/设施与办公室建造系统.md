## 一、建造系统基础

| 特性 | 说明 |
|------|------|
| **层数** | 最多10层+地下室，可建造多层办公大楼 |
| **网格系统** | **无网格**——所有物体可自由放置(Freedom Placement)，精度极高 |
| **克隆/复制** | 内置Clone Tool——选定区域一键复制粘贴，加速重复建造 |
| **建筑元素种类** | 墙壁/门/窗/地板/道路/停车场/自行车架/路径/电梯/楼梯/空中走廊/地下隧道 |
| **跨地块连接** | 空中走廊(跨道路连接两栋建筑)/地下隧道(同一地块跨区域) |
| **地块购买** | 可购买相邻地块扩大园区，地块间可打通连接 |

## 二、环境因素对生产力的影响

Software Inc. 最独特的设计之一：真实的物理环境参数直接影响员工工作效率。

| 环境因素 | 正面影响 | 负面影响 | 调节手段 |
|---------|---------|---------|---------|
| **温度(Temperature)** | 适宜温度→效率100% | 过热或过冷→效率显著下降 | 空调(AC/制冷)+暖气设备 |
| **噪音(Noise)** | 安静区域→高效率 | 吵闹设备(服务器/打印机/多人交谈)→周围员工效率下降 | 隔音墙/分区布置/机房独立隔间 |
| **光照(Light)** | 自然光(窗户)+适宜人工光→效率+ | 全暗→效率-；过亮→可能不适 | 窗户布局+人工照明设备 |
| **绿植(Plants)** | 改善环境质量→小幅提效 | 无 | 放置盆栽/绿植墙 |
| **装修材料质量** | 高档材料→环境评分↑→满意度↑ | 低级材料→环境评分↓ | 在建造时选择不同材质(价格×质量成正比) |
| **空气质量(Air)** | 待补充 | 密闭空间无通风→空气质量下降 | 通风系统(待补充) |

### 环境因素叠加规则

> 各环境参数独立计算，叠加作用于员工效率。例如: 一个工位同时处于"过热(效率-15%)"+"高噪音(效率-10%)"+"低光照(效率-5%)"区域，则该员工总体效率损失为累积叠乘——而非仅取最差项。

## 三、功能区规划

| 功能区 | 核心物件 | 功能 | 特殊需求 |
|-------|---------|------|---------|
| **办公区** | 工位(Desk)+电脑 | 核心工作区域。按项目/团队分区 | 每个员工需一个工位+一台电脑 |
| **服务器房** | 服务器机架 | 托管软件在线服务/源码控制/自有商店 | 需散热通风设备(空调/通风口)。高噪音→需隔音 |
| **会议室** | 会议桌+椅子 | 团队会议/商务谈判/合同签署 | 需足够容纳团队规模 |
| **食堂/厨房** | 厨具+用餐区 | 员工餐饮→满意度↑ | 需雇佣厨师(Cook)岗位。食物质量影响满意度 |
| **休息区** | 沙发/咖啡机/娱乐设施 | 员工休闲→恢复精力 | 可放置电视/游戏机等娱乐设备 |
| **打印间** | 打印机+纸张架 | 打印文档/包装材料 | 打印机产生噪音→远离办公区 |
| **接待区** | 接待台+等候区沙发 | 接收Deals/合同/访客 | **必须有接待员→才能接收Deals**。接待台等级→合同质量 |
| **洗手间** | 马桶/洗手池 | 员工基本需求 | 按员工数量规划足够坑位 |
| **仓库/存储** | 货架 | 存储实体产品/硬件 | 硬件制造商需要 |

## 四、环境效应详细分解

### 温度效应

| 温度区间 | 效应该 |
|:--------:|:------:|
| 18-26°C(舒适区间) | 无惩罚，效率100% |
| 26-32°C(偏热) | 效率-5%~-10% |
| 32°C+ (过热) | 效率-15%~-25% |
| 10-18°C(偏冷) | 效率-5%~-10% |
| 10°C-(过冷) | 效率-15%~-25% |

### 噪音传播

| 噪音源 | 影响半径 | 影响幅度 |
|-------|:-------:|:--------:|
| 服务器机架(单台) | 2-3格 | -5% |
| 服务器机架(满排) | 4-6格 | -10%~-15% |
| 打印机 | 3-4格 | -5%~-8% |
| 多人交谈区 | 2-3格 | -3%~-5% |
| 隔音墙 | 阻断传播 | 有效切断噪音路径 |

## 五、建造vs经营耦合

| 布局决策 | 运营影响 | 说明 |
|---------|---------|------|
| **食堂与办公区距离** | 员工移动耗时 | 食堂太远→员工来回30分钟→实际工作时间减少 |
| **服务器房位置** | 噪音扩散+散热成本 | 服务器房靠近办公区→噪音干扰；在顶层/独立隔间→散热成本高 |
| **接待区位置** | 商务效率 | 接待区靠近入口+会议室→客户动线流畅 |
| **电梯/楼梯密度** | 垂直交通效率 | 高层建筑电梯不足→高峰期拥堵→等待时间成本 |
| **工位布局** | 团队协作效率 | 同项目团队就近安排→沟通成本降低 |
| **动线设计** | 综合效率 | 员工从"入口→工位→会议室→食堂→洗手间→工位"的每日动线累积时间成本 |

> 建造系统与经营系统的"耦合度"在同类游戏中极高——每一个建筑决策都有可量化的经营影响。这不是装饰性建造，而是功能性建造。

## 六、设计特点分析

**"环境作为生产力杠杆"的设计创新**：绝大多数经营模拟游戏的建造系统是"装饰性"的——装修好看但无实际影响。Software Inc. 将环境参数(温/声/光/气/植/材)系统地挂钩到生产力，让办公室建造从"好看"变成"必须"。这是自《矮人要塞》的"质量-情绪-效率"链条以来，经营模拟中最深刻的环境-生产力耦合设计。

**无网格系统的自由度代价**：自由放置(无网格)提高了建造精度和创造自由度，但也增加了建造难度。玩家需要手动对齐物体、规划空间——没有网格辅助意味着每个建筑决策都需要更多时间和注意力。这是一个典型的高自由度+高认知负荷的设计取舍。

**"企业物理"模拟**：食堂距离影响工作时间的机制看起来是个细节，实际上定义了游戏的"空间管理"层面——时间就是金钱，员工走路的时间也是成本。这迫使玩家像真实企业管理者一样考虑办公空间的"动线效率"，而非仅仅"看起来好看"。

## 七、跨品类对比

**vs 双点医院**: 双点医院的建造系统同样有环境因素影响，但其影响面较窄——主要关注"美观度"和"温度"两个维度。Software Inc. 引入了噪音、光照、绿植、空气质量等更多维度，且每个维度都有独立的影响曲线。双点医院的建造是"走廊+房间"模板式建造，Software Inc. 是无网格自由建造，自由度更高但建造复杂度也更高。

**vs 过山车之星**: 过山车之星的建造系统是行业标杆级的，但其聚焦于景观/过山车轨道的美观性和乘客体验。Software Inc. 的建造系统聚焦于"办公效率"——同样的墙体/地板/装饰物件，出发点和评价标准完全不同。过山车之星建造的目的是"让游客开心"，Software Inc. 建造的目的是"让员工高效"。

**vs 开罗系列**: 开罗游戏的设施系统是"放置+升级"的极简模式——放下设施→升级→效果数值提升。几乎没有空间布局的考量(设施只要放下就生效，距离/噪音等都不存在)。Software Inc. 的"环境-效率耦合"设计在深度上远超开罗系列数十倍。
