# 营销与竞争系统 — Software Inc.

> 游戏版本: Software Inc. Beta 1.8 (PC/Steam, 2015+, Coredumping)
> 数据来源: [Software Inc. Wiki (Fandom)](https://software-inc.fandom.com), [Steam社区指南](https://steamcommunity.com/app/362620/guides/), [IndieDB](https://www.indiedb.com/games/software-inc/)

## 一、营销手段体系

Software Inc. 提供了四种核心营销手段，玩家在发布产品前通过持续积累热度来影响首发表现：

| 营销手段 | 成本 | 效果强度 | 解锁条件 | 最佳用途 |
|---------|:---:|:--------:|---------|---------|
| **新闻稿(Press Release)** | 低 | 基础 | 初期可用 | 日常积累粉丝，低成本维持曝光 |
| **媒体图(Press Photo)** | 中 | 中 | 需雇佣艺术家制作 | 中期营销，增强单次营销效果 |
| **媒体片(Press Video)** | 高 | 最高 | 需雇佣艺术家制作视频 | 冲刺期爆发，发布前集中使用 |
| **试用版(Trial Version)** | 中(开发成本) | 高(转化率) | 产品Alpha完成后 | 让用户试玩→转化为购买，降低购买决策门槛 |

### 营销手段的协同效应

| 组合策略 | 效果 | 推荐场景 |
|---------|------|---------|
| 新闻稿+媒体图 | 基础+增强 | 持续运营型营销 |
| 媒体片+试用版 | 爆发+转化 | 大型产品发布前冲刺 |
| 全手段叠加 | 最大化热度 | AAA级产品首发 |

## 二、粉丝(Follower)系统

粉丝系统是 Software Inc. 区别于其他商业经营游戏的核心设计之一——它不是"发布后才积累口碑"，而是"发布前就积累粉丝，发布时引爆"。

### 粉丝积累阶段

| 阶段 | 行为 | 粉丝增长量 | 策略意义 |
|:---:|------|:---------:|---------|
| 产品发布前 | 新闻稿/媒体图/媒体片 | 慢速积累 | 建立初始受众 |
| 产品发布前 | 试用版发布 | 中速增长 | 转化意向用户 |
| 产品发布时 | 粉丝→首日购买 | 一次性转化 | 积累的粉丝直接转化为销售额 |
| 产品发布后 | 口碑传播(Review Score) | 持续增长 | 好评带来自然增长 |
| 产品发布后 | 差评 | 粉丝流失 | 坏产品消耗品牌资产 |

### 粉丝转化率

| 产品评分 | 粉丝转化率 | 说明 |
|:-------:|:---------:|------|
| 9-10(杰作) | ~70-90% | 高评分引爆粉丝积累 |
| 7-8(优秀) | ~50-70% | 良好转化 |
| 5-6(一般) | ~20-40% | 转化不佳 |
| <5(差评) | ~0-10% | 几乎无转化，甚至流失 |

> **关键机制**: 发布前积累的粉丝数量直接决定首日销量。这模拟了真实游戏产业的"愿望单+首发"逻辑——Steam上愿望单最多的游戏首发销量最高，而不一定是品质最高的。

## 三、发行渠道

| 渠道 | 抽成比例 | 固定成本 | 覆盖面 | 解锁条件 |
|-----|:-------:|:--------:|:-----:|---------|
| **数字平台(Digital Platform)** | ~30% | 无 | 广 | 默认可用 |
| **实体压盘(Physical Printing)** | 较低 | 需打印间+包装设备 | 中(特定市场) | 需建设打印间 |
| **自有在线商店(Own Store)** | 0% | 需服务器基础设施 | 窄(需自行推广) | 需购买服务器架设 |

### 渠道策略对比

| 策略 | 利润 | 风险 | 推荐阶段 |
|-----|:---:|:---:|:--------:|
| 仅有数字平台 | 低(抽成高) | 最低 | 初创期，无基础设施 |
| 数字+实体 | 中 | 中 | 中型公司，有打印间 |
| 自有商店为主 | 最高(零抽成) | 高(服务器费+推广难) | 巨头期，品牌已建立 |
| 全渠道 | 综合最优 | 中 | 旗舰产品发布 |

## 四、合同/Deal 系统

合同系统是 Software Inc. 的"保底收入"机制——在自研产品没有收入时维持现金流。

### 合同类型

| 合同类型 | 收入 | 周期 | 所需员工 | 战略价值 |
|---------|:---:|:---:|---------|---------|
| **外包开发(Outsource Dev)** | 中 | 几周-几月 | 程序员/艺术家 | 维持现金流+练兵 |
| **移植工作(Porting)** | 中-高 | 中 | 程序员(跨平台技能) | 积累跨平台经验 |
| **技术支持(Support)** | 低但持续 | 无限(可取消) | 服务类员工 | 稳定被动收入 |
| **IP授权(Licensing)** | 持续版税 | 长期 | 需洽谈 | 不放弃所有权获得收入 |
| **IP出售(Sell IP)** | 一次性高额 | 一次性 | 需洽谈 | 快速获取大额资金但失去IP |

### 合同系统运营要点

- **接待台+接待员**: 必须设置接待台并雇佣接待员才能接收外部合同
- **合同谈判**: 合同价格可协商，取决于公司声誉和谈判技巧
- **合同与自研的平衡**: 接太多合同会占用开发资源→影响自研进度

## 五、IP交易系统

IP交易是 Software Inc. 的战略性决策系统，涉及"现在赚钱还是未来赚钱"的权衡：

| 决策 | 短期收益 | 长期收益 | 风险 |
|-----|:-------:|:-------:|:----|
| **出售IP** | 一次性大量现金 | 零(失去IP) | 竞争对手可能利用该IP |
| **授权IP** | 前期费用低 | 持续版税收入 | 授权方可能做得不好影响品牌 |
| **保留自用** | 零 | 可持续开发续作 | 需持续投入研发 |

> **设计意图**: IP交易系统让玩家面临"短期财务压力 vs 长期品牌价值"的真实商业抉择。初创期可能被迫出售优质IP换取现金流——这是模拟了现实中游戏工作室"卖IP求生"的困境。

## 六、竞争系统

### AI 公司行为模型

| 维度 | 行为 | 对玩家的影响 |
|------|------|------------|
| **产品竞争** | AI公司开发同类产品 | 直接争夺市场份额 |
| **价格战** | AI公司降价促销 | 挤压利润空间 |
| **抢先发布** | AI公司赶在玩家之前发布 | 抢占先发优势 |
| **专利封锁** | AI公司研究专利限制他人 | 阻止玩家使用特定技术 |
| **人才竞争** | AI公司高薪挖角 | 核心员工流失风险 |

### 竞争情报

| 情报类型 | 获取方式 | 信息内容 | 战略价值 |
|---------|---------|---------|:-------:|
| **财报** | 查看公开信息 | AI公司收入/利润/员工数 | 判断对手规模 |
| **产品线** | 市场分析 | AI公司正在开发/已发布的产品 | 预判竞争 |
| **股票信息** | 股票交易界面 | 市值/股价/流通股 | 收购决策依据 |
| **招聘信息** | 人才市场 | AI公司正在招聘的职位 | 推断发展方向 |

### 市场新陈代谢

Software Inc. 中的 AI 公司不是"永远存在的对手"——它们也会经历发展、衰退和倒闭：

| 生命周期阶段 | 特征 | 对市场的影响 |
|-----------|------|------------|
| 初创期 | 小团队，1-2个产品 | 低威胁 |
| 成长期 | 产品线扩展，扩张招聘 | 中等威胁 |
| 成熟期 | 多产品线，高市场份额 | 主要竞争对手 |
| **衰退期/倒闭** | 产品老化，破产或被收购 | **市场空间释放→玩家机会** |

## 七、股票交易与公司收购

这是 Software Inc. 在商业经营品类中的独有"金融化"维度：

### 股票操作

| 操作 | 条件 | 效果 |
|-----|------|------|
| **买入股票** | 有可用资金 | 持有目标公司股份 |
| **卖出股票** | 已持有股份 | 变现获利 |
| **持股超过50%** | 持续买入 | **触发收购选项** |
| **全资收购** | 持有100%或触发收购 | 获得该公司控制权 |

### 收购后策略

| 策略 | 效果 | 推荐场景 |
|-----|------|---------|
| **保留子公司独立运营** | 子公司继续运作，母公司获分红 | 对手盈利能力强，品牌价值高 |
| **吸收合并** | 获得子公司的IP/专利/员工/技术 | 对手有优质IP或核心技术 |
| **拆分出售** | 卖掉子公司资产获利 | 对手有价值资产但运营不佳 |

**对比 Game Dev Tycoon**: GD Tycoon 的营销是"研发完成后选择宣传力度"的单次决策——你决定投入多少钱做营销，然后得到一次性的销量加成。Software Inc. 的粉丝系统是完全不同的模型——它是"持续积累→发布时引爆"的长期运营模式。玩家必须在数月甚至数年的开发周期中不断运营粉丝，积累越多，回报越大。这意味着营销不是产品完成后的一次性选择，而是贯穿整个开发周期的持续工作。此外，GD Tycoon 没有股票/收购系统——你只能经营自己的公司。Software Inc. 的金融化维度让你可以"用钱买增长"，从经营一家公司升级为管理一个商业帝国。

## 八、股价与市值影响因素

| 因素 | 对股价影响 | 说明 |
|-----|:---------:|------|
| 公司盈利状况 | 强正相关 | 持续盈利→股价上涨 |
| 产品发布成功 | 短期大幅上涨 | 高评分产品→利好 |
| 产品发布失败 | 短期大幅下跌 | 低评分产品→利空 |
| 市场份额变化 | 中长期影响 | 份额增长→长期利好 |
| 行业趋势 | 系统性影响 | AI公司所在行业的整体趋势 |
| 专利/技术突破 | 中等正向 | 重要技术成果→利好 |

## 九、设计特点分析

**"愿望单经济"的模拟**: Software Inc. 的粉丝系统本质上是对真实游戏产业"愿望单+首发"的抽象模拟。在 Steam 上，游戏的首发销量很大程度上取决于发布前的愿望单数量——而不是发布后的口碑。Software Inc. 将这一现实转化为"粉丝积累→发布转化"的机制，让玩家在开发阶段就必须认真经营社区关系。

**合同系统作为"双刃剑"**: 合同/Deal 系统是维持现金流的重要工具，但它也是一把双刃剑——接太多合同会占用核心开发资源，导致新产品延期。这模拟了现实中外包公司"做别人的项目就没时间做自己的产品"的困境。玩家必须权衡"活下去"和"做想做的事"之间的平衡。

**竞争系统的"新陈代谢"设计**: AI 公司也会老去和倒闭，这保持了市场的动态活力。与许多商业经营游戏中竞争对手"永远存在、永远竞争"的设计不同，Software Inc. 的市场是流动的——旧公司倒闭为新公司腾出空间，玩家既可以通过提高实力击败对手，也可以耐心等待对手自然衰落。

**金融化维度的战略深度**: 股票交易+公司收购系统让游戏超越了"经营一家软件公司"的范畴，进入了"管理商业帝国"的层次。玩家可以做出纯粹的战略性金融操作——买入有潜力的竞争对手股票、等待其升值、在适当时机收购。这在商业经营品类中极为罕见，更多常见于纯金融模拟游戏。

## 十、与同类游戏对比

| 维度 | Software Inc. | Game Dev Tycoon | 开罗·游戏开发物语 |
|------|:------------:|:--------------:|:----------------:|
| 营销模式 | 持续积累粉丝→发布引爆 | 单次选择宣传力度 | 单次广告投入 |
| 竞争对象 | AI公司(有生命周期) | AI公司(固定) | 排名系统(无实体) |
| 股票/收购 | **完整系统** | 无 | 无 |
| IP交易 | 出售/授权/保留三选一 | 仅有续作 | 仅有续作 |
| 合同系统 | 完整(外包/移植/支持) | 无 | 无 |
| 粉丝转化 | 发布时粉丝→销量 | 粉丝→保底销量 | 粉丝→保底销量 |
| 竞争深度 | 价格战/专利/抢先/挖角 | 评分竞争 | 评分竞争 |
