## 一、营销手段效果表

### 1.1 营销方式对比

| 手段 | 单次费用($) | 粉丝增量 | 持续效果 | 冷却时间 | 推荐使用时机 |
|:---:|:----------:|:--------:|:-------:|:-------:|------------|
| **新闻稿(Press Release)** | $500–$2,000 | +20–80/天 | 持续约1–3周(递减) | 约3–6个月 | Alpha阶段→发布前全程，早期建立初始热度 |
| **媒体图(Press Screenshot)** | $2,000–$5,000 | +40–150/天 | 持续约2–4周(递减) | 约3–6个月 | Beta阶段→发布前，展示产品视觉效果 |
| **媒体片(Press Trailer/Video)** | $5,000–$15,000 | +100–500/天 | 持续约3–6周(递减) | 约6–12个月 | 发布前冲刺(1–3个月前)，最大化发布热度 |
| **试用版(Press Build/Demo)** | 开发成本(额外2–10天) | 转化率+15%~30% | 发布时一次性爆发 | — | 发布时，将粉丝转化为实际购买 |

### 1.2 叠加效果与注意事项

| 策略 | 效果 | 风险 |
|:---:|:----:|:----:|
| 单次新闻稿 | 少量稳定粉丝增长 | 无显著风险 |
| 连续2次新闻稿(同一阶段) | 第二次效果减半 | 可能触发"垃圾信息"惩罚导致粉丝流失 |
| 新闻稿+媒体图(同一阶段) | 效果叠加(约1.3×) | 合理使用无惩罚 |
| 全手段同时使用(发布前) | 效果叠加(约2×) | 费用高，需现金流充足 |
| 同一产品在不同阶段重复营销 | 每次营销效果递减约20% | 需间隔足够长时间 |

> 关键发现：**试用版(Press Build)** 被社区公认为最有效的营销手段，对粉丝增长的贡献远大于新闻稿。许多资深玩家认为新闻稿"基本上是浪费钱"。

### 1.3 营销对销量的影响模型

```
发布当日销量 = 基础销量 × (1 + Hype系数) × (1 + 粉丝转化率)
Hype系数 = min(累积营销效果 / 阈值, 上限)
粉丝转化率 = 粉丝数 × 基础转化率(5–15%) × 评分修正 × 定价修正
```

| Hype等级 | 累积Hype值 | 销量加成 |
|:--------:|:---------:|:-------:|
| 低 | 0–100 | +0%~+20% |
| 中等 | 100–500 | +20%~+60% |
| 高 | 500–2,000 | +60%~+150% |
| 狂热 | 2,000+ | +150%~+250% |

### 3.2 Deal系统的关键规则

| 规则 | 说明 |
|:---:|------|
| **解锁条件** | 需要雇佣一名接待员(Receptionist)并设置接待室 |
| **收益特点** | Deal提供稳定被动收入，且Design Deal仅需设计师无需程序员 |

## 四、粉丝(Follower)与销量转化

### 4.1 粉丝→销量转化率

| 影响因素 | 影响范围 | 说明 |
|:-------:|:-------:|------|
| 基础转化率 | 5%–15% | 未受任何修正的基础值 |
| Hype系数修正 | ×0.5–×2.0 | Hype越高，粉丝购买意愿越强 |
| 产品评分修正 | ×0.3–×2.0 | 高分产品转化率大幅提升 |
| 定价修正 | ×0.5–×1.5 | 价格合理(不高于市场均价20%)时转化率最高 |
| 竞争强度修正 | ×0.3–×1.0 | 同品类竞品越多，转化率越低 |
| 时代契合度修正 | ×0.7–×1.3 | 产品类型与当前时代热度匹配加分 |

```
实际转化率 = 基础转化率 × Hype系数 × 评分修正 × 定价修正 × 竞争修正 × 时代修正
```

### 4.2 Hype(Hype/热度)的来源与衰减

| Hype来源 | 单次增加量 | 衰减速度 |
|:--------:|:---------:|:--------:|
| 新闻稿 | +10–40 | 快(每天-2~-5) |
| 媒体图 | +20–80 | 中(每天-1~-3) |
| 媒体片 | +50–250 | 慢(每天-0.5~-1.5) |
| 试用版 | +100–500 | 一次性(发布时消耗) |
| 社交媒体(被动) | +1–10/天 | 极慢(基于粉丝数的自然增长) |
| 续作品牌加成 | +200–2,000 | 极慢(品牌效应持久) |

## 五、营销预算推荐

### 5.1 按开发阶段的营销投入占比(占产品总预算)

| 开发阶段 | 推荐营销投入占比 | 说明 |
|:-------:|:--------------:|------|
| **Design阶段** | 0%–2% | 此时营销为时过早 |
| **Alpha阶段** | 5%–10% | 开始新闻稿营销，建立初步认知 |
| **Beta阶段** | 10%–20% | 增加媒体图和预告片，提升Hype |
| **发布前冲刺** | 20%–35% | 试用版+全手段组合，最大化发布效果 |
| **发布后** | 5%–10%(维持) | 维持曝光，但主要依赖口碑传播 |

### 5.2 按公司规模的年度营销预算

| 公司规模 | 年营销预算建议 | 占年收入比例 |
|:-------:|:-------------:|:----------:|
| 单人创业 | $0–$5,000 | 0%–5% |
| 小团队(4–15人) | $5,000–$50,000 | 5%–15% |
| 中型公司(16–50人) | $50,000–$500,000 | 10%–20% |
| 大型公司(50+人) | $500,000–$5,000,000+ | 10%–25% |

### 5.3 预算分配策略

| 策略类型 | 适用场景 | 分配方式 | 预期效果 |
|:-------:|---------|:-------:|:-------:|
| 平衡型 | 大多数情况 | 新闻稿30%+媒体图30%+媒体片40% | 稳定的粉丝增长和转化 |
| 冲刺型 | 发布前1–2个月 | 媒体片60%+试用版30%+新闻稿10% | 短时间内Hype爆发式增长 |
| 经济型 | 现金流紧张 | 新闻稿70%+自然增长30% | 低投入，慢但稳定 |
| 品牌型 | 续作/大项目 | 试用版40%+媒体片30%+媒体图20%+新闻稿10% | 以粉丝转化和口碑为核心 |

## 六、营销策略指导

### 6.1 常见营销误区

1. **过度投放新闻稿**：连续在同一阶段发送2次以上新闻稿，会触发"垃圾信息"惩罚，反而流失粉丝
2. **忽视试用版**：试用版是最有效的营销手段，但许多新手玩家跳过这一步
3. **过早营销**：在Design阶段大肆营销，Hype在发布前就已衰减殆尽
4. **发布后停止营销**：巅峰期的持续营销可大幅延长产品生命周期

### 6.2 最佳实践

```
理想营销节奏(以6个月开发周期为例):
第1–2月(Alpha初期): 第1次新闻稿
第3–4月(Beta阶段): 第2次新闻稿 + 媒体图
第5月(Beta末期): 媒体片(预告片)
第6月(发布前2周): 试用版开放
发布日: 所有渠道同步发布
发布后1–3月: 持续轻度营销(如有资金)
```
