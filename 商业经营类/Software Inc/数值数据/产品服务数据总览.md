## 二、功能依赖链 (Feature Dependency)

### 2.1 功能类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **时间锁(Time-locked)** | 随日历年份推移自动解锁 | 8-bit Audio(1980s) → HD Audio(1990s) |
| **依赖锁(Dependency-locked)** | 需要其他软件类型的某个功能 | Game Engine需要OS的3D Rendering |
| **升级树(Tree)** | 功能间存在替换/演进关系 | PC Speaker → 8-bit Audio → 16-bit Audio → HD Audio |

### 2.2 关键功能依赖关系

```
Operating System
  ├── File System (FS) → 基础
  ├── GUI → 2D Rendering → 3D Rendering
  ├── Networking → TCP/IP
  ├── Multitasking
  ├── Audio: PC Speaker → 8-bit → 16-bit → HD
  └── 64-bit Support

Game Engine (依赖OS)
  ├── 需要OS的: GUI/2D Rendering/3D Rendering/Audio/Networking
  ├── 提供: Physics/Shaders/Visual Scripting/VR
  └── 游戏依赖Game Engine + Visual Tool + Audio Tool

Game (依赖Game Engine + Visual Tool + Audio Tool)
  ├── 2D Graphics / 3D Graphics
  ├── AI / Multiplayer / MMO
  ├── Open World / Mod Support
  └── Cutscenes / Motion Capture
```

> 使用自有产品(如自己的OS、Game Engine)可免除许可费(License Fees)，大幅提升利润率。

## 四、开发周期参考表

| 阶段 | 典型时长 | 员工角色 | 说明 |
|------|:--:|------|------|
| **Design (设计)** | 5-30天 | Designer(Lead Designer创造力影响) | 设计文档进度条需居中——过少= worthless，过多= overly complicated |
| **Alpha (Alpha开发)** | 30-120天 | Programmer + Artist | 功能实现阶段，并行开发 |
| **Delay (延期)** | 0-30天 | 可选 | 可跳过，但跳过增加Bug数量 |
| **Beta (Beta测试)** | 10-60天 | Programmer | Bug修复阶段，Bug影响评分和销量 |
| **Release (发布)** | — | Service | 正式上线 |
| **Support (支持)** | 持续 | Service + Programmer | 发布后更新和Bug修补 |

> 注：以上时长受团队规模/技能等级/Lead效率/功能数量影响，为中等复杂度产品的典型值(社区估算)。

## 七、Lead Designer 创造力系统

| 参数 | 影响 | 备注 |
|------|------|------|
| **Creativity (创造力)** | 决定产品设计质量 | 最高5星(社区估算) |
| 高创造力→Innovation提升 | 卖得更久 | 产品滑块：Innovation/Stability/Usability |
| 高创造力→Usability提升 | 公司声望增长 | |
| 低创造力→产品平庸 | 评分低/销量差 | |

### 产品三滑块平衡

| 滑块 | 影响 | 备注 |
|:--:|------|------|
| **Innovation (创新)** | 产品能卖多久 | 高创新→长生命周期 |
| **Stability (稳定性)** | Bug数量/技能点获取 | 高稳定→少Bug |
| **Usability (易用性)** | 公司知名度(Brand Popularity) | 高易用→快积累声望 |

## 八、专利与版税

| 项目 | 收入占比(估算) | 说明 |
|------|:--:|------|
| 专利授权(Patent Licensing) | ~2-5%总收入 | 将技术专利授权给其他公司 |
| 版税(Royalty) | 每份产品$0.5-5(社区估算) | 按销量收取固定版税 |
| IP交易(一次性出售) | 不定 | 出售完整IP可获得大笔一次性收入 |

---

> **总结**：Software Inc. 产品系统核心在于软件类型选择+功能依赖链管理+Lead Designer创造力+三滑块平衡。成功策略通常是先用小品类(2D Editor/Compiler)积累资金，中期打造OS+Game Engine基础设施，后期通过自研游戏/硬件实现规模化变现。
