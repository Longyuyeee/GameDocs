# 属性养成与训练系统 — 漫画物语

> 游戏版本: The Manga Works (PC/Steam, Kairosoft, 2022)
> 数据来源: [Kairosoft Fandom Wiki](https://kairosoft.fandom.com/wiki/The_Manga_Works), [Steam商店](https://store.steampowered.com/app/1921770/), [萌娘百科](https://zh.moegirl.org.cn), [百度百科](https://baike.baidu.com)

## 一、核心四维属性

漫画物语中漫画家有四个核心属性，共同决定漫画的品质。

| 属性 | 英文名 | 影响维度 | 对应漫画品质 |
|------|:--:|------|------|
| **画技** | Drawing | 画面质量、易读性、视觉效果 | 读者对"好不好看"的第一印象 |
| **故事** | Story | 叙事质量、剧情编排、节奏控制 | 读者对"好不好看"的第二层判断 |
| **角色魅力** | Character | 角色塑造的深度、角色吸引力 | 读者是否喜欢作品中的角色 |
| **个性** | Personality | 作品的独特性、风格辨识度 | 作品"一看就知道是谁画的" |

### 四维的权重关系

不同阶段和不同用途的漫画，四维的权重可能有差异：

| 创作类型 | 画技权重 | 故事权重 | 角色权重 | 个性权重 |
|------|:--:|:--:|:--:|:--:|
| **短篇参赛** | ★★★★ | ★★★ | ★★★ | ★★★★★ |
| **连载(一般)** | ★★★★ | ★★★★ | ★★★★ | ★★★ |
| **连载(追求最高人气)** | ★★★★★ | ★★★★★ | ★★★★★ | ★★★★★ |
| **单行本销量** | ★★★ | ★★★★ | ★★★★★ | ★★★ |

> **注**: 具体权重的数值公式待补充。但整体而言，画技和故事是"基础分"，角色和个性是"加分项"——在基础分满足后，后两者决定上限。

## 二、工作量(Workload)——被忽视的关键属性

| 属性 | 英文名 | 功能 | 建议目标值 |
|------|:--:|------|:--:|
| **工作量** | Workload | 决定创作速度——越高画得越快 | 200+ |

工作量是**最被低估但最重要**的属性。它不直接提升漫画品质，但决定了创作效率——在5年时间限制下，创作速度直接决定了总产出量。

### 工作量提升途径

| 途径 | 效率 | 说明 |
|------|:--:|------|
| **训练** | ★★★★ | 消耗经验值直接提升 |
| **创作漫画** | ★★★ | 每完成一话微量提升 |
| **器材加成** | ★★ | 特定器材可能提供工作量加成 |
| **助手加成** | ★ | 部分助手可能提升工作量 |

> **设计洞察**: 工作量属性的存在传递了一个微妙的信号——"好"不只是品质问题，也是**速度问题**。在5年期限内，画得快意味着产出多，产出多意味着更多连载和更多单行本。这与现实漫画产业中"连载速度决定竞争力"的逻辑一致。

## 三、知识点(Plot Points)——创作的"燃料"

知识点虽然不被定义为"属性"，但它作为可消耗资源在创作中的核心地位使其值得在属性章节讨论。

### 知识点的作用

| 用途 | 说明 |
|------|------|
| **漫画创作** | 每话消耗知识点 → 决定篇幅和品质 |
| **连载维持** | 知识点递增策略 → 决定人气走势 |
| **特殊话触发** | 可能需要特定知识点投入(待确认) |

### 知识点的获取方式对比

| 来源 | 是否消耗金钱 | 是否消耗体力 | 是否消耗时间 | 初期推荐度 |
|------|:--:|:--:|:--:|:--:|
| **美术馆取材** | 是(最少) | 少量 | 是 | ★★★★★ |
| **咖啡厅取材** | 是(中等) | 少量 | 是 | ★★★★ |
| **图书馆取材** | 是(较高) | 少量 | 是 | ★★★ |
| **卡拉OK取材** | 是(最高) | 少量 | 是 | ★★(初期) |
| **调查流行(技能)** | 否 | 是 | 是(操作时间) | 多周目★ |
| **妄想(技能)** | 否 | 是 | 是(操作时间) | 多周目★ |
| **注视器材** | 否 | 是 | 是 | ★★ |

### 取材地点的选择策略

```
初期(第1年):   美术馆(最便宜, 性价比最高)
中期(第2-3年): 咖啡厅+图书馆(平衡花费和收益)
后期(第4-5年): 卡拉OK(高花费但收益上限高)
多周目:         技能"调查流行"+"妄想"(不花钱, 消耗体力)
```

## 四、训练系统

训练是提升四维属性和工作量的主要方式。

### 训练机制

| 要素 | 说明 |
|------|------|
| **消耗资源** | 经验值(通过漫画创作获得) |
| **训练目标** | 画技/故事/角色/个性/工作量 |
| **效果** | 直接提升目标属性 + 缩短未来创作时间 |
| **时间消耗** | 每次训练消耗一定游戏内时间 |
| **训练上限** | 属性本身有上限(待补充)，但多周目可突破 |

### 经验值的获取

| 来源 | 经验值获取量 | 频率 |
|------|:--:|:--:|
| **完成漫画** | 每话获得 | 每话 |
| **完美匹配(绝佳组合)** | 更高 | 使用最佳组合时 |
| **连载高人气排名** | 额外加成 | 排名越高经验加成越多 |
| **比赛获奖** | 一次性大量 | 获奖时 |

## 五、属性养成的综合策略

### 初期(第1年)——基础建设

```
优先: 工作量(Workload) → 200+
    ↑
次优先: 画技(Drawing) → 提升画面基础品质
    ↑
取材: 美术馆(低花费/高性价比)
```

### 中期(第2-3年)——均衡发展

```
画技: 主力提升(确保连载品质)
故事: 第二优先(叙事决定读者留存)
角色: 跟随提升(角色→单行本销量)
个性: 适度投入(独特性的边际收益)
工作量: 维持200+ → 继续提升至300+
```

### 后期(第4-5年)——冲刺配置

```
全属性均衡发展:
- 画技/故事/角色: 优先到较高水平
- 个性: 追求独特性的上限
- 工作量: 越高越好(决定5年内总产出)
```

## 六、器材与助手的属性加成

属性和训练不是孤立系统——它与设施和员工系统紧密耦合：

```
属性提升来源 = 训练(消耗经验)
              + 器材加成(注视升级)
              + 助手加成(薪资投资)
              + 基础属性(多周目继承)
```

## 七、设计特点分析

1. **"工作量"作为隐藏优先级** — 工作量不是四维属性之一，但它的实际重要性与四维并列——甚至更高(在初期尤其如此)。不优先提升工作量的玩家会发现创作速度极慢，5年内根本完不成足够的连载话数。这是漫画物语中的**隐藏数值引导**——游戏没有明确提示"请先升工作量"，但实际体验会教给玩家这个教训。

2. **四维结构 = 漫画品质的"多因子模型"** — 四维属性不是简单的"堆数值"，而是分别映射到漫画的不同评价维度。这种设计与游戏开发物语的"四维评分"(娱乐性/创意/画面/音效)同构，但更贴合漫画产业——好漫画需要画功(画技)+剧情(故事)+角色(角色)+风格(个性)。

3. **知识点的"双重定位"** — 知识点既是"创作燃料"(创作消耗)，又是"维护成本"(连载递增)。这种双重角色让它超越了普通资源——它既是增长引擎又是持续性投入。知识点不足时，玩家不能"暂时不做连载"——因为连载中断会触发休刊/腰斩。

4. **训练的"间接收益"** — 训练的直接收益是提升属性(品质提升)，但它的间接收益同样重要：更短的创作时间=更快的话数产出=更多的单行本=更高的总销量。这个"时间×品质"的复利效应是属性养成系统的深层策略。

## 八、跨游戏对比：与游戏开发物语的属性系统

| 对比维度 | 漫画物语 | 游戏开发物语 |
|------|------|------|
| **核心属性** | 画技/故事/角色/个性(4维) | 编程/编剧/美术/音效(4维) |
| **特殊属性** | 工作量(创作速度) | 体力/灵感(开发效率) |
| **可消耗资源** | 知识点(Plot Points) | 研究数据(Research Data) |
| **资源获取** | 外出取材+技能 | 合同外包+开发积累 |
| **训练方式** | 消耗经验值训练 | 培训课程(花钱) |
| **外部加成** | 器材(注视升级)+助手 | 办公室设备+道具 |
| **属性上限突破** | 多周目继承 | 无(单周目) |
| **属性与产出关系** | 属性→漫画品质+创作速度 | 属性→游戏评分 |
| **培养周期** | 5年(短周期→多周目) | 21年(长周期→单周目) |

> **核心差异**: 游戏开发物语的属性系统服务于"团队能力管理"——你培养的是不同职能的员工(谁编程、谁编剧)。漫画物语的属性系统服务于"个人创作者养成"——你培养的是你扮演的漫画家本人。前者向外看(团队组合)，后者向内看(自我提升)。漫画物语更像RPG式的人物培养，而游戏开发物语更像策略式的资源配置。
