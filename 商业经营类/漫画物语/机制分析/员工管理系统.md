## 一、助手系统概述

漫画物语中的"员工"就是**助手(Assistant)**——帮助漫画家完成漫画创作的辅助角色。与游戏开发物语的8人团队不同，漫画物语的助手数量有限且以"个人助理"形式存在。

### 助手的基本特征

| 特征 | 说明 |
|------|------|
| **数量上限** | 受房间扩建限制（需助手房） |
| **雇佣条件** | 房间扩建后解锁助手房 → 在助手房中雇佣 |
| **核心功能** | 提供属性加成（画技/故事/角色/个性） |
| **薪资模式** | 月薪制（每月自动支付） |
| **薪资增长** | 续聘一次 → 薪资递增约50% |
| **继承性** | 助手属性可多周目继承 |

---

## 二、薪资体系——指数的财务陷阱

### 薪资递增公式

```
助手月薪 = 基础月薪 × (1.5)^(续聘次数)
```

| 续聘次数 | 薪资倍率 | 假设基础$2,000 | 占初期收入比 |
|:--:|:--:|:--:|:--:|
| 0(首次雇佣) | 1.0x | $2,000 | ~20% |
| 1 | 1.5x | $3,000 | ~30% |
| 2 | 2.25x | $4,500 | ~40% |
| 3 | 3.375x | $6,750 | ~60% |
| 4 | 5.0625x | $10,125 | ~80%+ |
| 5 | 7.59375x | $15,187 | >100% |

> **关键风险**: 第4-5次续聘后，助手单人的月薪可能超过初期连载的全部稿费收入。如果玩家同时雇佣2-3名助手且都续聘多次，薪资指数增长将迅速压垮现金流。

### 多助手的叠加压力

| 助手数 | 每人平均月薪(假设续聘2次) | 总月薪支出 | 财务压力等级 |
|:--:|:--:|:--:|:--:|
| 1 | $4,500 | $4,500 | ★★(可控) |
| 2 | $4,500 | $9,000 | ★★★(需注意) |
| 3 | $4,500 | $13,500 | ★★★★(危险) |
| 3人各续聘4次 | $10,125 | $30,375+ | ★★★★★(致命) |

## 三、助手的属性加成

助手对漫画家有直接的属性加成。不同类型的助手可能在特定属性上有侧重（具体数值待补充）。

| 属性维度 | 助手加成方式 | 游戏开发物语类比 |
|------|------|------|
| **画技(Drawing)** | 直接提升画面质量 | 程序员(Code) |
| **故事(Story)** | 直接提升叙事品质 | 编剧(Scenario) |
| **角色(Character)** | 直接提升角色塑造 | 美术设计师(Pixel Art) |
| **个性(Personality)** | 直接提升独特性 | 音效师(Sound) |

### 助手与器材的协同效应

```
助手加成(基础属性提升)
    +
器材加成(注视升级提升)
    =
漫画家总属性 → 漫画品质
```

> **设计分析**: 助手和器材提供的是**同一维度的叠加加成**——它们都提升画技/故事/角色/个性四维。这意味着"格子管理"(器材)和"薪资管理"(助手)是同一套属性体系的两种投资方式：器材需要格子+注视时间，助手需要薪资+助手房。玩家需要权衡：同样的空间，放器材还是建助手房？

## 四、雇佣策略分析

### 初期(第1年)：不建议雇佣助手

| 理由 | 说明 |
|------|------|
| 薪资负担重 | 初期稿费收入低，$2,000+的月薪是沉重负担 |
| 无助手房 | 需扩建后才能解锁助手房 |
| 属性需求低 | 初期漫画对属性要求不高，单人足以应付 |
| 优先投资方向 | 资金应优先投入器材购买和房间扩建 |

### 中期(第2-3年)：考虑1名助手

| 考虑因素 | 说明 |
|------|------|
| 连载开始后 | 连载需要更稳定的产出效率，助手可提升属性 |
| 优先短期雇佣 | 完成特定高要求话数时可临时雇佣 |
| 选择高初始属性助手 | 优先选择画技/属性高的助手(具体数值待补充) |

### 后期(第4-5年)：可维持1-2名助手

| 条件 | 说明 |
|------|------|
| 收入稳定 | 多连载+单行本版税可覆盖薪资 |
| 瞄准高等级 | 冲刺漫画之神需要全属性最大化 |
| 留意薪资递增 | 长期雇佣的薪资增长可能超过收入增速 |

## 五、多周目继承——助手属性的延续

| 继承项 | 是否继承 | 说明 |
|------|:--:|------|
| **助手属性(等级)** | 是 | 助手在上一周目积累的属性部分保留 |
| **助手薪资** | 待确认 | 薪资状态可能重置或部分继承 |
| **雇佣状态** | 否 | 新周目需要重新雇佣 |

> 多周目继承意味着：**越早培养一个助手，他在后续周目的价值就越高**。虽然一周目中期雇佣助手可能造成财务压力，但考虑到多周目继承，长期的属性积累回报可能超过短期薪资支出。这是一个"短期痛苦→长期收益"的设计。

## 六、设计特点分析

1. **指数级薪资的设计意图** — 助手薪资递增50%/次不是数值设计失误，而是刻意的**财务压力阀**。它制造了"可以雇但不能无限制续聘"的抉择——开罗想让玩家思考"什么时候该换一个新助手而不是一直续聘同一个"。这类似于游戏开发物语中"员工会退休"的设计——逼迫人才流动。

2. **助手作为"资产"而非"耗材"** — 多周目继承让助手从"每月花钱的负担"变成了"可以长期投资的资产"。这是开罗系列中少见的"跨周目员工培养"设计——大多数开罗游戏(包括游戏开发物语)的员工都是单周目的。

3. **雇佣时机的经济权衡** — "初期不要雇，中期谨慎雇，后期有限雇"的节奏是开罗精心设计的。太早雇会被薪资拖垮，太晚雇又错过属性积累。这个"时机窗口"创造了一局游戏中的策略弧线。

4. **与游戏开发物语的本质差异** — 漫画物语的助手是**个人助理**（辅助一个人），游戏开发物语的员工是**团队成员**（各司其职）。漫画物语不需要"项目管理"——助手不负责独立任务，只提供属性加成。这简化了管理深度，但保留了"薪资压力"这个核心经济决策。

## 七、跨游戏对比：与游戏开发物语的员工系统

| 对比维度 | 漫画物语 | 游戏开发物语 |
|------|------|------|
| **员工角色** | 助手(个人助理) | 开发者(团队成员) |
| **数量上限** | 少量(受助手房限制) | 8人(团队上限) |
| **核心功能** | 属性加成(4维) | 分工开发(编程/编剧/美术/音效) |
| **薪资模式** | 月薪+续聘递增50% | 年薪(每年3月固定) |
| **薪资压力** | 指数增长(续聘陷阱) | 固定增长(随招聘新员工) |
| **多周目继承** | 助手属性继承 | 完全不继承 |
| **培养系统** | 无"转职"概念 | 完整转职树(6路线×5等级) |
| **雇佣方式** | 助手房直接雇佣 | 招聘(签约金+年薪) |
| **辞退成本** | 待补充 | 无(直接解雇) |

> **核心差异**: 游戏开发物语的员工系统是"生产线的齿轮"——每个人在自己岗位上产出(程序员写代码、编剧写剧情)。漫画物语的助手系统是"创作者的Buff"——每个人不是独立产出，而是提升漫画家的属性。前者是团队管理模拟（多个角色各司其职），后者是个人能力强化（外挂属性包）。这种差异源于两个产业的本质区别：游戏开发是团队协作，漫画创作（至少在这个游戏里）是个人艺术家+辅助。
