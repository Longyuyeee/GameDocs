## 一、时间结构——5年的漫画家生涯

漫画物语以**天**为最小行动单位，在**5年**的总时限内展开。这与大多数开罗游戏的"年/周"回合制不同——天数制带来了更细腻的时间管理和更直接的压力体验。

### 基本时间单位

| 单位 | 内容 | 与游戏开发物语的对比 |
|------|------|:--:|
| **1天** | 最小行动单位。每天可执行1次主要行动 | 游戏开发物语无"天"概念 |
| **1话周期** | 连载创作通常需要数天(取决于工作量和模式) | 游戏开发以"周"为单位 |
| **1月** | 约30天，包含固定事件 | 4周/月(游戏开发物语) |
| **1年** | 365天，漫画家年龄+1 | 12个月(游戏开发物语) |
| **5年(总长)** | 25岁生日=游戏结束 | 21年(游戏开发物语) |

## 二、3种工作模式

玩家可以切换工作模式，平衡"工作产出"和"体力消耗"。

| 工作模式 | 工作时间 | 体力消耗速度 | 工作产出 | 适用场景 |
|------|:--:|:--:|:--:|------|
| **休闲(Relaxed)** | 9:00-18:00 | 慢 | 低 | 初期体力不足/取材日 |
| **普通(Normal)** | 9:00-23:00 | 中 | 中 | 日常创作/连载期 |
| **残酷(Cruel)** | 9:00-次日4:00 | 快 | 高 | 赶稿/冲刺连载截止日 |

### 模式切换策略

```
连载截止日前: 残酷模式(高产出但体力消耗大)
    ↓
体力低时: 切换休闲模式(恢复体力/减少消耗)
    ↓
自由时间: 取材/训练/购物(休闲或普通模式)
```

> **设计特点**: 三种工作模式是漫画物语"时间管理"的核心杠杆。它们将"体力"和"时间"两个资源连接起来——你可以在短时间内高强度产出(残酷模式)，但要以体力为代价。这创造了一个经典的策略抉择：**你想活得久(低强度但时间长)还是死得快(高强度但短命)**？

## 三、体力系统——"别把自己画进医院"

体力(Stamina)是漫画物语的核心生存资源。

| 体力状态 | 效果 | 应对策略 |
|------|------|------|
| **正常(>30%)** | 正常工作 | — |
| **低(<30%)** | 工作效率下降(可能) | 减少工作量/切换休闲模式 |
| **耗尽(=0)** | 送入医院 | 住院期间无法工作 |

### 住院的后果

```
体力=0 → 住院(数天无法工作)
         ↓
    连载截止日逼近 → 延期/休刊
         ↓
    出版社催稿 → 好感度降低
         ↓
    休息恢复体力 → 出院后追赶进度
```

**住院的双重惩罚**：
1. **时间损失** — 住院期间完全无法工作，连载进度落后
2. **关系损失** — 出版社催稿，好感度降低

## 四、连载截止日与催稿压力

每话连载都有严格的截止日。

| 截止日要素 | 说明 |
|------|------|
| **截止日设定** | 每话开始连载时设定（通常为7-30天内，取决于连载节奏） |
| **剩余天数显示** | 玩家界面上持续显示剩余天数（持续的视觉压力） |
| **延期机制** | 超时后最多延期1天。延期后截止日自动延后但人气会下降 |
| **连续延期后果** | 连续多话延期=休刊触发条件 |
| **出版社催稿** | 截止日逼近时出版社开始催促（文本提示/好感度压力） |

### 截止日的时间管理

```
连载第1天: 充满信心(剩余时间充裕)
    ↓
连载中期: 开始紧张(剩余时间不足一半)
    ↓
截止日前1-2天: 切换残酷模式赶稿
    ↓
截止日: 提交/延期(如果未完成)
```

## 五、5年期限——游戏结束与多周目

### 结局触发

| 条件 | 结果 |
|------|------|
| **漫画家25岁生日** | 游戏结算+结束 |
| **结算内容** | 总销量/最高等级/总收入/出版社关系等 |
| **多周目选项** | 可选择继承部分数据重新开始 |

### 5年设计的心理学

| 维度 | 分析 |
|------|------|
| **时间紧迫感** | 5年≈60个月≈1800天，给玩家"倒计时"的压力 |
| **一周目不完美** | 正常一周目不可能达到漫画之神——需要多周目 |
| **继承的诱惑** | 多周目继承让每一周目都"离终点更近一步" |
| **重玩价值** | 每次重开都可以尝试不同的策略路线 |

> **设计特点**: 5年时间限制是开罗游戏中最短的周期之一（游戏开发物语21年、温泉物语无限）。这种短周期设计迫使玩家做出**高密度的决策**——没有"等等看"的时间。同时，它也是"多周目"的完美载体：一周目探索和学习，二周目开始认真规划，三周目冲刺目标。

## 六、自由时间的利用

不画画的时候（等待连载期间、取材日），玩家可以从事多种活动：

| 活动 | 体力消耗 | 时间消耗 | 收益 |
|------|:--:|:--:|------|
| **外出取材(美术馆/咖啡厅等)** | 少量 | 半天-1天 | 知识点 |
| **训练** | 中量 | 半天-1天 | 属性提升 |
| **购物** | 无 | 半天 | 购买器材/主题/舞台 |
| **注视器材** | 少量 | 数小时 | 器材升级 |
| **休息** | 恢复 | 半天-1天 | 体力恢复 |

## 七、压力地图——5年的压力曲线

```
压力等级
    ↑
高  |                    [第4-5年]
    |                  /  多连载+多人催稿
    |       [第2-3年] /
中  |     / 连载维持压力  /
    |   / 初期成长压力 /
    |  / 资金紧张   /
低  | / [第1年]   /
    |/ 探索期    /
    +------------------------→ 时间
    第1年  第2年  第3年  第4年  第5年
```

| 阶段 | 时间 | 核心压力源 | 压力等级 |
|------|:--:|------|:--:|
| **探索期** | 第1年 | 资金紧张、知识不足 | ★★(可承受) |
| **成长痛苦期** | 第2-3年 | 连载维持、知识点递增、助手薪资膨胀 | ★★★★(最危险) |
| **成熟运营期** | 第4-5年 | 多出版社管理、冲刺等级目标 | ★★★(可控压力) |

## 八、设计特点分析

1. **天数制的差异化体验** — 大多数开罗游戏使用"周"或"月"作为基本时间单位，而漫画物语使用"天"。这带来了更细腻的时间管理：玩家需要规划每一天做什么——今天取材还是画画？今天休息还是赶稿？这种"一天一行动"的节奏让玩家更有"创作者的一天"的代入感。

2. **体力作为"高压阀"** — 体力系统是漫画物语中的"不能无视"的限制器。它不像游戏开发物语那样"可以无限加班"——在这里拼命工作真的会进医院。这种设计传递了明确的信息：**漫画家需要照顾好自己**。现实中的漫画家因过劳住院的案例并不少见，这个机械层面的设计也起到了叙事层面的共鸣作用。

3. **5年短周期的策略意义** — 5年周期在所有开罗游戏中属于最短之一。短周期意味着每局游戏都能在相对短时间内完成(现实时间2-4小时)，使得"再来一局"的门槛很低。同时，1亿销量的漫画之神目标在5年内几乎不可能达成——多周目继承成为必然，这又增加了重玩动机。

4. **催稿系统的"软压力"** — 漫画物语的催稿不是游戏结束的硬惩罚(如破产)，而是持续的好感度软压力。出版社不会因为你延期一次就腰斩你，但连续延期会逐渐恶化关系。这种"温水煮青蛙"式的压力设计比硬性惩罚更具沉浸感——它模拟了现实中编辑对作者的那种"礼貌但持续"的催促。

## 九、跨游戏对比：与游戏开发物语的时间系统

| 对比维度 | 漫画物语 | 游戏开发物语 |
|------|------|------|
| **基本时间单位** | 天 | 周 |
| **总时长** | 5年(到25岁生日) | 21年 |
| **工作模式** | 3种(休闲/普通/残酷) | 无工作模式(固定效率) |
| **体力系统** | 有(体力=0→住院) | 无(员工无体力概念) |
| **截止日系统** | 连载截止日+延期 | 合同截止日(无延期) |
| **终局条件** | 5年时限 | 21年时限 |
| **多周目** | 有(器材/助手继承) | 无 |
| **时间压力风格** | 每日管理(细腻) | 每周管理(宽松) |
| **失败惩罚类型** | 住院+腰斩(双重) | 合同失败+低评分(财务) |

> **核心差异**: 游戏开发物语是"CEO时间"——你管理一家公司，时间以周为单位流过，关注的是项目和团队。漫画物语是"创作者时间"——你管理自己的职业生涯，时间以天为单位流过，关注的是体力、截止日、连载节奏。前者像经营战略游戏，后者像生存模拟游戏。漫画物语在开罗体系中是**时间压力最密集**的作品之一。
