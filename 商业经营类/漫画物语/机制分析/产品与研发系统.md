## 一、漫画创作流程

漫画是游戏中的"产品"。创作核心流程分为四步：

```
选择主题 → 选择舞台 → 投入知识点(Plot Points) → 完稿并选择用途
```

### 创作流程详解

| 步骤 | 操作 | 影响因素 | 产出影响 |
|------|------|------|------|
| **1. 选主题** | 从已解锁主题中选择 | 主题来源：初始/商店购买/短篇委托/比赛解锁 | 匹配度→初始品质 |
| **2. 选舞台** | 从已解锁舞台中选择 | 舞台来源：与主题相同渠道 | 匹配度→初始品质 |
| **3. 投入知识点** | 选择投入点数(消耗Plot Points) | 当前拥有的知识点总量 | 投入越多→篇幅越长+品质越高 |
| **4. 完稿用途** | 选择存稿/自荐/参赛 | 当前策略+出版社状态 | 决定收入/曝光/风险 |

## 二、主题×舞台组合系统

这是漫画物语的**核心组合公式**——与游戏开发物语的"类型×题材"同构。

### 组合匹配度等级

| 匹配度等级 | 效果 | 出现条件 |
|------|------|------|
| **绝佳!(Perfect!)** | 初始品质最高、经验最多、篇幅最长 | 主题与舞台高度匹配 |
| **有创意(Interesting)** | 较高初始品质 | 主题与舞台较匹配 |
| **还不错(Not Bad)** | 中等初始品质 | 主题与舞台有一定关联 |
| **嗯...(Hmm)** | 较低初始品质 | 主题与舞台关联弱 |
| **不怎么样(No Good)** | 最低初始品质 | 主题与舞台不匹配 |

> **匹配度判断方法**: 游戏不直接展示匹配度等级，玩家需要**通过试错或查表**来掌握最佳组合。这是一个"半隐藏规则"设计——驱动社区自发整理最佳组合表(UGC)。

### 主题与舞台的来源

| 解锁方式 | 说明 |
|------|------|
| **初始已有** | 游戏开始时提供基础的主题和舞台 |
| **商店购买** | 花费金钱购买新的主题/舞台组合包 |
| **短篇委托奖励** | 完成特定短篇委托→解锁该委托相关的主题/舞台 |
| **比赛获奖奖励** | 赢得漫画比赛→解锁特殊主题/舞台 |
| **多周目继承** | 已解锁的主题/舞台可在新周目继续使用 |

## 三、知识点(Plot Points)管理系统

知识点是漫画创作的"燃料"——它决定了每一话的篇幅和品质。

### 知识点的获取

| 获取途径 | 说明 | 初期效率 | 后期效率 |
|------|------|:--:|:--:|
| **外出取材(美术馆)** | 花费金钱+时间，获得知识点 | ★★★★★ | ★★ |
| **外出取材(咖啡厅)** | 花费金钱+时间，获得知识点 | ★★★★ | ★★★ |
| **外出取材(图书馆)** | 花费金钱+时间，获得知识点 | ★★★ | ★★★★ |
| **外出取材(卡拉OK)** | 花费金钱+时间，获得知识点+特殊事件 | ★★ | ★★★★★ |
| **"调查流行"技能** | 消耗体力换取知识点(多周目技能) | 初期不可用 | ★★★★★ |
| **"妄想"技能** | 消耗体力换取知识点(多周目技能) | 初期不可用 | ★★★★★ |
| **注视器材** | 部分器材注视时可能产生知识点 | ★★ | ★★ |
| **随机事件** | 各类日常事件触发知识点奖励 | 随机 | 随机 |

### 知识点的消耗

| 用途 | 消耗量 | 说明 |
|------|:--:|------|
| **创作漫画** | 每话至少消耗1点 | 投入越多→篇幅越长→稿费越高 |
| **连载维护** | 续话需投入 | 连载期间每话都需要知识点 |
| **特殊话触发** | 特定条件 | 某些特殊话需要额外知识点触发(待确认) |

## 四、短篇委托系统

短篇委托是出版社单方面发布的任务：

| 委托要素 | 说明 |
|------|------|
| **触发方式** | 出版社主动发来委托要求 |
| **主题要求** | 特定主题/舞台组合(如"冒险×海贼") |
| **时间限制** | 需要在截止日前完成 |
| **报酬** | 固定稿费(高于普通投稿) |
| **核心奖励** | 完成委托→解锁新的主题/舞台 |
| **失败惩罚** | 超时可能降低出版社好感度 |

> **短篇委托的策略意义**: 解锁新组合是漫画物语内容推进的核心方式。初期主题/舞台有限，不通过短篇委托解锁新组合，后续漫画的品质上限会被锁定。因此短篇委托不只是"赚钱任务"，更是**内容扩展的关键路径**。

## 五、存稿/自荐/参赛三路径

详见[顾客与市场系统](./顾客与市场系统.md)的投稿渠道章节。此处简要列出对"研发"层面的影响：

| 路径 | 研发层面的意义 |
|------|------|
| **存稿** | 自动排队无需操作，适合批量创作后的"产能释放" |
| **自荐** | 研发成果直接接受检验，反馈最快 |
| **参赛** | 研发成果接受评审，但等待期长——期间可继续研发新作品 |

## 六、创作效率的影响因素

| 因素 | 影响方式 |
|------|------|
| **工作量(Workload)** | 越高→创作速度越快(优先升到200+) |
| **画技属性** | 影响画面质量和完成度 |
| **故事属性** | 影响叙事品质和篇幅效果 |
| **角色属性** | 影响角色塑造深度 |
| **个性属性** | 影响独特性和评价 |
| **器材加成** | 各类器材提供属性增益→间接提升创作效率 |
| **助手加成** | 助手提供属性增益→间接提升创作效率 |

## 七、设计特点分析

1. **主题×舞台 = 开罗组合公式的漫画版** — 这是开罗最核心的"配方"设计模式的又一次应用。与游戏开发物语的"类型×题材"、拉面店的"汤底×面条×配料"完全同构——玩家需要探索最佳组合、解锁新组合、然后用高匹配度组合产出高品质产品。

2. **知识点作为"创作燃料"的经济循环** — 知识点是漫画物语中最核心的可再生资源。它的获取(取材/技能)和消耗(创作)形成了一个"能量循环"：时间→知识点→漫画→收入→更多时间(通过器材减轻工作量)→更多知识点。这种循环与游戏开发物语的"研究数据"系统同构，但更贴近"创作者的灵感管理"主题。

3. **半隐藏规则驱动社区** — 主题×舞台的最佳匹配表不是直接给出的，而是需要玩家自己试验和总结。这与其他开罗游戏一样，是故意设计的"半隐藏规则"——它驱动玩家在社区中分享、整理、验证组合表，形成UGC循环。

4. **短篇委托作为内容解锁器** — 短篇委托是漫画物语区别于其他开罗游戏的独特设计。它不是一个"可选支线"，而是**主线内容扩展的必经之路**。没有短篇委托解锁新主题/舞台，玩家的创作很快就会陷入"重复同样组合"的瓶颈。

## 八、跨游戏对比：与游戏开发物语的产品研发系统

| 对比维度 | 漫画物语 | 游戏开发物语 |
|------|------|------|
| **核心组合** | 主题×舞台(匹配度5级) | 类型×题材(匹配度5级) |
| **研发资源** | 知识点(Plot Points) | 研究数据(Research Data) |
| **资源获取** | 外出取材+技能 | 合同外包+开发积累 |
| **品质决定因素** | 匹配度+知识点投入量+4维属性 | 匹配度+员工能力+平台性能 |
| **产品出口** | 存稿/自荐/参赛 | 直接发布(选择平台) |
| **多周目继承** | 继承解锁的主题/舞台 | 不继承(完全重开) |
| **内容解锁路径** | 短篇委托+比赛获奖 | 无特定的"解锁路径"(随年份自动开启) |
| **委托/合同系统** | 短篇委托(解锁新内容) | 合同外包(赚取研究数据) |

> **核心差异**: 游戏开发物语的产品研发是**团队协作**（员工能力决定品质），而漫画物语是**个人创造**（漫画家本人的属性和知识点决定品质）。前者多了"人员管理"的维度，后者多了"灵感/知识管理"的维度。但底层的组合公式（A×B→匹配度→品质）完全一致——这是开罗模板迁移的最佳例证。
