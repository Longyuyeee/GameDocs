## 一、顾客类型一览表

| 类型 | 出现条件 | 预算范围(社区估) | 购买偏好 | 出现概率 | 特殊行为 |
|------|---------|:--------------:|---------|:-------:|---------|
| 普通顾客 (Regular) | 初始即出现 | 中(约$10~$50/次) | 均衡——食品/饮料/日用品 | ~85~90% | 标准购物行为：进店→选购→排队→结账 |
| 批发型顾客 (Bulk Buyer) | 店等级~20+ | 高(约$50~$200+/次) | 大宗采购，偏好基础食材/饮料 | ~3~5% | 购买量显著大于普通顾客，多次往返货架 |
| 小偷 (Thief) | 店等级~15+开始出现 | N/A(不付钱) | 高价值小体积商品(寿司/牛排/酒类) | ~1~5%(随等级提高) | 行为详见第六节 |
| 配送客户 (Online Delivery) | v1.0解锁配送系统~Lv50+ | 中(线上订单) | 随机商品组合 | N/A(非到店) | 线上建单，需玩家/员工打包配送 |
| 挑剔顾客 (Picky) | 所有等级 | 低~中 | 对价格极度敏感 | 隐藏属性 | 价格略高即触发投诉 |

> 注：顾客类型数据主要来自社区观察，游戏内无明确分类界面。配送客户为v1.0新增，数据待进一步验证。

## 二、顾客行为状态表

| 状态阶段 | 动作描述 | 持续时间(社区估) | 触发条件 | 影响因素 |
|---------|---------|:--------------:|---------|---------|
| 进店 (Enter) | 从门口进入商店 | ~1~3秒 | 每日开店后分批进入 | 店等级(决定总客流) |
| 取购物篮 (Get Basket) | 在门口处取篮 | ~1~2秒 | 进店后自动触发 | 无(必执行) |
| 浏览选购 (Browse) | 在货架间行走并选取商品 | ~30秒~3分钟 | 根据商品需求和货架位置 | 商品多样性、货架布局、过道宽度 |
| 排队等待 (Queue) | 在收银台前排队 | ~1秒~10分钟 | 选购完成后寻找最短路队 | 收银台数量、收银员效率 |
| 结账 (Checkout) | 逐件扫描商品并支付 | ~5秒~2分钟(视商品数) | 到达收银台 | 收银员速度等级、商品数量 |
| 付款 (Pay) | 现金/刷卡支付 | ~3~5秒 | 扫描完毕 | 支付方式(现金需要找零) |
| 离店 (Exit) | 携带商品离开商店 | ~2~5秒 | 付款完毕 | 收银台位置(靠近出口最优化) |

## 三、投诉规则表

| 投诉类型 | 触发条件(社区共识) | 惩罚 | 规避方法 |
|---------|-----------------|------|---------|
| 价格过高 (Overpriced) | 售价超过市价一定阈值(约+5%~+10%以上开始出现) | 失去这笔销售；扣除1店评分/次 | 保持售价不超过+5%(零投诉)或接受有限投诉 |
| 缺货 (Out of Stock) | 货架空置超过一定时间(社区估：对应商品连续缺货2~3天) | 失去潜在销售；扣除店评分 | 确保补货及时，不要购买无法持续供货的许可证 |
| 排队过长 (Long Queue) | 排队人数超过N人(社区估：等待>3~5人) | 顾客直接离店不购买；扣店评分 | 增加收银台/雇佣收银员/部署自助结账 |
| 环境脏乱 (Dirty) | 地面有污渍未清理 | 小幅扣分 | 定期拖地 |

### 投诉对游戏进度的量化影响

| 指标 | 影响 |
|------|------|
| 单次投诉 | **-1 店评分** |
| 店评分与等级的关系 | 店等级提升需要累积店评分(经验值) |
| 低等级时期(Lv<30) | 投诉会显著拖慢等级提升速度 |
| 高等级时期(Lv30+) | 金钱成为主要瓶颈，投诉影响相对较小 |
| 顾客流失 | 频繁投诉的顾客可能不再来店(社区估) |

### 客流量关键机制

| 机制 | 说明 |
|------|------|
| 客流量决定因素 | **店等级是唯一主要决定因素**（开发者确认）。店铺面积、产品种类数量对客流量的影响极小或为零 |
| Lv45软上限 | 约在Lv45~75之间，客流量停留在约60人/天，成长期望落差最大的阶段 |
| 折扣对客流的加成 | **1%折扣 = +50%客流上限**（注意：是"上限"而非直接+50%），折扣力度超过1%不会带来额外客流增益 |
| 折扣上限 | 最高额外增加+50%客流（即1%折扣即可触发最大值，更高折扣无额外收益） |
| 客流量与店评分的关联 | 评分高可能略微提升客流（社区推测），但核心仍然是店等级 |

## 五、折扣对客流的加成详细数据

| 折扣力度 | 客流加成 | 备注 |
|:-------:|:-------:|------|
| 0%(无折扣) | 基准客流 | 100%基础客流 |
| 1%折扣 | **+50%** | 已达上限，更高的折扣无额外客流收益 |
| 5%折扣 | +50%(已达上限) | 无额外增益，反而降低利润率 |
| 10%折扣 | +50%(已达上限) | 不推荐——不必要的利润损失 |
| 20%折扣 | +50%(已达上限) | 强烈不推荐 |
| 40%折扣×5件商品 | +50%(已达上限) | 部分社区玩家用此方法布局"促销区" |

> 结论：1%的最小折扣即可获得全部客流加成，因此策略上只需对1件商品设1%折扣即可。

## 六、小偷行为参数

| 参数 | 数值(社区估算) | 说明 |
|------|:------------:|------|
| 出现等级 | Lv15+开始出现 | 低等级时不会遇到小偷 |
| 基础出现概率 | ~1~5%/天(随等级提高) | 店等级越高，小偷出现概率和频率增加 |
| 偷窃目标偏好 | 高价值小体积商品 | 社区报告：寿司、牛排、酒类(威士忌/红酒)是主要目标 |
| 发现难度 | 中(需注意行为异常) | 小偷特点：不拿购物篮、四处张望、不走寻常路线 |
| 偷窃速度 | ~2~3秒完成 | 从触碰到收入口袋很快 |
| 被发现后行为 | 加速跑向出口 | 未标记时不加速，被标记(摄像头/玩家注视)后可能加速逃逸 |

### 安防反制效率

| 安防手段 | 效率(社区评价) | 说明 |
|---------|:------------:|------|
| 无安防 | 极低 | 小偷可自由偷窃后离开 |
| 摄像头(Camera) $500 | 高 | 自动标记进入视线的小偷。一个摄像头+合理摆放≈全店覆盖 |
| 报警门禁(Alarm Gate) $350 | **低(有Bug风险)** | 出门触发警报，但小偷会超速逃跑，保安可能卡地图外——社区普遍不建议购买 |
| 保安巡逻(Security Guard) | 中~高 | 自动追逐已标记的小偷并将其拦截；雇员费用持续支出 |
| 玩家手持棒球棒手动拦截 | 高(但费时) | 玩家手动标记并击打小偷迫使其掉落赃物 |
| 组合：摄像头+保安 | **最高** | 社区公认最佳组合：摄像头自动标记→保安自动拦截 |
