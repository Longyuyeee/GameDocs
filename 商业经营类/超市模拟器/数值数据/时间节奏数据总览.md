## 1. 每日时间线详细表

游戏内一天分为"营业前→营业→关店结算"三个大段。顾客仅在营业时段进入。以下时间线基于游戏默认设置（无加速/暂停修改）。

| 时间段 | 游戏时间 | 活动类型 | 玩家操作重点 |
|------|:--:|------|------|
| 营业前 | 7:00-8:00（约3分钟现实时间） | 补货窗口 | 检查库存→从储藏室搬运商品→摆放上架→检查冷链设备运行正常 |
| 开店 | 8:00 | 切换状态 | 走到收银台→切换至"营业中"状态→首批顾客涌入 |
| 早高峰 | 8:00-10:00 | 收银+补货 | 收银为主→客流密度高→空隙时间快速补货 |
| 平峰段 | 10:00-14:00 | 综合运营 | 补货→清洁→整理货架→注意小偷→（可选）前往批发市场 |
| 午高峰 | 14:00-17:00 | 收银压力期 | 客流再次上升→收银压力增大→必要时从补货岗位切换回收银 |
| 晚高峰 | 17:00-20:00 | 最忙时段 | 全天客流峰值→收银+补货+清洁三线并行→最容易出现"排队投诉"的时段 |
| 关店 | 20:00 | 结算 | 自动切换至"已关店"→当日利润结算→显示关键数据指标→进入下一天 |

**关键时间节点说明**：

| 节点 | 意义 | 策略提示 |
|------|------|------|
| 7:00-7:59 | 唯一无顾客的补货窗口 | 优先补货冷链商品，常温商品可稍后边营业边补 |
| 8:00 | 第一批顾客进店 | 确保收银台有人值守 |
| 10:00 | 早高峰消退 | 可利用此时去批发市场（往返耗时少） |
| 14:00 | 午高峰开始 | 确保所有收银台正常运行 |
| 17:00 | 晚高峰开始 | 如有多名员工，确保他们在岗；自助结账机开启 |
| 19:30 | 最后一批顾客进店 | 之后只出不进，排队顾客会逐渐减少 |
| 20:00 | 强制结算 | 无论是否完成补货，立即结算 |

## 2. 周循环结构

与牧场经营类游戏（如《星露谷物语》有周末集市/节日）不同，超市模拟器**没有周循环机制**。

| 维度 | 情况 | 说明 |
|:--:|------|------|
| 周末 vs 工作日 | 无区别 | 顾客行为模式与客数每天相同 |
| 节假日 | 无内置节假日 | 无圣诞/新年等促销事件（等待社区Mod支持） |
| 周期性活动 | 无 | 纯粹每日重复循环，无周/月活动 |

**设计含义**：游戏节奏完全由"日"这一时间单位驱动。玩家每天面对相同的运营挑战——补货→收银→清洁→进货的循环。变化来源于：
- 玩家自身的扩张（新许可证→新商品→客数自然增长）
- 随机事件（小偷出现频率、特定商品销量波动）
- 1.0新增的折扣和配送系统引入了每天的决策变量

## 3. 批发采购节奏

| 参数 | 数值（估算） | 说明 |
|------|:--:|------|
| 推荐采购频率 | 每2-3天一次 | 取决于商品种类数和销量 |
| 单次耗时（现实时间） | 约5-10分钟 | 包含开车往返+选购+装载 |
| 装满一车可支撑 | 2-4天营业 | 取决于店规模与销量 |
| 批发市场位置 | 开车约30秒（游戏内） | 出店→上车→驾驶→到达 |
| 最佳出发时机 | 上午平峰（10:00-11:00）或关店前 | 避开早晚高峰以免错失客流 |

**各阶段的采购策略**：

| 阶段 | 采购策略 | 说明 |
|:--:|------|------|
| 初期（Lv1-12） | 每日进货 | 储藏室小+商品种类少，基本每天都要去批发市场 |
| 中期（Lv13-42） | 隔天进货 | 储藏室升级到Lv3-4后，可支撑2天销量 |
| 后期（Lv43-70） | 2-3天进货一次 | 储藏室Lv5-6+批量采购效率最高 |
| 终局（Lv70+） | 3-4天进货一次 | 满级储藏室+大批量采购，但需要精心规划采购清单 |

## 4. 每日时间分配比例

以下为手动运营（无员工或少量员工）情况下的典型时间分配：

| 活动 | 时间占比 | 说明 |
|:--:|:--:|------|
| 补货 | ~30% | 从储藏室搬货→上架→整理，冷链商品需优先处理 |
| 收银 | ~40% | 高峰期几乎独占收银台，平峰时段可由员工替代 |
| 清洁 | ~10% | 每2-3小时需要清洁一次，使用拖把/清洁剂 |
| 进货 | ~10% | 去批发市场采购，频率约每2-3天一次，分摊到日均 |
| 防盗/巡逻 | ~5% | 关注可疑顾客，查看监控画面 |
| 其他（定价/管理等） | ~5% | 调整价格、检查电脑管理页面、规划扩张 |

**员工替代时间**：

| 员工配置 | 释放的玩家时间（约） | 建议玩家新活动 |
|:--:|:--:|------|
| 1名收银员 | 收银时间减少~40% | 专注补货+进货 |
| 1名补货员 | 补货时间减少~30% | 专注收银+管理 |
| 1名保安 | 防盗时间减少~80% | 专注运营决策 |
| 3名以上员工 | 总手动操作减少~60% | 收银+补货均可交给员工，玩家做战略决策 |

## 5. 账单日与财务节律

| 项目 | 结算周期 | 说明 |
|:--:|:------|------|
| 日营收 | 每日关店后（20:00） | 显示当日总销售额 |
| 日利润 | 每日关店后 | 营收 - 进货成本 - 租金 - 电费 - 工资 - 贷款 |
| 租金 | 每日扣除 | 自动从日利润中扣除 |
| 电费 | 每日扣除 | 与设备数量/类型相关（冰箱/冰柜电费更高） |
| 工资 | 每日扣除 | 每名员工固定日薪 |
| 贷款还款 | 每日扣除（如有贷款） | 初始可申请贷款，分期偿还 |
| 配送收入 | 每单完成时 + 日结汇总 | 1.0在线配送的额外收入 |

> 注：游戏无7天或30天的大周期结算，所有财务指标均为日维度，与游戏的"日循环"设计一致。

## 6. 游戏时间 vs 现实时间

| 维度 | 数值 | 说明 |
|:--:|:--:|------|
| 1游戏小时 | 约45-60秒现实时间 | 经验值，可用于估算 |
| 1游戏天 | 约15-25分钟现实时间 | 含营业前补货+全天营业+结算 |
| 1游戏月（30天） | 约7.5-12.5小时现实时间 | 约等于中度玩家的日均游戏时长 |
| 基础通关 | ~10小时现实时间 | Lv30+10许可证 |
| 全成就 | ~111小时现实时间 | Steam社区统计 |

**时间压缩比**：

| 活动 | 游戏内时间流逝 | 说明 |
|:--:|:--:|------|
| 正常营业 | 1x（实时） | 游戏内时间与操作时间基本同步 |
| 开车去批发市场 | 约2x（压缩） | 实际开车约30秒，游戏时间约1小时 |
| 关店结算 | 瞬间 | 结算画面不计入游戏时间 |
| 升级/购买菜单 | 暂停 | 查看电脑/手机菜单时时间暂停 |

## 7. 时间压力分析

| 压力维度 | 压力水平 | 说明 |
|:--:|:--:|------|
| 补货时间压力 | 中 | 必须在营业前完成冷链补货，但常温商品可随时补 |
| 收银时间压力 | 高 | 高峰期排队增长快，超过4人即触发投诉 |
| 清洁时间压力 | 低-中 | 清洁可随时进行，但太脏会影响顾客 |
| 进货时间压力 | 低 | 可自由选择去批发市场的时机 |
| 总时间压力 | 中 | 游戏设计了"繁忙但不慌乱"的节奏——有足够时间完成必要操作 |

**时间压力的缓解路径**：
1. **短期**：雇佣收银员 → 释放玩家的收银时间用于补货
2. **中期**：升级设备（Large→Premium货架/冰箱）→ 减少单次补货频率
3. **长期**：自助结账机 + 多名员工 → 玩家从日常操作中解放，专注于扩张决策

> 本文档时间数据基于正常玩法节奏估算，实际时长受玩家操作效率、是否使用时间加速Mod等因素影响。
