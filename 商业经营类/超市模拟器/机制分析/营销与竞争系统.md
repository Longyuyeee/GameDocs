## 一、折扣系统（1.0核心新增）——唯一主动营销手段

超市模拟器的营销系统极为精简，**折扣是玩家唯一的主动营销手段**。折扣系统在1.0正式版中引入，此前游戏甚至没有任何营销功能。

### 1.1 折扣机制参数

| 参数 | 数值 | 说明 |
|------|:----:|------|
| 单商品折扣幅度 | 1% 起可调 | 每件商品可独立设置折扣百分比 |
| 客流吸引上限 | +50% | 无论折扣多深，额外客流不超过+50% |
| 1%折扣效果 | ~+50%顾客 | 即使最低折扣也能达到客流上限 |
| 折扣叠加规则 | **不叠加** | 多件商品折扣不累积，取最高值 |
| 总折扣异常值 | -40% x 5件 = -200% | 虽显示-200%，但实际仍只取50%上限 |

> **关键洞察**: 1%折扣即达客流上限意味着"深度折扣没有营销意义"——折扣系统的设计重点不在"如何吸引更多人"而在"如何控制利润损失"。

### 1.2 最佳折扣目标商品

| 商品类型 | 推荐理由 | 风险 |
|---------|---------|:----:|
| **便宜的高流转商品**（茶/酸奶/盐） | 单价低→折扣绝对损失小，但客流提升显著 | 补货频率剧增 |
| **高利润商品**（寿司/牛排） | 利润空间大→即使打折仍有盈利 | 略 |
| **大宗日用品**（米/油） | 购买量大→快速消耗库存 | 仓库容量压力 |
| **不建议打折**（昂贵酒类/书籍） | 高单价×折扣=大量利润流失 | 利润显著下降 |

### 1.3 折扣的风险循环

```
设置折扣 → 客流+50% → 货架消耗速度翻倍
                                    ↓
   投诉增加 ← 缺货频发 ← 补货员跟不上节奏
        ↓
   客流增速放缓 ← 隐性评分下降
```

折扣系统设计的核心悖论：**吸引更多顾客反而是最大的风险来源**。在超市模拟器中，客流不是越多越好——玩家的补货能力、收银处理能力和员工效率决定了最佳客流上限。这是与其他商业经营游戏（如开罗系列的"客流永远不嫌多"）的本质区别。

### 1.4 折扣系统操作界面

| 操作方式 | 路径 |
|---------|------|
| 设置折扣 | 电脑管理页面 → 商品列表 → 选择商品 → 设置折扣百分比 |
| 取消折扣 | 同一界面将折扣归零 |
| 批量操作 | 不支持批量设置，需逐件操作 |
| 生效时间 | 设置后次日营业开始生效 |

## 二、在线配送系统（1.0新增）——第二收入来源

1.0版本增加在线配送系统，为超市开辟了**第二收入维度**。

### 2.1 配送流程

| 步骤 | 动作 | 所需时间（估算） |
|:--:|------|:--:|
| 1 | 电脑接收线上订单 | 即时 |
| 2 | 按订单拣货打包 | 1-3分钟（视商品数量） |
| 3 | 亲自开车配送 | 3-5分钟/单 |
| 4 | 返回超市 | 2-3分钟 |

### 2.2 配送经济模型

| 维度 | 数值 |
|------|:----:|
| 配送费加成 | 商品售价的~15-25%（待精确测算） |
| 配送时间成本 | 每单约5-10分钟 |
| 对店内运营的影响 | 配送期间店内无人管理 → 收银/补货/安防暂停 |
| 与店内销售对比 | 配送单笔利润更高但效率更低 |
| 最佳场景 | 店内客流低时（如营业末尾）或紧急需要现金时 |

### 2.3 配送的战略意义

- **时间≠金钱是统一货币**: 配送系统强化了游戏的核心设计哲学——时间是玩家最宝贵的资源。花时间配送=赚更多钱，但牺牲店内管理和扩张节奏
- **解决"客流上限"问题**: 店内客流有物理空间上限，配送不受此限制
- **与折扣形成互补**: 折扣引客流到店，配送把商品送到顾客——线下+线上双渠道

> **跨品类对比**: 与《卡牌店模拟器》的"举办活动影响市场价格"相比，超市模拟器的在线配送是实实在在的运营维度——不是投机的金融操作，而是真实的物流工作。这反映了两个游戏对"零售"的不同理解：超市模拟器=物质流管理，卡牌店模拟器=信息流管理。

## 三、竞争系统——无竞争AI设计

### 3.1 竞争环境总览

| 维度 | 说明 |
|:----:|------|
| 竞争对手 | **无**——镇上唯一的超市 |
| 市场竞争 | 无同行AI，无价格战，无市场份额争夺 |
| 垄断地位 | 玩家天然垄断本地零售市场 |
| 紧迫感来源 | 内部运营压力而非外部竞争 |

### 3.2 "无竞争"的设计意图

超市模拟器选择了**纯PvE（玩家vs环境/系统）**的竞争模式，而非PvP（玩家vs AI对手）。这与其他商业经营游戏形成鲜明对比：

| 游戏 | 竞争模式 | 对手类型 |
|------|:--------:|---------|
| **超市模拟器** | 纯PvE | 系统压力（客流/投诉/运营成本） |
| **双点医院** | 弱PvE | 无显式对手，只有评级竞争 |
| **开罗系列** | 弱PvE | 排名系统但无实际对抗 |
| **Game Dev Tycoon** | 弱PvE | AI对手公司（存在但不直接对抗） |
| **过山车之星** | 可选PvE | 沙盒模式可选竞争对手公园 |
| **Winkeltje: The Little Shop** | 纯PvE | 无竞争（同类游戏对比） |

### 3.3 无竞争系统的优劣

| 优势 | 劣势 |
|------|------|
| 专注核心玩法——补货/收银/清洁的体力劳动循环 | 缺乏外部驱动力，中后期可能感到"为升级而升级" |
| 降低新手上手门槛 | 缺少"战胜对手"的成就感 |
| 贴合"小镇唯一超市"的叙事逻辑 | 竞争带来的策略变数缺失 |
| 压力来自内部管理，更接近真实零售体验 | 玩家缺少"竞争威胁下优化操作"的动力 |

## 四、口碑机制——隐性评价系统

超市模拟器**无显式口碑/评分界面**，但通过以下方式间接反映顾客满意度：

### 4.1 口碑指标

| 指标 | 显示方式 | 触发条件 |
|------|---------|---------|
| 顾客投诉 | 头顶气泡表情（红色愤怒） | 价格过高/缺货/排队过长/商品变质 |
| 客流变化 | 每日客流量 | 口碑差→客流增长放缓甚至下降 |
| 顾客离店 | 空手离开 | 排队超时/找不到想要的商品 |
| 商品遗留 | 收银台废弃商品 | 顾客排队不耐烦→放下商品走人 |

### 4.2 影响口碑的因素

| 因素 | 影响程度 | 控制方式 |
|------|:-------:|---------|
| 商品定价过高 | ★★★★ | 参考市场价调整 |
| 货架空缺 | ★★★★★ | 增加补货频率/雇佣补货员 |
| 收银排队过长 | ★★★★★ | 增加收银台/自助结账机/雇佣收银员 |
| 地板脏污 | ★★★ | 定期拖地/雇佣保洁员 |
| 商品变质 | ★★★★ | 正确分类存储（常温/冷藏/冷冻） |
| 小偷未被阻止 | ★★ | 安装安防设施/雇佣保安 |

## 五、店名与外观定制——纯装饰性营销

| 定制项目 | 内容 | 是否影响游戏性 |
|---------|------|:------------:|
| 店名 | 自由命名 | 否—纯装饰 |
| 建筑外观颜色 | 可选择多种配色方案 | 否—纯装饰 |
| 招牌样式 | 预设招牌模板 | 否—纯装饰 |
| 店内装修 | 地板/墙面颜色可选 | 否—纯装饰 |

外观定制作为营销手段的作用十分有限——因为没有"竞争对手"也没有"顾客评价界面"，外观改变不影响任何游戏数值。装饰系统的存在更多是为了**玩家自我表达**和**直播间/视频内容差异化**。

## 六、营销漏斗分析

### 6.1 超市模拟器的营销漏斗

```
        认知（无——小镇人尽皆知）
           ↓
        兴趣（折扣吸引→线上订单）
           ↓
        购买（店内选购→线上配送）
           ↓
        忠诚（投诉管理→满意度维持）
           ↓
        复购（顾客次日自动回流）
```

### 6.2 与其他品类的对比

| 阶段 | 传统商业经营游戏 | 超市模拟器 |
|------|----------------|-----------|
| 认知 | 广告投放/电视宣传/活动造势 | **无**——默认全知 |
| 兴趣 | 特色商品/主题活动/限时促销 | 仅折扣——且1%即达上限 |
| 购买 | 价格+品质+服务综合竞争 | 定价权+补货速度决定销售额 |
| 忠诚 | VIP系统/会员卡/积分 | **无**——顾客无忠诚度概念 |
| 复购 | 靠品牌粘性 | 靠"镇上只有一家超市"的垄断 |

## 设计特点分析

超市模拟器的营销系统呈现出一种**刻意精简**的设计美学。作为1.0新增的系统，折扣和配送并没有让游戏变成"营销模拟器"，而是保持了核心体验——**体力劳动经营的纯粹性**。

**设计意图1: 聚焦内部管理而非外部竞争**
游戏的设计者（Nokta Games）明确选择了"无竞争"路线。这意味着游戏的所有压力来源都是**内部运营问题**：缺货了、排队了、地板脏了、商品变质了。这种设计将玩家的注意力始终拉回到"亲手干活"上——而不是在抽象的管理面板上做营销决策。这种选择贴合了游戏"第一人称手动操作"的核心卖点。

**设计意图2: 折扣作为压力调节阀而非增长引擎**
1%折扣即达+50%客流上限是一个非常有趣的设计决策。它告诉玩家：折扣不是为了"最大化客流"——而是为了"在客流不足时补足"。更深的折扣没有任何额外收益，只会增加运营压力和利润损失。这种设计把折扣从一个"要好好研究的系统"变成了"紧急情况下的开关"。

**设计意图3: 配送作为"兜底安全网"**
在线配送系统在1.0中加入，其核心价值可能不在于"增加收入"，而在于**提供一种不受店内客流限制的稳定收入来源**。当店内运营出现问题时（如大规模缺货导致客诉频发），配送订单仍然可以持续产生收入。

**对比参考**: 在《双点医院》中，玩家通过调整科室布局和治疗质量来"营销"——好的治疗率自然吸引患者。在《卡牌店模拟器》中，玩家通过举办比赛和拆包直播来造势。而超市模拟器的营销系统最为精简——甚至可以说几乎没有营销系统。这种设计差异源于品类定位的不同。

**结论**: 超市模拟器的营销系统不是为了最大化利润而设计的，而是为了**维持一个可管理的客流水平**而设计的。这是"店主"视角和"CEO"视角的根本区别——前者关心"能不能忙得过来"，后者关心"能不能赚得更多"。
