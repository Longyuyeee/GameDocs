## 一、满意度系统——隐性评分机制

超市模拟器**没有显式的星级评分UI**（与《双点医院》的星级评价、《开罗系列》的月度评分不同），但后台运行着一个**隐性满意度系统**。

### 1.1 满意度影响因素

| 影响因素 | 影响方向 | 触发机制 | 影响程度 |
|---------|:-------:|---------|:-------:|
| 商品价格合理 | 正向 | 价格在市场基准价±10%范围内 | ★★★ |
| 商品价格过高 | 负向 | 定价显著高于市场价 | ★★★★ |
| 货架有货 | 正向 | 顾客找到想要的商品 | ★★★ |
| 货架空缺 | 负向 | 顾客找不到想要的商品 | ★★★★★ |
| 收银等待合理 | 正向 | 排队不超过3人 | ★★ |
| 收银等待过长 | 负向 | 排队超过5人→顾客放下商品离开 | ★★★★★ |
| 店面清洁 | 正向 | 地板无明显污渍 | ★★ |
| 店面脏污 | 负向 | 地板可见污渍 | ★★★ |
| 商品新鲜 | 正向 | 冷链商品未变质 | ★★★ |
| 商品变质 | 负向 | 顾客买到变质商品 | ★★★★★ |

### 1.2 隐性评分的作用

```
评分高 → 客流自然增长 → 每日客数增加 → 收入上升
                              ↓
评分低 → 客流增长放缓/下降 → 每日客数减少 → 收入下降
```

隐性评分不直接显示，但通过以下方式间接可感：
- **每日客流量变化趋势**：比历史数据高→评分好；比历史数据低→评分差
- **顾客头顶气泡**：红色愤怒表情→当前不满
- **顾客行为**：摸到商品后放回→价格不满；走到货架前停留后离开→缺货；排队中突然离开→等待时间过长

### 1.3 投诉的视觉反馈

| 投诉类型 | 视觉表现 | 背后原因 |
|---------|---------|---------|
| 价格投诉 | 看商品→皱眉→放回→离开 | 定价超过顾客接受上限 |
| 缺货投诉 | 走到货架前→摊手→离开 | 对应品类售罄 |
| 排队投诉 | 看排队→摇头→放下商品→离开 | 等待时间过长 |
| 清洁投诉 | 踩到污渍→关注脚下→做厌恶表情 | 地板脏污 |
| 变质投诉 | 拿起变质商品→愤怒表情→找店员 | 冷链管理失败 |

## 二、Steam成就系统

### 2.1 成就总览

超市模拟器在Steam平台拥有约**30+项成就**，覆盖经营流程中的各个里程碑。全成就预估需要约**111小时**游戏时间。

### 2.2 成就分类

| 成就类别 | 数量（约） | 代表成就 | 难度 |
|---------|:---------:|---------|:---:|
| **等级成就** | 5+ | 达到Lv10/Lv25/Lv50/Lv75/Lv90 | ★★~★★★★ |
| **收入成就** | 3+ | 单日利润达到$500/$2,000/$10,000+ | ★★★~★★★★★ |
| **产品成就** | 4+ | 解锁5/10/20/26张许可证 | ★~★★★★ |
| **销售成就** | 3+ | 销售1,000/10,000/100,000件商品 | ★~★★★★★ |
| **手工成就** | 3+ | 亲自收银50名顾客/亲自补货200次 | ★★~★★★ |
| **扩张成就** | 3+ | 升级储藏室/购买大店面/全设备升级 | ★★★ |
| **安保成就** | 2+ | 阻止10个小偷/安装全套安防设施 | ★★★ |
| **配送成就**(1.0新增) | 2+ | 完成首次配送/完成50次配送 | ★★~★★★ |
| **杂项成就** | 5+ | 自定义店名/雇佣首个员工/首次批发采购 | ★ |

### 2.3 核心里程碑成就详解

#### 等级成就

| 成就名称 | 要求 | 难度 | 达成时间（估算） |
|---------|:----:|:----:|:--------------:|
| 开业大吉 | 达到Lv5 | ★ | 1-2小时 |
| 小有规模 | 达到Lv25 | ★★ | 10-15小时 |
| 零售精英 | 达到Lv50 | ★★★ | 30-40小时 |
| 超市大亨 | 达到Lv75 | ★★★★ | 60-70小时 |
| 零售之王 | 达到Lv90 | ★★★★★ | 90-100小时 |

#### 手工成就——最具特色的成就设计

| 成就名称 | 要求 | 设计意图 |
|---------|:----:|---------|
| Hardworking Cashier | 亲自结账50名顾客（不能使用收银员） | 鼓励玩家体验收银的核心玩法 |
| Shelf Master | 亲自补货200次 | 鼓励玩家体验补货的体力劳动 |
| Clean Freak | 亲自拖地100次 | 鼓励玩家关注清洁系统 |

> **设计洞察**: 手工成就是超市模拟器成就系统最具特色的部分。它们不是在奖励"效率"或"成绩"，而是在奖励"亲自干活"——这与游戏"体力经营"的核心哲学高度吻合。在大多数Steam游戏中，成就会推动玩家去探索游戏的最优策略；但在超市模拟器中，手工成就推动玩家去**不要雇佣员工，自己干活**。

#### 产品成就

| 成就名称 | 要求 | 难度 |
|---------|:----:|:----:|
| 品类拓展 | 解锁第5张许可证 | ★ |
| 品类专家 | 解锁第15张许可证 | ★★★ |
| 全品类覆盖 | 解锁全部26张许可证 | ★★★★★ |

#### 利润成就

| 成就名称 | 要求 | 达成阶段 | 策略提示 |
|---------|:----:|:--------:|---------|
| 收支平衡 | 单日利润>$0 | 第1天 | 自动达成 |
| 日入斗金 | 单日利润>$500 | Lv15-20 | 引入冷藏+预制餐品类 |
| 日进万金 | 单日利润>$2,000 | Lv30-40 | 引入酒类+高利润品类 |
| 超市印钞机 | 单日利润>$10,000+ | Lv60+ | 全品类覆盖+高效率运营 |

### 2.4 成就系统设计特点

| 特点 | 说明 |
|------|------|
| **递进式解锁** | 成就按等级自然解锁——玩家不需要刻意刷成就 |
| **无时间限制** | 所有成就均可无限期追求，无错过型成就 |
| **无排行榜** | 纯单机，无任何联网排名机制 |
| **无隐藏成就** | 所有成就条件在Steam界面可见（待确认） |
| **无剧情相关** | 全部成就基于经营指标，无叙事成就 |

## 三、终局与完美标准

### 3.1 超市模拟器的"通关"定义

由于游戏无剧情结局，"完美通关"被社区定义为以下状态的达成：

| 完美标准 | 达成条件 | 预估时间 |
|---------|---------|:--------:|
| 店等级Lv90 | 达到游戏最高等级 | 90-100小时 |
| 全部26张许可证 | 解锁所有可售品类 | 与Lv90同步 |
| 全部设备最高级 | 货架/收银台/冰箱/冰柜/储藏室全满级 | Lv70+ |
| 全部Steam成就 | 30+项成就全部达成 | ~111小时 |
| 最大店面 | 购买游戏中最大的店面 | Lv60+ |
| 最优布局 | 社区标准"高效布局" | 主观标准 |

### 3.2 终局体验

| 维度 | 描述 |
|------|------|
| 继续经营 | 达到Lv90后游戏不会结束——进入"沙盒模式" |
| 目标缺失 | 无新内容解锁后，游戏变成纯粹的"维持运营" |
| 玩家自定目标 | 社区玩家挑战/效率极限/成就补完 |
| 与同品类对比 | 《双点医院》终局类似——全三星后无新挑战；《开罗》终局有"通关画面"后New Game+ |

## 四、无排行榜与社交系统

| 社交功能 | 存在与否 | 替代方案 |
|---------|:-------:|---------|
| 全球排行榜 | 无 | 纯单机体验 |
| 好友排行榜 | 无 | 无社交竞争 |
| 成就展示 | Steam成就系统 | 可在Steam个人资料展示 |
| 截图分享 | Steam F12 | 社区内容分享 |
| 创意工坊 | 无 | 无mod支持（当前版本） |

> **跨品类对比**: 在《过山车之星》中，玩家可以上传公园到Steam创意工坊，被其他玩家"参观"——这是一种间接的社交竞争。在《Game Dev Tycoon》中，游戏结束时会给玩家一个"全球排名"。超市模拟器完全放弃了社交维度——它在本质上是一个**单人、离线、无竞争的经营体验**。这与游戏"店主独自劳动"的核心体验一致——你不是在经营一个"帝国"，你只是在经营一家"店"。

## 五、社区自建评分体系

由于游戏内置评分系统缺失，社区自发形成了非官方的评分标准：

### 5.1 社区自评维度

| 评分维度 | 评价标准 | 数据来源 |
|---------|---------|---------|
| 运营效率 | 每日利润/客单价/员工利用率 | 个人记录+论坛分享 |
| 布局质量 | 顾客动线合理度/空间利用率 | 截图分享+社区讨论 |
| 补货管理水平 | 补货及时性/缺货率 | 个人记录 |
| 安防完整性 | 防盗成功率/安防设施覆盖率 | 个人记录 |
| 升级速度 | X天内达到Lv90的用时记录 | 社区挑战（Speedrun） |

### 5.2 Speedrun社区

超市模拟器在Speedrun.com上有专门的排行榜，挑战内容包括：
- **Any% (任意条件)**: 最快达到Lv90
- **No Employee (无员工)**: 不雇佣任何员工完成全成就
- **100%**: 全成就+全许可证+全设备升级

## 设计特点分析

超市模拟器的评分与成就系统体现了**"劳动奖励"而非"效率奖励"**的设计哲学。

**设计意图1: 隐性评分的"去焦虑化"设计**
没有显式星级评分是一个深思熟虑的设计选择。在《双点医院》中，星级评分是玩家焦虑的核心来源——你需要不断优化才能拿到三星。在超市模拟器中，没有评分界面意味着玩家不会因为"评分下降"而感到挫败。游戏的压力来自**实际的运营问题**（缺货/排队/变质）而不是**抽象的数字**。这让体验更加沉浸——你是"店主在解决问题"，而不是"玩家在刷评分"。

**设计意图2: 手工成就推动"非最优"玩法**
Normal游戏中，成就系统通常会引导玩家走向"最优策略"。超市模拟器的"手工成就"却恰恰相反——它在鼓励玩家**不雇佣员工**，亲自去做那些"效率低下"的体力活。这种设计非常聪明：它让玩家在追求全成就的过程中，完整体验了游戏的核心机制（体力劳动），而不是通过外包来跳过核心玩法。

**设计意图3: 无排行榜保持"单人叙事"的完整性**
没有排行榜意味着玩家永远不会看到"别人比你做得好"的对比焦虑。每一个超市都是玩家自己的、独一无二的经营故事。这与《动物园之星》等游戏中的"社交媒体评分"形成鲜明对比——后者的成就感来自"别人的认可"，前者的成就感来自"自己做到了"。

**对比参考**: 在《卡牌店模拟器》中，评分系统直接决定了卡牌市场的走向——高评分→更多稀有卡牌到店→更多顾客。评分是经济系统的核心输入。在超市模拟器中，评分虽然也会影响客流，但作用远没有那么决定性——即使评分低，超市仍然在营业，顾客仍然会来（只是少一些）。**超市模拟器的评分是"缓坡"的调节器，而非"硬约束"**。

**结论**: 超市模拟器的评分与成就系统不是为了让玩家"竞争"或"比较"，而是为了让玩家**确认自己的劳动成果**。它是游戏"体力经营"哲学在反馈层面的一致性体现——成就奖励的是劳动，评分反映的是管理质量，终局状态是玩家个人努力的完整展示。
