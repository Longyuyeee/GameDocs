## 一、顾客行为链（Customer AI Behavior Chain）

《超市模拟器》中的顾客遵循一套完整的"进店→选购→结账→离店"行为链，每个环节都有独立的AI决策逻辑。

| 阶段 | 行为 | 触发条件 | 耗时因素 |
|------|------|---------|---------|
| 1. 进入 | 从店门口进入并取购物篮 | 店铺营业中且有空间 | 门口到货架区的步行距离 |
| 2. 寻路 | 按购物清单导航至目标商品区域 | 进入店内后 | 店铺布局复杂度、过道宽度 |
| 3. 选购 | 从货架上取走所需商品放入购物篮 | 抵达对应货架且商品有库存 | 商品寻找时间（与货架布局相关） |
| 4. 排队 | 前往收银台排队等待结账 | 购物清单完成 | 排队人数、收银台开放数量 |
| 5. 结账 | 扫描商品 + 付款（现金或刷卡） | 轮到该顾客 | 收银员速度 / 自助机效率 |
| 6. 离店 | 携带商品离开店铺 | 付款完成 | 出口距离 |

### 行为链异常中断

| 异常情况 | 触发原因 | 后果 |
|---------|---------|------|
| 找不到商品 | 货架空置或商品被摆放在错误区域 | 放弃购买该商品，继续下一项 |
| 全部放弃 | 所需商品全部缺货 | 直接离开（零消费） |
| 放弃结账 | 排队时间过长（阈值待补充） | 放下购物篮离开，造成损失 |
| 价格拒购 | 商品定价超过其预算或高于阈值 | 放弃该商品 |

## 二、购物清单系统

每位顾客在生成时携带一份随机商品需求清单，这是决定其购物行为的核心数据。

| 参数 | 说明 |
|------|------|
| 生成时机 | 顾客生成时（进入店铺前） |
| 清单长度 | 每位顾客1~5件（估算，具体范围待补充） |
| 商品选择范围 | 已解锁的所有可售商品中随机抽取 |
| 优先级机制 | 待补充（是否存在核心商品必买逻辑） |
| 与货架关系 | 无论商品摆放在哪里，顾客会主动寻找 |
| 缺货反应 | 找不到则放弃该项，继续寻找下一项 |

### 购物清单生成逻辑推测

```
顾客生成 → 随机抽取N件商品 → 按品类分区导航 → 逐项采购
                                           ↓
                                   如有缺货 → 跳过该项
                                           ↓
                                   清单完成 → 前往收银台
```

> 注：购物清单的具体生成算法（是否加权热门商品、是否受折扣影响等）需进一步验证。

## 三、投诉系统

顾客在遭遇不良体验时会触发投诉，直接影响店铺评分和后续客流。

### 投诉触发条件

| 条件 | 阈值（社区估算） | 触发率 | 后果 |
|------|----------------|-------|------|
| 商品价格过高 | 高于市价约+15%以上 | 较高 | 失去该笔销售 + 扣分 |
| 商品缺货 | 货架空置，顾客找不到想要的商品 | 中等 | 失去该笔销售 + 扣分 |
| 排队时间过长 | 等待超过约30~60秒（估算） | 较低（取决于排队长度） | 顾客放弃购物直接离开 |

### 投诉对评分的影响

| 评分等级 | 特征 | 客流影响 |
|---------|------|---------|
| 高评分 | 投诉少，口碑好 | 客流稳定增长 |
| 中等评分 | 偶有投诉 | 客流增长放缓 |
| 低评分 | 频繁投诉 | 客流明显下降 |
| 极低评分 | 长期经营不善 | 客流持续萎缩 |

> 注：游戏中不显示显式满意度分数，评分状态需通过客流趋势间接推断。

## 四、顾客数量与客流控制

### 每日顾客数量上限

| 店铺等级 | 每日顾客数（估算） | 备注 |
|---------|------------------|------|
| 初期（Lv1~10） | ~20-40人/天 | 受店铺面积和设施限制 |
| 中期（Lv11~30） | ~40-60人/天 | 逐步解锁更多商品和货架 |
| 后期（Lv31~50） | ~60-80人/天 | 需考虑补货和收银效率 |
| 满级 | ~110-120人/天 | 无mod状态下上限 |
| 超过上限 | 客流不再增加 | 每日刷新，不累计 |

### 客流控制机制

| 机制 | 效果 | 说明 |
|------|------|------|
| 折扣吸引 | 最多+50%客流 | 1%折扣即可触发 |
| 店铺等级提升 | 自然增长客流上限 | 随游戏进度逐步解锁 |
| 商品种类增加 | 间接吸引更多顾客 | 满足多样化需求 |
| 负面因素 | 减少客流 | 高投诉/高定价/频繁缺货 |
| 季节性因素 | 无 | 待补充（目前确认不存在季节性客流变化） |

## 五、顾客类型

游戏中存在多种顾客类型，各自消费行为差异显著。

| 顾客类型 | 出现条件 | 消费特征 | 特殊行为 | 识别方法 |
|---------|---------|---------|---------|---------|
| 普通顾客 | 始终存在 | 按清单购买，预算适中 | 正常排队结账 | 无特殊特征 |
| 批发顾客 | 解锁批发商品后 | 只购买大量低价商品 | 一次性购买多件同类商品 | 购买行为异常大量 |
| 小偷 | 任意阶段 | 伪装成普通顾客 | 偷窃商品不付款 | 需保安/摄像头识别 |
| 高消费顾客 | 解锁高端商品后 | 预算高，购买高价商品 | 倾向于购买酒类/寿司等 | 购买商品类型偏向高端 |
| 折扣敏感顾客 | 设置折扣后 | 只购买折扣商品 | 直奔打折区域 | 待确认 |

### 小偷机制详述

| 参数 | 说明 |
|------|------|
| 出现频率 | 随店铺等级提升而增加（待确认具体频率） |
| 偷窃方式 | 将商品放入口袋（非购物篮），出店时不经过收银台 |
| 检测方式 | 摄像头监控区域 → 保安追踪 → 报警天线 |
| 拦截后果 | 保安追上后小偷放下商品逃跑，商品回收 |
| 未拦截后果 | 商品损失（等价于被偷走） |
| 玩家干预 | 玩家也可手动追赶小偷 |

## 六、满意度系统

《超市模拟器》采用隐性满意度机制，不提供显式满意度数值条。

| 满意度影响因素 | 权重（估算） | 说明 |
|---------------|-----------|------|
| 价格合理度 | 高 | 价格高于预期越多，满意度越低 |
| 商品可得性 | 高 | 缺货率越低，满意度越高 |
| 排队等待时间 | 中 | 等待越久，满意度越低 |
| 店铺清洁度 | 中 | 地面污渍/垃圾影响体验 |
| 店铺布局合理性 | 低 | 过道太窄或动线混乱影响购物体验 |
| 商品种类丰富度 | 中 | 可选择范围越大越满意 |

### 满意度的间接观测方法

由于游戏不显示满意度数值，玩家需通过以下指标间接推断：

1. **客流增长趋势**：客流持续增长 → 满意度尚可
2. **客流停滞或下降**：满意度可能出现问题
3. **顾客抱怨动画**：顾客头顶出现愤怒图标 → 即时不满
4. **收银台废弃购物篮**：多名顾客弃购离开 → 严重问题

## 七、设计特点分析

《超市模拟器》的顾客与客流系统设计有其独特的"有限模拟"哲学，既不追求极致拟真，也不过度简化。

**设计意图一："购物清单"机制创造了经营思考的深度**
每位顾客携带随机购物清单的设计，让"商品多样性"和"库存充足度"成为了关键的经营变量。这不是一个"商品越多越好"的简单加法——玩家需要确保每件上架商品都有稳定的供应，因为任何一个缺货商品都可能导致顾客的投诉。这一设计比《卡牌店模拟器》中"顾客随机浏览购买"的模式要求更高的供应链管理能力。

**设计意图二：隐性满意度降低数值焦虑**
刻意隐藏满意度数值的设计，避免了玩家过度关注单一数字指标，而忽略了经营的整体性。这一选择与许多模拟游戏（如《双点医院》中的显式满意度条）形成鲜明对比——设计者显然希望玩家通过观察顾客的实际行为（是否抱怨、是否弃购）来"读懂"店铺的状况，而非机械地堆砌数值。

**设计意图三：小偷机制作为"可控风险"调节器**
小偷的加入为日常经营增加了一层"安全管理"维度，但其出现频率被控制在"需要防范但不至于崩溃"的范围内。这一机制的设计目标并非惩罚玩家，而是在中后期店铺规模扩大后提供新的挑战维度——当补货和收银已成常规操作后，安保成为了新的注意焦点。

**设计意图四：顾客上限防止了指数级膨胀**
设定每日客流上限（满级约110-120人）的设计，防止了无限制客流增长对游戏性能的冲击，同时也避免了"客流越多=赚钱越容易"的单向增长曲线。这一上限迫使玩家在达到客流天花板后，转向优化客单价和利润率来提升收入。

## 八、跨游戏对比

### 与《卡牌店模拟器》的顾客系统对比

| 对比维度 | 超市模拟器 | 卡牌店模拟器 |
|---------|-----------|-------------|
| 购买行为 | 按清单购买（目标明确） | 随机浏览购买（冲动消费） |
| 购物篮机制 | 有（取篮→选购→结账） | 无直接类比 |
| 投诉机制 | 三维度（价格/缺货/排队） | 主要以价格敏感度为主 |
| 顾客类型 | 4+种（含小偷） | 以卡牌偏好划分顾客 |
| 客流增长 | 等级+折扣双驱动 | 以店铺评分和卡牌种类驱动 |
| 排队系统 | 明确的排队+结账流程 | 无排队机制 |
| 小偷系统 | 有专门安保应对机制 | 无 |

### 与《开罗游戏·温泉物语》的对比

| 对比维度 | 超市模拟器 | 温泉物语 |
|---------|-----------|---------|
| 顾客目标 | 购物清单导向（功能性） | 休闲体验导向（满意度） |
| 客流上限 | 硬上限（约120人/天） | 逐步增长无硬上限 |
| 顾客AI复杂度 | 高（多阶段行为链） | 低（简单的入住→体验→离开） |
| 客群细分 | 按购买行为划分 | 按年龄/性别/偏好划分 |
| 排队系统 | 真实的排队+结账等待 | 无排队概念 |
