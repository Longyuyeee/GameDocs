## 一、员工角色体系

《超市模拟器》的员工系统由三种核心角色构成，各自负责超市运营的不同环节。

| 角色 | 英文名称 | 核心职责 | 对经营效率的影响 | 出现阶段 |
|------|---------|---------|----------------|---------|
| 收银员 | Cashier | 操作收银台为顾客结账 | 直接影响顾客排队时间和顾客流失率 | 早期可招聘 |
| 补货员 | Stocker / Restocker | 从储藏室取货→搬运→摆放到货架 | 直接影响商品可得性和销售效率 | 中期核心 |
| 保安 | Security Guard | 巡逻威慑小偷、门口检查、监控室观察 | 减少商品损失，间接影响利润率 | 中后期 |

### 角色重要性随游戏阶段的变化

```
重要性
  ↑
  │   初期          中期           后期
  │  收银员★★★★★  补货员★★★★★   补货员★★★★★
  │  补货员★★★    收银员★★★★    保安★★★★★
  │  保安★         保安★★★      收银员★★★
  │
  └──────────────────────────────→ 游戏进度
```

- **初期（Lv1~10）**：玩家自营为主，收银员为首要雇佣目标（解放玩家从收银台）
- **中期（Lv10~40）**：商品种类增多，补货需求暴增，补货员成为效率瓶颈
- **后期（Lv40+）**：店铺规模大，小偷增多，保安的重要性显著提升

## 二、招聘系统

### 招聘流程

```
店内电脑 → 管理页面 → 招聘选项 → 查看可用候选人 → 确认雇佣
```

| 参数 | 说明 |
|------|------|
| 招聘渠道 | 店内电脑（管理界面） |
| 候选人池 | 系统自动生成（有无随机性待确认） |
| 招聘所需条件 | 店铺等级达到要求（如有） |
| 招聘费用 | 可能存在一次性招聘费（待确认） |
| 招聘周期 | 即时雇佣（无需等待） |
| 员工数量上限 | 由店铺等级决定（具体上限待补充） |
| 解雇方式 | 通过管理界面解雇（确认是否存在解雇费用） |

## 三、薪资制度

### 日薪制详解

| 参数 | 说明 |
|------|------|
| 支付方式 | 每日营业结束时自动结算 |
| 支付对象 | 所有在岗员工 |
| 薪资计算 | 固定日薪 × 员工人数（按角色和等级可能有差异） |
| 支付优先级 | 与其他账单同时扣除 |
| 薪资调整 | 是否存在调薪机制待确认 |

### 不同角色的日薪对比（估算）

| 角色 | 初始日薪（估算） | 升级后日薪（估算） | 备注 |
|------|----------------|------------------|------|
| 收银员 | $20~30/天 | 随升级可能提升 | 初级岗位，薪资较低 |
| 补货员 | $20~30/天 | 随升级可能提升 | 与收银员同级 |
| 保安 | $30~40/天 | 随升级可能提升 | 高级岗位，薪资略高 |

> 注：具体薪资数值需进一步精确确认。

### 人力成本占总成本比例

```
早期（1-3名员工）: 人力占比较低 (~10-20%)
中期（4-8名员工）: 人力占比显著 (~20-35%)
后期（10+名员工）: 人力成为主要成本之一 (~30-40%)
```

## 四、员工升级系统

### 升级路径

| 等级 | 速度标识 | 效率提升 | 解锁条件 |
|------|---------|---------|---------|
| Lv1 | Slow（慢） | 基础效率 | 初始状态 |
| Lv2 | Medium（中） | 中等提升 | 工作经验积累 |
| Lv3 | Fast（快） | 显著提升 | 较高经验要求 |
| Lv4 | Very Fast（很快） | 最高效率 | 满级 |

### 升级效果详细对比

#### 收银员升级效果

| 等级 | 每小时处理顾客数（估算） | 排队减少效果 | 对日收入的影响（估算） |
|------|----------------------|------------|-------------------|
| Slow | ~15-20人 | 基准 | 基准 |
| Medium | ~20-30人 | 排队减少~25% | 收入提升~15-20% |
| Fast | ~30-40人 | 排队减少~50% | 收入提升~30-40% |
| Very Fast | ~40-50人 | 排队大幅减少 | 收入提升~50%+ |

#### 补货员升级效果

| 等级 | 补货速度 | 单次补货耗时减少 | 对缺货率的影响 |
|------|---------|----------------|--------------|
| Slow | 基础速度 | 基准 | 基准 |
| Medium | 较快 | ~20-30% | 缺货率降低 |
| Fast | 快速 | ~40-50% | 缺货率显著降低 |
| Very Fast | 极速 | ~60%+ | 缺货率降至最低 |

#### 保安升级效果

| 等级 | 巡逻效率 | 威慑范围 | 拦截成功率 |
|------|---------|---------|-----------|
| Slow | 基础 | 较小 | 基准 |
| Medium | 中等 | 扩大 | 提升 |
| Fast | 高效 | 较大 | 显著提升 |

## 五、自助结账机 vs 收银员

自助结账机是游戏中唯一可以替代人工的自动化设备，构成了"一次性投资 vs 持续成本"的重要决策点。

| 对比维度 | 自助结账机 | 收银员 |
|---------|-----------|--------|
| 成本类型 | 一次性购买（固定成本） | 持续日薪（可变成本） |
| 长期成本 | 仅电费+维护费 | 日累积显著 |
| 效率 | 固定（不可升级） | 可升级（Very Fast达到最高） |
| 灵活性 | 固定位置不可移动 | 可分配至任何收银台 |
| 故障风险 | 需要维护（待确认故障率） | 无（始终可用） |
| 顾客接受度 | 部分顾客偏好人工（待确认） | 所有顾客接受 |
| 投资回报期 | 初始成本较高，长期回本 | 持续投入无前期成本 |

### 决策场景分析

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 早期资金紧张 | 招聘1~2名低等级收银员 | 前期投入低 |
| 中期客流增长 | 收银员升级 + 补充自助机 | 混合方案平衡成本和效率 |
| 后期规模经营 | 自助机为主 + 少量高级收银员 | 长期成本最优 |

## 六、员工AI行为详解

### 补货员AI

| 行为 | 触发条件 | 优先级 | 说明 |
|------|---------|-------|------|
| 检测空货架 | 持续扫描 | 高 | 自动识别库存不足的货架 |
| 前往储藏室取货 | 某货架需要补货 | 高 | 自动导航至对应商品存放区 |
| 搬运至目标货架 | 取到商品后 | 高 | 沿最优路线搬运 |
| 摆放商品 | 抵达货架后 | 高 | 逐个摆放至整齐 |
| 处理空纸箱 | 货品摆放完成后 | 低 | 丢弃空纸箱 |

### 收银员AI

| 行为 | 触发条件 | 说明 |
|------|---------|------|
| 呼叫下一位顾客 | 当前顾客完成结账 | 自动喊话"Next Please" |
| 扫描商品 | 顾客将商品放到收银台 | 逐件扫描条码 |
| 处理支付 | 扫描完成后 | 根据顾客选择现金或刷卡 |
| 找零 | 现金支付 | 自动计算并找零 |

### 保安AI

| 行为 | 触发条件 | 效果 |
|------|---------|------|
| 常规巡逻 | 持续进行 | 随机在店内移动，起到威慑作用 |
| 识别可疑行为 | 摄像头覆盖范围内 | 将注意力转向可疑顾客 |
| 追逐小偷 | 确认偷窃行为 | 快速移动至小偷位置 |
| 拦截小偷 | 追上后 | 小偷放下商品逃跑，商品回收 |
| 门口检查 | 有报警天线 | 对触发警报的顾客进行检查 |

## 七、员工管理策略

### 最小员工配置模型

| 店规模 | 最低配置 | 推荐配置 | 理由 |
|--------|---------|---------|------|
| 小型（~20种商品） | 1收银员 | 1收银员 + 玩家自补货 | 初期资金有限 |
| 中型（~60种商品） | 1收银员 + 1补货员 | 2收银员 + 2补货员 | 客流和补货需求增加 |
| 大型（~100种商品） | 2收银员 + 2补货员 + 1保安 | 3收银员 + 3补货员 + 1保安 | 全方位运营需求 |
| 超大型（全商品） | 3收银员 + 3补货员 + 1保安 | 4收银员 + 4补货员 + 2保安 | 满负荷运营 |

### 员工升级优先级

```
第一优先: 补货员升级
  理由: 补货效率直接影响商品可得性和销售额

第二优先: 收银员升级
  理由: 减少排队时间，降低顾客流失

第三优先: 保安升级
  理由: 中后期才有显著需求
```

## 八、设计特点分析

**设计意图一：补货员作为"核心瓶颈"角色——与真实超市运营逻辑对齐**
在《超市模拟器》中，补货员是三个员工角色中最重要的一个，因为他们直接决定了"商品是否可得"这一最基础的经营要素。这一设计真实反映了实体零售业的核心痛点——不是没有人买，而是货架空了没有及时补上。与《卡牌店模拟器》中员工主要用于收银和清洁不同，《超市模拟器》将补货效率定位为经营效率的瓶颈，这让员工管理不再是简单的"雇人干活"，而是需要玩家思考"如何安排补货员的工作节奏"。

**设计意图二：自助结账机作为"自动化权衡"的典型教学**
自助结账机的引入创造了一个经典的经济学决策——用固定成本（购买设备）替代可变成本（员工工资）。这一设计在游戏中巧妙地教授了"资本替代劳动"的基本商业概念，同时为玩家提供了一个有意义的中期战略选择。自助机的效率固定且不可升级，这意味着玩家不能简单地用机器完全替代人工——高级收银员的效率最终会超过自助机，但成本更高。这种"效率vs成本"的持续权衡是员工管理系统的核心乐趣。

**设计意图三：员工升级系统的"隐性投资回报"**
员工升级（Slow→Very Fast）是游戏中少数几种"隐性投资"之一——玩家花费时间让员工积累经验，获得的是效率提升而非显性的数值增长。这一设计让老员工变得更有价值，也创造了"解雇老员工换新人是否划算"的思考空间。与许多游戏中"员工只是生产单元"的设计不同，《超市模拟器》赋予了员工成长属性，增加了玩家与员工之间的情感连接。

**设计意图四：保安角色的"防御性管理"设计**
保安是游戏中唯一一个"不直接创造收入"的员工角色，其价值体现在"减少损失"而非"增加收入"。这一设计引入了"防御性管理"的概念——玩家需要决定是否投入资源来防止潜在损失，而这一决策在游戏早期可能看起来"不划算"，但随着店铺规模扩大，偷窃损失会逐渐变得不可忽视。保安系统的存在让"什么都不做"成为了一种有风险的选择。

**设计意图五：与《双点医院》员工系统的对比——专精vs通用**
在《双点医院》中，员工具有多种技能可以学习（诊断、治疗、魅力等），一个员工可以胜任多种角色。而在《超市模拟器》中，员工角色是高度专精的——收银员不会补货，补货员不会收银，保安不会做其他事。这一设计简化了员工管理（不需要考虑员工的技能搭配），但也意味着玩家需要更精准地预估每种角色的需求数量。
