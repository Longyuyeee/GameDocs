## 三、与传统研发系统的对比

| 对比维度 | 超市模拟器 | 传统模拟经营游戏 | 差异说明 |
|---------|-----------|----------------|---------|
| 解锁方式 | 许可证购买（花钱解锁） | 研发点数+时间投入 | 超市=纯金钱驱动，无时间等待 |
| 研发树结构 | 线性等级解锁 | 分支树状结构 | 超市=单线前进，无分支选择 |
| 配方研究 | 不存在 | 常见（如开罗游戏） | 超市=所有同种商品完全相同 |
| 品质等级 | 不存在 | 常见（如双点医院的设备等级） | 超市=无品质差异 |
| 研发速度控制 | 等级决定解锁权 | 玩家可调配资源加速 | 超市=被动等待升级 |
| 研发成本 | 仅金钱 | 金钱+时间+材料 | 超市=成本结构简单 |
| 可选择放弃的路径 | 无 | 有（可跳过某些研发） | 超市=必须按顺序全解锁 |
| 独占性路线 | 无 | 有（A研发和B研发互斥） | 超市=所有许可证最终全部解锁 |

### 为什么没有传统研发树？

《超市模拟器》放弃传统研发树的设计，基于以下考虑：

1. **现实参照**：现实中的超市经营不涉及"研发"商品，而是采购和销售——许可证制度更贴近现实
2. **简化认知**：移除研发树让新玩家更容易理解游戏的核心循环（进货→销售→盈利→扩张）
3. **聚焦核心**：游戏的核心挑战不在于"选择研发什么"，而在于"如何管理已解锁的商品"

## 四、库存管理流程

### 三级库存流转模型

```
┌──────────┐    进货      ┌──────────┐    补货      ┌──────────┐
│ 供货商   │ ────────→   │ 储藏室   │ ────────→   │ 商店货架  │
│ (批发/送) │             │ (纸箱存放)│             │ (销售端)  │
└──────────┘             └──────────┘             └──────────┘
                              │                        │
                              │ 管理决策:               │ 管理决策:
                              │ 储藏室扩容              │ 上架商品种类
                              │ 纸箱存放顺序            │ 货架位置分配
                              │                        │ 各商品库存量
```

### 库存管理的关键决策

| 决策点 | 选项 | 影响 |
|-------|------|------|
| 进货数量 | 多进 vs 少进 | 资金占用 vs 缺货风险 |
| 储藏室容量 | 扩容 vs 维持 | 一次性投资 vs 未来补货效率 |
| 上架商品组合 | 多样化 vs 聚焦热门 | 满足不同顾客 vs 简化补货 |
| 商品排面分配 | 每种商品多少排面 | 陈列量 vs 补货频率 |
| 缺货容忍度 | 频繁补货 vs 容忍短暂缺货 | 员工效率 vs 顾客满意度 |

### 库存周转效率

| 指标 | 计算公式 | 优化目标 |
|------|---------|---------|
| 库存周转率 | 销售量 / 平均库存量 | 越高越好（资金效率高） |
| 缺货率 | 缺货商品数 / 总商品数 | 越低越好（顾客满意度高） |
| 补货效率 | 单次补货耗时 | 越短越好（员工效率高） |
| 资金占用率 | 库存货值 / 可用资金 | 适中（太低=缺货，太高=现金流紧张） |

## 五、商品品质与经济价值

### 同质化商品特征

| 特征 | 说明 | 设计影响 |
|------|------|---------|
| 无品质分级 | 所有同种商品完全一样 | 简化了采购决策，不需要考虑"进哪种品质" |
| 无品牌差异 | 游戏内不使用现实品牌 | 避免授权问题，同时简化商品管理 |
| 无保质期 | 商品不会过期（变质仅由存储错误导致） | 降低库存管理复杂度 |
| 无商品等级 | 无"高级面粉""普通面粉"之分 | 升级意味着解锁新品类而非提升品质 |

### 商品的定价价值逻辑

```
商品价值 = 品类基础价值 × 品类系数
            ↑              ↑
        由商品类型决定    类型内无差异
```

不同品类的单价差异构成了游戏的经济分层——低价商品（基础食材）保障基础客流，高价商品（酒类/预制餐食）提升客单价，两者缺一不可。

## 六、设计特点分析

**设计意图一："花钱解锁"替代"研发树"——降低认知门槛，聚焦经营本质**
将传统模拟游戏中的"研发点数+时间投入"替换为单纯的"金钱解锁"，是《超市模拟器》最具辨识度的设计决策之一。这一选择显著降低了新玩家的学习成本——不需要理解研发树的分支逻辑，不需要权衡"先研发A还是B"，所有的产品解锁都简化为一个决策："我是否有足够的钱购买这个许可证？"这种简化让玩家能够将注意力集中在经营管理的核心挑战——库存管理、定价策略、店铺布局——而非在研发树中做选择。

**设计意图二：许可证与等级的强绑定创造线性成长曲线**
许可证解锁与玩家等级严格绑定，意味着无论玩家积累了多少资金，都不能"跳级"解锁高级商品。这一设计的核心意图是控制游戏节奏——防止资金充裕的玩家过早解锁高端商品（如Lv30的酒类）而跳过中期的品类扩张阶段。这一设计使得各阶段的游戏体验更加均衡，但也减少了玩家的策略选择空间。

**设计意图三：150+商品丰富度保障中后期内容不空洞**
26张许可证对应约150种可售商品，确保了玩家在长达90级的成长过程中，每隔几个等级就有新内容可探索。每个新品类不仅是货架上的新面孔，还可能带来新的存储需求（如Lv13的冷冻品）、新的定价策略（如Lv30的高价酒类）、或新的顾客类型（如Lv70的宠物食品可能吸引特定顾客群）。

**设计意图四：无品牌/无品质的抽象化处理**
游戏中刻意省略了品牌和品质差异，所有同种商品在功能上完全等同。这一抽象化处理将真实超市中极为复杂的SKU管理简化为可管理的游戏系统，与《双点医院》中对疾病和治疗的抽象化处理异曲同工。
