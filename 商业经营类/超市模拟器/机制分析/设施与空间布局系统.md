## 一、三级存储体系

《超市模拟器》的核心存储机制分为三种类型，每种对应不同的商品品类和温度要求。这是游戏中最重要的设施系统，也是区别于同类模拟游戏的关键设计。

| 存储类型 | 对应商品品类 | 温度要求 | 违规后果 | 可升级 | 初始解锁 |
|---------|-------------|---------|---------|-------|---------|
| 普通货架（Shelf） | 面粉、意面、零食、清洁用品、书籍 | 常温 | 无（正常存储） | 是（容量/外观） | 初始可用 |
| 冰箱（Fridge） | 牛奶、奶酪、鸡蛋、酸奶、果汁、饮料、寿司、肉类 | 冷藏 | 常温存放→变质无法销售 | 是（容量/数量） | Lv3许可证 |
| 冰柜（Freezer） | 炸薯条、披萨、冰淇淋、冷冻蔬菜 | 冷冻 | 常温存放→变质无法销售 | 是（容量/数量） | Lv13许可证 |

### 冷链违规惩罚机制

| 违规类型 | 后果 | 经济损失估算 | 能否挽回 |
|---------|------|------------|---------|
| 冷藏品放常温货架 | 商品不可销售 | 约等于商品进价 | 需丢弃损坏商品 |
| 冷冻品放常温货架 | 商品不可销售 | 约等于商品进价 | 需丢弃损坏商品 |
| 冷藏品放冰柜（反之亦然） | 待确认 | 待补充 | 待补充 |
| 长时间断电 | 冷冻/冷藏品变质 | 视库存量而定 | 需重启电源并清理损坏品 |

> 注意：游戏中未提供违规放置的警告提示，玩家需自行记忆每种商品的正确存储类型。

### 存储容量与升级

| 存储单位 | 基础容量 | 升级后容量 | 升级方式 |
|---------|---------|-----------|---------|
| 普通货架单元 | 4~6件商品（视尺寸） | 可通过购买更大型号扩展 | 购买替换 |
| 冰箱单元 | 4~6件冷藏品 | 可购买额外冰箱增加总容量 | 购买新增 |
| 冰柜单元 | 4~6件冷冻品 | 可购买额外冰柜增加总容量 | 购买新增 |
| 储藏室货架 | 6~8个纸箱 | 可升级扩大储藏空间 | 购买升级 |

## 二、收银系统

### 收银方式对比

| 维度 | 手动收银台 | 自助结账机 |
|------|-----------|-----------|
| 解锁条件 | 初始可用 | 达到特定等级后解锁 |
| 操作方式 | 玩家/收银员手动扫描条码 | 顾客自助操作 |
| 支付方式 | 现金（需找零）+ 刷卡 | 刷卡为主（待确认是否支持现金） |
| 效率 | 取决于收银员速度等级 | 固定效率，略低于最高级收银员 |
| 成本 | 购买收银台（一次性）+ 收银员工资（持续） | 购买设备（一次性）+ 电费（持续） |
| 维护需求 | 无需特殊维护 | 需定期维护和通电 |
| 排队效果 | 取决于收银员数量和效率 | 减少排队压力的有效手段 |
| 顾客满意度影响 | 排队时间影响满意度 | 减少等待提升满意度 |

### 收银流程分解

```
顾客排队 → 轮到 → 收银员/玩家逐件扫描商品条码
                                      ↓
                    计算总价 → 顾客出示支付方式
                                      ↓
                    现金支付 → 收银员找零或收银员操作刷卡
                                      ↓
                    打印收据 → 顾客取走商品离店
```

### 收银员效率等级

| 等级 | 速度标识 | 每小时处理顾客数（估算） |
|------|---------|----------------------|
| Slow（慢） | 初始状态 | ~15-20人 |
| Medium（中） | 首次升级 | ~20-30人 |
| Fast（快） | 二次升级 | ~30-40人 |
| Very Fast（很快） | 最高等级 | ~40-50人 |

| 设施类型 | 功能 | 解锁条件 | 覆盖范围 | 成本 |
|---------|------|---------|---------|------|
| 摄像头 | 监控特定区域，记录偷窃行为 | 待确认等级 | 有限视角范围 | 中等 |
| 报警天线（门禁） | 检测未付款商品经过门口 | 解锁保安系统后 | 门口区域 | 中等 |
| 保安巡逻 | 主动威慑和追逐小偷 | 招聘保安 | 全店范围（移动式） | 日薪成本 |
| 监控室 | 集中查看所有摄像头画面 | 与摄像头配套 | N/A（查看功能） | 包含在摄像系统中 |

### 安防协同效应

```
摄像头覆盖区域 → 识别可疑行为 → 保安调度至目标区域 → 拦截小偷
                                                      ↓
                             未雇佣保安 → 仅记录不拦截 → 商品仍可能被偷
```

## 四、储藏室系统

储藏室是三级存储体系的中转枢纽，承担库存缓冲功能。

| 参数 | 说明 |
|------|------|
| 功能定位 | 存放进货纸箱的独立空间 |
| 初始面积 | 默认赠送 |
| 扩展方式 | 购买额外储藏室空间 |
| 内部设施 | 储藏室货架（需额外购买） |
| 货架容量 | 每个货架可存放6~8箱 |
| 补货流程 | 纸箱→储藏室货架→搬运至商店货架→拆箱→摆放商品 |
| 升级选项 | 面积扩大、货架数量增加 |

### 补货操作流程

```
进货到达 → 纸箱搬运至储藏室
              ↓
     纸箱存放至储藏室货架
              ↓
     补货员从储藏室取货 → 搬运至对应商店货架
                            ↓
               拆箱 → 将商品逐个摆上货架
                            ↓
               空纸箱 → 丢弃或回收（待确认）
```

## 六、动线设计原则

虽然《超市模拟器》采用纯自由布局（无相性加成系统），但有效的动线设计对经营效率有显著间接影响。

### 布局原则对比

| 原则 | 说明 | 效果 |
|------|------|------|
| 合理分区 | 同类商品集中摆放（饮料区/零食区/乳制品区等） | 顾客寻路效率提升 |
| 仓库就近 | 高周转商品靠近储藏室 | 缩短补货员搬运距离 |
| 收银台位置 | 靠近出口、避免堵塞主通道 | 减少排队拥堵、优化离店动线 |
| 过道宽度 | 预留足够空间供两人并行 | 避免顾客拥堵造成的投诉 |
| 冷链集中 | 冰箱和冰柜集中在同一区域 | 减少冷链设备分散造成的电力浪费（真实性待确认） |
| 入口缓冲区 | 门口预留缓冲区，避免顾客一进门即面对货架 | 营造宽敞感（主观感受） |

### 与同类游戏的布局系统对比

| 对比维度 | 超市模拟器 | 开罗·温泉物语 | 双点医院 |
|---------|-----------|--------------|---------|
| 布局自由度 | 完全自由 | 网格约束 | 房间约束 |
| 相性加成 | 无 | 有（物品相邻加成） | 无 |
| 布局对效率的影响 | 间接（动线合理性） | 直接（数值加成） | 直接（病人流动效率） |
| 布局工具 | 手动拖拽放置 | 网格铺设 | 房间绘制 |
| 垂直空间 | 无（单层） | 可多层 | 可多层 |
| 装饰系统 | 基础（外观美化） | 丰富（景观影响满意度） | 中等（声誉影响） |

## 八、设计特点分析

**设计意图一：三级存储体系创造"供应链管理"核心玩法**
普通货架/冰箱/冰柜的三级分类并非简单的装饰性设计，而是整个游戏供应链管理的核心约束。玩家需要在购买每种商品时思考其存储需求——这不仅影响货架布局，还影响储藏室的存放规划和补货员的搬运路线。这一设计将现实超市中"冷链管理"的复杂性以简化的方式融入了游戏机制，比《卡牌店模拟器》中"所有商品统一上架"的模式增加了战略深度。

**设计意图二：无相性加成的自由布局传达"真实感优先"**
与开罗游戏系列中"相邻放置获得加成"的抽象化设计不同，《超市模拟器》刻意避免了任何数值化的布局加成系统。布局的优劣完全通过顾客动线和补货效率间接体现——过道太窄会导致拥堵，饮料区离储藏室太远会增加补货时间，收银台位置不当会造成客流堵塞。这种"物理合理性而非数值最优化"的设计哲学，是游戏整体"真实模拟"定位的体现。

**设计意图三：手动补货的操作细节强化沉浸感**
与其他简化补货过程的模拟游戏不同，《超市模拟器》要求玩家/补货员经历"从储藏室取纸箱→搬运到货架→拆箱→逐个摆放"的完整流程。这一设计看似繁琐，实则通过"操作成本"创造了有意义的选择——玩家需要在"扩大规模"和"保持运营效率"之间权衡，因为更大的店铺意味着更长的搬运距离和更多的补货需求。

**设计意图四：安防系统作为中后期的新增管理维度**
小偷和安防系统的引入时机被设计在游戏中后期——此时玩家已经熟悉了基本的补货和收银流程，需要一个全新的挑战来维持游戏的新鲜感。摄像头/保安/报警天线的多层次安防体系，让玩家可以从多个维度构建自己的防盗策略，而非单一的"放置即生效"。

## 九、跨游戏对比

### 与《卡牌店模拟器》的设施系统对比

| 对比维度 | 超市模拟器 | 卡牌店模拟器 |
