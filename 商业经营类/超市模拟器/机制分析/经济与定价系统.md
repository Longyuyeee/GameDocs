## 一、市场价波动机制

### 每日市场价格变动

每日开店前系统对所有商品进行一次随机价格调整，进价和售价均以该市场价为基准。

| 参数 | 说明 |
|------|------|
| 触发时机 | 每日营业开始时自动刷新 |
| 变动方式 | 每件商品独立随机涨跌，无固定周期或规律 |
| 显示方式 | 绿色向上箭头（涨价）、红色向下箭头（降价）、灰色横线（持平） |
| 进价联动 | 进货价格直接跟随市场价变动 |
| 玩家售价 | 玩家可自行设定高于或低于市场价的零售价 |
| 变动幅度 | 游戏内未公示具体幅度范围，社区估算约为±5%~±15% |

### 社区验证的最优定价策略

| 定价策略 | 利润率 | 顾客投诉率 | 销售额影响 | 可持续性 |
|----------|--------|-----------|-----------|---------|
| 市价-10%（低价走量） | 极低 | 极少 | 高客流但单利薄 | 可持续但利润低 |
| 市价-3%（微幅优惠） | 低 | 几乎为零 | 适中 | 可持续 |
| **市价+10%（推荐）** | **最优** | **可接受** | **最大化** | **可持续（社区共识）** |
| 市价+15%~+20% | 较高 | 明显增加 | 开始下降 | 中期不可持续 |
| 市价+30%以上 | 虚高 | 极高 | 大幅下降 | 不可持续 |

> 说明：以上最优定价策略来自社区15次独立测试验证。+10%在投诉率与利润之间取得最佳平衡点。

### 定价精确度分析

百分比定价与固定金额定价在不同基准价格商品上的效果差异显著。

| 商品 | 基准市价 | +$0.50固定加价换算百分比 | 效果 |
|------|---------|-------------------------|------|
| 苏打水 | ~$2.00 | +25% | 暴利定价，顾客拒绝购买 |
| 面粉 | ~$3.00 | +16.7% | 偏高，投诉风险大 |
| 寿司 | ~$25.00 | +2% | 利润损失严重 |
| 高端酒类 | ~$60.00+ | +<1% | 几乎无意义 |

**结论**：游戏中百分比定价策略显著优于固定金额定价，建议按市价百分比设定售价。

## 二、折扣系统（v1.0新增）

| 参数 | 说明 |
|------|------|
| 折扣触发 | 在管理界面设置特定商品的折扣百分比 |
| 折扣粒度 | 单个商品设定折扣比例 |
| 客流影响 | 1%折扣即吸引约+50%更多顾客 |
| 客流上限 | 最多吸引+50%额外顾客（达到上限后更高折扣无效） |
| 批量折扣 | 设置-40%×5件商品同样达到+50%客流上限 |
| 补货风险 | 客流量激增可能导致热门商品迅速售罄，补货员来不及上架 |
| 策略建议 | 1%~5%小额折扣即可触发客流增长，无需大幅让利 |

### 折扣策略对比

| 策略 | 利润率影响 | 客流增长 | 综合推荐度 |
|------|-----------|---------|-----------|
| 1%折扣×少量商品 | 几乎无损 | +50%（达上限） | 强烈推荐 |
| 5%~10%折扣×热门商品 | 轻微损失 | +50%（达上限） | 推荐 |
| 20%以上折扣 | 明显损失 | +50%（达上限，无额外增益） | 不推荐 |
| 批量商品低折扣 | 因商品而异 | +50%（达上限） | 视利润率而定 |

## 三、收入来源

| 收入来源 | 解锁时机 | 收入占比（估算） | 说明 |
|---------|---------|----------------|------|
| 店内零售 | 初始可用 | ~70%-80% | 核心收入来源，顾客到店购买商品 |
| 在线配送（v1.0） | 达成特定等级后解锁 | ~15%-25% | 线上订单自动配送，无需玩家干预 |
| 批发转售 | 解锁批发市场后 | ~5%-10% | 低价批量进货→加价转售，利润空间大但耗时 |

### 店内零售收入公式

```
单笔收入 = 商品售价 × 购买数量
日总收入 = Σ(单笔收入) - 当日成本
```

### 成本结构

| 成本项目 | 类型 | 支付频率 | 说明 |
|---------|------|---------|------|
| 进货成本 | 可变成本 | 每次进货时 | 按批发价/送货价购买商品的费用 |
| 租金 | 固定成本 | 每日结算 | 店铺租赁费用，随店铺等级提升而增加 |
| 电费 | 可变固定成本 | 每日结算 | 随店内设备（冰箱/冰柜/照明）增加而上升 |
| 贷款利息 | 固定成本 | 每日结算 | 如使用贷款功能，每日需支付利息 |
| 员工工资 | 固定成本 | 每日结算 | 按员工数量与级别计算日薪 |
| 许可证费用 | 一次性成本 | 解锁时 | 购买新商品许可证的一次性投资 |
| 设备费用 | 一次性成本 | 购买时 | 货架/冰箱/收银台等设备购置费 |

## 四、批发市场 vs 送货到店

| 对比维度 | 批发市场自提 | 送货到店 |
|---------|-------------|---------|
| 价格 | 便宜约50% | 较贵 |
| 耗时 | 需玩家开车前往并亲自挑选 | 自动送达，节省时间 |
| 利润空间 | 可翻3-4倍 | 利润较薄 |
| 批量购买 | 适合大宗采购 | 无限制 |
| 操作复杂度 | 需驾驶+搬运等手动操作 | 一键下单 |
| 推荐使用时机 | 资金充裕且时间充足时 | 紧急补货或时间紧张时 |

## 五、账单与结算系统

| 参数 | 说明 |
|------|------|
| 结算周期 | 每日营业结束后自动结算 |
| 支付顺序 | 自动从账户余额扣除所有应付费用 |

## 六、设计特点分析

《超市模拟器》的经济系统设计采取了**极高真实度 + 高风险管理的混合策略**，这与同类游戏形成鲜明对比。

**设计意图一：用"真实市场波动"创造日常决策压力**
每日随机价格波动的设计让定价不再是"设一次用到通关"的静态策略，而是需要玩家每日关注市场动态并做出微调。这一设计有效延长了中后期游戏的内容寿命——即使在店铺扩张完成后，每日价格检查与定价调整仍构成规律的游戏循环。相比之下，许多开罗经营游戏（如《温泉物语》）的定价是静态的，设好价格后几乎不需要再调整。

**设计意图二：用"百分比优先"引导玩家思考经济学原理**
游戏中固定加价与百分比加价的差异效应，实际上在潜移默化地教育玩家一个基础经济学概念——同样的绝对加价额在不同基价商品上代表不同的利润率。设计者刻意不提供"建议售价"自动计算功能，迫使玩家自行理解和适应这一机制，这是模拟类游戏中较为少见的"教学性设计"。

**设计意图三：折扣系统的"阈值天花板"防止经济系统崩溃**
1%折扣即可触发+50%客流上限的设计极具匠心——它既给予了玩家一个低成本的引流工具，又通过上限机制防止了"恶性价格战"的出现（无限折扣=无限客流=补货系统崩溃）。这一设计与《卡牌店模拟器》中"降价竞争直接导致亏损"的硬约束不同，提供了更灵活的操作空间。

**设计意图四：批发与送货的双轨制衡**
批发自提（低价高利润但耗时）与送货到店（高价低利润但省时）的二元选择，构成了游戏中最重要的"时间vs金钱"博弈。这一设计暗示了设计者希望玩家在"沉浸式经营体验"与"效率优先"之间做出有意义的权衡，而非单纯地数值最优解。

## 七、跨游戏对比

### 与《卡牌店模拟器》的经济系统对比

| 对比维度 | 超市模拟器 | 卡牌店模拟器 |
|---------|-----------|-------------|
| 定价自由度 | 完全自由定价（可高于/低于市价） | 固定市场价为主 |
| 价格波动 | 每日随机涨跌 | 无价格波动 |
| 折扣系统 | 百分比折扣+客流上限机制 | 无系统性折扣 |
| 成本结构 | 多维度（租金/电费/工资/利息） | 以进货成本为主 |
| 高利策略 | 市价+10%为最优 | 无此类策略空间 |
| 经济风险 | 每日结算，可能负余额倒闭 | 风险较低 |
| 批发机制 | 自提便宜/送货贵 | 固定供货价 |

### 与《双点医院》的对比

| 对比维度 | 超市模拟器 | 双点医院 |
|---------|-----------|---------|
| 定价调整 | 每日需关注微观价格波动 | 固定服务定价 |
| 经济压力源 | 多成本维度挤压利润 | 以治愈率驱动的收入 |
| 玩家控制力 | 强（可调每件商品价格） | 弱（通过政策间接调控） |
