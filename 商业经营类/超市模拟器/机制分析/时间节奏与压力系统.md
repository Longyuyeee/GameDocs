## 一、每日时间结构

超市模拟器以**天**为基本时间单位，每一天分为三个明确的阶段：

### 1.1 每日三阶段

| 阶段 | 时间 | 主要活动 | 玩家可操作内容 |
|:----:|:----:|---------|:------------:|
| **开店前** | ~08:00前 | 补货时间窗口 | 从仓库补货上架、整理冷藏冷冻、处理配送订单 |
| **营业时间** | 08:00 - 22:00 | 顾客进店购物 | 收银、清洁、防盗、补货（持续进行） |
| **关店结算** | 22:00后 | 强制关店→日结 | 查看当日收入/支出/利润报表 |

### 1.2 营业时间内的客流节律

```
客流密度曲线（概念模型）
          高峰期①              高峰期②
   高  ┃    ██                    ██
    ↑  ┃    ██       █████       ██
   客  ┃    ██     █████████     ██
   流  ┃    ██   ███████████   ████
   量  ┃  ███████████████████ ██████
   低  ┃████████████████████████████████
       └──────────────────────────────→ 时间
         08:00        15:00        22:00
```

| 时段 | 客流特征 | 运营重点 |
|:----:|---------|---------|
| 08:00-10:00 | 开门高峰——第一批顾客集中到店 | 确保货架在开店前补满 |
| 10:00-12:00 | 客流回落至平稳 | 可抽时间补货/清洁 |
| 12:00-14:00 | 午间小高峰 | 收银压力上升 |
| 14:00-17:00 | 平稳期 | 最佳批发采购时间窗口 |
| 17:00-19:00 | 傍晚高峰——下班后客流 | 收银台压力最大时段 |
| 19:00-22:00 | 逐渐回落至关店 | 准备关店结算 |

> **设计洞察**: 两段式的客流高峰（开门+傍晚）模拟了真实超市的客流节律。与《双点医院》中患者"随时到达，随时诊断"的连续流不同，超市模拟器的客流更接近实际零售业的周期性波动。

## 二、多重并行压力系统

超市模拟器的核心体验来自于**多重压力同时作用于玩家**的设计。游戏没有单一的"血量"或"分数"，而是通过多个维度的压力共同创造经营紧张感。

### 2.1 压力源全景图

| 压力类型 | 触发条件 | 后果 | 缓解方式 | 强度评级 |
|---------|---------|------|---------|:-------:|
| **补货压力** | 货架商品售罄 | 顾客投诉→客流减少 | 自有补货/雇佣补货员 | ★★★★★ |
| **收银排队** | 顾客多于收银台处理能力 | 顾客离店→商品废弃→收入损失 | 增加收银台/自助结账/收银员 | ★★★★★ |
| **清洁维护** | 地板出现污渍 | 顾客满意度下降 | 拖地/雇佣保洁员 | ★★★ |
| **安防压力** | 小偷随机出现 | 商品被盗→经济损失 | 手动拦截/摄像头/保安 | ★★★★ |
| **冷链变质** | 冷藏/冷冻商品放错区域 | 商品损失→利润蒸发 | 正确分类存储 | ★★★ |
| **配送压力** | 1.0新增线上订单 | 配送期间店内无人管理 | 配送时间选择/权衡 | ★★★ |

### 2.2 补货压力——最核心的压力源

补货是超市模拟器中最持久、最核心的压力。游戏设计通过以下机制强化补货压力：

| 机制 | 效果 |
|------|------|
| **前一天遗留** | 关店时货架状态保留到次日——补货不是"自动重置"而是玩家责任 |
| **物理搬运** | 必须亲手从仓库拿货放到货架上——比抽象界面操作耗时多10倍 |
| **体积守恒** | 商品占货架物理空间——不是"按按钮补满"而是"逐个摆放" |
| **补货员AI限制** | 补货员效率有限/有巡逻范围——无法完全替代玩家 |
| **仓库到货架距离** | 店越大→补货路程越长→时间消耗越大 |

**补货压力曲线（店面规模与压力的关系）**:

```
压力
高 ┃                          ██
   ┃                       ██  ██
   ┃                    ██    ██
   ┃                 ██      ██
中 ┃              ██        ██
   ┃           ██          ██
   ┃        ██            ██
   ┃     ██              ██
低 ┃  ██                ██
   ┗━━━━━━━━━━━━━━━━━━━━━━━→ 店面规模
     小                  大
     ↑ 自家补货能搞定     ↑ 必须雇补货员
```

> **跨品类对比**: 在开罗系列的商业经营游戏中，"补货"通常是一个按钮或一个自动流程。玩家只需确保库存数量>0。而在超市模拟器中，补货是核心玩法——从仓库搬运、寻找正确的货架、逐个摆放、处理过期商品。这种设计的代价是"琐碎"，收获是"沉浸感"。

### 2.3 收银排队压力——"人潮恐慌"的来源

| 阶段 | 排队长度 | 玩家应对 |
|:----:|:--------:|---------|
| 早期（小店） | 3-5人 | 自己收银即可 |
| 中期（中等店） | 8-15人 | 雇佣1名收银员+自己协助 |
| 后期（大店） | 20+人 | 多个收银台+自助结账机+多名收银员 |

**排队超时的连锁反应**:

```
顾客等太久 → 放下商品离开 → 商品留在收银台 → 需手动放回货架
                                      ↓
                                浪费的补货时间 → 其他货架空缺
                                                  ↓
                                            更多投诉
```

收银排队压力与补货压力形成**资源竞争**：玩家花时间收银就不能补货，花时间补货就不能收银。这种竞争是超市模拟器经理体验的核心困境。

### 2.4 清洁维护——被忽视的细节压力

| 清洁等级 | 状态表现 | 对顾客影响 |
|:-------:|---------|:---------:|
| 干净 | 地板光亮 | 无影响 |
| 轻微污渍 | 少量碎屑/痕迹 | -5%满意度（估算） |
| 明显污渍 | 多处污迹 | -15%满意度（估算） |
| 肮脏 | 大面积污渍 | -30%满意度（估算） |

清洁压力在游戏中期开始变得显著——店面增大→踩踏面积增大→污渍产生速度加快。玩家需要每隔15-20分钟拖一次地。

### 2.5 小偷与安防——不可预测的压力

小偷以**随机事件**的形式出现，混入普通顾客中：

| 行为特征 | 识别方式 | 后果 |
|---------|---------|------|
| 随机出现 | 无明显外观区别，无法提前识别 | 商品丢失 |
| 偷窃动作 | 拿起商品直接走向门口 | 需玩家或保安拦截 |
| 频率 | 随店规模增大而增加 | 大店可能每天数个 |
| 拦截失败后果 | 商品被盗+顾客无法追回 | 经济损失 |

### 2.6 冷链变质——系统性压力

1.0版本中，商品有三层温度等级体系：

| 温度等级 | 典型商品 | 存放要求 | 变质时间（估算） |
|:-------:|---------|---------|:--------------:|
| 常温 | 罐头/零食/饮料/书籍 | 普通货架 | 无限期 |
| 冷藏 | 乳制品/肉类/预制餐 | 冷藏冰箱 | 约1-2游戏日 |
| 冷冻 | 冰淇淋/冷冻披萨 | 冷冻冰柜 | 约2-3游戏日 |

将商品放错温区会导致加速变质——损失商品成本+占用容量。

## 三、批发采购节奏

每周玩家需要处理的**批发采购**是游戏重要的节奏节点：

| 采购方式 | 成本 | 时间消耗 | 适合场景 |
|---------|:---:|:--------:|---------|
| **批发市场自提** | 商品价~50% off | 约10-15分钟（往返+搬运） | 大宗采购/非高峰时段 |
| **送货上门** | 市场价 | 免时间 | 急需补货/不想离店 |

**批发采购的时间权衡**:

```
选择批发自提:
  ↓
收益: 节省~50%商品成本
代价: 离开店铺10-15分钟 → 期间无法补货/收银/防盗
      → 可能导致缺货/排队,产生隐性损失
```

> **设计洞察**: 批发采购是超市模拟器中最经典的"策略权衡"设计。它让成本管理从"看数字"变成了"做选择"。玩家需要在"省钱但离店"和"花钱但守店"之间做决策——这比传统经营管理游戏中"点击'批发价进货'按钮"的体验丰富得多。

## 四、无季节/年份系统

与大多数商业经营游戏不同，超市模拟器**不存在季节循环或年份推进**系统：

| 游戏中常见的时间系统 | 超市模拟器 | 代表游戏（有该系统的） |
|----------------------|:---------:|---------------------|
| 四季循环 | 无 | 动物园之星（季节影响游客和动物） |
| 年度事件 | 无 | 双点医院（年度评奖） |
| 节日活动 | 无 | 开罗系列（特定节日促销） |
| 昼夜交替 | 有（每日营业循环） | 同品类基础设计 |
| 等级推进 | 有（升级解锁） | 替代时间推进的进度系统 |

**设计意图**: 无季节系统意味着游戏没有"外部时间压力"——玩家永远不会遇到"冬天客流量下降"或"节日促销季"的情况。所有压力都是**玩家自身运营节奏的产物**，而非外部注入的。这保持了游戏"自驱动"的风格——玩家的节奏由自己控制。

## 五、压力曲线与员工雇佣的决策点

随着店等级提升，压力不仅在增大，而且在**质变**：

| 店等级 | 主动力 | 关键决策 |
|:-----:|-------|---------|
| Lv1-5 | 补货 | 自己搞定一切 |
| Lv5-15 | 补货+收银 | 首次考虑雇收银员 |
| Lv15-25 | 补货+收银+清洁 | 是否雇补货员？ |
| Lv25-40 | 补货+收银+清洁+安防 | 是否装摄像头+雇保安？ |
| Lv40-60 | 四重全压+配送 | 自助结账机替代人工？ |
| Lv60+ | 规模化管理 | 需要全套员工+设备配合 |

**员工雇佣的ROI困境**:

```
雇佣员工 → 日薪支出 → 利润下降
              ↓
       得到更多自由时间
              ↓
       可以花时间批发采购 → 降低成本
       可以花时间配送 → 增加收入
       可以有时间优化布局 → 提升效率
              ↓
       长期ROI：取决于玩家如何利用自由时间
```

## 设计特点分析

超市模拟器的时间与压力系统可以概括为：**"经营体力活"的时间管理游戏**。

**设计意图1: 将"管理"还原为"劳动"**
传统商业经营游戏把管理抽象为"点按钮"(《双点医院》的升级房间、《开罗》的放置建筑)。超市模拟器反其道而行之——把每个管理动作都变成体力劳动：搬箱子、扫条码、拖地。这种设计的核心意图是让玩家体验到"店主的一天"而不是"CEO的一天"。时间的"流逝感"来自于玩家身体在空间中的移动，而不是日历页的翻动。

**设计意图2: 多重压力=玩家总有事做**
超市模拟器几乎没有"等待时间"——补货、收银、清洁、防盗四个维度确保玩家在任何时候都有事可做。这种设计让游戏节奏紧凑到几乎没有喘息空间，这正是其吸引力所在：玩家总是在"下一件事"的路上。这与《双点医院》中患者在走廊里走来走去、玩家等待诊断完成的节奏形成鲜明对比。

**设计意图3: 自动化的代价设计**
游戏中的所有自动化（收银员、补货员、自助结账机）都有代价——要么是直接成本（工资），要么是效率损失（AI效率低于玩家）。这让"雇佣员工"成为一个真实的策略选择而不是"无脑升级"。玩家永远面临一个问题：自己动手省钱但费时，还是花钱省时间？

**对比参考**: 在《过山车之星》中，时间压力来自游客的满意度——他们等排队太久会抱怨，但不会真正"惩罚"玩家。在超市模拟器中，顾客的耐心是有限的——排队太久他们直接走人，而且会放下商品导致额外工作量。后者的"错误代价"更高，也更即时。

**结论**: 超市模拟器的时间节奏系统是围绕"你有多能干"设计的——游戏不考验你的策略规划能力，它考验的是**你能否在有限时间内高效完成一系列体力劳动**。这是一个"动作游戏"式的时间管理，而非"策略游戏"式的规划管理。
