## 一、游戏总时长：20年周期

### 时间框架

| 项目 | 说明 |
|------|------|
| **游戏周期** | 20年(第20年3月为"结局") |
| **结局后** | 可继续无限游玩——无时间限制 |
| **时间单位** | 以"月"为基本操作周期 |
| **结算** | 每年年底(12月)进行年度结算 |

### 20年周期的设计价值

```
第1-3年: 起步期 → 学习基础运营、稳定出刊
第4-8年: 成长期 → 团队扩张、首次办公室搬迁
第9-15年: 扩张期 → 城镇建设、公共事业、店铺邀请
第16-20年: 巅峰期 → 冲刺大奖、最大化指标
```

> **设计特点**: 20年周期是开罗游戏的经典时间设计。短于20年会显得内容不足，长于20年会超出轻量游戏的最佳体验时长。20年恰好覆盖"起步→成长→扩张→巅峰"四个阶段，每个阶段约5年，既给玩家足够的时间体验每个阶段的内容，又不至于让任何一个阶段显得冗长。

### "结局"的特殊设计

| 版本 | 结局条件 | 结局后 |
|------|------|------|
| 原版(手机版) | 第20年3月 | 显示结局画面和数据统计，可继续游玩 |
| Steam版 | 第20年3月 | 同手机版，可继续无限游玩 |

> **与游戏开发物语的对比**: 游戏开发物语的结局在第21年3月(或4月?)，杂志物语缩短到了第20年3月。这个1年的差异可能反映了杂志物语的内容密度更高(每月出刊带来的更紧密节奏)，也可能只是版本迭代的调整。无论如何，内核设计逻辑相同：**明确的终局时间创造"冲刺意识"**，而结局后继续游玩满足"还想再玩一会"的需求。

## 二、出刊节奏：每月一期

### 基础节奏

| 节奏要素 | 说明 |
|------|------|
| **出版频率** | 每月一期杂志 |
| **制作周期** | 上期发行后立即开始下期制作 |
| **操作节奏** | 每期杂志需完成: 策划→取材→制作→校对→发行→推广→广告结算 |
| **年产量** | 12期/年，20年共240期 |

### 每月出刊的操作压力

```
每月固定节奏:
  Week 1: 企划会议(选择三要素)
  Week 2: 制作进行(分配员工)
  Week 3: 校对+调整
  Week 4: 发行+推广+结算
```

### 截稿压力

| 压力类型 | 来源 | 影响 |
|------|------|------|
| **时间压力** | 每月固定出刊——无论如何都要按时发行 | 迫使玩家在有限时间内做出决策 |
| **品质压力** | 品质不够→销量下降→成长放缓 | 激励员工培养和内容投资 |
| **错别字压力** | 错字多→读者流失 | 校对成为每期必修课 |

> **设计分析**: 每月一期的高频率节奏是杂志物语的核心压力来源——你永远无法停下来"慢慢准备"，而是必须持续产出。这与游戏开发物语的"数周开发周期"(一年约2-4款游戏)形成对比：游戏开发是项目制(一个项目周期内可以深度打磨)，杂志出版是周期性(每月必须出刊)。这种差异塑造了完全不同的游戏节奏——杂志物语更紧凑、更忙碌；游戏开发物语更有计划性。

## 三、南瓜商店(Pumpkin Products)

### 出现规律

| 项目 | 说明 |
|------|------|
| **首次出现** | 第2年5月第2周 |
| **出现频率** | 每年5月第2周 |
| **出售物品** | 转职手册、Dead Bull(体力饮料)等限定道具 |
| **价格变化** | 随办公室搬迁次数递增 |

### 南瓜商店的战略价值

```
每年5月 → 南瓜商店出现
    ↓
重要道具(转职手册等)限时出售
    ↓
需要提前储备资金 → 影响前几个月的经济策略
    ↓
价格随办公室升级上涨 → 早期购买更划算
    ↓
"要不要现在买"的决策压力
```

> **设计特点**: 南瓜商店是开罗游戏的标志性系统——一个每年固定时间出现的周期性事件，出售独家道具。它为游戏增加了"定时决策"的节奏点：你需要在前几个月就开始储备资金，在商店出现时做出购买决策。这种"事件节奏"打破了一般经营游戏的"均匀节奏"，每12个月产生一个小高潮。

## 四、取材季节与时机

### 取材系统的时间维度

| 时间因素 | 影响 | 说明 |
|------|:--:|------|
| **季节** | 可能影响取材结果 | 不同季节在某地取材可能有不同发现(待补充具体数据) |
| **员工状态** | 取材消耗员工体力 | 取材归来后员工需要恢复时间 |
| **时机选择** | 在出刊前/后的取材安排 | 出刊前取材可能影响制作进度 |

### 取材时间管理

派遣员工外出取材是一个时间消耗型活动：

- 取材需要占用员工时间(离开编辑部)
- 取材期间该员工不参与杂志制作
- 需要平衡"取材获取新内容"和"保持制作团队完整"的关系

> **待补充**: 季节对取材结果的具体影响数据需要更多的游戏内验证。

## 五、多周目继承

### 结局后继承

第20年第3月"结局"后，可以开启新游戏(+)，继承部分数据：

| 继承项目 | 说明 | 能否继承 |
|:------:|------|:--:|
| 主题(Theme)等级 | 已解锁的主题等级继承 | 是 |
| 角度(Angle)等级 | 已解锁的角度等级继承 | 是 |
| 地点(Location) | 重置——需要重新发现 | 否 |
| 资金 | 重置——从零开始 | 否 |
| 员工属性/职业 | 重置——需要重新培养 | 否 |
| 订阅者 | 重置——从零开始 | 否 |
| 城镇数据 | 重置——需要重新建设 | 否 |
| Conch Shell道具 | 保留 | 是 |
| 已解锁的广告商/店铺类型 | 部分保留 | 是 |
| Alpaca Bank贷款记录 | 保留 | 是 |

### 多周目的隐藏角色

| 角色 | 解锁条件 |
|------|------|
| **Kairobot(开罗君)** | 使用Conch Shell招募——多周目继承后可直接招募 |

### 多周目的策略意义

```
多周目继承:
  保留内容库(主题/角度等级) → 开局即可制作高质量杂志
  保留Conch Shell → 可直接招募Kairobot
  重置经济 → 体验更高效的经营过程

目标:
  在更短的时间内冲击顶峰
  尝试不同的城镇建设路线
  追求更高的最终评分
```

> **跨游戏对比 - 多周目设计**: 几乎所有开罗游戏都有多周目继承机制，但继承的具体内容因游戏而异：
>
> | 游戏 | 核心继承内容 | 重置内容 |
> |------|------|------|
> | 杂志物语 | 主题/角度等级、Conch Shell | 资金、员工、订阅者 |
> | 游戏开发物语 | 方针点数、部分类型/题材 | 资金、员工、办公室 |
> | 漫画物语 | 已解锁题材/画风 | 资金、员工 |
> | 温泉物语 | 部分设施、金钱 | 布局、顾客 |
>
> 杂志物语的继承策略聚焦在"内容库"(主题和角度的等级)——因为杂志品质高度依赖内容多样性，保留内容库意味着新周目的开局可以快速产出高品质杂志。这与游戏开发物语的"方针点数"继承同构——都是保留"知识/经验"而非"物质资产"。

## 六、时间压力的设计价值

### 节奏分析的四个层次

| 节奏层次 | 周期 | 压力类型 |
|------|:--:|------|
| **出刊节奏** | 每月 | 持续稳定产出的截稿压力 |
| **年度事件** | 每年 | 南瓜商店(5月)+杂志大赛(年度) |
| **扩张节奏** | 数年 | 办公室搬迁/城镇建设的阶段性跃迁 |
| **终局节奏** | 20年 | 最终冲刺的倒计时压力 |

### 与其他开罗游戏的节奏对比

| 游戏 | 核心周期 | 年产量 | 终局时间 | 节奏特征 |
|------|:--:|:--:|:--:|------|
| 杂志物语 | 每月 | 12期杂志 | 20年 | **高频紧凑**——每月必须出刊 |
| 游戏开发物语 | 数周 | 2-4款游戏 | 21年 | **项目制**——深度打磨后上线 |
| 漫画物语 | 每周/每月 | 12-52话 | 20年 | **连载制**——持续创作 |
| 温泉物语 | 每日 | 365天 | 20年 | **日经营**——日常运营 |

> **设计总结**: 杂志物语的时间节奏是开罗游戏中**最紧凑**的——每月一期杂志的高频节奏使玩家始终处于"正在制作下一期"的状态。相比游戏开发物语的"项目制"(做一款游戏需要数周的深度投入)，杂志物语更像是"流水线"(每月一期的快速产出)。南瓜商店作为年度事件的节奏调节器，20年周期作为终局的倒计时压力——三者共同构成了杂志物语的"永远有事做+定期有大事+最终有目标"的时间节奏体系。
