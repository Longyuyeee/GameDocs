# 员工管理系统 — 杂志物语

> 游戏版本: Magazine Mogul (PC/Steam, Kairosoft, 2022)
> 数据来源: [Kairosoft Wiki (Fandom)](https://kairosoft.fandom.com/wiki/Magazine_Mogul), [Kairosoft Wiki Manual](https://kairosoft.fandom.com/wiki/Transcript:Manual_(Magazine_Mogul)), [Steam商店](https://store.steampowered.com/app/2054800/)

## 一、员工属性体系

每个员工有四个核心属性：

| 属性 | 英文 | 作用 | 重要性 |
|------|------|------|:--:|
| **写作(Writing)** | Writing | 影响文章质量 + 广告文案质量。高写作=杂志能**长期稳定销售** | ★★★★★ |
| **设计(Design)** | Design | 影响杂志视觉设计 + 广告设计质量。高设计=**初期销售爆发力** | ★★★★ |
| **企划(Planning)** | Planning | 影响企划会议方案质量 + 杂志趣味度。决定杂志的"好看"程度 | ★★★★★ |
| **取材(Scouting)** | Scouting | 影响外出取材的效果。高取材=更好的发现(新主题/作家/训练方式) | ★★★★ |

> **设计特点**: 四属性的不对称设计——写作和企划是"长期引擎"，设计是"爆发力"，取材是"研发"。这种不对称性创造了员工分工的策略空间：你不可能让每个员工四项全高，必须根据岗位需求培养不同方向的专才。

### 属性对角色的影响

| 岗位 | 首要属性 | 次要属性 | 说明 |
|------|:--:|:--:|------|
| **文章编辑(Article)** | 写作 | 企划 | 负责杂志文章内容，写作属性直接决定文章质量 |
| **广告业务(Advertising)** | 设计 | 写作 | 负责为店铺设计广告，设计属性决定广告品质→广告奖金 |

## 二、岗位分配系统

### 文章岗位 vs 广告岗位

员工可以通过 **Assign(分配)** 菜单在"文章岗位"和"广告岗位"之间切换：

| 岗位 | 核心产出 | 属性依赖 | 对收入影响 |
|------|------|:--:|:--:|
| **文章岗位** | 杂志文章内容 | 写作 > 企划 | 影响杂志品质→销售收入(间接) |
| **广告岗位** | 为店铺设计广告 | 设计 > 写作 | 广告质量→广告奖金(直接) |

### 岗位分配策略

```
初期(团队小):
  所有员工设文章岗位 → 确保杂志品质优先
  广告收入靠自然增长

中期(团队5-7人):
  2-3人广告岗位 + 其余文章岗位
  开始追求广告收入补充现金流

后期(团队8-10人):
  3-4人广告岗位 + 其余文章岗位
  广告收入成为重要收入来源
```

> **设计分析**: 岗位分配系统是杂志物语员工管理中最关键的策略决策——它本质上是在"销售收入"和"广告收入"之间分配人力资本。游戏开发物语中没有类似的分配系统(员工按照开发阶段轮换)，漫画物语中比较接近的是"作者定位"(主力/辅助)。这是杂志产业特有的"内容vs广告"二元收入结构的映射。

## 三、训练系统

### 训练方式

消耗知识点(Research Data)可以训练员工，提升属性：

| 训练等级 | 消耗 | 效果 | 解锁方式 |
|:--:|------|------|------|
| Lv.1 | 少量知识点 | 属性小幅提升 | 初始可用 |
| Lv.2 | 中等知识点 | 属性中幅提升 | 办公室升级后解锁 |
| Lv.3 | 较多知识点 | 属性大幅提升 | 取材发现新训练方式 |
| Lv.4 | 大量知识点 | 属性大幅提升 | 后期解锁 |
| Lv.5 | 大量知识点 | 属性最大提升 | 终局解锁 |

### 训练的限制

| 限制因素 | 说明 |
|------|------|
| **等级上限** | 每个员工每项属性最高可训练到Lv.5 |
| **知识点消耗** | 训练消耗的知识点递增——高等级训练性价比递减 |
| **效果衰减** | 训练到一定程度后，继续提升效果不再明显 |

### 训练效果衰减机制

当某项属性被训练到较高水平后，继续训练的边际收益递减——这是一个防止"无限堆属性"的软上限设计。玩家需要权衡：是继续堆一个员工的单项属性，还是培养更多员工的基础属性。

## 四、职业/转职系统

### 职业体系

员工有不同职业(Career)，职业影响属性偏向和成长方向：

| 职业 | 属性偏向 | 定位 |
|------|:--:|------|
| **记者(Reporter)** | 取材+写作 | 外勤型人才，适合取材发现新内容 |
| **编辑(Editor)** | 写作+企划 | 内勤型人才，适合文章制作和企划 |
| **设计师(Designer)** | 设计 | 广告/视觉方向专才 |
| **其他职业** | 按职业不同 | 待补充详细列表 |

注：职业的具体分类和数量需要进一步确认(不同版本可能有差异)。

### 转职(使用道具)

使用道具"转职手册(Career Change Manual)"可以改变员工职业：

| 项目 | 说明 |
|------|------|
| **获取方式** | 南瓜商店(Pumpkin Products)每年5月第2周出售 |
| **使用效果** | 员工职业改变→属性偏向调整 |
| **策略价值** | 将前期"用顺了"的员工转型到后期更需要的岗位 |

> **跨游戏对比 - 与游戏开发物语的转职树**: 游戏开发物语有完整的6条职业路线构成的转职树(程序员→总监→硬件工程师→黑客)，是开罗游戏中最复杂的职业体系。杂志物语的转职系统**更简单**——使用道具直接改变职业，而非通过等级解锁路线。这种简化反映了游戏体量和产业特性的差异：游戏开发需要"跨职能成长"(从程序员到制作人)，而杂志社的职业路径更扁平。但简化的代价是失去了游戏开发物语中"转职前先拉满属性"的策略深度。

## 五、作家签约系统

### 作家的发现与签约

| 阶段 | 说明 |
|------|------|
| **发现** | 派遣员工外出取材时发现作家 |
| **签约** | 与作家签订连载协议→支付稿费 |
| **连载** | 作家定期为杂志提供连载内容 |

### 作家的价值

| 价值维度 | 说明 |
|------|------|
| **内容补充** | 作家连载丰富杂志内容，不需要消耗员工精力 |
| **品质加成** | 连载对杂志有正面品质加成 |
| **读者留存** | 知名连载作家吸引固定读者群 |
| **知名度** | 签约知名作家提升杂志品牌 |

### 作家稿费

作家稿费是持续性的支出，作家知名度越高→稿费越高。但高稿费作家带来的品质加成和读者吸引力通常远超其成本——前提是杂志已经有一定的读者基础能够"消化"高知名度作家的内容。

> **设计特点**: 作家系统是"外部内容引入"的设计——员工是内部生产力，作家是外部内容提供者。两者的平衡决定了杂志的品质上限。这种"自制+外购"的内容策略模型与真实的杂志出版业高度吻合。

## 七、员工培养策略

### 四阶段培养路线

| 阶段 | 焦点 | 行动 |
|------|:--:|------|
| **起步期(1-3年)** | 取材属性优先 | 提升取材→发现新内容→丰富杂志内容库 |
| **成长期(4-8年)** | 平衡培养 | 写作+设计均衡提升，开始关注企划 |
| **扩张期(9-15年)** | 企划+写作 | 为冲击大奖做准备，打磨核心团队 |
| **巅峰期(16-20年)** | 针对性特化 | 根据短板重点提升，追求极限属性 |

> **设计总结**: 杂志物语的员工管理系统是一个**四属性×双岗位×外部作家**的三层结构。与游戏开发物语的转职树相比，杂志物语的员工培养更横向(多面手)而非纵向(专精深)。岗位分配(文章vs广告)是核心策略决策，替代了游戏开发物语中"谁负责哪个开发阶段"的分配策略。整体而言，员工管理复杂度适中，聚焦于"人的分配"而非"人的成长路径"。
