# 团队与开发历程 — 温泉物语 (Hot Springs Story / ゆけむり温泉)

> **版本**: Steam 版 (2022.03) | **引擎**: 开罗社自研 (C++/Java) | **平台**: PC, Switch, iOS, Android
>
> **来源**: [Kairosoft 官方 - 温泉物语](https://kairosoft.net/game/hotspring/) | [Steam 商店页](https://store.steampowered.com/app/1823710/) | [Wikipedia - Hot Springs Story](https://en.wikipedia.org/wiki/Hot_Springs_Story) | [Famitsu 开罗社采访](https://www.famitsu.com/)

## 1. 开发团队 — 开罗社 (Kairosoft)

### 1.1 公司背景

温泉物语由**开罗社 (Kairosoft Co., Ltd.)**开发。详见[游戏开发物语-团队与开发历程](../游戏开发物语/特色文档/团队与开发历程.md)中的开罗社完整背景。

### 1.2 温泉物语在开罗产品线中的定位

| 维度 | 说明 |
|:----:|------|
| **开发顺序** | 开罗第 10+ 款作品，但在全球市场是**第 2 款获得广泛认可的游戏** |
| **主题选择** | 日本温泉旅馆经营，避开了 Game Dev Story 的"游戏开发"国际化主题，选择了更本土化的题材 |
| **目标市场** | 首发针对日本本土市场，但因前作口碑迅速获得全球关注 |
| **重要性** | 温泉物语回答了开罗社最关键的问题：**"开罗公式是否可迁移到不同主题？"** 答案是一个响亮的"是" |

### 1.3 团队规模

开罗社在 2011 年前后的员工数量约为 10-15 人。温泉物语的开发由其中的一个小团队（4-6 人）核心完成，其他成员并行开发其他项目。这是开罗多项目流水线模式的早期体现。

## 2. 开发历程

### 2.1 温泉物语诞生背景

2010 年 Game Dev Story 在 iOS 上的意外爆红让开罗社面临一个选择: 是做一款"游戏开发公司"的直接续作，还是运用同一套公式做一个新主题？

臼井和之选择了后者。他认为：
- 开罗的核心竞争力不是"游戏开发"这个主题，而是**"用参数化引擎做模拟经营游戏"**的能力
- 如果第二款游戏选择完全不同的主题（温泉旅馆）并同样成功，就能证明开罗公式的通用性
- 这也为开罗后续的"高频发游戏"策略（每年 3-5 款新作）铺平了道路

### 2.2 时间线

| 年份 | 事件 |
|:----:|------|
| 2010.07 | Game Dev Story iOS 版全球爆红 |
| 2010.12 | 开罗社决定启动新项目——温泉物语 |
| **2011.02** | **温泉物语 iOS/Android 日本版发布** — 距离前作仅 7 个月 |
| 2011 年中 | 英文本地化版全球发布 |
| 2018 | Nintendo Switch 版发布 |
| **2022.03.27** | **Steam 版发布** (与 Game Dev Story Steam 版同日) |
| 2022.12 | 《温泉物语2》Steam 发布 — 约 11 年后的正式续作 |

### 2.3 从 Game Dev Story 到温泉物语的设计进化

温泉物语不是 Game Dev Story 的"换皮"，而是对开罗公式的第一次重大迭代：

| 设计维度 | Game Dev Story | 温泉物语 | 进化方向 |
|:--------:|:--------------:|:--------:|---------|
| **空间系统** | 无 (纯工位制) | **完整的景观辐射 + 契合度系统** | 加入了"空间"维度 |
| **员工系统** | 复杂 (6 职业线转职树) | 简单 (分配制) | 弱化以平衡新增的空间复杂度 |
| **客人系统** | 统计数据 (无个体) | **49 种独立偏好客人** | 从"统计"到"个体" |
| **竞争系统** | GGA 年度大奖 | 8 大杂志排名 | 从"单一年奖"到"多维度评比" |
| **空间契合度** | 无 | 设施之间的"契合度"决定组合加成 | 开罗首次引入网格化空间管理 |
| **终局** | 21 年固定结算 | 15 年结算 + 可自由继续 | 更灵活 |

### 2.4 空间系统: 开罗公式的第一次重大迭代

温泉物语最大的设计贡献是**"空间即产品"**的理念。在 Game Dev Story 中，玩家的决策主要是"雇佣谁"和"开发什么游戏"——是一个关于"人和时间"的管理游戏。而在温泉物语中，决策核心变成了"在哪里放什么设施"——是一个关于"空间和布局"的游戏。

这一转变让开罗的模拟经营游戏从"数据管理"进阶到"空间规划"，影响深远:
- 后续的《百货商场物语2》继承了商场楼层布局
- 《都市大亨物语》继承了城市功能分区
- 《住宅梦物语》继承了公寓房间布置

## 4. 温泉物语 2 (2022) 的传承

2022 年 12 月，开罗社发布了《温泉物语 2》。与 1 代相比:

| 维度 | 温泉物语 (2011) | 温泉物语 2 (2022) |
|:----:|:---------------:|:-----------------:|
| 画面 | 经典像素 2D | 像素增强 HD |
| 设施数 | 约 50 种 | 约 100+ 种 |
| 客人类型 | 49 种 | 更多个性化客人 |
| 空间系统 | 平面网格 | 多层建筑 + 室外扩展 |
| 温泉功效 | 跨周目叠加 | 更丰富的组合效果 |

2 代使用了开罗新一代引擎，增加了更多现代 UX 设计，但核心公式——空间布局 × 客人满意度 × 多维度排名——与 1 代保持一致。

## 5. 设计启示

1. **开罗公式的可迁移性证明** — 温泉物语回答了"开罗的成功是 Game Dev Story 的偶然，还是开罗方法的必然"这个问题。它证明了开罗的核心竞争力是"做模拟经营游戏的方法论"，而不是某个特定的主题

2. **空间布局作为核心玩法的可行性** — 在温泉物语之前，很少有"放置设施→观察效果→优化布局→评分反馈"这种纯空间规划的经营游戏。它开辟了"空间即产品"的子类型

3. **主题多样性是独立工作室的抗风险策略** — 开罗通过覆盖"游戏开发、温泉旅馆、百货商场、拉面店、漫画出版"等不同主题，在任何一个细分市场出现波动时都有其他产品支撑

4. **续作周期的智慧** — 11 年后才出温泉物语 2，既不是"趁热打铁"也不是"放弃 IP"，而是让 1 代充分释放价值后再用新技术拥抱新玩家。这种"慢续作"策略在移动游戏时代极为罕见
