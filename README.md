## 采集规则

> 📋 **全局强制规范**: [GameDocs 采集守则 v2.1](./采集策略/GameDocs采集守则.md) — 面向所有品类的质量等级标准、来源规范、文档深度要求。**采集前必读。**
>
> 各品类的具体维度定义、数据表结构和信息源策略参见各自的采集策略文件。

### 核心原则

- **三条铁律**：每个数字必须有来源 / 文档深度 > 数量 / 不凭空捏造
- **质量三级制**：A级(⭐) / B级 / C级(⚠红线)，目标全品类B级以上
- **来源优先级**：官方Wiki(L1) → 社区Wiki(L2) → Steam指南(L3) → 视频攻略(L4) → 社区讨论(L5) → 评测分析(L6) → 数据挖掘(L7)

### 游戏筛选标准（全局）

| 标准 | 要求 |
|------|------|
| 平台 | PC / 主机（排除纯手游 / 页游） |

## 补充说明

### 关于文档数量差异

不同游戏的文档数量存在差异，原因包括：
- **游戏类型差异**：跨界游戏（是的陛下、偶像经理人、大正对称爱丽丝）因玩法与传统养成不同，维度有所调整
- **信息可得性**：部分游戏（如莱尔兰德物语）缺乏公开 Wiki 或详尽攻略，数据文档基于评测交叉验证
- **持续迭代**：所有文档均标注数据来源与版本，随新信息持续完善

### 关于文档命名

- 游戏文件夹名：中文（如 `火山的女儿`、`星露谷物语`）
- 机制分析文档：中文描述（如 `日程与行动系统.md`、`经济系统.md`）
- 数值数据文档：中文 + "数据总览"后缀（如 `活动属性数据总览.md`）
- 策略与索引文件：英文下划线前缀（如 `_index.md`）
