## 📋 采集规则

> 📄 **全局强制规范**: [GameDocs 采集守则 v2.2](./采集策略/GameDocs采集守则.md) — 面向所有品类的质量等级标准、来源规范、文档深度要求。**采集前必读。**
>
> 各品类的具体维度定义、数据表结构和信息源策略参见各自的采集策略文件。

### 三条铁律

| # | 规则 |
|:--:|------|
| 1 | **每个数字必须有来源URL** — 没有来源的数据不允许写入文档 |
| 2 | **文档深度 > 文档数量** — 一篇 5KB 的详细数据表胜过三篇 1KB 的空壳 |
| 3 | **真实数据 > 善意推测** — 无法验证的数据标注"待补充", 不凭空捏造 |

### 第四条铁律（v2.2 新增）

> **P0/P1/P2 仅代表对用户的重要度/参考优先级，不影响采集深度。** 所有游戏无论优先级一律按最高标准采集。优先级决定先做哪个，不决定做到什么程度。

### 质量三级制

| 等级 | 标志 | 机制文档 | 数据文档 |
|:--:|:--:|------|------|
| **A 级** | ⭐ | ≥10KB + 5+数据表 + 设计意图分析 + 跨游戏对比 | ≥5KB + 全结构化表格 + 性价比/策略分析 |
| **B 级** | — | ≥3KB + 3+数据表 + 设计特点分析 | ≥2KB + 表格主体 + 来源标注 |
| **C 级** | ⚠ | <3KB 或纯文字无表格 | <2KB 或数据稀疏 |

- **目标**: 所有游戏达到 B 级以上，品类标杆达到 A 级
- **红线**: 任何文档不得为 C 级——宁缺毋滥

### 来源优先级

```
官方Wiki(L1) → 社区Wiki(L2) → Steam指南(L3) → 视频攻略(L4) → 社区讨论(L5) → 评测分析(L6) → 数据挖掘(L7)
```
