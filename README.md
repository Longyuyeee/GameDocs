## 采集规则

> 📋 **全局强制规范**: [GameDocs 采集守则 v2.0](./采集策略/GameDocs采集守则.md) — 面向所有品类的质量等级标准、来源规范、文档深度要求。采集前必读。

### 标准维度

每款游戏覆盖 **8 项标准机制维度**，每维度一份独立分析文档：

| # | 维度 | 核心内容 |
|:--:|------|------|
| 1 | **日程与行动** | 时间结构（年 / 月 / 周 / 日）、活动槽数量与类型、疲劳 / 压力 / 体力机制、行动→属性映射 |
| 2 | **属性与成长** | 属性种类与子属性、计算公式与效率因子、属性阈值→事件触发、隐藏属性 / 特质 |
| 3 | **事件与叙事** | 触发机制（属性 / 日期 / 前置 / 随机）、事件密度与总量、选择→后果分支、多周目事件差异 |
| 4 | **社交与关系** | NPC 数量与角色定位、好感度等级与阈值、礼物偏好表、关系→结局关联 |
| 5 | **经济系统** | 货币类型与获取途径、消费出口与性价比、各阶段经济策略、经济与属性的关联 |
| 6 | **结局系统** | 结局总数与分类、判定逻辑与属性阈值、优先级 / 互斥规则、隐藏结局条件 |
| 7 | **服装 / 装备 / 道具** | 服装种类与获取、属性 / 活动效率修正、消耗品 / 礼物 / 特殊道具、武器系统（如有） |
| 8 | **美术与情感化设计** | 角色美术风格（立绘 / 动画 / 年龄变化）、UI 信息架构、情感化设计手法、音频表现 |

> 如某维度在游戏中不具备，文档标注「无此系统」并简要说明原因。

### 信息源优先级

```
1. 官方 Wiki / 攻略本           ← 最权威
2. 社区 Wiki / Fandom           ← 信息量最大
3. Steam 社区指南 / 深度攻略     ← 玩家验证
4. B站 / YouTube 攻略视频       ← 可视化参考
5. 贴吧 / NGA / TapTap          ← 社区共识
6. 知乎 / 豆瓣                  ← 设计分析
7. 数据挖掘 / 玩家工具           ← 精确数值
```

### 质量标准

- **可追溯**：每条信息必须能回溯到原文 / 原链接
- **数值准确**：涉及数值时注明具体数字和范围
- **版本标注**：注明信息对应的游戏版本号
- **机制完整**：不只有"有什么"，更要有"怎么运作"和"为什么这么设计"
- **数据矛盾处理**：以官方 Wiki 为最终仲裁依据
- **中文优先**：中文行文优先，英文 / 日文原名首次出现时括号附注

### 游戏筛选标准

| 标准 | 牧场经营类 | 培养模拟类 |
|------|------|------|
| 平台 | PC / 主机（排除纯手游 / 页游） | 同左 |
| 评价 | Steam 好评率 > 85% 或 Metacritic > 75 | 同左 |
| 核心玩法 | 以农场 / 牧场经营为核心循环 | 以培养 / 角色成长为核心，有日程安排与多结局 |
| 知名度 | 品类中有一定影响力或独特参考价值 | 同左 |

## 补充说明

### 关于文档数量差异

不同游戏的文档数量存在差异，原因包括：
- **游戏类型差异**：跨界游戏（是的陛下、偶像经理人、大正对称爱丽丝）因玩法与传统养成不同，维度有所调整
- **信息可得性**：部分游戏（如莱尔兰德物语）缺乏公开 Wiki 或详尽攻略，数据文档基于评测交叉验证
- **持续迭代**：所有文档均标注数据来源与版本，随新信息持续完善

### 关于文档命名

- 游戏文件夹名：中文（如 `火山的女儿`、`星露谷物语`）
- 机制分析文档：中文描述（如 `日程与行动系统.md`、`经济系统.md`）
- 数值数据文档：中文 + "数据总览"后缀（如 `活动属性数据总览.md`）
- 策略与索引文件：英文下划线前缀（如 `_index.md`）
