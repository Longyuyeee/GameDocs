Winds of Anthos 是 Harvest Moon 系列中**首部真正开放世界的作品**，彻底告别了传统"固定区域+加载切换"的地图设计。

## 二、传送系统

本作的地图极为广阔，因此传送系统是**核心移动手段**而非奢侈功能。

| 传送点 | 解锁方式 | 说明 |
|:------:|:--------:|------|
| 水井 | 初始可用 | Lectenbury |
| 灯塔 | 探索解锁 | Lilikala |
| 大树 | 探索解锁 | 帕斯蒂亚区域 |
| 烤炉 | 探索解锁 | Herbstburg |
| 日晷 | 探索解锁 | Zimagrad |
| 天气雕像 | 探索解锁 | 火山区域 |
| 丰收女神之门 | 主线解锁 | Providence |
| 玩家农场 | 始终可用 | 自宅 |

> 传送点的具体数量社区统计不一，至少 8 个以上为主流看法。部分传送点需要通过主线剧情或特定任务解锁。

## 三、农场选址与搬迁

### 3.1 农场搬迁机制

| 要素 | 说明 |
|:----:|------|
| **搬迁成本** | 消耗**一整天**（搬迁当日不能做其他事） |
| **搬迁地点** | 可在所有已解锁的农场间选择 |
| **原农场状态** | 保留原状，农作物/动物数据不丢失（推测） |
| **战略意义** | 搬迁至永久季节农场可实现全年高价值作物生产 |

> 农场的"同时使用"问题：搬迁后原农场是暂停状态还是自动运行——社区对此存在分歧。多数观点认为是暂停状态（类似休眠）。

## 五、DocPad（游戏内辅助系统）

本作内置了 DocPad 作为玩家的辅助系统，整合地图、任务跟踪、收集进度等功能。

| DocPad 功能 | 说明 |

## 七、设计特点分析

### 7.1 与传统牧场游戏的对比

| 维度 | Winds of Anthos | 星露谷物语 | 牧场物语: 矿石镇 | SoS系列 |
|:---:|:--------------:|:----------:|:----------------:|:--------:|
| **地图结构** | 开放世界（无缝大地图） | 固定区域+加载 | 固定区域+加载 | 固定区域+加载 |
| **农场数量** | 8+（可搬迁） | 1（不可移动） | 1（不可移动） | 1（不可移动） |
| **传送系统** | 有（8+传送点） | 有（少量） | 无 | 有（部分作品） |
| **Wisp导航** | 有（开放式收集） | 无 | 无 | 无 |
| **DocPad** | 有（游戏内辅助） | 无 | 无 | 无 |
| **拍照系统** | 有 | 有 | 无（原版） | 部分有 |

### 7.2 核心创新

1. **开放世界的首次尝试**：Winds of Anthos 是 Natsume 自研 Harvest Moon 系列中首个真正的开放世界，标志着系列从"小镇模拟器"向"世界探索器"的转型。

2. **多农场系统**：不是"一个农场做到满"而是"探索→发现新农场→战略搬迁→专业化生产"。农场成为了探索的奖励而非起点的约束。

3. **景观即玩法**：每个区域的永久季节是核心玩法机制，而非装饰。选择搬迁至 Herbstburg 就是选择了"全年秋作物路线"。

### 7.3 局限性

| 问题 | 说明 |
|:----:|------|
| 开放世界地图过大而内容填充略显稀疏 | 部分区域除 Wisp 刷新外缺少互动点 |
| 传送点覆盖不够密集 | 中间区域的步行距离偏长 |
| 农场搬迁的'一整天'消耗偏高 | 限制了玩家尝试不同农场的意愿 |
| 标志性地标数量有限 | 地图导航主要依赖传送点而非视觉地标 |

---

> **数据真实性声明**：地图区域描述基于社区探索报告和官方放出的信息。精确的耕地数量和传送点坐标社区仍在收集中。DocPad 和拍照系统的功能清单来自玩家体验报告。由于本作开放世界内容量较大，部分偏远区域的完整数据可能尚未被充分记录。
