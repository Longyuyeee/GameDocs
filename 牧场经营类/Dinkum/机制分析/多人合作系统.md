# 多人合作系统 - Dinkum

> 游戏版本: Dinkum v1.0 (PC/Steam, Nintendo Switch, 2025正式版) | 开发商: James Bendon (单人)
> 数据来源: [PlayCoopGame Dinkum](https://playcoopgame.com/dinkum/) | [TheGamer Multiplayer Guide](https://www.thegamer.com/dinkum-multiplayer-co-op-friends-explained-tutorial-guide/) | [LionadeGames Co-op Guide](https://lionadegames.com/understanding-the-mechanics-of-co-op-in-dinkum-a-comprehensive-guide/) | [AttackOfTheFanboy Guide](https://attackofthefanboy.com/guides/dinkum-multiplayer-guide-how-to-play-with-friends/) | [SirusGaming Guide](https://sirusgaming.com/dinkum-multiplayer-co-op/)

## 1. 联机参数

| 参数 | 设定 |
|:----:|:----:|
| 最大玩家数 | **4人**(1主机+3访客) |
| 在线联机 | 支持(Steam好友邀请) |
| LAN联机 | 支持 |
| **分屏合作** | **不支持** |
| **同屏本地** | **不支持** |
| 跨平台 | 待确认 |
| 存档归属 | 仅主机存档——访客进度在主机世界保存 |

> 每位玩家需要自己的游戏副本和PC/主机——不支持一台设备上双人玩。

## 2. 联机解锁流程

| 步骤 | 说明 |
|:----:|------|
| 1. 单人开局 | 每位玩家先各自完成新手教程 |
| 2. 完成第一天 | 完成Fletch的第一个任务, 然后睡觉(保存游戏) |
| 3. 选择模式 | 主机从主菜单选择"Multiplayer" |
| 4. 设置权限 | 选择隐私设置: Friends Only / Invite Only / Public |
| 5. 邀请好友 | 通过Steam好友列表邀请 |
| 6. 加入游戏 | 访客从多人菜单选择主机的公开游戏 |

> 第一天教程结束后才能解锁多人——这是防止玩家跳过新手引导的必要设计。

## 3. 主机与访客权限

### 3.1 主机(Host)权限

| 权限 | 说明 |
|:----:|------|
| 世界所有权 | 主机世界=存档所有者——访客离开后, 建好的内容仍保留 |
| 建筑放置/移动 | 只有主机可以放置和移动建筑 |
| NPC管理 | 主机决定NPC建筑契约的购买和放置 |
| 动物交易 | 只有主机可以购买和出售动物 |
| Kick权限 | 主机可以踢出不受欢迎的玩家 |

### 3.2 访客(Visitor)权限

| 权限 | 可用 | 不可用 |
|:----:|:----:|:------:|
| 采集资源 | ✓ 钓鱼/采矿/伐木/采集 | — |
| 使用工具 | ✓ 所有工具 | — |
| 战斗 | ✓ 狩猎/战斗 | — |
| 烹饪/制作 | ✓ 使用已有工作台 | — |
| 使用储物 | ✓ 取用和放入物品 | — |
| 对话NPC | ✓ 可与所有NPC互动 | — |
| 建筑放置 | — | ✗ 不能放置/移动建筑 |
| 建筑契约 | — | ✗ 不能购买建筑契约 |
| 动物交易 | — | ✗ 不能购买/出售动物 |

> **重要**: 访客可以从储物箱中取走物品——包括主机存放的所有物品。建议只邀请可信任的好友。主机有权踢出不希望的玩家。

## 4. 协作内容

### 4.1 采集协作

| 活动 | 单人效率 | 4人效率 | 提升倍数 |
|:----:|:--------:|:--------:|:--------:|
| 采矿 | 1人挖矿 | 4人分工(挖+运输+战斗) | ~3-4x |
| 伐木 | 1人砍树 | 4人同时砍树 | ~4x |
| 钓鱼 | 1人钓鱼 | 4人不同水域钓鱼 | ~4x |
| 采集 | 1人跑图 | 4人分散采集 | ~4x |

### 4.2 建设协作

| 活动 | 协作方式 | 效果 |
|:----:|---------|------|
| 石材/木材收集 | 分工: 2人伐木+2人采矿 | 建材收集速度翻倍+ |
| 建筑建造 | 多人同时贡献材料 | 建筑契约完成更快 |
| 地形改造 | 多人同时铺路/挖地 | 城镇建设大幅提速 |
| 设施操作 | 同时使用多个加工台 | 生产流水线化 |

### 4.3 战斗协作

| 敌人 | 单人策略 | 多人策略 |
|:----:|:--------:|:--------:|
| Croco(鳄鱼) | 远程风筝+大量走位 | 1人吸引+3人集火 |
| Shark(鲨鱼) | 避免下水 | 多人乘船围猎 |
| Alpha级敌人 | 难(需要高级装备+大量回复) | **推荐多人**(坦克+输出+回复) |
| Deep Mine怪物 | 逐个击杀, 谨慎推进 | 分工(探路+输出+后勤) |

> 战斗是多人合作提升最大的领域——敌人对不同玩家的仇恨机制让团队战斗变得简单。

### 4.4 NPC任务协作

| 任务类型 | 单人 | 多人 |
|:--------:|:----:|:----:|
| 收集型 | 单人收集 | 多人分头收集→更快完成 |
| 狩猎型 | 逐个击杀 | 团队围猎 |
| 送货型 | 单人跑腿 | 多人可同时接多个任务 |

## 5. 多人经济

### 5.1 共享与独立

| 经济系统 | 共享/独立 | 说明 |
|:--------:|:---------:|------|
| Dinks(金币) | **按主机世界存档** | 所有玩家赚取的金币归入主机世界 |
| 物品 | **共享储物** | 所有玩家可存取同一套储物系统 |
| 许可证 | **访客无法独立购买** | 访客使用主机的许可证进度 |
| 任务奖励 | **归主机世界** | 任务奖励进入主机经济系统 |

### 5.2 玩家间交易

| 交易方式 | 说明 |
|:--------:|------|
| 丢物品 | 玩家可将物品丢在地上→其他玩家捡起 |
| 储物共享 | 所有玩家使用同一套储物箱系统 |
| 工具分享 | 访客可以使用主机的工具和装备 |
| Dinks转移 | 理论上可通过丢物品间接转移价值 |

> Dinkum没有内置的玩家交易UI——所有交易通过丢物品/共享储物实现。这是很多玩家反馈希望改进的地方。

## 6. 日/夜循环与睡眠机制

| 机制 | 单人 | 多人 |
|:----:|:----:|:----:|
| 睡眠 | 玩家独立决定睡觉 | **所有玩家需同意**才能推进到第二天 |
| 时间流动 | 单人游戏正常流动 | 游戏时间**暂停直到所有人都准备好** |
| 不同步 | — | 如果某玩家未准备好, 其他人不能强制睡 |
| 深夜 | 体力消耗极快 | 同上(所有玩家都受影响) |

> 睡眠共识机制确保了所有玩家对时间推进有平等的控制权——避免了"一个玩家急于睡觉, 其他人还没准备好"的冲突。

## 7. 网络要求

| 项目 | 最低要求 | 推荐 |
|:----:|:--------:|:----:|
| 网络类型 | 宽带互联网 | 低延迟宽带 |
| 上传速度(主机) | 5 Mbps | 10+ Mbps |
| 下载速度(访客) | 5 Mbps | 10+ Mbps |
| NAT类型 | 开放/中等(Open/Moderate) | Open |
| 跨区域 | 可行但延迟更高 | 同区域最佳 |

## 8. 多人常见问题

| 问题 | 原因 | 解决 |
|:----:|:----:|:------|
| 无法加入游戏 | NAT类型限制/防火墙 | 检查网络设置/端口转发 |
| 延迟高 | 跨区域联机/主机上传不足 | 选择同区域好友/升级网络 |
| 物品丢失 | 访客取走了物品 | 仅邀请可信任好友 |
| 访客不能睡觉 | 设计如此 | 需要主机推进到第二天 |
| 访客任务不保存 | 设计如此 | 访客的本地任务进度不保存 |

## 9. 设计特点分析

### 9.1 访客=帮手模式, 而非独立玩家

Dinkum的多人模式采用了**"访客帮手"模型**——访客是主机世界中的协作方, 而非独立的"第二个镇长"。这与《星露谷物语》的多人模式(所有玩家共享农场, 各自独立操作但共用资源)和《动物森友会》的多人模式(访客不能改造岛屿)都有所不同。

| 维度 | 星露谷物语 | Dinkum | 动物森友会 |
|:----:|:--------:|:------:|:--------:|
| 玩家角色 | **共同农场主** | 访客/帮手 | 访客 |
| 建筑权限 | 所有玩家 | **仅主机** | **仅岛主** |
| 储物 | 独立+共享 | **完全共享** | 独立 |
| 进度 | **各自保存** | **仅主机保存** | 各自动物森友会存档 |
| 最大玩家 | 4人 | 4人 | 8人(上岛) |

Dinkum的模型更接近"协助型合作"——访客来帮忙, 而不是来"经营自己的世界"。

### 9.2 无分屏的决策

Dinkum不支持分屏/同屏合作——这是一个明确的设计决策。原因可能包括:

1. **性能**: 4人分屏对渲染性能要求极高
2. **视距**: Dinkum的开放世界不适合分屏(玩家可能分处地图不同角落)
3. **单人开发的资源限制**: James Bendon单人开发——实现高质量分屏的成本过高

### 9.3 睡眠共识机制

"所有玩家必须同意才能睡觉"的机制是多人合作游戏中的最佳实践——它确保:
- 没有人被"拖入"第二天
- 所有人都准备好了时间推进
- 避免了"队友突然消失"的挫败感

但这也带来了一个潜在问题: 如果某个玩家AFK(暂离), 其他三人无法推进时间。

### 9.4 访客物品安全

Dinkum没有实现访客物品权限控制——访客可以取走主机储物箱中的所有物品。这既是优点(协作方便)也是隐患(不信任的玩家可偷窃)。建议:
- 只与可信任的好友联机
- 在公共服务器上注意贵重物品
- 利用主机Kick功能驱逐不当行为的玩家

### 9.5 跨游戏对比: 珊瑚岛(Coral Island)

珊瑚岛的多人模式(2024年发布)同样采用4人合作模型, 但珊瑚岛允许所有玩家独立推进自己的进度, 访客可以建造自己的房屋和农场。Dinkum选择了更集中的"一人世界, 多人协作"模式——各有优劣。

Dinkum模式的优势在于: **城镇建设更一致**(不会有"各建各的"导致风格冲突); 劣势在于: **访客的成就感较低**(不能留下永久标记)。

---

> **设计启示**: Dinkum的多人模式选择了"集中式协作"而非"独立式并行"——访客是帮手而非主人。这种模式适合"共同建设一个城镇"的合作体验, 但牺牲了访客的独立成就感。对于单人独立开发者来说, 这种简化是正确的取舍——降低了网络同步的复杂度, 同时保留了"和朋友一起玩"的核心体验。
