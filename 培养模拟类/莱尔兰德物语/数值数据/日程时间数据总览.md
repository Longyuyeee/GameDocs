# 日程时间数据总览 - 莱尔兰德物语(Lair Land Story)

> 游戏版本: Lair Land Story v1.0 (PC/Steam, 2020; Switch, 2022)
> 原始版本: 2007年
> 数据来源: Rice Digital, LadiesGamers, DigitallyDownloaded, Siliconera

## 一、全年日程结构

| 时间维度 | 结构 | 说明 |
|------|------|------|
| 培养周期 | **4年** | 收养到成年 |
| 每年 | **4季**(Spring/Summer/Autumn/Winter) | 季节循环 |
| 每季 | 多周 | 具体周数未公开 |
| 每周 | **3个活动槽** | 学习/工作/重建/休息任意组合 |
| 每周额外 | **3个城镇访问** | 地点探索(与活动槽独立) |

## 二、全程活动槽总量估算

```
假设每季约12周:
4年 × 4季/年 × 12周/季 ≈ 192周
192周 × 3活动槽/周 = 576个活动槽

假设每季约8周(更保守):
4年 × 4季/年 × 8周/季 ≈ 128周
128周 × 3活动槽/周 = 384个活动槽
```

> 实际周数未公开，以上为大致量级的估算。

## 三、城镇访问总量估算

```
192周 × 3次/周 = 576次城镇访问
(或128周 × 3次/周 = 384次)
```
每次访问可触发对话或事件。

## 四、每周时间分配策略

### 4.1 均衡型(新手推荐)
```
活动: 1学习 + 1工作 + 1休息
城镇: 1广场 + 1商店 + 1随机探索
```

### 4.2 学术冲刺型
```
活动: 2学习 + 1休息
城镇: 1图书馆 + 1神殿 + 1探索
风险: 金币可能不足
```

### 4.3 财富积累型
```
活动: 1学习 + 2工作
城镇: 1商店 + 1广场 + 1社交
风险: 属性增长慢
```

### 4.4 城市重建型
```
活动: 1学习 + 1工作 + 1重建
城镇: 1广场(市民) + 1宫殿 + 1探索
```

## 五、疲劳-热情管理节奏(推断)

| 模式 | 连续学习/工作周 | 休息间隔 |
|------|:--:|:--:|
| 高强度 | 3-4周 | 每4周休1周 |
| 中等强度 | 2-3周 | 每3周休1周 |
| 低强度(安全) | 1-2周 | 每2周休1周 |

### 疲劳管理最佳实践
```
检测疲劳接近阈值 → 下周插入休息活动
检测热情下降 → 更换活动类型(学习→工作→休息)
```

## 七、Bonus Goal时间线

```
每季开始 → 设定Bonus Goal
     ↓
整个季度 → 执行活动达成目标
     ↓
每季结束 → 结算→获得/未获得Bonus Points
     ↓
下季开始 → 新Bonus Goal
```

## 八、长期规划参考

| 年份 | 阶段定位 | 主要目标 |
|:--:|------|------|
| 第1年 | 基础期 | 了解系统+建立金币基础+确定专业方向 |
| 第2年 | 成长期 | 深化专业属性+开始城市重建+推进关系线 |
| 第3年 | 发展期 | 属性冲刺+完成关键事件线+做出关键关系选择 |
| 第4年 | 收尾期 | 补齐结局阈值+最终事件线+结局判定 |

## 九、设计特点

1. **每周3+3的双层时间分配**: 活动安排(养成层)+城镇访问(叙事层)→时间管理的双重维度
2. **活动槽=稀缺资源**: 每周仅3个选4大类→每次选择都有机会成本→专业化自然产生
3. **城镇访问=从菜单到空间**: 不是点菜单→是"去城镇的不同地点"→增加了空间感和发现感
4. **无明确AP制**: 不同于梦幻魔法公主的15AP点数→此游戏用"活动槽"作为限制→更简单直接
