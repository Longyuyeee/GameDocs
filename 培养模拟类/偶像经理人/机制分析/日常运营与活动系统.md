# 日常运营与活动系统 - 偶像经理人(Idol Manager)

> 游戏版本: Idol Manager v1.0 (PC/Steam, 2021年7月27日)
> 数据来源: GamePretty攻略, NGA攻略, PTT, Steam社区

## 1. 时间结构

| 参数 | 设定 |
|------|------|
| 游戏周期 | **无限年**(持续经营→征服娱乐圈) |
| 每日行动 | 1次日常活动(演出/宣传/SPA) |
| 每周循环 | 训练安排+合同处理+设施管理 |
| 每月循环 | 单曲制作/演唱会/财务管理 |
| 年度循环 | 年度奖项+总选举 |

## 2. 每日活动(三选一)

| 活动 | 效果 | 消耗 |
|------|------|:--:|
| **演出(Performance)** | 按等级获得金币 | 偶像体力 |
| **宣传(Promotion)** | 按等级获得粉丝 | 偶像体力+金币 |
| **SPA** | 最多5名偶像恢复体力(3天最佳) | 金币 |

## 4. 合同系统

### 4.1 合同类型

| 类型 | 主要收益 | 次要收益 | 消耗 |
|------|------|------|:--:|
| **写真合同** | ★★★★ 金币 | 粉丝↑(少量) | 偶像体力 |
| **媒体合同** | ★★ 金币 | **知名度↑** | 偶像体力 |
| **商务合同** | ★★★ 金币 | — | 偶像体力 |

### 4.2 合同策略
- 前期: **写真合同为主**(稳定金币+不需要知名度)
- 中期: 写真+媒体(提升知名度→解锁更多机会)
- 后期: 媒体为主(知名度品牌价值)

## 6. 演唱会系统

### 6.1 准备条件
- 至少**10首单曲**
- 预计上座率≥60%
- 足够的偶像体力储备

### 6.2 节目编排
```
1-3: 开场(中规中矩)
4-6: 中段(稳定输出)
7-8: 高潮前(建立期待)
9: 休息
10: 压轴(最有把握的节目!)←最重要
```

### 6.3 事故处理
- 事故率>30%→使用减事故率卡
- 不同类型事故需要不同应对
- C位体力不足→使用加体力卡

## 8. 设计特点分析

1. **无限年≠无限内容**: 持续经营→但核心循环是"做单曲→赚粉丝→做更多单曲"→后期可能缺乏新挑战
2. **每日三选一=简洁但有效**: 演出/宣传/SPA→只有三个选择但每个有明确的战略目的
3. **单曲制作管线=生产链管理**: 不是一键出歌→要管理作词/作曲/编曲/编舞四个步骤→模拟真实音乐制作的复杂性
4. **演唱会=高潮事件**: 10首歌的编排+事故处理+压轴设计→制作了"紧张的高潮体验"
5. **年度奖项=中长期目标**: 5个不同维度的奖项→引导玩家尝试不同的策略方向
