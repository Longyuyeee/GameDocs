| 参数 | 说明 |
|------|------|

## 2. 团体属性计算

### 2.1 加权平均公式
```
团体属性 = Σ(成员属性 × 站位权重) / 总权重
```

| 站位 | 权重 |
|:--:|:--:|
| **C位(中心)** | 最高 |
| 前排 | 中 |
| 后排 | 低 |

### 2.2 团体规模的权衡

| 规模 | 优势 | 劣势 |
|:--:|------|------|
| 小团(5人) | 属性集中→粉丝群聚焦 | 单点故障风险 |
| 中团(8-10人) | 覆盖面广 | 管理复杂 |
| 大团(15人) | 最大粉丝覆盖面 | **属性被稀释→每个粉丝群都不够聚焦** |

> 关键洞察: 盲目扩团≠更强。聚焦的粉丝群>分散的覆盖面。

## 5. 设计特点分析

1. **属性=粉丝群体的函数**: 不像传统养成的"属性→结局"→这里的属性直接决定**哪些人成为你的粉丝**→商业化的属性设计
2. **年龄=属性的自然约束**: 可爱的有效期→性感的准入门槛→巅峰后的衰退→年龄是核心的战略考量
3. **团体属性=数学优化问题**: C位权重×成员属性=团体输出→站位不是装饰而是数学
4. **39种特性=偶像的不可复制性**: 每个偶像有随机特性→不是"培养模板"→每个偶像都是独特的个体
5. **俄罗斯开发者做日本偶像游戏=跨文化视角**: 一个非日本团队做的日系偶像游戏→对行业有更"外部"的观察(有评测提到游戏触及了偶像产业的黑暗面)
