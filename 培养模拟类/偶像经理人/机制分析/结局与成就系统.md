# 结局与成就系统 - 偶像经理人(Idol Manager)

> 游戏版本: Idol Manager v1.0 (PC/Steam, 2021年7月27日)
> 数据来源: Steam商店页, GamePretty, NGA

## 1. 结局总览

| 参数 | 说明 |
|------|------|
| 结局性质 | **开放式**(持续经营→无"通关结局") |
| 失败状态 | **破产=Game Over**(资金归零+无法偿还债务) |
| 胜利状态 | 主观(征服娱乐圈/打造最强团体/成为年度最佳雇主) |
| 单局时长 | 10-80+小时(取决于策略) |

## 2. 广义结局

### 2.1 破产结局(失败)

| 触发原因 | 常见情景 |
|------|------|
| 过度扩张 | 贷款太多→利息压垮 |
| 忽略成本管理 | 偶像薪水失控/员工太贵 |
| 前期出单曲 | 无粉丝基础→单曲亏本→资金断裂 |
| 偶像大规模离队 | 关系管理失败→核心成员出走 |

### 2.2 "偶像帝国"(成功)

| 指标 | 说明 |
|------|------|
| 年收入 | 高且稳定 |
| 偶像团体 | 多团/多代偶像 |
| 年度奖项 | 多次获奖 |
| 市场地位 | 行业顶端 |

## 3. 年度奖项作为"阶段性结局"

| 奖项 | 象征意义 |
|------|------|
| 最多公演 | 勤劳/量产的象征 |
| 最佳演唱会 | 品质的象征 |
| 最佳新人 | 造血能力的象征 |
| 最活跃团体 | 持续产出的象征 |
| 最佳雇主 | 管理能力的象征 |

## 4. 游戏中的选择≠多结局

与养成游戏不同，偶像经理人不提供"51个结局"——它提供的是**持续经营过程中的不同状态**:

| 决策维度 | 可能的结果 |
|------|------|
| 偶像路线(可爱vs性感vs帅气) | 不同的粉丝群体构成 |
| 管理风格(压榨vs温情) | 不同的偶像满意度和留存率 |
| 扩张策略(谨慎vs激进) | 不同的财务风险 |
| 恋爱选择 | 与不同偶像的关系发展(或不发展) |

## 5. Steam成就(部分)

- 第一次发行单曲
- 第一次举办演唱会
- 获得年度奖项
- 偶像知名度达到顶级
- 事务所规模达到XX
- 等等(具体列表未公开)

## 6. 设计特点分析

1. **无终局=持续经营的自由**: 不同于固定年数的养成游戏→"你想玩多久就玩多久"→适合喜欢优化的玩家
2. **破产=唯一的真正失败**: 只要不破产→你可以一直玩下去→失败条件简单但明确
3. **年度奖项=短期的"结局"代用品**: 每年都有新目标→给持续经营提供节奏和方向
4. **无终局的设计问题**: 评测反映后期缺乏挑战→"钱太多不知道花哪了"→无限游戏的常见问题
5. **与养成品类的本质差异**: 养成=培养一个对象→有终点(18岁/毕业)。经营=管理一个系统→没有自然终点。这决定了它更接近"模拟经营"而非"子女养成"
