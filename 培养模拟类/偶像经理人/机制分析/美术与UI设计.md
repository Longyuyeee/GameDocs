# 美术与UI设计 - 偶像经理人(Idol Manager)

> 游戏版本: Idol Manager v1.0 (PC/Steam, 2021年7月27日)
> 数据来源: Steam商店页, 豆瓣评测, 18183评测, PTT, Steam社区

## 1. 美术风格

### 1.1 整体风格
| 要素 | 特点 |
|------|------|
| 绘画风格 | **2D日系动画风格**(二次元) |
| 角色设计 | 各具特色的偶像立绘(可更换服装/发型) |
| 界面风格 | **简洁商务风+日系元素混合** |
| 色调 | 明亮/活泼(偶像产业氛围) |

### 1.2 偶像角色美术
- 2D立绘(可换装)
- 每个偶像有独特的角色设计
- 服装/发型/妆容可变更→视觉定制
- 随机生成的偶像有不同的外观

### 1.3 场景设计
- 事务所各楼层(俯视图)
- 训练室/化妆室/舞蹈室/录音室等
- 演唱会舞台

## 2. UI/UX设计

### 2.1 界面特征
| 要素 | 说明 |
|------|------|
| 主界面 | **楼层视图**(俯视事务所) |
| 偶像面板 | 8属性+体力+满意度+特性 |
| 财务面板 | 收入/支出/贷款/现金流 |
| 粉丝面板 | 12种粉丝群的分布 |

### 2.2 信息呈现
- 偶像属性通过**数字+进度条**双重显示
- 粉丝按性别×年龄×忠诚度的矩阵呈现
- 财务有周报/月报/年报
- 事件有弹窗提示

### 2.3 评测反映的UI问题
| 问题 | 表现 |
|------|------|
| **后期操作繁琐** | 偶像数量多了→每天重复操作 |
| **信息过载** | 同时管理太多偶像→UI不够清晰 |
| **缺乏自动化** | 升级经理人前→大量手动操作 |

## 3. 音频设计

| 要素 | 说明 |
|------|------|
| 音乐 | 偶像风格的日系流行乐/J-Pop风格 |
| 音效 | 标准UI音效+活动音效 |
| 语音 | **无**全程语音(独立小团队限制) |
| 演出 | 演唱会时有音乐表演 |

## 4. 偶像产业的视觉呈现

### 4.1 对日本偶像产业的美学复制
- 日系二次元立绘
- 偶像的"可爱"视觉化
- 握手会/演唱会/总选举的视觉再现

### 4.2 批判性视觉
- 评测提到: 游戏不只是美化偶像产业→也展现了产业的**剥削性**
- 偶像薪水可以被压到1元→这是视觉上"正常"的但机制上是批判的
- 俄罗斯开发者对日本偶像产业的"外部观察"

## 6. 设计特点分析

1. **楼层俯视图=模拟经营的经典UI**: 不是养成游戏的"女儿站立绘"-而是经营游戏的"俯瞰视图"→UI本身就定义了游戏的经营本质
2. **偶像立绘的可定制=收集驱动**: 可以换装/换发型/换妆容→视觉定制增加了偶像的"个性感"
3. **无语音=小团队的取舍**: 独立团队无法承担全程语音→但通过2D立绘+文字+音乐完成了偶像氛围
4. **后期UI疲劳=经营的单调**: 评测普遍反映后期重复操作→缺乏自动化→这是经营游戏常见的问题
