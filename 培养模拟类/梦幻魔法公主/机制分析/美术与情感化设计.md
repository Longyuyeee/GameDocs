# 美术与情感化设计 - 梦幻魔法公主(Magical Princess)

> 游戏版本: Magical Princess v1.0 (PC/Steam, 2026年4月27日)
> 数据来源: Steam商店页, 微信公众号试玩评测, PTT C_Chat, Stage1st, TapTap

## 1. 美术风格

### 1.1 角色美术
| 要素 | 特点 |
|------|------|
| 绘画风格 | **手绘(Hand-drawn)** 日系动画风格 |
| 动态效果 | **Live2D**动画，角色有呼吸/表情动态 |
| 角色设计 | 30+独特性格角色，日系美少女风格 |
| 女儿形象 | 从幼年到少女的渐进成长变化 |
| 服装变化 | 购买服装后立绘实时更新 |

### 1.2 场景/背景
- 魔法学院(主场景): 教室/图书馆/训练场/庭院
- 王都: 街道/商店/教堂/王宫
- 战斗场景: 古树林/赤月之夜战场/大迷宫
- 季节变化: 场景随季节变化

### 1.3 CG收集
- 画廊系统(Gallery): 包含相册-家人分类
- 大量可收集CG:
  - 父女出游特殊CG
  - 恋爱CG(好感100触发)
  - 生日CG(善恶≤30有额外内容)
  - 结局CG
- 部分CG可错过(有玩家反馈全成就后仍有4张CG未解锁)

## 2. UI/UX设计

### 2.1 信息架构
- **主界面**: 女儿立绘居中，属性面板分布四周
- **属性面板**: 4大主属性+16子属性→分层折叠展示
- **日程界面**: 月度三阶段清晰划分
- **画廊界面**: 职业(结局条件查看)+相册(CG收集)

### 2.2 便捷功能
| 功能 | 说明 |
|------|------|
| 大量存档位 | 支持分阶段/分路线存档 |
| 内置结局条件 | 画廊→职业中查看，消除攻略依赖 |
| 课程加速 | 按回车键加速课程动画 |
| 全语音 | 日语全语音+中/英/繁中字幕 |

### 2.3 信息呈现
- "一个人的所有属性"通过4×4网格+关键数值(HP/名望/善恶/父女感情)呈现
- 月度报告展示本月成长摘要
- 结局预览(画廊)显示条件→玩家可提前规划

## 3. 情感化设计

### 3.1 女儿的表情与反应
| 情境 | 情感表达 |
|------|------|
| 每月聊天 | 根据对话选项产生不同的情绪反应(开心/失落/生气) |
| 压力过高时 | 疲惫/沮丧的表情和语气 |
| 好感事件 | 害羞/喜悦/感动等细腻表情变化 |
| 战斗 | 紧张/坚定/受伤等战斗表情 |
| 父女出游 | 开心的特殊CG+对话 |

### 3.2 父女感情的传达
- **每月聊天的对话选择**: 不是简单的"讨好女儿"→选项需要体现真正的关心
- **父女出游**: 0AP消耗的特殊时光→纯体验导向(非功利)
- **女儿的成长反馈**: 属性提升/事件完成后女儿的表现变化
- **"我的女儿天下第一可爱"**: 玩家的情感投入是游戏设计的核心目标

### 3.3 善/恶路线的情感差异
- 善路线: 温暖/正面的父女互动
- 恶路线(善恶≤30): 聊天气氛变化+不可父女出游+专属恶路线CG
- 恶路线的"代价"不仅是游戏机制上的(压力管理变难)，也是情感上的(失去温暖的互动)

### 3.4 全语音的情感增强
- 日语全语音覆盖
- 女儿的声音表演传达情绪层次
- 关键剧情场景有完整的语音表演

## 5. 设计特点分析

1. **Live2D+手绘=静态CG的动态化**: 不是简单的静态立绘→角色有呼吸/表情变化→增加情感传达
2. **内置结局条件=反攻略依赖**: 不依赖外部wiki→玩家在游戏中自然发现目标→减少"查攻略-执行"的机械化体验
3. **全语音=情感沉浸的核心**: 女儿的声音让玩家感受到"对话"而非"点击选项"→语音是养成游戏情感传达的关键杠杆
4. **画廊=收集驱动+攻略提示**: 既是成就展示也是游戏内攻略→双重用途
5. **CG可错过+100成就=硬核收集挑战**: 不是所有内容都在一次游玩中可获得→驱动多周目+社区交流
6. **开发团队仅3人=小团队高品质**: 3人核心团队在11天内销量20万+→小团队做养成游戏满足品质标准的参考案例
