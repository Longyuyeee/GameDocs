## 一、属性体系架构

### 1.1 4×4双层属性结构

```
体力(Stamina) = 力量 + 意志 + 敏捷 + 生命
智力(Intelligence) = 魔术 + 信仰 + 算术 + 文学
魅力(Charm) = 美貌 + 社交 + 礼仪 + 道德
感性(Sensibility) = 想象 + 创意 + 音乐 + 美术
```

## 四、属性成长曲线特征

### 4.1 课程进度→属性提升
```
单次课程进度达到100% → 对应主属性+子属性提升
提升量取决于: 课程基础值 × 技能加成 × 天赋修正
```

### 4.2 属性→Rank→技能点
```
属性累积到一定值 → Rank提升(E→D→C→B→A→S)
每次Rank提升 → 获得技能点
Rank A → 解锁该属性EX技能
```

## 六、隐藏属性/影响因子

| 隐藏因子 | 说明 |
|------|------|
| 父女感情(Bond) | 显示数值(0-100)，影响月聊天的对话选项和出游效果 |
| 课程进度条 | 每门课程独立(0-100%)，100%才提升属性 |

| 随机种子 | 影响偷窃/翻垃圾/赌博的结果，可利用伪随机机制预测 |

## 七、属性培养效率分析

### 最高效培养策略
```
1. 选择对应学级→课程进度加速
2. 点满「积极听课」/「超积极听课」
3. 点出「额外修习」增加上课次数
4. 单次课程进度保持≥34%(3次完成)
5. 属性Rank提升→获取技能点→继续点效率技能→正向循环
```

### 全能型(Semi-Balanced)路线
- 不追求单一属性制霸
- 各类属性均衡提升至2000-3000范围
- 优点: 解锁大量中等要求结局，多结局可选
- 缺点: 很难达到极端阈值(体力5000/魅力5000等)
