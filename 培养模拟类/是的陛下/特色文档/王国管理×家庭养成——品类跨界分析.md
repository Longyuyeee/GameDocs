# 王国管理×家庭养成——品类跨界分析 - 是的陛下(Yes, Your Grace)

> 游戏版本: Yes, Your Grace v1.0 (PC/Steam, 2020年3月6日)
> 数据来源: Steam社区, 多篇评测, entertainment14攻略

## 一、为什么这款游戏被纳入采集范围

是的陛下不是一款传统的子女养成游戏。它被纳入的原因如采集策略文档所述: "养成对象不仅是女儿也包括整个王国"——这是一款**双重养成**的跨界实验品。

## 二、双重养成机制

### 2.1 两个"养成对象"

| 养成对象 | 机制层 | 情感层 |
|------|------|------|
| **女儿们** | 有限决策(嫁不嫁/留不留熊) | 情感核心(爱/保护/无力) |
| **王国(达文)** | 核心玩法(52周的资源管理) | 责任(国王的义务) |

### 2.2 两者的耦合

| 耦合方式 | 说明 |
|------|------|
| **女儿=外交筹码** | 嫁女儿→获得兵力/金币→王国强大 |
| **王国资源→女儿安全** | 没有足够的资源→无法保护女儿 |
| **女儿的命运→玩家的情感投入** | 玩家不只是管理数值→是为了"保护她们"而管理王国 |

## 三、与纯养成游戏的根本区别

| 维度 | 传统养成 | 是的陛下 |
|------|------|------|
| 时间框架 | 固定年数(3-8年) | **52周(~1年)** |
| 核心行动 | 安排课程/活动 | **听取请愿/分配资源** |
| 培养对象 | 1个女儿 | **1个王国+3个女儿** |
| 属性系统 | ~10项可培养属性 | **4维资源(不可直接"培养")** |
| 你的角色 | 父亲/监护人 | **国王**(决策者) |
| 失败条件 | 通常无Game Over | **4维资源任一归零=Game Over** |
| 重玩驱动 | 不同结局/收集 | 体验不同的叙事选择 |
| 情感核心 | 父爱/培养成就感 | **责任/牺牲/保护** |

## 四、王国管理系统的构成

### 4.1 请愿系统
```
每天→臣民排队→国王听取→决策:
  - 帮助(消耗资源)
  - 派遣代理人
  - 拒绝
```

### 4.2 外交系统
```
写信→领主回应→建立/维持/失去同盟
联姻→嫁女儿→获得军事/经济回报
```

### 4.3 战争系统
```
准备阶段: 积累兵力+补给+盟友
战斗阶段: 战术选择→多阶段事件
```

## 五、跨界设计的成功之处

1. **家庭=冰冷政治中的人性温度**: 在纯王国管理游戏中→容易沦为数字游戏。有了女儿→你的每一个政治决策都有了"为什么"→是为了她们
2. **52周的紧迫感**: 不是"无限年"→每一周的选择都有时间压力→紧凑的叙事节奏
3. **洛苏莉亚的必然死亡=品类的情感创新**: 养成游戏很少"失败"→但也几乎没有"必然的失去"。这个设计打破了养成游戏的隐含契约→制造了真实的情感冲击

## 六、跨界设计的问题

1. **女儿不是"养"的是"保护"的**: 你没有在培养她们的属性→你只是在做有限的二进制决策(嫁/不嫁/留/不留)→缺乏传统养成的"培养深度"
2. **核心玩法是王国管理**: 玩惯了纯养成游戏的玩家可能会感到"这不是养成游戏"
3. **5小时=叙事体验≠养成体验**: 传统养成游戏的重复可玩性来自不同培养路线→这里的重玩价值来自"我想看看不同的选择"
4. **线性主线=有限的重玩驱动**: 核心剧情不可改变→即使做了不同选择→主要剧情不变→重玩的叙事回报有限

## 七、设计启示

1. **双重养成=情感×机制的耦合**: 当"养成对象A"的机制决策直接影响"养成对象B"的命运时→耦合是成功的。如果是两个独立系统拼在一起→会分散注意力
2. **不是所有含"女儿"的游戏都是养成游戏**: 是的陛下更接近"王国管理RPG"而非"养成模拟"→它的核心循环不是"培养→成长→结局"而是"决策→后果→新的决策"
3. **有限控制=不同的情感体验**: 养成游戏给玩家"我掌控一切"的幻觉→是的陛下给玩家"即使我是国王也有我做不到的事"的真实→不同的情感设计
4. **52周=精炼的优点**: 不是越长越好→52周的紧凑叙事=一个完整的故事→适合没有时间投入400小时的玩家
