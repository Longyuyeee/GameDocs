# 叙事结构与路线系统 - 大正对称爱丽丝(Taisho x Alice)

> 游戏版本: Taisho x Alice Ep 1-3 + Epilogue (PC/Steam, 2019-2021) | 原始版: 2015年(PC分章)
> 开发商: Primula | 发行商: PRODUCTION PENCIL / PROTOTYPE
> 数据来源: VNDB, Noisy Pixel评测, Steam社区, 知乎, PTT RomanceGame, rekowiki, Bilibili攻略

## 1. 游戏类型声明

大正对称爱丽丝是一款**纯视觉小说(Visual Novel)**，**没有属性培养、没有日程安排、没有资源管理**。采集策略将其纳入P2(边缘参考)的原因是其代表了"养成"的极端叙事化形式——**通过对话选择"培养"关系，而非通过数值"培养"能力**。

## 2. 分章结构与强制顺序

| 章节 | Steam发布日期 | 可攻略角色 | 约游戏时长 |
|------|:--:|------|:--:|
| **Episode 1** | 2019年11月 | 仙杜瑞拉(Cinderella) + 小红帽(Red Riding Hood) | 9-18小时 |
| **Episode 2** | 2020年9月 | 辉夜姬(Kaguya) + 格雷特(Gretel) | 10-20小时 |
| **Episode 3** | 2021年3月 | 白雪(Snow White) + 魔法师(Wizard) | 10-20小时 |
| **Epilogue** | 2021年6月 | **爱丽丝(Alice)**——真相线 | 8-15小时 |
| FANDISC | 2022年(HEADS&TAILS) | 后日谈+IF路线 | — |

> **强制顺序**: Ep1→Ep2→Ep3→Epilogue，不可跳章。每周目解锁更多真相。

## 3. 路线选择机制

### 3.1 无共通线
- 序章结束后→**直接选择攻略角色路线**
- 无需通过共通线选项分支进入个人线
- 这种设计降低了早期选错的挫败感

### 3.2 选择肢系统
| 参数 | 说明 |
|------|------|
| 每路线选项数 | **较少**(比传统乙女游戏少) |
| 选项分布 | 分散在路线全程 |
| BE触发 | 错误选项→魔法师出现举手提示→即时BE |
| HE条件 | 关键选项选择正确 |
| 选项追踪UI | 显示已选/未选选项→方便回收 |

### 3.3 选项类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **态度选择** | 对攻略角色的态度(抬杠/顺从/主动) | 仙杜瑞拉: 跟他作对抬杠→增加好感 |
| **关键决策** | 决定性选项→直接影响HE/BE | 辉夜姬: 菜刀刺向自己→HE; 选日记本→BE |
| **情感表达** | 表达理解/原谅/拒绝 | 小红帽: "原谅他"→解开关键心结 |

## 4. BE系统设计

### 4.1 BE的特征
- **即时触发**(不像某些游戏的延迟BE)
- 魔法师角色出场举手→提示"这是坏结局"
- 可SL重新选择
- 部分路线BE极多(辉夜姬有**11个BE**)

### 4.2 BE的收集价值
- 需要收集全部BE+HE→全成就
- BE通常提供额外剧情片段
- 魔法师在BE中的出场是主线伏笔

## 5. 叙事架构

### 5.1 章节间的叙事递进

| 章节 | 叙事色调 | 剧情深度 |
|:--:|:--:|:--:|
| Ep 1 | 轻松/幽默 | 浅(引入世界观) |
| Ep 2 | 开始转向黑暗 | 中(开始揭示异常) |
| Ep 3 | 黑暗/心理 | 深(心理创伤主题) |
| Epilogue | 真相/治愈 | 最深(全貌揭示) |

### 5.2 男主视角SIDE
- 部分路线包含男主视角剧情
- 用**不同颜色UI**区分(女主=粉色/男主=蓝色等)
- 提供对称叙事视角

## 6. 真相线的"养成"隐喻

### 6.1 核心设定(含剧透)
> 所有可攻略角色=男主角爱丽丝泰尔的**DID(解离性身份障碍)分裂人格**。
> 女主角=现实中"女王"在爱丽丝梦境中创造的人格。
> **通过与每个人格恋爱来治愈心伤**→唤醒沉睡的爱丽丝。

### 6.2 "养成"的叙事化
在传统养成游戏中你通过**属性培养**来塑造未来。
在大正爱丽丝中你通过**选择对的回应**来治愈创伤。
两个过程在"培养一个更好的未来"这一点上同构——但手段完全不同。

## 7. 设计特点分析

1. **无养成系统=养成品类的极端边界**: 这款游戏定义了一个问题"如果完全没有属性、没有日程，只有选择，它还是养成吗?"——答案是它保留了养成的情感核心(培养/治愈)但完全放弃了机制外壳
2. **选项极少=选择的重量增加**: 不是数百次微选择→而是少量关键选择→每个选项的分量更重
3. **BE=学习工具**: 魔法师的举手提示="这里不对，换一个选项"→BE不是惩罚而是导航
4. **分章=渐进的信息揭示**: 不是一章50小时→而是4章逐层深入→叙事驱动的章节化设计
5. **DID设定=养成游戏的终极隐喻**: "他破碎了→你进入他的梦境→通过爱与每个碎片对话→整合为一个完整的人"→
这就是养成最极致的隐喻
