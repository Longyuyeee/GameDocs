# 选择系统与分支设计 - 大正对称爱丽丝(Taisho x Alice)

> 游戏版本: Taisho x Alice Ep 1-3 + Epilogue (PC/Steam, 2019-2021)
> 数据来源: Noisy Pixel, Steam社区指南, Bilibili攻略, 知乎

## 1. 选项系统设计

### 1.1 与传统乙女/养成游戏的对比

| 维度 | 传统乙女游戏 | 传统养成游戏 | 大正爱丽丝 |
|------|------|------|------|
| 选项数量 | 中-多 | 极高(数百次活动安排) | **极少** |
| 选项频率 | 频繁 | 每日/每周 | 稀疏(关键节点) |
| 错误选项后果 | BE(需重新来) | 属性偏了→不同结局 | **即时BE**(魔法师提示) |
| 选项的重量 | 中 | 低(每次微决策) | **极高**(每个选项都很关键) |
| 攻略依赖度 | 中-高 | 低-中 | **中**(关键选项需要攻略) |

### 1.2 "选项极少=选择极重"

在养成游戏中你有数百次选择(每次安排课程/活动)→每个选择的影响很小。
在大正爱丽丝中你只有10-30个选项→**每个选项都能改变结局**。

这是两种完全不同的选择设计哲学:
- 养成: 累积效应(每次+1→总共+500)
- 大正爱丽丝: 关键时刻效应(选错→BE)

## 2. 选项类型分析

### 2.1 三类选项

| 类型 | 表现 | 决策逻辑 | 示例 |
|------|------|------|------|
| **态度型** | 对角色说话的语气/态度 | "我应该怎么回应他?" | 抬杠vs顺从 |
| **决策型** | 关键行动选择 | "在这个时刻我应该做什么?" | 菜刀vs日记本 |
| **情感表达型** | 表达理解/共情/拒绝 | "我是否真正理解了他?" | "我原谅你"vs"我不能原谅" |

### 2.2 选项的"正确性"来源

| 选项正确≠道德正确 | 选项正确=**理解了他的需求** |
|------|------|
| 仙杜瑞拉: 跟他抬杠→正确 | 因为他的问题是"从不被真实对待" |
| 辉夜姬: 用菜刀刺向自己→正确 | 因为她的问题是"不相信有人愿意为她付出一切" |
| 小红帽: "原谅他"→正确 | 因为他的创伤来自"不被原谅" |

## 3. BE系统的设计

### 3.1 BE=导航而非惩罚

| 传统BE | 大正爱丽丝BE |
|------|------|
| "你失败了" | "这里不是正确的路" |
| 挫败感 | 学习感(魔法师举手=换条路试试) |
| 隐藏(没攻略不知道) | **可见**(选项追踪UI) |
| 需要从存档重新来 | 魔法师提示后可SL |

### 3.2 BE是叙事进度的一部分

- 看到魔法师→你就知道"这条路不对"
- 魔法师在BE中的出现是**主线伏笔**(他是观察者)
- 收集全部BE=理解全部"不是这样"→才能理解"是怎样"

## 4. 选项追踪UI

| 功能 | 说明 |
|------|------|
| 显示已选/未选选项 | 方便知道"还有哪些选项没试过" |
| BE标记 | 已触发的BE被记录 |
| 回收辅助 | 不需要攻略也能知道全选项位置 |

> 这是对纯VN游戏的便捷性提升——在没有养成系统的游戏中，UI的透明化替代了"培养"的过程感。

## 5. "心理解谜"型的游戏玩法

### 5.1 每个路线的核心"谜题"

| 角色 | 核心谜题(你需要发现的) |
|------|------|
| 仙杜瑞拉 | 他不是在逃避责任→他是在逃避"不被看见" |
| 小红帽 | 他不是在拒绝信任→他是曾被信任的人背叛 |
| 辉夜姬 | 她的自我伤害不是"想死"→是"不相信有人在乎我" |
| 格雷特 | 他对姐姐的执着不是"依赖"→是"害怕被丢弃" |

### 5.2 选项=你的诊断

每个选择选项的过程实际上是:
1. 观察这个人的行为
2. 推断他的心理创伤
3. 选择**最能治愈这个创伤**的回应
4. 正确→HE/错误→BE

这是一个**心理诊断+选择治疗**的过程。这与传统养成游戏中的"观察属性→选择课程→属性提升"在结构上同构。

## 6. 设计特点分析

1. **选项极少=每个选项都是"关键帧"**: 不是累积效应→是关键时刻效应→选择的分量最大化
2. **BE=教育而非惩罚**: 魔法师=你的导航员→BE告诉你"你误解了他"→下一次你更有机会理解正确
3. **心理解谜=养成的最纯粹形式**: 不是培养属性→是理解一个人→治愈他的创伤→这是"养成"在心理学意义上的实现
4. **选项追踪UI=透明化设计**: 在没有养成进度条的情况下→选项追踪提供了"进度感"→替代了传统养成的属性面板
