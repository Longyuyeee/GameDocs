# 父嫁争议与父女叙事 - 美少女梦工厂2(Princess Maker 2)

> 游戏版本: Princess Maker 2 Refine (PC, 1993/2016年)

## 一、概述

PM2中最具争议性的设计是**"父嫁"(父亲与养女结婚)**结局——在1993年的游戏中，玩家(父亲)可以与亲手养大的女儿发展恋爱关系并最终结婚。这个设计引发了几十年的争议，但也确立了PM系列最具记忆点的叙事身份。

## 二、父嫁的触发条件

| 条件 | 说明 |
|------|------|
| 亲密度最高 | 与父亲的关系超过所有其他NPC |
| 道德偏低 | 不是极低(纯暗黑会触发暗黑结局) |
| 特定事件 | 触发父女关系特殊事件 |
| 其他婚姻未触发 | 王子/管家/魔王/龙嫁优先级高于父嫁 |

## 三、为什么父嫁存在

| 原因 | 说明 |
|------|------|
| **情感代入** | 玩家花了8年(现实数十小时)培养女儿→产生了真实的情感投入 |
| **独占欲的满足** | "我养大的女儿=我最了解她=她应该和我在一起" |
| **叙事实验** | 1993年，开发者(Gainax)以大胆著称——父嫁是他们的"我们敢做" |
| **话题性** | 争议=传播——父嫁成为PM系列最被记住的结局 |

## 四、父嫁的争议

| 批评 | 辩护 |
|------|------|
| 鼓励乱伦 | "养女"(非亲生)——没有血缘关系 |
| 物化女性 | 女儿=独立的个体——有权利选择(包括选择父亲) |
| 不健康的权力关系 | 这是游戏——不是现实——玩家可以"选择不触发" |

## 五、父嫁对品类的影响

| 影响 | 说明 |
|------|------|
| **每个养成游戏都被问"有父嫁吗?"** | PM2确立了这个"品类期待" |
| 火山的女儿 | 没有父嫁——有父亲角色但明确"父女关系"不可改变 |
| 后来的PM系列 | PM3/PM5保留了特殊婚姻但弱化了父嫁 |
| **品类身份的锚点** | "父嫁"成为PM2的代名词——无论好坏——无法被忘记 |

## 六、设计分析

1. **父嫁=养成游戏中"情感投资"的终极回报**: 如果玩家在8年(288回合)中投入了大量情感→父嫁提供了一种"独占"的满足
2. **选择权在玩家**: 不是"强制父嫁"——需要特定条件+玩家主动触发——道德选择由玩家决定
3. **话题性=免费营销**: 30年后仍在被讨论——证明了设计的"记忆点"力量
4. **火山的女儿的"安全"选择**: 移除父嫁——避免争议——但也失去了"被讨论30年"的力量
