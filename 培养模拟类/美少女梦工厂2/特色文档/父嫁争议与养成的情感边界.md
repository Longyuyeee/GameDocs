# 父嫁争议与养成游戏的情感边界 - 美少女梦工厂2(Princess Maker 2 Refine)

> 游戏版本: Princess Maker 2 Refine (PC/Steam, 1993/2016)
> 数据来源: princessmaker.fandom.com, PTT PM版, 3DM社区, 知乎

## 一、什么是"父嫁"

在PM2的5个特殊婚姻结局中，"父嫁"（女儿嫁给养育她的父亲/玩家角色）优先级排第二，仅次于王嫁。

### 触发条件
```
父感 = 100 (必须)
道德 ≥ 100
信仰 ≥ 100
罪孽 < 100
智力 ≥ 道德
气质 ≥ 道德
魅力 ≥ 道德
(智力 + 气质 + 魅力) × 2 > (信仰 + 道德)
```

这不是一个"容易"达成的结局——它需要8年的刻意培养。

## 二、为什么有争议

### 2.1 伦理问题
- 你养了她8年 → 然后娶了她？
- 这不是"青梅竹马" → 这是**养育者与被养育者**的关系
- 权力不对等(父亲-女儿) → 同意是否真正自由？

### 2.2 法律问题
- 在大多数现代国家 → 养父娶养女是违法的
- 游戏发布时(1993年)在日本引发讨论

### 2.3 品类的自我审查
- 后续PM系列游戏(PM3/PM4/PM5) → 父嫁结局被移除或弱化
- 火山的女儿 → 没有父嫁结局
- 大多数现代养成游戏 → 回避了这个问题

## 三、父嫁的设计意图

### 3.1 情感上的"终极验证"
从设计角度看，父嫁是对"父女关系"的极端测试:
- 父感=100 → 女儿完全信任父亲
- 道德/信仰≥100 → 父亲是有道德的人
- 智力/气质/魅力≥道德 → 父亲不仅仅是一个"好人"，也是有魅力的个体

### 3.2 玩家的选择 = 设计者的挑战
开发者赤井孝美和Gainax敢于放入这个结局 → 不是因为他们"支持"父嫁 → 而是他们在问玩家: **"你对她的感情是什么?"**

### 3.3 "父亲"的多重含义
- PM2中的"父亲"≠ 真实的父亲
- 她是"从天而降的养女" → 不是血缘关系
- 游戏在设定上为这个结局留了一些叙事空间

## 四、与当代养成游戏的情感边界对比

| 游戏 | 玩家与养成对象的关系 | 可否与养成对象恋爱 |
|------|------|:--:|
| PM2 | 养父-养女 | **✅ 可以(父嫁)** |
| PM3-5 | 养父/养母-养女 | ❌(被移除) |
| 火山的女儿 | 父亲-女儿 | ❌ |
| 梦幻魔法公主 | 父亲-女儿 | ❌ |
| 偶像经理人 | 经理人-偶像 | ✅(成年偶像+好感度) |
| 大正爱丽丝 | 恋人(女主角-攻略对象) | ✅(这就是游戏的目标!) |

## 五、当代视角下的评价

### 5.1 批评方的观点
- 这正常化了一种不健康的权力关系
- 玩家被置于"既是父亲又是丈夫"的矛盾角色
- 女儿没有说"不"的空间(她是被设计的程序)

### 5.2 辩护方的观点
- 游戏明确标注了这不是真实的父女关系(养女/从天而降)
- 父嫁的达成条件非常苛刻 → 不是默认结局
- 这是一个"选择" → 不是"鼓励"
- 游戏作为艺术形式 → 有权探索不舒服的主题

### 5.3 设计遗产
无论立场如何，PM2的父嫁结局提出了一个至今没有答案的问题:

> **养成游戏的情感边界在哪里？**

当代养成游戏普遍选择了安全的路径(父亲≠恋爱对象) → PM2的勇敢(或不智)→ 至少定义了这个品类的边界。

## 六、设计启示

1. **争议性设计可以是品类的标志**: PM2被讨论了30年 → 一定程度上是因为父嫁 → 它是品类的"不可忽视的存在"
2. **情感边界的游戏化需要谨慎**: 不是所有关系都适合"游戏化" → 父女关系可能是其中之一
3. **玩家 ≠ 设计者的价值观**: 你可以放入一个有争议的结局 → 但不意味着你支持它 → 你只是保留了玩家的选择权
4. **当代养成游戏的安全性 = 品类的成熟**: 没有父嫁 ≠ 缺乏勇气 → 可能是品类找到了更适合的情感表达方式
