# Kenshi — 势力与世界状态系统

> 游戏版本: Kenshi v1.0 (PC/Steam, Lo-Fi Games, 2018)
> 数据来源: [Kenshi Wiki (Fandom)](https://kenshi.fandom.com/wiki/Factions), [Kenshi Wiki (独立)](https://kenshi.wiki/factions), [Steam 社区指南 - 势力与外交](https://steamcommunity.com/sharedfiles/filedetails/?id=2332765105), [游研社](https://www.yystv.net/p/6008)

## 一、系统概述

Kenshi 的世界不是静态的背景板——它是一个**由势力博弈驱动的动态生态系统**。游戏中有 3 大主要势力和数十个小势力，它们之间通过好感度（Reputation）系统相互关联。最核心的特色是**世界状态系统（World State）**：杀死或囚禁势力领袖会永久改变世界的政治格局——城镇易主、NPC 重组、商店变更、废墟新生。玩家的行为不只是"影响世界"，而是"彻底重塑世界"。

## 二、三大主要势力

### 2.1 势力总览对比

| 维度 | 神圣王国 (Holy Nation) | 联合城市 (United Cities) | 沙克王国 (Shek Kingdom) |
|:----:|:---------------------:|:-----------------------:|:----------------------:|
| 英文名 | Holy Nation | United Cities | Shek Kingdom |
| 首都 | 巨石城 (Blister Hill) | 扒手之傲 (Heft) | 阿德马克 (Admag) |
| 种族偏好 | 仅人类（绿原人>焦土人） | 全种族容忍 | 沙克人优先 |
| 骨人态度 | **敌对(立即攻击)** | 容忍但歧视 | 中立 |
| 非人类态度 | 女性=次等/沙克敌对/蜂人敌对 | 容忍 | — |
| 政体 | 神权专制 | 腐败封建/奴隶制 | 军事寡头 |
| 主要军队 | 圣骑士(Paladin)+审判官(Inquisitor) | 武士(Samurai)+守卫 | 战士(Invincible) |
| 区域 | 西北平原(肥沃) | 东部沙漠(干旱) | 西南山区(贫瘠) |
| 好感起始 | 0（人类）/-50（非人类/骨人） | 0 | 0 |

### 2.2 神圣王国 (Holy Nation)

| 项目 | 内容 |
|------|------|
| 信仰 | 奥克兰（Okran）——一位男性神祇 |
| 社会结构 | 神权等级制——高阶圣骑士 > 圣骑士 > 审判官 > 信徒 |
| 核心教义 | 人类至上（绿原人最高）、女性次于男性、排斥非人种族 |
| 经济模式 | 农业为主（区域土地肥沃） |
| 军事力量 | 遍布巡逻队——玩家在神圣王国领地内几乎无时无刻不被监视 |
| 对玩家的影响 | 若玩家队伍中有骨人/沙克/女性角色→被巡逻队盘问/攻击/监禁 |
| 主要领袖 | 凤凰(Phoenix) |
| 好感关系 | 与沙克王国：**-100（世仇）** / 与联合城市：-50 |
| 特殊规则 | 每周有"祈祷日"——若不在城中祈祷会引发圣骑士巡逻 |
| 友好方式 | ①捐赠食物/物品 ②击败沙克王国 ③只带纯人类男性队伍 |
| 完全敌对触发 | 携带骨人进入领地 / 杀死凤凰 / 帮助其他势力攻击神圣王国 |

#### 神圣王国"祈祷日"机制

| 项目 | 内容 |
|:----:|------|
| 触发周期 | 每7天一次（游戏时间） |
| 触发地点 | 神圣王国控制的城市 |
| 要求 | 在城中/城附近→被圣骑士要求一同祈祷 |
| 拒绝后果 | 圣骑士敌对→攻击玩家 |
| 错过后果 | 错过祈祷日→好感降低 |
| 骨人角色 | 骨人在祈祷日被强制要求离开城市——否则被攻击 |

> 设计特点分析: 神圣王国的"祈祷日"机制是一个精妙的"文化冲突"设计工具。它不是一个"好 vs 坏"的道德判断，而是一个"文化规则"——如果你不理解/不遵守，就会被排斥。这迫使玩家设身处地理解一个神权社会的运作逻辑，而非简单将其归类为"反派"。这种"文化差异而非善恶二元"的叙事手法是Kenshi的一大特色。

### 2.3 联合城市 (United Cities)

| 项目 | 内容 |
|------|------|
| 政体 | 名义上的帝国——实际是各大贵族封地的松散联盟 |
| 社会结构 | 贵族(高)>武士阶级>商人>自由民>奴隶(底层) |
| 核心特征 | **合法奴隶制**——可合法买卖奴隶 |
| 经济模式 | 商业+采矿为主 |
| 军事力量 | 强大的武士军团但分布松散（贵族私兵为主） |
| 对玩家的影响 | 不检查种族→对所有人开放。但**每周收税** |
| 收税机制 | 玩家在UC领地拥有建筑/基地→约3000 Cats/周被征收 |
| 主要领袖 | 天狗皇帝(Tengu) |
| 好感关系 | 与沙克王国：**-75（仇恨）** / 与神圣王国：-50 |
| 友好方式 | ①缴税 ②不帮助反UC势力 ③完成UC任务 |
| 完全敌对触发 | 杀死天狗皇帝 / 帮助造反农民推翻UC / 拒绝缴税并攻击收税官 |

#### 联合城市"奴隶制"机制

| 项目 | 内容 |
|:----:|------|
| 合法运营 | UC控制的城市内有奴隶商店——可买卖奴隶 |
| 奴隶来源 | 战俘/债务人/罪犯/被绑架者 |
| 玩家作为奴隶主 | 购买奴隶→分配劳动+配给食物 |
| 玩家作为奴隶 | 在UC领地犯罪→被逮捕→判为奴隶→强制劳动 |
| 奴隶逃跑 | 在UC领地外跑掉→被通缉（奴隶=UC合法财产） |
| 反奴运动 | 可帮助"造反农民"(Rebel Farmers)攻击奴隶主→触发世界状态 |

> 设计特点分析: 联合城市的奴隶制设计不是为了"政治不正确"，而是为了创造一个**道德复杂性**——你在UC可以合法拥有奴隶，在政治上是"中立"行为（不被任何执法者追责），但你可能因此与反奴势力（造反农民）交恶。Kenshi不评判你选择支持还是反对奴隶制——它只提供后果。这与《骑马与砍杀2》中"赎买奴隶"的中性设计不同——骑砍的奴隶是纯粹的"商品-利润"关系，而Kenshi的奴隶是一个完整的"生存状态"（可打工/逃跑/被释放/加入你）。

### 2.4 沙克王国 (Shek Kingdom)

| 项目 | 内容 |
|------|------|
| 政体 | 军事部落联盟——领袖为最强者 |
| 社会结构 | 战士(核心)>工匠>农民 |
| 核心信条 | **力量即正义**——弱者不值得同情 |
| 军事力量 | 强大但人少（与神圣王国战争后人口锐减） |
| 对玩家的影响 | 不主动攻击→但要求证明实力 |
| 入城规则 | 首次进城→被守卫挑战→要求击败沙克战士（1v1决斗） |
| 主要领袖 | Esata the Stone Golem |
| 好感关系 | 与神圣王国：**-100（世仇）** / 与联合城市：-75 |
| 友好方式 | ①击败沙克战士展示武力 ②帮助对抗神圣王国 ③捐赠武器/食物 |
| 完全敌对触发 | 杀死Esata / 与神圣王国同盟 / 帮助UC攻击沙克 |

#### 沙克王国"决斗"机制

| 项目 | 内容 |
|:----:|------|
| 触发条件 | 玩家角色首次进入沙克控制的城市 |
| 对手 | 一名沙克战士（等级与玩家队伍最高等级相当） |
| 规则 | 1v1——其他队员不可干预 |
| 获胜奖励 | 沙克王国好感+10~+20，可通过后续挑战继续提升 |
| 失败后果 | 被打晕→被关进笼子（不会死亡）→醒来后可选择再挑战 |
| 设计意图 | 沙克王国尊重强者——通过战斗才能赢得尊重 |

### 3.2 中立组织功能详解

| 组织 | 主要城市分布 | 关键服务 | 加入条件 | 加入福利 |
|:----:|:----------:|---------|:--------:|---------|
| 科技猎人 | 所有主城+前哨站 | 出售地图/研究图纸/古代科技情报 | 无（自动中立） | 可购买高级科技蓝图 |
| 忍盗团 | 主要城市有塔楼 | 销赃/出售忍者装备/提供床铺 | 10,000 Cats | 永久庇护+销赃+训练 |
| 商人行会 | 全图所有商店 | 中间商/物流 | 无 | 开放贸易 |
| 奴隶商人 | 所有主城 | 买卖奴隶 | 无 | 合法奴隶交易 |

> 跨游戏对比: Kenshi的小势力系统与《骑马与砍杀2》的"中立组织"（如商队/雇佣兵/学者）类似，但功能性更强——Kenshi中忍盗团的"销赃"功能是潜行玩法的核心支柱，而科技猎人的"古物交易"是后期装备的主要来源。相比之下，骑砍2的中立组织多作为"氛围装饰"存在，Kenshi的小势力则直接参与"玩法循环"。

## 四、好感度系统

### 4.1 好感度等级与效果

| 好感度区间 | 等级标签 | 势力对玩家态度 | 实际效果 |
|:---------:|:-------:|:-------------:|---------|
| -100 ~ -71 | 不共戴天 | 主动攻击 | 进入领地即被追杀——不可交易/不可进城 |
| -70 ~ -31 | 敌对 | 主动攻击 | 进入领地即被攻击 |
| -30 ~ -1 | 不友好 | 冷淡/警惕 | 守卫盘问，但不主动攻击 |
| 0 ~ 29 | 中立 | 正常 | 标准交互——交易/进城/接任务 |
| 30 ~ 49 | 友好 | 友善 | 打折/特殊对话/部分区域开放 |
| 50 ~ 100 | 同盟 | 盟友 | 同盟支援/特殊任务/资源共享 |

### 4.2 好感度变化方式

| 变化方式 | 数值变化 | 适用范围 |
|---------|:-------:|---------|
| 捐赠物品(Cats/食物/装备) | +1~+5/次 | 所有非敌对势力 |
| 完成势力任务 | +5~+20/次 | 取决于任务难度和类型 |
| 击败势力敌人 | +2~+10/次 | 敌对势力之间互斥 |
| 释放俘虏 | +5~+15/次 | 释放特定势力成员 |
| 杀死势力成员 | -10~-30/次 | 任何势力 |
| 偷窃势力物品 | -1~-5/次 | 任何势力 |
| 与敌对势力同盟 | -20~-50/次 | 连锁反应 |
| Pacifier调停 | 付费提升1~20 | 特定NPC（酒吧随机刷新） |

### 4.3 Pacifier（调停者）机制

| 项目 | 内容 |
|:----:|------|
| 刷新位置 | 全图酒吧随机刷新 |
| 可调停的势力 | 所有主要势力（神圣王国/UC/沙克）+部分小势力 |
| 调停费用 | 与好感度差距正相关——差越多越贵 |
| 调停效果 | 固定提升好感度（具体数值取决于Pacifier等级） |
| 仅限一次 | 每个Pacifier只能使用一次 |
| 设计意图 | 给玩家一个"赎罪"机制——误伤势力成员不至于永久敌对 |

> 设计特点分析: Pacifier机制是Kenshi系统设计"仁慈"的一面——它防止玩家因为一次误操作或新手期犯错而与某个势力永久敌对。这种"容错设计"对于Kenshi这样一个高难度游戏尤为重要：如果因为不小心打了某个势力成员就被全世界追杀，游戏体验会迅速恶化。Pacifier提供了一条"虽然你搞砸了，但你还有机会弥补"的出路。这与《环世界》的"派系礼物"功能类似——环世界中你可以通过空投礼物来缓解派系关系，但Kenshi的Pacifier是"中介模式"（需要特定NPC），增加了交互的"人味"。

## 五、世界状态系统（核心特色）

### 5.1 世界状态触发条件

| 触发事件 | 世界状态变化 | 影响范围 |
|---------|:-----------:|---------|
| 杀死/囚禁凤凰(Phoenix) | 神圣王国崩溃→领地分割 | 西北全境 |
| 杀死/囚禁天狗皇帝(Tengu) | 联合城市覆灭→军阀割据 | 东部全境 |
| 杀死/囚禁Esata | 沙克王国灭亡→领地重置 | 西南全境 |
| 杀死虫之主(Bugmaster) | 科技猎人解锁高等对话 | 仅科技猎人相关区域 |
| 帮助造反农民推翻UC | UC城镇被反叛军占领 | 各UC控制的城市 |
| 帮助神圣王国镇压异端 | 神圣王国扩张 | 被镇压区域 |

### 5.2 世界状态具体效果

| 世界状态 | 城镇变化 | NPC变化 | 商店变化 | 玩家影响 |
|---------|---------|---------|---------|---------|
| 神圣王国崩溃 | 圣国城镇变为废墟/被科技猎人占领 | 圣骑士消失→科技猎人/流浪者生成 | 圣国专属商店消失 | 西北不再有宗教压迫，但失去贸易伙伴 |
| UC覆灭 | UC城市被造反农民/军阀占领 | 武士消失→叛军出现 | 奴隶商店关闭 | 奴隶制结束但治安恶化 |
| 沙克灭亡 | 沙克城镇变为流浪者营地 | 沙克战士分散 | 沙克武器商消失 | 失去强力潜在盟友 |
| 虫之主死亡 | 蠕虫之地怪物减少 | 科技猎人好感提升 | 科技猎人解锁特殊商品 | 可购买古代科技蓝图 |

### 5.3 世界状态连锁反应

```
杀死凤凰(Phoenix)
  ├→ Blister Hill(首都)变为废墟 / 被科技猎人占领
  ├→ Stack(小城)被科技猎人接管
  ├→ Bad Teeth(边境堡垒)成为无主之地
  ├→ 神圣圣骑士巡逻队消失
  ├→ 每周祈祷日取消
  ├→ 骨人/非人类可在西北自由活动
  ├→ 沙克王国向西北扩张（与其好感相关）
  └→ 联合城市向西北推进（与其好感相关）
```

```
帮助造反农民推翻UC
  ├→ 各UC城镇被造反农民占领
  ├→ 奴隶商店被关闭
  ├→ 武士守卫被替换为叛军守卫
  ├→ 收税机制取消（叛军不收税）
  ├→ 治安水平下降（强盗增多）
  ├→ 商人行会撤离该区域（商业环境恶化）
  └→ 神圣王国/沙克王国趁机扩张
```

> 设计特点分析: Kenshi的世界状态系统实现了**"玩家行为对世界产生因果链式反应"**——杀死领袖不只是一个"任务完成"的标记，而会引发一系列逻辑上合理的后续变化。你杀死神圣王国凤凰→圣国崩溃→骨人被解放→沙克人和UC趁虚而入→原本的神圣王国城市变成废墟或改旗易帜。这个世界不会"等你来拯救"——它在你行动之前就在自我运转，你的行动只是打破了原有的平衡。

> 与《骑马与砍杀2》世界状态对比:
> - 骑砍2中杀死领袖→该家族继承→势力不灭（除非所有家族成员死亡）
> - Kenshi中杀死领袖→**势力直接崩溃**→城市被其他势力瓜分或变为废墟
> - 骑砍2的世界状态变化是"王朝更替"（一家倒台另一家上位），本质是线性替换
> - Kenshi的世界状态变化是"熵增"（秩序瓦解→混乱加剧）——势力崩溃后不一定会被其他完整势力接管，可能变为无政府状态
> - Kenshi的"世界状态"比骑砍2更"硬核"——因为势力崩溃后，你失去了该势力的所有商店/服务/任务，这是不可逆的永久损失

## 六、外交连锁反应

### 6.1 好感连锁表

| 玩家行为 | 神圣王国 | 联合城市 | 沙克王国 | 造反农民 | 忍盗团 |
|---------|:-------:|:--------:|:--------:|:--------:|:-----:|
| 与神圣王国同盟 | **+50** | -20 | -50 | 0 | 0 |
| 与联合城市同盟 | -20 | **+50** | -20 | -50 | 0 |
| 与沙克王国同盟 | -50 | -20 | **+50** | 0 | 0 |

## 七、设计特点分析总结

| 维度 | 设计特点 | 设计意图 |
|------|---------|---------|
| 三大势力生态 | 彼此敌对、互不信任、形成三角势力 | 玩家不可能同时讨好所有势力——必须在阵营间做选择 |
| 文化差异化 | 神圣王国=神权/UC=奴隶商业/沙克=军事 | 每个势力有独特的"文明逻辑"——没有绝对善恶 |
| 世界状态系统 | 杀死领袖=永久改变世界格局 | 玩家行为有"后果权重"——杀死领袖是真正的"改变世界" |
| 连锁反应 | 一个势力崩溃→其他势力重新分配领土 | 世界是"活"的——你在废土上留下的空白会被其他势力填补 |
| Pacifier调停 | 提供赎罪机制 | 防止误操作导致永久对立——维持玩家"选择权" |
| 无主线剧情 | 势力状态不绑定任何"主线" | 玩家自由决定何时/是否介入势力政治 |

| 跨游戏对比 | 相似点 | 差异点 |
|-----------|--------|--------|
| 骑马与砍杀2 | 杀死领袖/占领城镇改变版图 | 骑砍2=家族继承制(势力不灭)→Kenshi=领袖即势力(一击崩盘) |
| 环世界 | 派系好感度影响袭击/贸易 | 环世界=袭击派系有固定生成逻辑→Kenshi=世界状态由玩家行动触发 |
| Elona | 特定行为影响城镇/势力状态 | Elona=固定城镇+固定势力(不可摧毁)→Kenshi=城镇可易主可废墟化 |
