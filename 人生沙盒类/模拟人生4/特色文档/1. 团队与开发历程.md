## 1. 开发团队

### Maxis 的起源

- **1987年**：由 **Will Wright**（威尔莱特）和 **Jeff Braun**（杰夫布劳恩）在美国加州奥林达创立
- **1997年**：被 Electronic Arts 以 **$1.25亿美元** 收购
- **核心基因**：从《模拟城市》(SimCity, 1989)开始，Maxis 就确立了"用游戏模拟真实系统"的设计哲学——让玩家成为系统的构建者而非被动的通关者

### 团队演变的关键节点

| 时间 | 事件 | 影响 |
|:----:|:----|:------|
| 2000年 | The Sims 1 发售 | Maxis 创造了史上最成功的PC游戏系列之一 |
| 2004年 | The Sims 2 发售 | 3D化+基因遗传系统+生命周期 |
| 2009年 | Will Wright 离开 Maxis | 灵魂人物离开，转向Stupid Fun Club |
| 2009年 | The Sims 3 发售 | "Sims Studio"开发(非原Maxis团队)+开放世界 |
| 2013年 | EA 重组，品牌重新整合为 Maxis | 回归统一品牌 |
| 2015年 | **Maxis Emeryville 工作室关闭** | 原Sims 3团队被解雇，整合至EA Redwood Shores |
| 2020s | Maxis Texas 成立 | 新工作室承担Sims 4后续开发和"Project Rene" |

### 当前运营团队

|:----|:------|

## 2. 开发历程

### 第一阶段：代号"Olympus" — 被取消的MMO (2010~2013)

Sims 4 最初的开发方向完全不同于最终的产品：

| 项目 | 细节 |
|:----|:------|
| **项目代号** | "Olympus" (奥林匹斯) |
| **初始定位** | **在线多人在线游戏(MMO)** |
| **核心驱动** | EA当时的CEO **Frank Gibeau** 公开表示"不再做单机游戏"，要求所有系列必须联网 |
| **设计方向** | 类似 Second Life 的在线社交世界 |
| **行业背景** | 2010年代初"所有游戏都要在线化"的行业狂热期 |

> 关键矛盾: EA高层认为"单机游戏已死"，但Maxis团队的核心能力是"做模拟"。联网要求与模拟游戏的离线深度存在根本性冲突。

### 第二阶段：SimCity 灾难与被迫转向 (2013)

转折点来自同门兄弟《SimCity》(2013)的灾难性发布：

| 事件 | 影响 |
|:----|:------|
| SimCity 2013 强制在线 | 发售时服务器崩溃，玩家无法进入，媒体差评如潮 |
| EA 被迫反思在线策略 | 强制联网模式被玩家和行业双重抵制 |
| Olympus 项目被叫停 | 高层意识到做一个"在线Sims"的风险过高 |
| **从MMO→大改为单机** | 仅剩一年半时间将Olympus改成一款可发售的单机Sims游戏 |

### 第三阶段：紧急转向与首发内容的砍掉 (2013~2014)

由于从MMO转为单机的时间紧迫，大量Sims 3时代已有的功能被砍掉：

| 被砍内容 | 补回时间 | 砍掉原因 |
|:---------|:--------:|:---------|
| **泳池 (Swimming Pools)** | 2014年11月(首发后2个月) | 转向期内未完成 |
| **幼儿 (Toddlers)** | 2017年1月(首发后2年4个月) | 引擎未准备好幼儿AI |
| **开放世界** | **从未补回** | SmartSim引擎不支持无缝加载 |
| **Create-a-Style** | **从未补回** | 经济考虑(简化=减少材质数量) |
| **地形编辑** | 部分(通过秘籍) | 引擎限制 |
| **地下世界** | **从未补回** | 地段制限制 |
| **螺旋楼梯** | **从未补回*(2025年有修改| | 技术限制 |

> 砍掉内容的后果: Sims 4 首发时媒体评分 **70/100**(Metacritic)，远低于Sims 3的86/100。玩家社区愤怒，Metacritic用户评分仅3.5/10。EA被批评为"用半成品骗钱"。

### 第四阶段：长期更新与口碑修复 (2014~2022)

| 时间 | 关键更新 | 意义 |
|:----:|:---------|:------|
| 2014.11 | 免费更新: 泳池 | 快速回应玩家核心诉求 |
| 2015.06 | 免费更新: 新世界(Newcrest) | 添加空白世界供自由建造 |
| 2016.06 | 免费更新: 地精/公会系统 | 为同欢共乐资料片做铺垫 |
| 2017.01 | **免费更新: 幼儿(Toddlers)** | 最大规模的一次免费更新，标志着态度转变 |
| 2018.06 | Seasons 资料片 | 社区最想要的资料片之一 |
| 2020.11 | 免费更新: 平台(Platforms) | 建造模式重大升级 |
| 2022.07 | 免费更新: 狼人(额外内容) | 持续内容支持 |
| 2022.10 | **基础版免费(F2P转型)** | 转折点——从买断制转为免费+内购 |

## 3. 技术与工具

### SmartSim 引擎

| 项目 | 说明 |
|:----|:------|
| **引擎名称** | SmartSim (内部自研) |
| **核心特性1** | **情绪系统** — 16种情绪状态×Moodlets叠加×情绪光环 |
| **核心特性2** | **多任务AI** — 市民可同时做多件事(如边跑步边聊天) |
| **核心特性3** | **地段制加载** — 取消Sims 3的开放世界，改为按地段加载 |
| **渲染** | DirectX 9 → DirectX 11(2019年更新) |
| **跨平台** | PC/Mac引擎一致 + PS4/Xbox One的移植版本 |

### 非开放世界的技术抉择

| 对比项 | Sims 3 (开放世界) | Sims 4 (地段制) |
|:-------|:------------------|:----------------|
| 世界的连续性 | 无缝大地图 | 分割地段+加载画面 |
| 性能表现 | 大量NPC导致性能灾难 | 仅加载当前地段，性能稳定 |
| 模组数量 | 无限制(导致卡顿) | 每地段最多20个市民 |
| 建造自由度 | 受地形限制 | 极高(可自由编辑室内外) |
| 加载速度 | 慢 | 快(Solid State 5~15秒) |

> Sims 4 的"地段制"被许多人视为退步(相对于Sims 3的开放世界)，但实际上是为了性能稳定和后续10年内容更新做出的工程权衡。

### 工具与管线

| 工具 | 用途 |
|:----|:------|
| 内部版本引擎 | 关卡设计/物品制作/动画绑定 |
| Sim rig (角色骨架) | 支持多任务动画混合的市民骨架系统 |
| Procedural animation | 程序化动画(如市民走路姿态的动态调整) |
| Python + XML | 脚本系统(主要语言) |
| Gallery API | 社区分享平台的后端服务 |

## 6. 设计启示

### 从Sims 4学到的10个设计教训

1. **MMO→单机的灾难性转向**: 高层的外部压力(强制联网)如果不在项目早期被挑战，后期转向的成本极其高昂

2. **砍掉内容比延迟发布更致命**: Sims 4首发缺少泳池/幼儿→玩家的"第一印象"需要数年才能修复

3. **免费更新是最好的公关**: 泳池(2个月)和幼儿(2年)的免费更新最终修复了大部分口碑损失

4. **情绪系统的创新价值**: Sims 4首创的情绪驱动系统是其最具独创性的设计——从"效率驱动"到"情感驱动"是一条从未被充分探索的路

5. **DLC生态=无限游戏**: 10年/30+付费扩展的模式证明"游戏即平台"比"游戏即产品"更有商业可持续性

6. **F2P转型=第二次生命**: 一款8年老的游戏在免费后玩家数翻倍——传统买断制与F2P之间没有不可逾越的鸿沟

7. **地段制是技术妥协也是设计选择**: 性能稳定的代价是开放世界的失去，但对建造模式的深度有着正向推动

8. **社区为内容付费的意愿**: Sims 4的DLC商业模式证明——如果扩展包能"改变你能活成什么样"，玩家愿意持续买单

9. **10年寿命的基础**: 选择"简化但稳定"的技术路线使Sims 4比Sims 3更能支撑10年+的持续开发

10. **下一代"Project Rene"**: EA承诺会更早与社区合作——Sims 4首发的教训将直接影响Sims 5的设计方式

### 与同类游戏的对比

| 对比维度 | Sims 4 | 其他人生沙盒 |
|:--------|:-------|:-----------|
| 开发模式 | AAA大团队+10年服务 | 大多数为独立/小团队制作 |
| 内容量 | 极多(30+DLC) | 少(或MOD社区补充) |
| 情感系统 | 独家(16种情绪) | 大多无情绪系统 |
| 建造自由 | 极高 | 大多数有限 |
| 商业模式 | F2P+DLC | 买断制为主 |
| 社区平台 | Gallery内置 | 依赖外部平台 |
