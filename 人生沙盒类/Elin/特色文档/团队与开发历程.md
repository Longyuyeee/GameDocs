# 团队与开发历程 — Elin

> noa (Lafrontier) — 日本独立游戏开发者
> Steam EA: 2024年11月1日 | 引擎: C# + Unity | 团队规模: **1人**

---

## 一、从Elona到Elin的17年

| 时间 | 事件 |
|------|------|
| 2007年8月 | Elona v1.0发布(HSP引擎，免费) |
| 2010年11月 | noa因孤独和缺乏反馈停止Elona开发，公开源代码 |
| 2012-2015年 | 多次尝试制作续作/衍生作，均因"资源和个人动力不足"在1-2年内崩溃 |
| 2016年 | **Elin正式公布**，作为Elona十周年纪念项目 |
| 2016-2023年 | Elin缓慢开发（引擎从HSP迁移到C#/Unity） |
| 2023年 | Kickstarter众筹成功 |
| **2024年11月1日** | **Elin Steam EA上线** |
| 2024-2025年 | 首年200+次更新/35万+销量/1,500+MOD |

## 四、开发哲学

### noa的核心原则

1. **"深而非广"**——宁可10个深度系统，不添加20个浅层功能
2. **"系统交互优于系统数量"**——一个能与所有现有系统交互的新系统胜过3个独立系统
3. **"玩家应该能做出设计师都想不到的事"**——涌现性是最高的设计成就
4. **"规范玩家的行为是毫无意义的"**——不做硬禁止，只做软后果

### Elin的开发重点转变

| Elona的重点 | Elin的重点 |
|------|------|
| 定义"可以做什么"的边界 | **给你空间去"建造什么"** |
| 混沌中的自由 | **秩序中的自由**（基地建设需要结构） |
| 一个人的世界 | **有社区的世界**（居民/城镇） |

## 五、noa的回归故事

> "经过了将近20年的游戏开发，我在Elona时期播下的种子终于发芽了。与很久以前的'姐妹们'(玩家们)重逢，是我现在最大的动力。"

这段来自AUTOMATON采访的话概括了整个故事——一个开发者因为孤独放弃了游戏，17年后因为社区的支持重新找回了热情。Elona的源代码公开创造了Elona+（一个持续15年的社区分支），而现在Elin的成功证明了那个17年前的设计DNA从未过时。

---

## 六、设计启示

1. **技术栈会过时，系统不会**——HSP语言今天无人使用，但Elona的系统设计逻辑在17年后仍然领先
2. **社区是孤独的解药**——从0反馈到35万玩家，noa的故事是关于"社区"价值的证明
3. **EA模式让单人开发可持续**——每日更新的收入流让一个人可以全职开发一款深度RPG
4. **开放源代码创造了第二生命**——Elona+让原版游戏多活了15年，显示了开源对游戏生命的延长效应
