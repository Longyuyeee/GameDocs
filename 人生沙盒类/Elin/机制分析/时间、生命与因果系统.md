# 时间、生命与因果系统 — Elin

> 游戏版本: Elin Steam EA (2025.6)
> 数据来源: [Ylvapedia Wiki](https://ylvapedia.wiki/), [Steam Tips Guide](https://steamcommunity.com/sharedfiles/filedetails/?id=3401513288)

## 一、与Elona时间/生命系统的差异

| 维度 | Elona | Elin |
|------|------|------|
| 睡眠功能 | 恢复HP/MP+做梦(提升潜力+宝藏梦) | **仅恢复体力**+仅法术梦 |
| 食物腐败 | 无 | **4阶段腐败**(新鲜→微陈→陈→腐坏) |
| 同伴饥饿 | 不需要进食 | **需要喂食**(不致死) |
| 死亡惩罚 | 随机降属性/技能+掉金币 | **物品不掉落**——仅降属性/技能 |
| 负重致死 | 超重=不能移动 | 采集中被压垮→HP降至0但不死 |
| 命周期 | 老化系统 | 保留(但EA中细节待确认) |

## 二、食物腐败系统（核心新增）

### 腐败阶段

| 阶段 | 可食用 | 效果 | 处理策略 |
|:--:|:--:|------|------|
| 新鲜(Fresh) | 是 | 全效果 | 直接食用/出售 |
| 微陈(Slightly Stale) | 是 | 轻微降低 | 尽快加工 |
| 陈(Stale) | 风险 | 大幅降低 | 酿酒/制作保存品 |
| 腐坏(Rotten) | 否 | 可能致病 | 丢弃或特殊用途 |

### 腐败驱动的行为改变

腐败系统改变了玩家的资源管理行为：
- **及时加工**——收获后立即加工(蘑菇酿酒/鱼做柴鱼片)
- **不要囤积**——过量新鲜食材=浪费
- **规划产出**——种刚好需要的数量

## 三、睡眠系统简化

### Elin的睡眠

| 功能 | Elona | Elin |
|------|:--:|:--:|
| 回复体力 | 是 | **是(主要功能)** |
| 回复HP/MP | 是(缓慢) | 有限 |
| 提升技能潜能 | **是(做梦)** | **仅法术梦** |
| 提升属性潜能 | **是(做梦)** | 有限 |
| 宝藏梦 | 有 | **无** |

### 简化意图

Elin的睡眠简化与"每日祈祷全队回满"机制互补——在Elona中睡眠是HP/MP/潜能的全功能恢复站，在Elin中这些功能被分散到了不同的机制中（祈祷/训练/药草）。这是"解耦恢复功能"的设计——防止单一机制过于万能。

## 四、死亡惩罚减轻

| 惩罚 | Elona | Elin |
|------|:--:|:--:|
| 属性降低 | 是(随机) | 是 |
| 技能降低 | 是(随机) | 是 |
| 金币损失 | 是 | 是(减少) |
| **物品掉落** | **是** | **否** |
| 当场复活 | 卷轴/魔法 | 保留 |

### "物品不掉落"的设计影响

这是Elin对Elona死亡惩罚最重大的改动——装备不会因为死亡而丢失。这意味着：
- 玩家敢于使用珍贵装备冒险（而非一直锁在仓库）
- "恐惧损失装备"不再是冒险的障碍
- 但属性/技能的随机降低仍然保留了死亡的重量

## 五、采集中的"假死"机制

Elin新增：采集任务中超重被压垮→HP降至0但不真正死亡：
- 防止"为了一堆蘑菇而暴毙"的尴尬
- 同时为任务走私创造了可利用的机制（卡时机超额采集）
- 体现noa的"少惩罚玩家，多给有趣的下场"设计哲学

## 六、设计特点分析

**食物腐败=即时加工的经济循环驱动器**：腐败系统不是为了让游戏更难——它是为了让"种植→收获→加工→出售"成为一个**必须及时完成**的循环。如果食物不会腐败，玩家可以囤积1000个蘑菇然后一次性加工——那就会失去"日常经营"的节奏感。

**睡眠简化+"每日祈祷"=两个恢复锚点的交响**：睡眠恢复体力（短期恢复），每日祈祷恢复HP/MP（每日一次大恢复）。两个机制分工明确——你需要频繁睡眠但你只能每天祈祷一次。这创造了一个"短期/日常"的两层恢复节奏。

**物品不掉落=鼓励冒险的温和设计**：这是Elin比Elona更"温和"的少数几个重大改动之一。在Elona中，"装备会掉落"的恐惧让许多玩家不敢穿最好的装备下地城——全身穿备用品。Elin消除了这个障碍，鼓励玩家"拿出你最好的装备去冒险"。

---

> 跨品类对比: Elin的腐败系统比Don't Starve更温和(只有4阶段/不会立即惩罚)——设计重心是"驱动加工"而非"制造生存压力"。死亡不掉落装备让Elin更接近"现代Roguelite"(Hades式的死后保留)而非"传统Roguelike"(NetHack式的丢失一切)。
