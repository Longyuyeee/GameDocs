## 一、"可以做什么"的完整谱系

### Elona允许你做的非常规事情

| 行为类别 | 具体可做的事 | 所需条件 |
|------|------|------|
| **暴力** | 攻击任何NPC(包括友好) | 无限制 |
| **谋杀** | 杀害商人/守卫/国王/神祇 | 战斗能力 |
| **偷窃** | 扒窃任何角色、偷商店物品 | 偷窃技能 |
| **毁灭** | **在城镇中引爆核弹**——全城变为废墟 | 获得核弹道具 |
| **绑票** | 使用支配术捕获任何人形NPC | 支配术/精灵球 |
| **基因实验** | 将两个生物融合成一个新物种 | 基因工程技能+基因机器 |
| **开店** | 自己定价、雇店主、经营连锁 | 开业许可+柜台 |
| **演奏** | 街头卖艺、接表演任务 | 演奏技能+乐器 |
| **农耕** | 购买牧场→种地→收获→出售 | 农场+园艺技能 |
| **钓鱼** | 全世界任何水域钓鱼 | 钓竿+钓鱼技能 |
| **结婚** | 与宠物/NPC结婚 | Soul Mate好感 |

| **奴隶交易** | 购买/出售人形"商品" | 访问德尔菲 |
| **篡改祭坛** | 将任何祭坛转换为你的信仰 | 献祭 |
| **自杀** | 监狱中有绞索(逃脱手段之一) | — |
| **赌博** | 赌场玩骰子游戏 | 赌场 |

## 二、自由度的"边界"设计

### 不做硬限制，做软后果

Elona几乎不禁止你做任何事情，但**一切都有后果**：

| 行为 | 没有硬禁止 | 但有后果 |
|------|:--:|------|
| 杀NPC | 任何NPC都可被杀 | 守卫敌对/城镇功能丧失/善值下降 |
| 偷窃 | 任何NPC都可偷 | 失败→善值-5/被发现→永久敌对 |
| 引爆核弹 | 任何城镇都可炸 | 善值-80(非罪犯)/全镇变为废墟 |
| 不吃不喝 | 不进食不会死 | 属性惩罚/体力耗尽 |

### 设计哲学："规范玩家的行为是毫无意义的"

noa的设计哲学可以从一件事中看出——游戏允许无限重掷初始属性。为什么？因为"你想花时间掷完美属性，那是你的自由"。同样的逻辑延伸到整个游戏：**如果玩家想屠城，让他们屠——但世界会记住。**

## 三、涌现性叙事

### 非脚本故事的生成

Elona没有"主线剧情"（虽然有一个最小主线）。大部分玩家故事是涌现式的——由系统交互自发产生：

| 涌现类型 | 示例 |
|------|------|
| **意外悲剧** | 在以太风季节穿着以太装备外出→迅速感染致命症状组合→角色被太病毁了 |
| **宠物故事** | 一只精心培育的火幼龙在进化到最终阶段前被火球误杀(友军伤害) |
| **经济冒险** | 从一只老鼠起家→烹饪产业链→成为北泰里斯最富有的厨师 |
| **犯罪生涯** | 偷窃失败被抓→被守卫追捕→只能躲在德尔菲活动→通过赎罪最终重返社会 |
| **爱情故事** | 宠物从Fellow好感一路培养到Soul Mate→结婚→在战斗中配偶牺牲→再未结婚 |

### 涌现性的支撑条件

1. **系统之间的连锁反应**（偷窃→善值↓→商店拒绝→经济来源被切断→只能走犯罪路线）
2. **NPC的持续存在与记忆**(被杀的人不会复活，被偷的人会记住)
3. **不可逆的后果**(核弹炸掉的城镇真的变成废墟)
4. **随机元素的介入**(以太病症状组合是随机的，每个角色的"故事"都不同)

## 四、自由与引导的平衡

### 新手引导策略

| 方式 | 说明 |
|------|------|
| **主线线索** | 莱西墨苏的探索提供了一个方向性目标(但不强制) |
| **任务板** | 随机任务提供短期目标和白金币获取途径 |
| **以太风的倒计时** | 创造持续的"你需要在下一轮以太风前做好准备"的紧迫感 |
| **社区攻略** | 游戏故意不全面解释系统——依靠玩家社区传递知识 |

### "无所适从"问题的解决

纯沙盒游戏的常见问题是"我该做什么？"Elona通过以下方式解决：
- **明确但可选的主线**(去莱西墨苏)
- **硬件倒计时**(以太风)
- **持续的小奖励循环**(每清一个Nefia都有Boss+白金币+宝箱)
- **多方向的可玩性**(打不过地牢就去开店，开腻了就去钓鱼，鱼钓够了回去打地牢)

## 五、设计特点分析

**"核弹炸城=自由的边界测试"**：在城镇中引爆核弹可能是Elona中"可以做的最极端的事情"——它能将一座繁华的城镇永久变为废墟，全城NPC死亡。这不是一个实用性的机制——它存在的意义是告诉玩家"规则不是限制你的，是你可以打破的"。这是自由度设计的哲学宣言。

**软后果=道德的重量**：不是"不能杀NPC"的弹窗警告，而是"你杀了武器商人，以后再也买不到他卖的那些稀有武器了"——这种基于**物理后果**而非**道德说教**的设计让"恶"有了真实的重量。你做出的每一个暴力行为都在缩小你自己未来的选择空间。

**系统交互的乘法效应**：4个独立系统（以太病×装备材质×天气×城镇）可以产生16种不同的交互场景。当系统数量增长到Elona的15+个独立系统时，可能的涌现交互是天文数字——这就是涌现性的数学基础。

---

> 跨品类对比: 与RimWorld的AI叙事者驱动的涌现不同，Elona的涌现更依赖系统交互而非外部事件触发器。与太吾绘卷的"每个NPC有自己的生活=涌现"不同——Elona的涌现更多来自玩家行为×多系统交互。与Kenshi的无主角光环沙盒不同——Elona给玩家"主角光环"但允许你主动选择不使用它。
