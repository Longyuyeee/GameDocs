# 财富驱动威胁：反膨胀的游戏经济设计 — 环世界

> 大多数游戏中"更多财富=更好的处境"。RimWorld中"更多财富=更大的威胁"——这是经营游戏中最激进的"反膨胀"设计。

---

## 一、财富如何转化为威胁

| 财富来源 | 对威胁的影响 | 隐性风险 |
|------|:--:|------|
| 建筑(墙壁/地板/家具) | 高 | 你的坚固防御工事本身 = 更大的袭击 |
| 物品(装备/材料/食物) | 高 | 囤积 = 不知不觉中放大了下次袭击的规模 |
| 殖民者 | 中高 | 更多人手 = 更大的威胁(但同时也是更多战斗力) |
| 动物 | 中 | 农场的牛群 = 财富值的隐性增加 |
| 尸体/垃圾 | 低(但累积) | 清理不及时 = 财富值缓慢累积 |

## 二、"财富陷阱"

```
新玩家直觉：建更多 = 更安全 ❌
现实：建更多 = 财富增加 = 更大袭击 = 当前的防御不够用 = 殖民地毁灭 ✓
```

RimWorld惩罚"过度建设"。最有效的策略是：**只建你立刻需要的东西**。不要囤积——多余的装备和材料尽快卖掉或送给其他派系以降低财富值。

## 三、财富的"最优水平"

RimWorld的元策略是找到"刚好够用的财富"——足够支持殖民地的生存和发展，但不会因为财富过高而触发无法应对的威胁。这不是一个固定的数值——它是**你的防御能力 × 殖民者战斗力 × 当前威胁缩放 = 可管理的最大财富**。

## 四、设计启示

财富驱动威胁机制是游戏设计中最优雅的"反膨胀"方案——它解决了沙盒游戏的根本问题：玩家最终会积累太多资源使游戏失去挑战。RimWorld不需要"难度递增设置"——它通过财富-威胁的负反馈循环，让挑战始终与玩家的状态匹配。

---

> 核心洞察: RimWorld的财富系统创造了一种罕见的游戏体验——你不仅在与敌人战斗，你还在与你自己的"贪婪"战斗。这是一场关于"什么是足够"的哲学课。
