# Roguelite+商店经营跨界融合 - 夜勤人(Moonlighter)

> 游戏版本: Moonlighter v1.0+ (PC/Switch/PS4/Xbox, 2018年)

## 一、概述

夜勤人是**商店经营×Roguelite地牢探索**的跨界标杆。两个系统通过"地牢获取物品→商店出售→金币升级装备→更强地牢探索"形成完美的正向循环。这不是"两种玩法拼在一起"而是"两种玩法相互需要"。

## 二、双循环结构

```
[白天] 商店经营循环:
  定价→观察反应→调整→出售→赚金币
    ↓ 用金币升级装备
[夜晚] 地牢探索循环:
  下地牢→杀怪→开宝箱→获取物品→(死亡风险)→返回
    ↓ 带回物品
[白天] 商店经营循环(重复)
```

## 三、两个系统的联动节点

| 联动点 | 机制 | 设计意图 |
|------|------|------|
| 地牢物品→商店库存 | 所有商品来自地牢采集 | 探索风险=采购成本 |
| 金币→装备升级 | 赚的钱提升战斗能力 | 经营成功=战斗力 |
| 装备→更深地牢 | 更强装备→更难地牢→更好物品 | 正向难度曲线 |
| 吊坠脱离 | 保物品→回商店→赚金币 | 风险vs收益的决策 |
| Jelly→药水 | 不出售Jelly→自制廉价药水 | 材料管理策略 |
| 愿望单 | 锁定装备→地牢中看到材料 | 目标驱动探索 |

## 四、Roguelite要素

| 要素 | 实现 |
|------|------|
| 程序化生成 | 每层房间/敌人/宝箱随机 |
| 死亡惩罚 | 失去背包中部分物品(保留装备) |
| 进度保留 | 装备/商店/金钱永久保留(非全部丢失) |
| 特殊房间 | 治疗池/秘密房间(跳入坑洞)/营地(剧情) |
| Boss不可刷 | 每Boss一局一次→推动前进 |
| 漫游者(Wanderer) | 每层时间限制→强制快节奏→反磨蹭 |

## 五、与传统Roguelite对比

| 维度 | 传统Roguelite(以撒等) | 夜勤人 |
|------|:-----:|:-----:|
| 资源用途 | 战斗内消耗 | **带回商店出售→赚金币** |
| 死亡损失 | 全部失去 | 仅失部分背包(保留装备+金币) |
| 进度驱动 | 通关 | 赚钱+升级商店+击败守护者 |
| 核心技能 | 战斗操作 | 战斗操作+**定价判断+库存管理** |
| 重玩动力 | 不同Build | 不同装备Build+更优定价策略 |

## 六、设计分析

1. **夜勤人=两种乐趣的融合**: Roguelite的"探索+紧张感"+商店经营的"策略+优化感"——同一个物品既是"战利品"也是"库存"——两种视角的满足
2. **轻量化Roguelite惩罚**: 不是"全丢光"而是"丢部分背包"→保留了Roguelite的紧张感但不至于崩溃→适合更广受众
3. **装备进度保留=RPG要素**: 传统Roguelite没有"装备永久升级"→夜勤人的装备系统更像ARPG——综合了Roguelite+RPG+经营
4. **Jelly悖论=跨系统策略深度**: 6g出售vs数千g药水制作——新手陷阱→需要跨系统(商店/地牢/制作)综合判断
5. **定价=后地牢玩法**: 地牢中的刺激(战斗+寻宝)→商店中的满足(定价+数钱)——两种"多巴胺"交替释放
