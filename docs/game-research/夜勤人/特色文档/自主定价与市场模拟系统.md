# 自主定价与市场模拟系统 - 夜勤人(Moonlighter)

> 游戏版本: Moonlighter v1.0+ (PC/Switch/PS4/Xbox, 2018年)

## 一、概述

夜勤人的核心创新在于将**价格发现**变成核心玩法。不是"系统给你固定价格→你卖"，而是"你定价→观察顾客反应→调整价格→找到最优区间"。配合隐藏的供需/流行度系统，创造了一个**动态的小型市场模拟器**。

## 二、定价四象限

```
           需求高(流行)
              ↑
    😡不满售出    |    😊满意(最优)
    (隐形损害)   |    (长期收益)
              |
  价格过高 ← ---+---→ 价格合适
              |
    😡愤怒拒买   |    😍狂喜
    (零收益)    |    (利润损失)
              ↓
           需求低(滞销)
```

## 三、价格发现的游戏性

### 3.1 首次定价(未知物品)
1. 给一个直觉价位
2. 观察首个顾客反应
3. 😍→提价 / 😊→锁定 / 😐→微降 / 😡→大幅降
4. 重复调整直到找到"😊满意"价位

### 3.2 价格笔记的局限
- 游戏提供"商人笔记本"记录历史价格
- 但笔记本**不动态更新**因供需变化导致的最优价变动
- 玩家必须**持续观察反应**→不能"设一次忘掉"

### 3.3 需求衰减的长期管理
- 同物品卖太多→需求下降→最优价下滑
- 😡不满交易→加速需求衰减
- 展示柜出售有概率**+1流行度**(恢复需求)
- →策略: 不要连续大量出售同一物品，轮换品类

## 四、顾客类型=定价维度

| 顾客 | 价格弹性 | 策略 |
|------|:--:|------|
| 普通顾客 | 敏感 | 标准最优价 |
| 英雄 | 中等 | 只买装备类 |
| 富人 | **零弹性** | 展示柜→2倍价 |

## 五、与其他游戏的经济系统对比

| 游戏 | 定价方式 | 市场动态 |
|------|:--:|:--:|
| 星露谷 | 固定价格(出货箱) | 无动态 |
| 符文工房4 | 固定价格(商店) | 无动态 |
| 波西亚/沙石镇 | 固定价格+浮动市场价 | 轻微浮动 |
| **夜勤人** | **完全自主定价** | **需求衰减/流行度/顾客类型四维动态** |
| 露塞提娅(Recettear) | 讨价还价系统 | 供需+顾客类型 |

## 六、设计分析

1. **定价=探索**: 每个新物品都是一次"价格发现"的迷你解谜——不是数据陈述而是玩法
2. **隐性惩罚>>显性奖励**: 不满售出→损害流行度→未来收益下降——很多玩家长时间没意识到为什么"明明卖出去了生意却变差"
3. **展示柜=策略性金融工具**: 2倍售价+防窃+恢复流行度——三重功能→不是纯装饰
4. **模拟真实市场的信息不对称**: 你不知道"真实价值"——只能通过顾客反馈推断——和真实市场一样
5. **堆叠影响定价**: 价格是按"桌子"设的，不是按"件"——定价策略受展示方式影响
