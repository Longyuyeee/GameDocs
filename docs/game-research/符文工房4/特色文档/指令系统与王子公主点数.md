# 指令系统与王子公主点数（Order System & Prince/Princess Points）
![RF4指令](https://www.runefactory4.com/img/system/ss_order.jpg)

## 概述

「指令系统（Order System）」与「王子/公主点数（Prince/Princess Points，简称 PP）」是《符文工房4》区别于系列前作最核心的创新设计。这一系统将城镇发展权交给玩家，通过完成委托和参与活动积累 PP，再消耗 PP 解锁各类功能、设施和权限，形成「付出→回报→发展」的驱动循环。PP 系统从根本上改变了 RF 系列的成长逻辑——玩家不再仅仅是「一个住在镇上的农民」，而是真正意义上的城镇统治者。

## 一、PP 系统基础

### 1.1 PP 的获取方式

| 获取途径 | PP 量 | 频率 | 说明 |
|---------|------|------|------|
| **Eliza 委托请求（Requests）** | 10~50 PP / 个 | 每日可接 | 主要获取方式，分为日常/周常/剧情 |
| **节日参与/优胜** | 50~200 PP / 次 | 节日日 | 优胜奖励更高 |
| **主线剧情推进** | 50~300 PP / 阶段 | 一次性 | 关键剧情节点给予 |

| **作物出货量里程碑** | 100~500 PP | 累计达成 | 如出货 1000 个作物 |
| **钓鱼/战斗里程碑** | 100~300 PP | 累计达成 | 对应统计数据达到阈值 |

### 1.2 PP 总量的阶段性积累

|---------|------------|---------|
| Act 1 初期（春~夏） | 200~500 PP | Eliza 委托为主 |

## 三、Eliza 委托系统

### 3.1 Eliza 的角色

Eliza 是赛尔菲亚城堡中负责发布委托的女性 NPC，玩家可以通过她获取 PP 和赏金。她是 PP 系统中最核心的交互对象。

- **位置**：赛尔菲亚城堡王座厅左侧
- **工作时间**：全天可用
- **核心功能**：发布请求（Requests）、追踪完成进度

### 3.2 委托类型

| 类型 | 内容示例 | PP 奖励 | 金币奖励 | 刷新频率 |
|------|---------|---------|---------|---------|
| **日常委托（Daily）** | 出货 3 个芜菁、采集 5 个药草 | 10~20 PP | 100~300 G | 每日刷新 |
| **周常委托（Weekly）** | 击败 10 只魔物、钓鱼 5 条 | 20~30 PP | 300~500 G | 每周刷新 |
| **挑战委托（Challenge）** | 在不使用道具的情况下击败指定 BOSS | 30~50 PP | 500~1,000 G | 一次性 |
| **成就委托（Achievement）** | 累计出货 1000 个作物、图鉴完成度 50% | 50~100 PP | 1,000~3,000 G | 里程碑达成时 |

### 3.3 委托策略

- 优先接取 PP 回报率高的委托（单位时间 PP 产出最大化）
- 日常委托和周常委托可以并行完成（如「出货作物」和「获得金币」可同时推进）
- 成就委托是 PP 大头，建议长期规划

## 四、PP 驱动的发展循环

PP 系统的核心设计理念是形成一条完整的玩家成长闭环：

```
                ┌────────────────────────┐
                │  完成委托/参与节日      │
                │    → 获得 PP            │
                └────────┬───────────────┘
                         │
                         ▼
                ┌────────────────────────┐
                │  消耗 PP 发布指令        │
                │    → 解锁新功能/设施     │
                └────────┬───────────────┘
                         │
                         ▼
                ┌────────────────────────┐
                │  新功能提升效率          │
                │    → 更高效地完成委托    │
                └────────┬───────────────┘
                         │
                         ▼
                ┌────────────────────────┐
                │  城镇发展 → 玩家成就感   │
                │    → 继续投入循环         │
                └────────────────────────┘
```

### 4.1 循环的核心意义

1. **玩家自主权**：完全由玩家决定城镇发展的优先级和方向
2. **渐进式解锁**：避免前期信息过载，每阶段解锁适量新内容
3. **目标引导**：PP 目标引导玩家尝试游戏内的各种系统（钓鱼、种植、战斗等）
4. **长期驱动**：后期仍然有高额 PP 目标（如全部指令解锁、全设施升级）

## 五、与 RF3 公告板系统的对比

### 5.1 对比表格

| 项目 | RF3（符文工房3）公告板系统 | RF4（符文工房4）PP / 指令系统 |
|------|--------------------------|---------------------------|
| **核心机制** | 在公告板接取任务赚取金币和声望 | 通过委托赚取 PP，消耗 PP 发布「指令」 |
| **任务来源** | 公告板固定刷新 | Eliza 委托 + 里程碑成就 |
| **奖励类型** | 金币 + 道具为主 | PP + 金币 + 城镇发展 |
| **城镇影响** | 完成任务提升小镇评价 | PP 指令直接影响城镇设施和功能 |
| **自由度** | 线性，任务选择有限 | 高度自由，玩家决定解锁顺序 |
| **深度** | 单层——接任务→完成→奖励 | 双层——赚 PP → 选择发展方向 → 解锁 → 新可能 |
| **终局内容** | 任务做完即止 | PP 持续驱动终局发展 |
| **玩家身份** | 冒险者 | **王子/公主（统治者）** |

### 5.2 RF4 PP 系统的核心创新

1. **身份转变**：从 RF3 的「外来者/冒险者」变成 RF4 的「王子/公主」，指令系统的设定与身份高度统一
2. **决策权重**：PP 的稀缺性（前期）迫使玩家做出有意义的选择——先解锁锻造还是先扩容背包？
3. **成就转化**：玩家在任意系统的投入都能转化为 PP，形成全系统联动
4. **后期价值**：普通 RPG 后期金币过剩，RF4 的 PP 在后期依然有价值（高额指令持续消耗 PP）

## 六、策略建议

### 6.1 PP 使用优先级

| 阶段 | 推荐优先级 | 理由 |
|------|-----------|------|
| **初期（0~500 PP）** | 背包扩容 Lv.1（500 PP） | 背包容量是制约效率的最大瓶颈 |
| **前期（500~1,500 PP）** | 锻造许可证（500 PP）→ 化学许可证（500 PP） | 制造系统是经济循环的核心 |
| **中期（1,500~3,000 PP）** | 魔物棚 × 2~3 → 田地扩张 Lv.1 | 扩大生产规模 |
| **后期（3,000~5,000 PP）** | 节日许可证 → 传送点 → 跨季种植 | 丰富游戏体验 |
| **终局（5,000+ PP）** | 全设施解锁 → 全指令完成 | 完美收集 |

### 6.2 PP 刷取建议

- **稳定刷 PP**：每天必接 Eliza 的日常委托，积少成多
- **节日前准备**：每个节日确保参加并争取优胜，PP 奖励丰厚
- **监控里程碑**：关注成就类委托条件，有针对性地推进
- **不要追求一次性解锁全部**：PP 系统设计本意是渐进式体验，分阶段解锁更有乐趣
