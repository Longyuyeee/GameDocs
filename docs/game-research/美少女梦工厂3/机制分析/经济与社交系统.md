## 4. 社交系统

### 4.1 信任(Trust)——核心关系属性

| 提升方式 | 降低方式 |
|------|------|
| 度假(Vacation) | 长期不理女儿 |
| 送礼 | 做女儿讨厌的事 |
| 自由行动(共度时间) | 特定事件选择 |

- 信任<50→**触发妖精女王访问**(关键事件!)
- 信任=妖精与女儿的关系衡量——不是"亲密度"而是"信任度"

## 6. 设计特点

1. **信任=PM3的情感核心**: 不是PM2的"父女亲密"→而是"妖精与人类女儿之间的信任"
2. **竞争对手=7个额外的社交维度**: 不仅是"打败她"→还可以和她成为朋友——社交的多样性
3. **节日=循环事件**: 3个节日按顺序循环——每年都有→提供稳定的社交机会
4. **度假=金币→信任转换**: 花费金币+时间→换取信任提升——经济与社交的资源转换
