# 风险与背包管理系统 - 夜勤人(Moonlighter)

> 游戏版本: Moonlighter v1.0+ (PC/Switch/PS4/Xbox, 2018年)
> 数据来源: moonlighter.fandom.com

## 1. 背包系统

### 1.1 基础机制

| 参数 | 说明 |
|------|------|
| 初始容量 | 有限格数(每个物品占用1格) |
| 堆叠上限 | 每种物品有独立堆叠上限(5或10) |
| 扩展 | 储物箱升级(小→中→大, 每级+28格) |
| 家中储物 | 制作材料可自动从家中存储读取 |

### 1.2 背包物品类型

| 类型 | 用途 | 建议 |
|------|------|:--:|
| 普通材料 | 制作/出售 | 优先出售(留制作所需) |
| Jelly系列 | 制作药水/附魔 | **绝对不出售** |
| Boss掉落 | 剧情/成就 | 保留(托莫想看) |
| 高价物品 | 出售 | 展示柜高价卖出 |

## 2. 死亡惩罚(风险核心)

| 情况 | 惩罚 |
|------|------|
| 地牢死亡 | 失去**背包中随机部分物品**(保留装备/武器) |
| 安全返回(吊坠) | 保留全部背包物品 |
| 正常通关返回 | 保留全部 |

## 3. 吊坠(Pendant)决策

- 地牢中随时可使用→立即返回商店
- **保留所有背包物品**
- 策略决策: 继续深入(可能获得更好物品但风险死亡) vs 安全撤退(锁定当前收益)

## 4. 地牢物品选择策略

| 物品价值 | 策略 |
|------|------|
| 高价材料(如魔像历史I-III: 2,750g) | 优先保留(背包满了→舍弃低价值) |
| 低价材料(如藤蔓: 3g) | 满背包时优先丢弃 |
| Jelly | **绝对不丢**(制作价值远高于出售价) |
| 愿望单物品 | 优先拾取(制作装备所需) |

## 5. 商店库存管理

| 因素 | 策略 |
|------|------|
| 高价物品 | 放展示柜→卖给富人→2倍价+防窃 |
| 中价物品 | 放普通桌子→测试最优价格 |
| 低价/滞销品 | 放入特价箱(Sale Box)→自动出售(75%→50%折扣) |
| 防窃布局 | 最贵物品放在展示柜(防窃)或靠近收银台(可快速反应) |

## 6. 设计特点

1. **死亡保装备丢背包**: 比全丢(传统Roguelike)宽容，比无惩罚(休闲)刺激——精妙的中间地带
2. **吊坠=风险决策按钮**: 不是"继续"和"离开"两选一——而是"继续有更多奖励但可能损失当前所有"
3. **物品价值差距巨大**: 3g到26,400g——背包满了必须决策→"丢掉什么？"
4. **Jelly悖论**持续考验: 6g出售价vs数千g的药水制作价值——新手陷阱
5. **展示柜=策略性防窃**: 最贵物品放在防窃展示柜vs需要策略布局→商店管理有空间规划深度
