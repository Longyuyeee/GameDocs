# 服务端经营设计分析 - 潜水员戴夫(Dave the Diver)

> 游戏版本: Dave the Diver v1.0+ (PC/Switch, 2023年)

## 一、概述

Dave the Diver属于"服务端经营"——不是"生产→出货"(星露谷)，也不是"定价→卖货"(夜勤人)，而是**在实时服务流程中管理厨房+服务+顾客体验**。这是最接近现实餐饮业模拟的经营模式。

## 二、服务端经营vs传统经营

| 维度 | 传统牧场(星露谷) | 商店经营(夜勤人) | **服务端经营(潜水员戴夫)** |
|------|:--:|:--:|:--:|
| 核心 | 生产→出货 | 定价→出售 | **实时服务流程** |
| 操作 | 浇水/收获(缓慢) | 定价/观察(静态) | **送餐/倒茶/清桌/补芥末(实时)** |
| 压力 | 无(无限天) | 低(顾客等待) | **高(红条顾客+芥末耗尽+高峰期)** |
| 品质 | 作物星级 | 定价精确度 | **Cooksta评级(味道+速度)** |
| 员工 | 无 | 店员(自动抓贼) | **招聘+培训+分配+派遣** |

## 三、服务流程的操作压力

```
18:00 开店
  ↓ 顾客涌入→分配桌子
18:05 菜单开始被点
  → 厨房做菜 → 服务员送餐
  → 倒绿茶(完美倒茶=最大金币)
  → 清理空桌
  → **芥末剩余量检查**(最优先!)
18:20 高峰期 → 同时5-8桌在等
  → 红条顾客开始出现
  → 必须快速送餐否则评级降
19:00 高峰期缓解
22:00 关店→统计→Artisan's Flame→强化
```

## 四、芥末=服务端经营的核心隐喻

- 芥末=寿司的"燃料"=厨房的"运作能力"
- 芥末耗尽=不能做任何菜=全厨房停摆
- 补充芥末=最优先的操作
- 隐喻: **服务业的"瓶颈资源"管理**

## 五、对比分析

| 游戏 | 经营类型 | 实时压力 | 品质驱动 |
|------|:--:|:--:|:--:|
| 星露谷 | 生产端 | 无 | 作物品质 |
| 夜勤人 | 销售端 | 低(静态) | 定价精确 |
| 露塞提娅(Recettear) | 销售端 | 高(讨价还价) | Pin Price命中 |
| **潜水员戴夫** | **服务端** | **最高(实时多任务)** | **Cooksta评级+味道** |

## 六、设计分析

1. **服务端=最接近真实工作的经营**: 倒茶/送餐/清桌/补芥末——不是"管理策略"而是"操作技能"
2. **高峰期=难度曲线**: 不是"越来越难的地牢"而是"越来越多的顾客"——自然的难度增长
3. **员工=难度缓解**: 培训员工→他们帮你做更多操作→从"手忙脚乱"到"如臂使指"
4. **Cooksta=经营等级**: 不是"赚了多少钱"而是"餐厅的声誉等级"——更高评级=更多顾客=更多压力=需要更好员工
