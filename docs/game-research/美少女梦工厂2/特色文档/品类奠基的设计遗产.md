# 品类奠基的设计遗产 - 美少女梦工厂2(Princess Maker 2)

> 游戏版本: Princess Maker 2 Refine (PC, 1993/2016年)

## 一、概述

PM2是**整个子女养成品类的蓝图**。几乎所有后续养成游戏的核心系统都能在PM2中找到原型——日程安排、多维属性、多结局分支、打工/课程经济、父女关系、比赛/事件系统。火山的女儿本质上是对PM2在现代技术条件下的全面复兴。

## 二、PM2首创的品类模板

| 设计 | PM2的实现 | 被继承的作品 |
|------|------|------|
| **月×3行动回合** | 每月安排3个活动 | 火山的女儿(完全一致!) |
| **8年培养周期** | 10岁→18岁 | 火山的女儿(12年)/PM5(10年) |
| **打工/课程双线** | 打工(赚金币)+课程(消耗金币/提升属性) | 所有养成游戏 |
| **多结局分支** | 79结局(7步判定) | 火山的女儿58结局 |
| **多维属性** | 10+属性×4评价 | 所有养成游戏 |
| **年度比赛** | 4季度比赛 | 火山的女儿(火山节) |
| **特殊婚姻** | 5种(王子/父亲/管家/魔王/龙) | 火山的女儿(18种情感结局) |
| **父亲角色** | 玩家=父亲 | 火山的女儿 |
| **守护神Build** | 9种开局选择 | 火山的女儿(简化/移除了) |
| **探索战斗** | PM2独有(系列唯一) | 火山的女儿(冥滩战斗=复兴!) |

## 三、PM2→火山女儿的进化线

| 维度 | PM2(1993) | 火山女儿(2023) |
|------|:--:|:--:|
| 结局数 | 79 | 58 |
| 结局选择 | 自动判定(7步) | **手动选择**(先职业→再情感) |
| 战斗系统 | 有(回合制RPG) | 有(简化回合制) |
| 社交系统 | 5特殊婚姻 | **18情感结局**(含百合线) |
| 道德维度 | 罪孽+道德 | 黑暗值 |
| 多周目 | 无(无继承) | **继承点+服装保留** |
| 美术 | 像素/手绘 | 现代2D动画 |
| 品质门槛 | 气品800=女王 | 8学科500+全属性2000=火山女神 |
| 守护神 | 9种开局Build | 3领主选择 |

## 四、PM2中未被继承的独特设计

| 设计 | 为什么没被继承 |
|------|------|
| 罪孽+道德双维度 | 太复杂——后续游戏简化为单一"黑暗值" |
| 打工年龄补正(×1.5-4.0) | 太复杂——火山的女儿没有年龄倍率 |
| 父亲职业选择 | 火山的女儿简化(仅指定初始身份→无职业选择) |
| 女儿离家出走机制 | 太残酷——火山的女儿移除了 |
| 比武大会→王子嫁 | 设计巧妙——但太特定(仅PM2有王子角色) |

## 五、设计分析

1. **PM2=品类中设计最"完整"的游戏**: 79结局/10属性/4评价/5婚姻/9守护神——系统密度在1993年难以置信
2. **火山的女儿=PM2的现代翻译**: 核心系统1:1映射——只是用现代美术+QoL重新包装
3. **288回合=有限资源的养成哲学**: PM2确立的核心理念——不是"无限天想做什么做什么"，而是"有限回合内做出选择"
4. **父嫁=品类最大胆的设计**: 1993年引入父亲恋爱结局——至今仍具争议性——但确立了"养成游戏中玩家与养成对象的感情可以超越父女关系"
