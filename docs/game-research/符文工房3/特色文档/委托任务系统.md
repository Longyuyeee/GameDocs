# 委托/任务系统 - 符文工房3 豪华版(Rune Factory 3 Special)(特色系统)
![RF3 委托板](https://manuals.marvelousgames.com/rf3sp/12/img/top.jpg)

> 游戏版本: 符文工房3 豪华版(Rune Factory 3 Special)(Switch / Steam, 2023年)

## 1. 系统概述

符文工房3(Rune Factory 3)的委托系统是连接社交、剧情与日常玩法的核心纽带。不同于传统RPG的"主线任务+支线任务"二分,符文工房3(Rune Factory 3)采用**公告板(Bulletin Board)制+角色专属委托链**的多层结构,每个候补都有一条独立的委托故事线。

## 2. 委托发布来源

### 2.1 三级公告系统

| 公告来源 | 位置 | 发布者 | 接受限制 |
|---------|------|--------|---------|
| **公告板(Bulletin Board)** | 希雅蕾斯(Sharance)广场 | 全体村民 | 每日1个 |
| **邮箱(Mailbox)** | 玩家家门口 | 各角色(含候补专线) | 按进度触发 |
| **猫头鹰(Owl)** | 尤尼维尔聚落(Univir Settlement) | 库鲁娜(Kuruna,独角兽族候补)及其他尤尼维尔/独角兽族(Univir)居民 | 每日1个 |

### 2.2 委托类型

| 类型 | 内容 | 奖励 |
|------|------|------|
| **日常委托** | 提交特定物品(作物/鱼/怪物掉落) | 金币+小道具 |
| **剧情委托** | 角色故事线推进,带对话和剧情 | 爱情点数(LP/Love Points)+5 + 解锁新事件 |
| **教程委托** | 引导玩家学习游戏系统 | 基础工具/种子 |
| **挑战委托** | 击败特定怪物/到达特定区域 | 稀有道具/配方面包(Recipe Bread) |

## 3. 候补专属委托链

### 3.1 基本规则
- 每个候补(11位)有 **9-10个专属委托**
- 委托按好感度逐级解锁(更高的爱情点数(LP/Love Points)等级→新的委托出现)
- 完成全部10个委托是**结婚的必须条件**
- 委托通过邮箱(Mailbox)(主要)或猫头鹰(Owl)(库鲁娜(Kuruna,独角兽族候补)的前4个)发布

### 3.2 库鲁娜(Kuruna,独角兽族候补)特殊委托链
库鲁娜(Kuruna,独角兽族候补)的委托来源不同(她是尤尼维尔/独角兽族(Univir)):

| 委托 | 来源 | 备注 |
|------|------|------|
| 委托 1-4 | 尤尼维尔猫头鹰(Univir Owl) | 前4个在尤尼维尔聚落(Univir Settlement)完成 |
| 委托 5-10 | 邮箱(Mailbox) | 统一祭(Unity Festival)后通过邮箱(Mailbox) |

### 3.3 委托链的设计意义
- **每个角色=一条支线剧情**: 不是"送礼→好感→结婚"的简单流程,而是通过10个委托逐步揭示角色背景
- 例如: Raven 的委托链涉及她的真实身份和过去

## 4. 每日限制与节奏控制

### 4.1 接受限制

| 限制项 | 规则 |
|--------|------|
| 公告板(Bulletin Board) | 每日最多接受 1 个 |
| 猫头鹰(Owl) | 每日最多接受 1 个 |
| 邮箱(Mailbox) | 无每日限制(按进度触发) |
| 最大同时进行 | 多个委托可同时持有 |

### 4.2 节奏设计意图
- 防止玩家一天内刷完所有委托
- 强制分配游戏内容到多天
- 配合符文点数(RP/Rune Points)消耗系统,一天内能做的事情本来就有限

## 5. 奖励结构

| 委托类型 | 典型奖励 |
|---------|---------|
| 日常委托(村民) | 50-500G,基础道具,种子 |
| 候补委托(前期) | 爱情点数(LP/Love Points)+5,少量金币 |
| 候补委托(后期) | 爱情点数(LP/Love Points)+5,稀有道具,剧情推进 |
| 教程委托 | 基础工具(水壶/锄头),种子 |
| 特殊委托 | 配方面包(Recipe Bread),稀有种子 |

## 6. 与同类游戏对比

| 维度 | 符文工房3(Rune Factory 3) | 星露谷物语(Stardew Valley) | 符文工房4(Rune Factory 4) |
|------|---------|-----------|----------|
| 任务来源 | 公告板(Bulletin Board)+邮箱(Mailbox)+猫头鹰(Owl) | 邮箱(Mailbox)+社区中心+公告板(Bulletin Board)(1.5) | 公告板(Bulletin Board)+邮箱(Mailbox)+指令系统(Order system) |
| 角色个人故事 | 10个委托链/候补 | 好感度事件(2-14心) | 恋爱事件+日常 |
| 每日限制 | 每日1个/来源 | 每日2个(公告板(Bulletin Board)) | 无严格限制 |
| 主线推动 | 公告板(Bulletin Board)(剧情)+迷宫击败 | 社区中心+Joja超市(Joja Mart) | 主线任务+指令系统(Order system) |
| 教程委托 | 有(引导系统) | 无(通过TV+图书馆自学) | 有(引导任务) |

## 7. 设计特点

1. **三级公告系统(公告板(Bulletin Board)+邮箱(Mailbox)+猫头鹰(Owl))**: 物理上将不同来源的委托区分,而不是一个UI全收
2. **角色委托链=深层叙事**: 每个候补10个委托不是凑数,而是角色故事——这是"社交系统"进化为"叙事系统"的关键
3. **双聚落差异化**: 希雅蕾斯(Sharance)和尤尼维尔/独角兽族(Univir)聚落各有独立公告系统,强化世界观
4. **每日1个限制=反刷设计**: 防止玩家一口气完成所有委托,保持每日新鲜感
5. **结婚必须完成10个委托**: 将社交任务化,给"追求结婚"一个可量化的路径
